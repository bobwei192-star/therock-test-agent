## 与你场景 (ring timeout → display freeze → GUI crash) 最相关的 Issue

**#5151** — 完全复现 queue eviction cascade + MES REMOVE_QUEUE 失败 + GPU reset 不恢复的完整链条，已开 11 个月仍 open，标签 "Under Investigation"。

注意: #6012 (queue eviction kills display compositor) 和 #5665 (Strix Halo AI+video encoding crash) 虽已 close，但都只有 `status: triage` 标签，无 `resolved/fixed` 标签。关闭原因是问题归属在 amdgpu 内核驱动 / MES 固件层面，非 ROCm 用户态可修。根本解决需要:
- MES 固件更新加入 compute/display 队列优先级调度
- kernel 侧 `amdgpu` 驱动增加防饿死 (starvation prevention) 机制
- 或硬件层面加入 GPU 硬件分区/虚拟化隔离
