- **URL:** https://github.com/ROCm/ROCm/issues/1309

System specs:

- Ubuntu: 20.04.1
- ROCm: 3.5.1
- GPU: Radeon RX 570 8 Gb
- CPU: AMD Ryzen 5 1600X Six-Core Processor
- TensorFlow version: tensorflow-rocm==2.2.0

Hi, I switched recently to ROCm 3.5.1 due to Radeon RX 570 being incompatible with ROCm 3.9 (#1265 , #1269 ,  [#2187](https://github.com/ROCm-Developer-Tools/HIP/issues/2187)). 

The thing is when I run a CNN benchmark, bandwidth seems to be very low. The test I run is the following: `python3 tf_cnn_benchmarks.py --num_gpus=1 --batch_size=8 --model=resnet50` from [tf_cnn_benchmarks.py](https://github.com/tensorflow/benchmarks/blob/master/scripts/tf_cnn_benchmarks/tf_cnn_benchmarks.py). The full log message is as follows: 
[rocm_benchmark.txt](https://github.com/RadeonOpenCompute/ROCm/files/5615294/rocm_benchmark.txt)
As you can see in the 13th line, `coreClock: 1.268GHz coreCount: 32 deviceMemorySize: 8.00GiB deviceMemoryBandwidth: -1B/s`, bandwidth is ridiculously small.
