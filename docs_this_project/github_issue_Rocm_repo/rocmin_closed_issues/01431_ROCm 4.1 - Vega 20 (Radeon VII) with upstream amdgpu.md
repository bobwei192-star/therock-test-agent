# ROCm 4.1 - Vega 20 (Radeon VII) with upstream amdgpu

- **Issue #:** 1431
- **State:** closed
- **Created:** 2021-03-26T22:54:30Z
- **Updated:** 2021-05-13T21:00:33Z
- **URL:** https://github.com/ROCm/ROCm/issues/1431

As documentation of ROCm 4.1 release mentions Vega20 based devices seem to be temporarily disabled when upstream AMDGPU kernel module is used. As expected, after upgrading ROCm 4.0 to 4.1 `rocminfo` reports:

> HSA Error:  Incompatible kernel and userspace, Vega 20 [Radeon VII] disabled. Upgrade amdgpu.

(OpenSUSE Tumbleweed with kernel 5.11.6-1)

What version of kernel/amdgpu is required to re-enable Vega20 with ROCm 4.1?
