 **Details**
 In addition to main, there are two functions that perform OpenMP offloading.

- whole:
  - maps 3 arrays statically declared in the global scope by their names alone
  - computes using these three arrays
- section:
  - maps 3 arrays statically declared in the global scope using either a “to” mapping or a “tofrom” mapping of the form map(to: x[0:nn-1], y[0:nn-1]), map(tofrom: z[0:nn-1])
  - computes using these three arrays

whole runs fine. section crashes.  It is up to the OpenMP compiler and runtime team to figure out what goes wrong with the mappings.
