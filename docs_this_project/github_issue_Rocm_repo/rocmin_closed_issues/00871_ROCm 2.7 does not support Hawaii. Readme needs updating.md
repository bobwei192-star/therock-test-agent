# ROCm 2.7 does not support Hawaii. Readme needs updating

- **Issue #:** 871
- **State:** closed
- **Created:** 2019-08-20T16:57:17Z
- **Updated:** 2021-11-23T04:58:54Z
- **URL:** https://github.com/ROCm/ROCm/issues/871

Support for Hawaii (gfx700/1) has been broken since ROCm 2.0 (#691 and also #640) which was released 8 months ago. The original issue appeared to be with firmware files (which was fixed) but the later issue (execution failure) has not been fixed since then and nothing has changed (#784).

At this point it really doesn't look like this is going to get fixed. Both OpenCL and HIP/HCC programs hang on execution and so ROCm is currently of no use on Hawaii GPUs.

I only revisited Hawaii today because I was going to make a tutorial showing installation and operation of ROCm/HIP. With this is mind it seems to be unhelpful and misleading to list Hawaii as a supported GPU ([even if "support is not guaranteed"](https://github.com/RadeonOpenCompute/ROCm#supported-gpus)).
