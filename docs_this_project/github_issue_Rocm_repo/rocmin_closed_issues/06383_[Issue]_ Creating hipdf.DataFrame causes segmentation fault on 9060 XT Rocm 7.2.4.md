### Problem Description

Hi,
I have been trying to use hipdf with rocm on my gpu to accelerate some data analysis.
However, even creating a simple hip.df dataframe crashes the python kernel with a segmentation fault. I'm running the following code:
`import os
import sys
os.environ["HSA_OVERRIDE_GFX_VERSION"] = "12.0.0" 
os.environ["HSA_XNACK"] = "1"
os.environ['HSA_ENABLE_SDMA']='1' 
os.environ['AMD_SERIALIZE_KERNEL']='3'
import hipdf

df = hipdf.DataFrame({
    'A': [1, 2, 3],
    'B': [4, 5, 6]
})
print(df.head())`

Which throws the following error:
`20:55:19.186 [error] Disposing session as kernel process died ExitCode: undefined, Reason: [IPKernelApp] WARNING | Kernel is running over TCP without encryption. All communication (including code and outputs) is sent in plain text and is susceptible to eavesdropping. Use IPC transport or launch with kernel manager-provisioned CurveZMQ keys to enable transport encryption.
Fatal Python error: Segmentation fault`

I don't know what I'm doing wrong in my setup or installation.
