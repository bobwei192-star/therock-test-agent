# ROCm 5.4.3: `FW bug: No PASID in KFD interrupt` on gfx7 Hawaii

- **Issue #:** 2032
- **State:** closed
- **Created:** 2023-04-10T01:58:00Z
- **Updated:** 2024-06-19T15:42:19Z
- **URL:** https://github.com/ROCm/ROCm/issues/2032

Environment: Ubuntu `22.10` with linux kernel `5.15.0` and amdgpu-pro `22.40-1538781.22.04` and rocm `5.4.3`.

Kernel command line options:

```
# tr ' ' '\n' < /proc/cmdline | grep amdgpu
amdgpu.si_support=1
amdgpu.cik_support=1
amdgpu.exp_hw_support=1
amdgpu.gpu_recovery=1
amdgpu.ppfeaturemask=0xffffffff
```

As described [there](https://github.com/RadeonOpenCompute/ROCm/issues/1624#issuecomment-1501290112), both `rocminfo` and `clinfo --list` now lists the gfx7 device, but complete `clinfo` hangs, and I get an error printed in `dmesg`, and if I interrupt `clinfo` with `Ctrl+C` I get more error logs in `dmesg`. At the moment I interrupt `clinfo` I get a screen freeze for a short time.

The fact OpenCL applications will hang is a problem.

@fxkamd does that other error mean something to you?
