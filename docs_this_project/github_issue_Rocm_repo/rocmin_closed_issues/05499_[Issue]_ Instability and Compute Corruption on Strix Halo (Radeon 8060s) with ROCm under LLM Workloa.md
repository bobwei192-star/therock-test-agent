# [Issue]: Instability and Compute Corruption on Strix Halo (Radeon 8060s) with ROCm under LLM Workload (Ollama) Environment

### Problem Description

When running Large Language Model (LLM) inference using Ollama on a new AMD Strix Halo platform, the ROCm compute stack appears to become unstable after a short period of use.

Initially, the models perform correctly. However, after 4-5 conversational turns, the model's output becomes corrupted and degenerates into a repetitive, single-character string (e.g., "GGGGGGGGGGGGGGGGGGGGGGGGGGGGGGGGGG").

This issue is not model-specific. It has been reproduced with multiple, unrelated models, including llama3:8b, phi3, deepseek-r1:1.5b, and deepseek-r1:70b. This strongly indicates the root cause is not the model but the underlying compute platform.

The failure seems to be triggered as the conversational context grows, placing a sustained computational and memory load on the GPU.


Crucial Diagnostic Finding: Isolation to GPU

The issue has been definitively isolated to the ROCm GPU stack. By forcing Ollama to run in a CPU-only mode, the instability completely disappears. The application runs slowly, as expected, but remains perfectly stable even during very long conversations that exceed the model's context window.

This proves that the bug is not within the Ollama application itself but is a result of a fault in the GPU compute process.
Logs and Evidence

I am attaching three log files that provide definitive proof of this behavior:

    ollama-gpu-failure.log: An Ollama debug log showing a typical session where the GPU is used and the model fails, producing the "GGGG..." output.

    ollama-cpu-success.log: An Ollama debug log from the CPU-only test. This log shows the application correctly falling back to the CPU and running stably for a long session.

    chat-log-failure-example.txt: An example of the user-facing chat session that leads to the failure.


Request

Please investigate potential instabilities or bugs in the ROCm driver for the new Radeon 8060s GPU on the Strix Halo platform when placed under sustained, memory-intensive compute workloads like those generated by LLM inference.

### Steps to Reproduce

Steps to Reproduce

    Install Ollama v0.12.3 on a clean AMD Strix Halo system with a Radeon 8060s GPU.

    Download any standard language model (e.g., ollama pull llama3:8b).

    Run the model in interactive mode (ollama run llama3:8b).

    Engage in a multi-turn conversation. Ask an initial question, then 3-4 follow-up questions that build on the previous context.

    After the 4th or 5th prompt, the model's output will become a corrupted, repeating string of characters.
