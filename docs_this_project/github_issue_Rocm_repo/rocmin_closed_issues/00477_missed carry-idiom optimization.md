# missed carry-idiom optimization

- **Issue #:** 477
- **State:** closed
- **Created:** 2018-07-29T15:24:14Z
- **Updated:** 2023-12-12T21:49:25Z
- **Labels:** Under Investigation, Compiler Performance Issue
- **URL:** https://github.com/ROCm/ROCm/issues/477

I use this idiom in OpenCL to express the carry resulting from a V_MAD_U64_U32:
uint a;
ulong b;
ulong x = a * (ulong) a + b;
uint carry = (x < b);

Yet ROC 1.9.2 generates this ISA:
	v_mad_u64_u32 v[6:7], s[0:1], v3, v3, v[8:9]
	v_cmp_lt_u64_e32 vcc, v[6:7], v[8:9]

What we see here is that the compiler discards the carry in the v_mad, just to compute it again below.

Expected behavior: the compiler understands the carry idiom, and maps it to the carry generated by the MAD.
