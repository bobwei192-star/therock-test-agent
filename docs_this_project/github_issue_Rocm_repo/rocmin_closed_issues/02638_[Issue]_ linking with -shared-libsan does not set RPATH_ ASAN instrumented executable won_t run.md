# [Issue]: linking with -shared-libsan does not set RPATH, ASAN instrumented executable won't run

- **Issue #:** 2638
- **State:** closed
- **Created:** 2023-11-11T18:08:30Z
- **Updated:** 2023-11-12T02:59:49Z
- **URL:** https://github.com/ROCm/ROCm/issues/2638

### Problem Description

I have build my HIP code following https://rocm.docs.amd.com/en/latest/understand/using_gpu_sanitizer.html#compiling-for-address-sanitizer.

It does not include an RPATH for libclang_rt.asan-x86_64.so. Running `ldd` shows:

```
        libclang_rt.asan-x86_64.so => not found
```

This shared library is not present in 
```
$ ls /opt/rocm-5.7.0/llvm/lib/libclang_rt*.so
ls: cannot access '/opt/rocm-5.7.0/llvm/lib/libclang_rt*.so': No such file or directory
```
so adding this path to `LD_LIBRARY_PATH` does not help.

Therefore, the code won't run, since it can't find the asan shared library:
```
./src/HydroBlast3D/test_hydro3d_blast: error while loading shared libraries: libclang_rt.asan-x86_64.so: cannot open shared object file: No such file or directory
```
