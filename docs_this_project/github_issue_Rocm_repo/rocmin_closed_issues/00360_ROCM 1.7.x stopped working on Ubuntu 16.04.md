# ROCM 1.7.x stopped working on Ubuntu 16.04

- **Issue #:** 360
- **State:** closed
- **Created:** 2018-03-14T12:08:39Z
- **Updated:** 2018-03-17T13:37:44Z
- **URL:** https://github.com/ROCm/ROCm/issues/360

There are no render-devices for AMD GPUs in /dev/dri/ when using 112 or 116 of the 4.4.0 kernel. Tried ROCm 1.7.1 and 1.7.0. 

Upgrading to 4.13 solves the problem. When using 1.7.1 with docker on kernel 4.13, there is another problem: https://github.com/RadeonOpenCompute/ROCm-docker/issues/36

As many people are still on 16.04 with kernel 4.4.0, I think this should be fixed.
