# ROCm 5.2, OpenCL: wrong codegen with 24bit mul

- **Issue #:** 1764
- **State:** closed
- **Created:** 2022-07-05T11:42:41Z
- **Updated:** 2022-07-07T10:20:39Z
- **URL:** https://github.com/ROCm/ROCm/issues/1764

Using ROCm 5.2, RadeonVII, Ubuntu 21.10, Linux kernel 5.18.8.
For this OpenCL kernel:
```
kernel void testKernel(global uint *io) {
  uint me = get_local_id(0);
  ulong x = io[me] & 0x00ffffffu;
  io[me] = (x * x) >> 32;
}
```
this ISA is generated:
```
	s_load_dwordx2 s[0:1], s[4:5], 0x0
	v_lshlrev_b32_e32 v0, 2, v0
	v_mov_b32_e32 v2, 0
	s_waitcnt lgkmcnt(0)
	v_mov_b32_e32 v1, s1
	v_add_co_u32_e32 v0, vcc, s0, v0
	v_addc_co_u32_e32 v1, vcc, 0, v1, vcc
	global_store_dword v[0:1], v2, off
	s_endpgm
```
As you see, the generated code disregards the input value, and simply writes 0 (v2). This is not correct, the expected behavior would be to write the high dword of the multiplication, as would be produced by e.g. v_mul_hi_u32_u24
