# ROCm 5.4.3 libstdc++ dependency version too high

- **Issue #:** 2084
- **State:** closed
- **Created:** 2023-04-25T18:41:31Z
- **Updated:** 2024-03-20T22:06:33Z
- **Labels:** Under Investigation
- **URL:** https://github.com/ROCm/ROCm/issues/2084

On SLES 15 SP4, using ROCm 5.4.3 is so difficult compared to 5.4.0 due to extremely tight libstdc++ version requirement.
amdclang and hip libraries both requires at least gcc12 equivalent libstdc++.
When I use gcc 11 as the host compiler and hip runtime library.
```
/soft/compilers/gcc/11.1.0/x86_64-suse-linux/bin/g++ main.cpp /soft/compilers/rocm/rocm-5.4.3/lib/libamdhip64.so.5.4.50403
/usr/bin/ld: /soft/compilers/rocm/rocm-5.4.3/lib/libamd_comgr.so.2: undefined reference to `std::condition_variable::wait(std::unique_lock<std::mutex>&)@GLIBCXX_3.4.30'
collect2: error: ld returned 1 exit status
```
main.cpp
```
#include <memory_resource>
int main()
{}
```
Please minimize the dependency version.

PS: no issue with amdclang due to its using ld.lld as the linker instead of ld and probably no symbol checks.
