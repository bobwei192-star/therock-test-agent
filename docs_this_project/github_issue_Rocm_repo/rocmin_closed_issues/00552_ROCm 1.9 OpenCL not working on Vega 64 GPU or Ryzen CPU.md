# ROCm 1.9 OpenCL not working on Vega 64 GPU or Ryzen CPU

- **Issue #:** 552
- **State:** closed
- **Created:** 2018-09-20T19:24:36Z
- **Updated:** 2018-12-11T19:44:14Z
- **URL:** https://github.com/ROCm/ROCm/issues/552

Hi,
I installed ROCm 1.9 on my Kubuntu 18.04 machine according to the instructions on this git repository, but OpenCL is not working for neither the Vega 64 GPU nor the Ryzen 1800X CPU. I haven't had ROCm installed previously.

When running `clinfo` it shows 0 supported platforms
```
Number of platforms                               0
```

It appears the kfd is not working properly, since it reports my GPU not being supported. This I find strange, since Vega 64 is stated as supported

```
dmesg | grep kfd
[    1.573162] kfd kfd: Initialized module
[    2.228953] amdgpu 0000:09:00.0: kfd not supported on this ASIC
```
The `amdocl64.icd` is present within `/etc/OpenCL/vendors/`, so I assume it has to do with the above output from the kfd module. As noted earlier, it also does not report the CPU as an OpenCL device.
`rocminfo` returns the following error
```
troels@troels-ryzen-mothership:~$ /opt/rocm/bin/rocminfo 
hsa api call failure at line 900, file: /home/jenkins/jenkins-root/workspace/compute-rocm-rel-1.9/rocminfo/rocminfo.cc. Call returned 4104
```
and the bundled `clinfo` returns:
```
terminate called after throwing an instance of 'cl::Error'
  what():  clGetPlatformIDs
Aborted (core dumped)
```
