# Please support +lds flag for vectory memory operations in ROCm

- **Issue #:** 630
- **State:** closed
- **Created:** 2018-12-03T07:36:43Z
- **Updated:** 2023-12-18T18:45:18Z

In section 12.18, the [Vega ISA guide](https://developer.amd.com/wp-content/resource/Vega_Shader_ISA_28July2017.pdf) states that vector memory operations support a flag called `lds` with the following effect:
>Data is transferred between LDS and Memory, not VGPRs.

I am interested in trying this out, but neither is this flag ever generated by the compiler (for example it will issue `global_load` followed by `ds_write` instead of `global_load … lds`) nor accepted when I write inline asm. I am using HIP/hcc.

Thanks!
