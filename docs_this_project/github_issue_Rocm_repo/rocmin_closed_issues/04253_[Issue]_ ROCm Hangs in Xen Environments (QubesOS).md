# [Issue]: ROCm Hangs in Xen Environments (QubesOS)

- **Issue #:** 4253
- **State:** closed
- **Created:** 2025-01-13T21:17:10Z
- **Updated:** 2025-01-14T20:54:32Z
- **Labels:** Under Investigation, ROCm 6.3.0
- **URL:** https://github.com/ROCm/ROCm/issues/4253

### Problem Description

hashcat, pytorch using ROCm inside qube(Xen-Based-VM) cause terminal hangs. so far, I've tested hashcat, openai-whisper, stable-diffusion-webui on QubesOS(fedora-40, debian-12) and also inside docker(rocm/pytorch). results are the same.

and I've tried on those distros in bare-metal setup, ROCm works just fine. not sure if its Xen's problem or ROCm's, the ROCm in Xen is problematic, but CUDA seems works well in Xen Environments. so I believe its related to ROCm.

### Operating System

QubesOS (debian-12-xfce+testing/firmware-amd-graphics, fedora-40-xfce) 

### CPU

Intel Core i7-14700K

### GPU

AMD Radeon RX7800XT

### ROCm Version

ROCm 6.3.0

### ROCm Component

ROCm

### Steps to Reproduce

Runs anything, for example, pytorch tests using ROCm in qube with AMD GPU. and the terminal hangs.
