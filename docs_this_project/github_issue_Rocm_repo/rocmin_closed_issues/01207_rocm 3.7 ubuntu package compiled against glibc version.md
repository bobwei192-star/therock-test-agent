# rocm 3.7 ubuntu package compiled against glibc version

- **Issue #:** 1207
- **State:** closed
- **Created:** 2020-08-27T21:18:29Z
- **Updated:** 2020-09-17T00:18:09Z

I am not sure if this is the right repository to report rocm package build issues.

I updated my rocm3.5 to rocm3.7 on Ubuntu 16.04. When running clang I get the following error:

```
$ which clang
/opt/rocm-3.7.0/llvm/bin/clang
$ clang
clang: /lib/x86_64-linux-gnu/libm.so.6: version `GLIBC_2.27' not found (required by clang)
```

It looks like clang is build against the wrong glibc version because `libm` on Ubuntu 16.04 is not supporting GLIBC 2.27

```
$ strings /lib/x86_64-linux-gnu/libm.so.6 | grep GLIBC
GLIBC_2.2.5
GLIBC_2.4
GLIBC_2.15
GLIBC_2.18
GLIBC_2.23
GLIBC_PRIVATE
```

System:

```
$ cat /etc/lsb-release 
DISTRIB_ID=Ubuntu
DISTRIB_RELEASE=16.04
DISTRIB_CODENAME=xenial
DISTRIB_DESCRIPTION="Ubuntu 16.04.7 LTS"
```
