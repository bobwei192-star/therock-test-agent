# rdc.h doesn't work on gcc 4 or gcc 8 with rhel7

- **Issue #:** 1641
- **State:** closed
- **Created:** 2021-12-16T20:08:45Z
- **Updated:** 2023-05-24T15:43:49Z
- **URL:** https://github.com/ROCm/ROCm/issues/1641

The definition of a const MAX_TEST_CASES in rdc.h causes gcc to fail with the error below.
One solution is illustrated by:
```
sed -i -e 's/^const uint32_t MAX_TEST_CASES =  RDC_DIAG_TEST_LAST - RDC_DIAG_TEST_FIRST + 1;/#define MAX_TEST_CASES (RDC_DIAG_TEST_LAST - RDC_DIAG_TEST_FIRST + 1)/' $(RDC_INC)/rdc/rdc.h
```

the error is:
```
/opt/rocm-4.5.0/rdc/include/rdc/rdc.h:479:28: error: variably modified 'diag_info' at file scope
     rdc_diag_test_result_t diag_info[MAX_TEST_CASES];
                            ^
```

Whether or not this is an excessive error detection by gcc I cannot say, but on my production hpc clusters gcc is what we compile with.
