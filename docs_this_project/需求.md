打造cli 级别的终端测试agent (loop engineering工程)


定义：测试任务：

usecase: 用户打开therock-test-agent终端工具：

输入：
     TheRock repo地址 （如果不填则智能体会自己下载）
     The rocm artifacts path（使用者必须填写）
     gpu model （使用者必须填写）
     The rock组件列表（（如果不填则智能体会默认所有组件都会测试）


自串行测试 The rock组件列表中的每个组件测试用例 


第一步 定义全局状态文件：
global_state.json


第二步： 排序  从quick standard comprehessive full 递增排序
将输入的组件名称按照docs_this_project/component_sort_order.json 里定义好的顺序进行排队 如果
如果用户传入的组件名为空列表 则 默认直接使用 docs_this_project/component_sort_order.json

  更新全局状态文件 global_state.json

第二步骤： 创建结果输出目录 更新全局状态文件 global_state.json

第三步骤：开始测试循环
一个一个的测试测试任务 每个都测试 更新全局状态文件 global_state.json
loop 需求描述  /docs_this_project/loop_engineering.md

并 输出测试结果 更新全局状态文件 global_state.json
测试结果定义：
1 汇总测试报告
2 测试log
3 单测试组建测试失败的详细报告
