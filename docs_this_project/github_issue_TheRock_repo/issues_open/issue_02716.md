We presently run a smaller subset of architectures for release branch Cherry picks. Running pre-submit CI for cherrypicks on all architectures supported for the release as part of the BOM would provide comprehensive coverage and avoid regressions before merging to release branch. This issue is raised to match the CI architecture runs based on the BOM file for releases.
