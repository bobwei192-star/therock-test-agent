Overarching feature ticket to increase usability of the test logs generated by TheRock CI. Work will be separated into separate, smaller pull requests.

- [ ] Similar to other fetch scripts in the repo, fetch the raw test logs from a given CI run ID. 
  - must be within github log retention period
  - a run can involve multiple gpu architectures, OS, and ROCm subprojects
  - organize the logs into a sensible directory structure to accommodate above matrix
- [ ] Parse and concatenate the test data from the logs into a unified file.
  - test logs will contain extra text
  - only capture important details and concatenate into a unified file, perhaps a .csv
  - sort the data within the unified file 
- [ ] Tool to compare unified files between one or more runs
  - allows for quick detection of trends such as recurring failures or long tests
  - potentially repurposed for uploading the data to a backend
