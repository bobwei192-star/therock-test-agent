Follows up on https://github.com/ROCm/TheRock/issues/2069 and https://github.com/ROCm/TheRock/issues/1761

Previous issues with `choco` were a result of being rate limited by choco community package servers, and having the internal package cache server, Nexus become inaccessible, which originally was the mitigation for rate limiting.

There has been some discussions about how to increase the reliability of the runner setup process such as:
- Add more of the choco packages to the Windows runner docker image.
- Use winget
- add community servers as a fallback to the internal package server
- Ensure internal package server is running (may require additional discussion, configuration and or management by OSSCI team) and understand why it was stopped as mentioned in issue 2069
- consolidate invocations of `choco` package installs (which has lead to a 503 error that caused the docker image build to erroneously successfully build. This in particular requires additional safeguards in place before an image should be used in production)
