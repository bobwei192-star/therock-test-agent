To help with the transition of running HIP on Windows via ROCR, there will be a period where hip-tests for Windows will run once through PAL and once through ROCR. This is controlled through the environment variable `GPU_ENABLE_PAL` being set to 1 or 0.

First step: Have hip-tests on Windows run with the flag set to 1, and then run again with the flag set to 0. This can be separated as two jobs. The PAL result can be used for pass/fail, while the ROCR result can be informational.

Once parity is achieved, the version used for the pass/fail signal can be reversed.
