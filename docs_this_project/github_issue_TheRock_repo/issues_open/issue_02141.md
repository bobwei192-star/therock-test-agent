The `build_linux_jax_wheels.yml`, triggered in `release_portable_linux_packages.yml`, gets a single Python version passed as an input

https://github.com/ROCm/TheRock/blob/aae9d1c6ed01cd441c0337ac16a501b8581afb81/.github/workflows/release_portable_linux_packages.yml#L309-L317

While `jax-rocm7-pjrt` is not build for a specific Python version, `jaxlib` and `jax-rocm7-plugin` are and thus are only available for Python 3.12. Instead of triggering `build_linux_jax_wheels.yml` for a single Python version, the triggered workflow needs to span a matrix in order to build `jaxlib` and `jax-rocm7-plugin` for Python 3.11, 3.12 and 3.13. The `jaxlib` is ideally excluded as it will otherwise get build multiple times with the same name and will be overwritten.

`release_portable_linux_pytorch_wheels.yml` can provide some guidance on how to span the matrix and is itself triggered in `release_portable_linux_packages.yml`.
