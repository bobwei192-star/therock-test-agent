### Summary
We've implemented a temporary fix by adding Google DNS servers (8.8.8.8, 8.8.4.4) to our Docker container configuration in the JAX wheel testing workflow. This resolved connectivity issues when downloading packages from CloudFront endpoints, but we need to understand the root cause and implement a proper solution.

### Current Situation
- **Temporary Fix Applied**: Added `--dns 8.8.8.8 --dns 8.8.4.4` to Docker container options
- **PR Reference**: [https://github.com/ROCm/TheRock/pull/2994]
- **Impact**: JAX wheel testing workflow was failing due to DNS resolution issues

### Problem Description
The GitHub Actions runner's Docker containers were unable to resolve CloudFront URLs for package downloads without explicit DNS configuration. This suggests either:
1. Default DNS configuration issues in the GitHub Actions runner environment
2. Docker daemon DNS configuration problems
3. Network policy or firewall restrictions affecting DNS queries

### Investigation Tasks
- [ ] Investigate Docker's DNS resolution mechanism in GitHub Actions environment
- [ ] Review any recent changes to infrastructure or our workflow configuration
- [ ] Test alternative DNS solutions (resolv.conf)
- [ ] requires permanent configuration
