## Summary

Two `hip-tests` shards failed on `gfx1151` with reproducible test assertion failures.
Both jobs were retried once (`RETRY_COUNT=1`) and the same tests failed on retry,
confirming these are **not flakes**.

> **Note:** The GitHub UI annotation `"Executing the custom container implementation failed"`
> is **misleading** — it fires whenever the test step exits with code ≠ 0. The root cause
> is test assertion failures, **not** runner/infra issues.
