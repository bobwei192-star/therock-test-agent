### Problem Description

The runners that get to run ROCgdb tests need to be configured properly to have the correct core file pattern. The core pattern is usually configured to redirect to a distro-specific bug-reporting tool.

In the case of Ubuntu this is apport:

2026-04-17 02:38:49,217 - INFO: Current kernel.core_pattern: |/usr/share/apport/apport -p%p -s%s -c%c -d%d -P%P -u%u -g%g -- %E
2026-04-17 02:38:49,217 - WARNING: Failed to configure kernel.core_pattern: [Errno 30] Read-only file system: '/proc/sys/kernel/core_pattern'

Given this is a host-wide setting, the ROCgdb test job cannot configure this setting from within the docker container. Therefore we need to adjust it for every host that gets used to run testing. Ideally this should be done by default at startup or changed permanently.

This should not impact any other tool as it is mainly the location where the core file will be put into.

The core file pattern should be set to /proc/%p/cwd/core.%p by default, which mean the core file will be put into the current directory of where the command ran.
