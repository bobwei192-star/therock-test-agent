## Summary

9 tests fail in the PyTorch test suite. The primary structural failure is `test_cuda.py::TestMemPool::test_mempool_empty_cache_inactive`, which calls `torch.utils.cpp_extension.load_inline` to JIT-compile a `dummy_allocator` C++ extension at test runtime. Compilation fails because `ATen/hip/Exceptions.h` (line 5, shipped inside the installed wheel) unconditionally includes `<hipblas/hipblas.h>`, but the hipBLAS development headers are not available in the test container (`ghcr.io/rocm/no_rocm_image_ubuntu24_04`).

Five additional test failures (`test_memory_compile_regions`, `test_memory_plots`, `test_memory_plots_free_segment_stack`, `test_memory_snapshot`, `test_memory_snapshot_script`, `test_memory_snapshot_with_cpp`) likely share the same root cause (they all attempt C++ extension JIT compilation that hits the same missing header).

`test_no_triton_on_import` and `test_layer_norm_backwards_eps` are separate failures (see below).
