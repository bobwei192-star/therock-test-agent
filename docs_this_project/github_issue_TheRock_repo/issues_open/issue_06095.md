### Problem Description

The latest numpy release (2.5.0, released Jun 21) changed the behaviour of `threshold` function, causing pytorch UT to fail: https://github.com/ROCm/rockrel/actions/runs/28071131699/job/83114331597

This isn't a problem in upstream CI jobs because pytorch pins numpy version, but the Rock CI doesn't. Relevant files:
https://github.com/ROCm/TheRock/blob/main/external-builds/pytorch/requirements-test.txt doesn't pin numpy version

https://github.com/ROCm/TheRock/blob/main/.github/workflows/test_pytorch_wheels.yml#L214-L217 and https://github.com/ROCm/TheRock/blob/main/.github/workflows/test_pytorch_wheels_full.yml#L333-L337 install dependencies
