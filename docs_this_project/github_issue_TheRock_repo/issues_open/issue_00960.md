# #960 - [python] Don't accept kwargs for initialize_process

In `rocm_sdk.initialize_process`, I opted to accept **kwargs in a not fully thought through attempt to enable forward compatibility. This has now bitten me three times with slightly misspelled arg names being silently swallowed.

The more proper way to do this is the good ol' `undef_ok` arg list for arguments that might be added by a future version and need to be consumed by an old one. If a kwarg is in that list as passed in, then it is ignored. Otherwise, it is an error.
