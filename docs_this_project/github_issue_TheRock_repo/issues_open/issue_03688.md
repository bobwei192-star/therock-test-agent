## Problem

[`build_tools/build_python_packages.py`](https://github.com/ROCm/TheRock/blob/main/build_tools/build_python_packages.py#L88-L136) has filter functions with hardcoded artifact name lists that must match the CMake-defined target names. When a CMake target is renamed, the Python filter silently drops it — no error, the artifact just disappears from the wheel.

This happened with `miopen-plugin` → `miopenprovider` and `hipblaslt-plugin` → `hipblasltprovider` (fixed in #3687).

## Possible approaches

### 1. C++ consumption test against packaged wheels

Add a small C++ app that builds and links against the installed wheel/tarball contents. This verifies that all expected libraries are present, compile, and link. For the provider plugins, we can use the backend API to query loaded plugins and verify they're available.

This would fit naturally into the CI workflows that already build wheels (e.g., [`build_portable_linux_python_packages.yml`](https://github.com/ROCm/TheRock/blob/main/.github/workflows/build_portable_linux_python_packages.yml)) — install the wheel, then build and run the consumption test against it.

### 2. Post-build artifact directory check

In CI workflows that produce artifacts, add a step that lists the actual artifact directories and asserts every name is accepted by exactly one filter (core, libraries, or devel catch-all). Most reliable for detecting filter drift, but only runs in build workflows.

### 3. Shared source of truth

Extract the name lists into a config file (TOML/JSON) that both the CMake and Python sides read. Eliminates drift by construction, but requires changes to both systems.

## Context

- Filter functions: [`core_artifact_filter`](https://github.com/ROCm/TheRock/blob/main/build_tools/build_python_packages.py#L88), [`libraries_artifact_filter`](https://github.com/ROCm/TheRock/blob/main/build_tools/build_python_packages.py#L117)
- Artifact naming convention: `{name}_{component}_{target_family}` (see [`_therock_utils/artifacts.py`](https://github.com/ROCm/TheRock/blob/main/build_tools/_therock_utils/artifacts.py))
