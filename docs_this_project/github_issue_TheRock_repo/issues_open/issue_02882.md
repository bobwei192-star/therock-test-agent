### Problem Description

Hello, when using nightly builds, the performance of matrix multiplication (f32 precision) seems slower than expected in PyTorch.

This is a benchmark of matmul with size 1024 (dtype float32).

Result:
```
torch.matmul(A,B)
  Median: 2.43 ms
  IQR:    0.01 ms (2.42 to 2.43)
  5 measurements, 100 runs per measurement, 1 thread
```

with `export HSA_OVERRIDE_GFX_VERSION=11.0.0`
```
torch.matmul(A,B)
  Median: 697.77 us
  IQR:    2.37 us (696.82 to 699.19)
  137 measurements, 10 runs per measurement, 1 thread
```

For size 512 or 4096, it is also faster with `HSA_OVERRIDE_GFX_VERSION=11.0.0`
When using types float16 or bfloat16 performance seems correct.

### Steps to Reproduce

Steps to reproduce : 
```bash
uv venv testrocm
source testrocm/bin/activate
uv pip install --index-url https://rocm.nightlies.amd.com/v2/gfx110X-all/ 'rocm[libraries]' torch
```
Then run
```python
import torch
import torch.utils.benchmark as benchmark
for N in (256, 512, 1024, 2048, 4096):
    A = torch.randn((N, N), dtype=torch.float, device="cuda")
    B = torch.randn((N, N), dtype=torch.float, device="cuda")
    t = benchmark.Timer(stmt='torch.matmul(A,B)', globals=globals())
    print(f"{N = }")
    t.blocked_autorange(min_run_time=1.0)
    print(t.blocked_autorange(min_run_time=2.0))
```
