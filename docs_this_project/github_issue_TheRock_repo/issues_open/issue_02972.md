# #2972 - Avoid auto-patching and branch switching for TheRock components (development workflow)

This week I've been doing some work using TheRock as part of my development flow, and I've encountered two behaviours that, while useful in the CI case, seem to me to be counterproductive in the development case.

The first is the automatic patching of components during builds. This obviously interferes with any patches that developers might be working on themselves, and complicates preparing patches for pull requests, etc.. Is there a way that the patching can be avoided?

The second is the tendency for the build to automatically switch branches during the build process: again, this makes it hard to test code modifications, and it would be good if there was a way to avoid it.

Ideally, there should be a mode where source code is considered immutable and a build can complete without touching the network.
