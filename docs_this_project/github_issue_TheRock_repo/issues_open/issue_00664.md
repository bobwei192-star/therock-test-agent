# #664 - [Windows] Explore ways to avoid git long path errors

**State**: open  
**Author**: ScottTodd  
**Created**: 2025-05-20T00:38:53Z  
**Updated**: 2025-06-23T23:10:09Z  
**URL**: https://github.com/ROCm/TheRock/issues/664  

---

Right now if a developer clones / runs `fetch_sources.py` without git long path support, some files may just be missing from the source tree, resulting in cryptic build errors (usually in the math libraries that have long file names).

Some ideas:

* Have `fetch_sources.py` check that long path support is enabled
* Have `fetch_sources.py` enable long path support in the submodules automatically (if possible)
* Scan for known long paths and ensure that they exist
* Add some configure or build time checks in CMake. I'd prefer to handle it at source checkout time rather than take the overhead hit for every configure/build.
