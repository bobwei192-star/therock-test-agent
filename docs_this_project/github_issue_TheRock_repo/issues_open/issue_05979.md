## Summary

  The "Run PyTorch smoketests" step in the `multi_arch_release_linux_pytorch_wheels.yml` workflow crashes immediately on `linux-gfx90a-gpu-rocm` runners before any tests execute. The HIP runtime issues a fatal `F`-level abort in `agent.cpp:245` because the runner's GPU isolation env-file sets both `HIP_VISIBLE_DEVICES=0` and `GPU_DEVICE_ORDINAL` to a different ordinal (1, 2, or 4 depending on the runner) simultaneously —conflicting values that the HIP runtime treats as a fatal error.

  ## Suspected Root Cause

  The container is started (from `test_pytorch_wheels.yml`) with the runner's env-file (`/etc/podinfo/gha-gpu-isolation-settings`) injecting both:

  ```
  HIP_VISIBLE_DEVICES=0
  GPU_DEVICE_ORDINAL=1   # varies per runner: 1, 2, or 4
  ROCR_VISIBLE_DEVICES=1 # matches GPU_DEVICE_ORDINAL, not HIP_VISIBLE_DEVICES
  ```

  These two variables refer to different GPU ordinals, and the HIP runtime's `agent.cpp:245` detects the conflict and issues a fatal abort. The `rocSHMEM libnuma` message is an unrelated warning on stderr (tracked in #5459); the actual killer is the `F0619 … agent.cpp:245` line.

  The code path is: `run_pytorch_smoke_tests.py` → `get_unique_supported_devices()` → `get_supported_and_visible_gpus()` → `subprocess.run([python,-c, "import torch; torch.cuda.device_count()…"])` → HIP fatal crash → `CalledProcessError` → `sys.exit(1)`.
