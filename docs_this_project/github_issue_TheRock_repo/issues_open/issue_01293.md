### Suggestion Description

Recently there was an ask from a customer to enable the Windows [Control Flow Guard](https://learn.microsoft.com/en-us/windows/win32/secbp/control-flow-guard) across the compiler and runtime components -- when building the compute stack on Windows.

We have verified that enabling CFG leads to minimal performance impact and have now enabled it by default in the internal builds. TheRock should either adopt the same or provide a command line option to enable Windows CFG when building the compiler and runtime artifacts (until the impact can be verified).

Possible `CMakeLists.txt` modifications :

```
set(CMAKE_C_FLAGS "${CMAKE_C_FLAGS} /guard:cf")
set(CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS} /guard:cf")
set(CMAKE_EXE_LINKER_FLAGS "${CMAKE_EXE_LINKER_FLAGS} /guard:cf /DYNAMICBASE")
```
