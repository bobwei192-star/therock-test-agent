# #387 - False dependency from rocSOLVER/rocSPARSE -> rocPRIM tests causes 10m+ per target arch critical build path regression.

**State**: open  
**Author**: stellaraccident  
**Created**: 2025-04-10T01:34:22Z  
**Updated**: 2025-04-10T01:34:22Z  
**URL**: https://github.com/ROCm/TheRock/issues/387  

---

Both rocSOLVER/rocSPARSE are in the library build critical path and take a non-trivial build time themselves. They both depend on rocPRIM, a header-only C++ template library. As a header-only library, this should have close to zero build time (just the cost of configure and moving some files around), but because we build tests as part of it, it costs 10+m per bundled target arch, and that goes straight to the overall critical path.

We should split the PRIM build to do the library build first followed by the build with tests enabled and reconfigure the PRIM artifact to pull from this kind of split build.
