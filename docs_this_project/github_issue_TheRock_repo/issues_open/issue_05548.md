## Summary

Two tests in `tests/test_rocm_sanity.py` fail on every ASAN nightly run since 2026-05-30.

### Failure 1 — `test_hip_printf` (skip guard broken)

The test has `@pytest.mark.skipif(is_asan(), reason="hipcc test fails with ASAN build, see TheRock#3313")` but the skip never fires. `is_asan()` checks `"asan" in ARTIFACT_GROUP`, but in `test_component.yml` the `ARTIFACT_GROUP` env var is set to `inputs.artifact_group = "gfx94X-dcgpu"` (from `matrix.family_info.amdgpu_family` in `multi_arch_ci_linux.yml`), which does not contain `"asan"`. The underlying binary fails at runtime:

```text
==1005==ASan runtime does not come first in initial library list; you should either link runtime to your application or manually preload it with LD_PRELOAD.
```

Root cause: `multi_arch_ci_linux.yml` L137 sets `artifact_group: ${{ matrix.family_info.amdgpu_family }}` for per-family test jobs instead of `${{ fromJSON(inputs.build_config).artifact_group }}` which would be `multi-arch-asan`. A TODO at `test_component.yml` L120–121 acknowledges the same issue for the setup step.

### Failure 2 — `test_amdsmi_suite` (real ASAN heap-use-after-free)

The `amdsmitst` binary (from `rocm-systems` submodule at `4a30df8`) crashes inside `amdsmitstReadOnly` with an ASAN heap-use-after-free:

```text
==1007==ERROR: AddressSanitizer: heap-use-after-free on address 0x7c205d212f08 at pc 0x5eb8073275e0
READ of size 8 at 0x7c205d212f08 thread T0
    #0 0x5eb8073275df  (amdsmitst+0xcdf5df)
    #1 0x5eb806d0580d  (amdsmitst+0x6bd80d)
    ...
freed by thread T0 here:
    #0  (libclang_rt.asan-x86_64.so+0x162952)
    #1 0x5eb807136851  (amdsmitst+0xaee851)
    ...
SUMMARY: AddressSanitizer: heap-use-after-free (amdsmitst+0xcdf5df)
```

Two additional Google Test subtests also fail independently:
- `amdsmitstReadOnly.TestSysInfoRead` — `amdsmi_get_gpu_vbios_info(nullptr)` returned `AMDSMI_STATUS_INVAL (1)`, expected `AMDSMI_STATUS_NOT_SUPPORTED (2)` (`sys_info_read.cc:96`)
- `amdsmitstReadOnly.TestIdInfoRead` — `amdsmi_get_gpu_asic_info` returned `AMDSMI_STATUS_NOT_SUPPORTED (2)`, expected `AMDSMI_STATUS_SUCCESS (0)` (`id_info_read.cc:119`)

## Suggested Fixes

**Fix 1 (test_hip_printf):** Thread the ASAN build variant through to the per-family test `artifact_group` input in `multi_arch_ci_linux.yml`, OR add a `BUILD_VARIANT` env var to `test_component.yml` and update `is_asan()` to also check it. Tracked in `TODO(#3313)`.

**Fix 2 (test_amdsmi_suite):** File a bug against `ROCm/rocm-systems` for the heap-use-after-free in `amdsmitst`. Short-term workaround: add `amdsmitstReadOnly.TestSysInfoRead` and `amdsmitstReadOnly.TestIdInfoRead` to `TESTS_TO_IGNORE` in `tests/test_rocm_sanity.py` for ASAN runs.
