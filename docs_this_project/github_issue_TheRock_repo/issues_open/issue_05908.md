### Description of errors

I noticed the naming scheme in our GPU index tables are inconsistent between multi-arch and per-family arch sections.

In the multi-arch section, GFX1103 is listed as "Ryzen 7 7840U / Ryzen 9 270" but in the per-family section, it is listed as "AMD Radeon 780M Laptop iGPU"

Similarly, GFX1102 is listed as "AMD Radeon RX 7600" in mulit-arch but listed as "AMD RX 7700S / Framework Laptop 16" in per-family

This makes it unclear whether the table is intended to associate architectures with APUs or with GPU products, or both.

The list is also missing the rest of the products that would fall under the GFX1102/1103 families. Take GFX1103 for example, it would include:

Phoenix (Ryzen 7040 Series)
- Ryzen 5 7640U / 7640HS → Radeon 760M
- Ryzen 7 7840U / 7840HS → Radeon 780M
- Ryzen 9 7940HS → Radeon 780M

Hawk Point (Ryzen 8040 Series)
- Ryzen 5 8640U / 8640HS → Radeon 760M
- Ryzen 7 8840U / 8840HS → Radeon 780M
- Ryzen 9 8945HS → Radeon 780M

Phoenix Desktop APUs (8000G Series)
- Ryzen 5 8600G → Radeon 760M
- Ryzen 7 8700G → Radeon 780M

Another issue caused as a byproduct of this inconsistency is the install experience. When I looked through the RELEASES.md file I CTRL+F to search for "780M" and installed the per-family ROCm wheels for GFX1103 instead of the multi-arch ones. The per-family wheels gave me a segmentation fault when running rocminfo whereas the multi-arch ones didn't.
