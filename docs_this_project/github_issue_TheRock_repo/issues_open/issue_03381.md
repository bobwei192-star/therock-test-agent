During `fetch_artifacts.py` script calls, `artifact_group` during ASAN runs does not retrieve files properly. An `artifact_group` for ASAN will look something like:

```
amdgpu_families: gfx94X-dcgpu
artifact_group: gfx94X-dcgpu-asan
```

Since the s3 file_names <b>do not</b> include asan in the artifact name, the script does not retrieve architecture based tar files properly.

The issue will need to be fixed here:
https://github.com/ROCm/TheRock/blob/8793649f1114fabd4ed34188a4d491db88d915d0/build_tools/fetch_artifacts.py#L72
