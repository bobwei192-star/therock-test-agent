We'd like to closely track the HEAD commits for each sub-project (mainline/develop/etc. branches).

## The existing manual process

The process for updating sub-projects to their upstream HEAD commits on their selected branches currently involves a few manual steps: https://github.com/ROCm/TheRock/blob/main/docs/development/git_chores.md. Notably, when there are conflicts with our local patches that cannot be automatically resolved, manual intervention is required.

Updates like https://github.com/ROCm/TheRock/pull/219 are currently ad-hoc, whenever a developer finds time for it. We want to turn this into a constantly running process using some mix of human control and automation.

## Background details

Each sub-project is currently included in TheRock as a git submodule, with some sub-projects carrying local patches.

* Submodules are defined here, each tracking some upstream branch: https://github.com/ROCm/TheRock/blob/main/.gitmodules 
* Patches are here: https://github.com/ROCm/TheRock/tree/main/patches.
  * Note that we may end up with multiple patch subfolders, one per major upstream version (e.g. mainline, 6.3, 6.4)
* The script to fetch sources and apply local patches is here: https://github.com/ROCm/TheRock/blob/main/build_tools/fetch_sources.py
* The script to save patches is here: https://github.com/ROCm/TheRock/blob/main/build_tools/save_patches.sh

## Automating

We can add a script that runs through the steps in the git chores document, failing if manual action is needed. Then, we can have a scheduled action (e.g. using github actions or a cron job) that runs that script, either creating a pull request if the script succeeds or sending some sort of alert if it fails. Once a PR is created (either by the automation or by a developer running the process / script themselves), we'll run our standard set of builds and tests and merge the PR if the results look good.
