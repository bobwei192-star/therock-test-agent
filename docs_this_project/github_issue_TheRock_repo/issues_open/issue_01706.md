# #1706 - LLVM Sanitizer Deps Need Flattening

When `SANITIZER_AMDGPU` is enabled, we do several reach-arounds, introducing circular-ish dependencies:


**compiler-rt/CMakeLists.txt:**

```
if(SANITIZER_AMDGPU)
  list(APPEND SANITIZER_COMMON_CFLAGS -DSANITIZER_AMDGPU=1)
  find_path(HSA_INCLUDE hsa.h HINTS ${SANITIZER_HSA_INCLUDE_PATH} ${CMAKE_INSTALL_PREFIX}/include/hsa ${CMAKE_INSTALL_PREFIX}/../include/hsa /opt/rocm/include/hsa PATH_SUFFIXES hsa)
  include_directories(${HSA_INCLUDE})
  find_path(COMgr_INCLUDE amd_comgr.h.in HINTS ${SANITIZER_COMGR_INCLUDE_PATH} ${CMAKE_INSTALL_PREFIX}/../include ${CMAKE_INSTALL_PREFIX}/include)
  if (NOT COMgr_INCLUDE)
    add_compile_definitions(AMD_COMGR)
    find_path(COMgr_INCLUDE amd_comgr.h HINTS ${CMAKE_INSTALL_PREFIX}/../include ${CMAKE_INSTALL_PREFIX}/include /opt/rocm/include)
  endif()
  include_directories(${COMgr_INCLUDE})
endif()
```

**offload/CMakeLists.txt:**

```
if(SANITIZER_AMDGPU)
  add_definitions(-DSANITIZER_AMDGPU=1)
  # Check for COMGr package , ASan requires COMGr with minimum version 2.4
  find_package(amd_comgr QUIET 2.4.0 HINTS ${CMAKE_INSTALL_PREFIX} PATHS /opt/rocm)
endif()
```

(there are other special cases, but these seem to be the ones that impact the build graph)

We need to get to the point that the build graph and sources needed to build are a strict DAG.

Currently, in TheRock, we do a multi-phase build of:

* llvm (libLLVM, clang, flang, compiler-rt, etc)
* amd-comgr
* hipify

We will be moving to a more disaggregated build in order to better isolate ancillary components and packaging (i.e. flang, mlir, comgr, possibly parts of omp/offload). Some of the above layering forces that (i.e. the offload dep on the amd_comgr library). That will need to account for these "hidden" deps between projects.

The compiler-rt/sanitizer deps on the headers of various projects is a bit easier dealt with. Here, I think we just need to eliminate ad-hoc path based `find_path` in favor of explicit directories and validation for needed header sources. This is partially done for COMgr_INCLUDE already and just needs some better validation and plumbing in from the top-level. The HSA_INCLUDE probing needs to be made explicit too.

We need to relegate all calls to `find_path` and especially all `CMAKE_INSTALL_PREFIX` and `/opt/rocm` based hints to be used with legacy flags, at the least. It is never valid to be building with dependencies from your own CMAKE_INSTALL_PREFIX, and that breaks all kinds of layering and good practices. And `/opt/rocm` is verboten for anything you care about being right vs "happens to work for the moment".

For the offload piece, we should invest in making `OPENMP_STANDALONE_BUILD` mode work and building the offload support on top of LLVM and as a separate invocation that explicitly depends on comgr.
