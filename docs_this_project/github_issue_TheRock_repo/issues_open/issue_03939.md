### Suggestion Description

Follows up on #2152. Waiting until current tests are stabilized before doing this split.

Several PyTorch tests compile C++ extensions at runtime (e.g. `dummy_allocator`) which require development headers like `hipblas/hipblas.h` from `rocm[devel]`. These tests are currently skipped because only `rocm[libraries]` is installed at test time.

Installing `rocm[devel]` for the entire test run is undesirable because it would mask regressions/issues in the torch runtime package configuration that end users have.

## Proposed Plan

Split the test [workflow](https://github.com/ROCm/TheRock/blob/main/.github/workflows/test_pytorch_wheels.yml) into two phases:

1. **Phase 1 - Core tests (without `rocm[devel]`):**
   Run the full test suite as-is with only `torch` installed (the current behavior). Tests that require devel packages remain skipped. This ensures we catch regressions in the runtime configuration.

2. **Phase 2 — Devel tests (with `rocm[devel]`):**
   Install `rocm[devel]` (version derived from the installed `rocm` package via `importlib.metadata`), set `ROCM_PATH` via `hipconfig --rocmpath`, and run only the tests that require `rocm[devel]`. This would re-enable the [currently skipped tests](https://github.com/ROCm/TheRock/blob/main/external-builds/pytorch/skip_tests/generic.py) that require these packages.
