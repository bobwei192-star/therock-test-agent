# #1830 - [Issue]: `rocm-sdk init` does not set any environment vars needed for CMake builds

### Problem Description

Installing ROCm wheels and running `rocm-sdk init` does not any `ROCM_*` or `HIP_*` environment variables:
```
wibking@login11:~> rocm-sdk init
Devel contents expanded to '/ccs/home/wibking/.local/lib/python3.11/site-packages/_rocm_sdk_devel'
wibking@login11:~> amdclang++ --version
AMD clang version 20.0.0git (https://github.com/ROCm/llvm-project.git a7d47b26ca0ec0b3e9e4da83825cace5d761f4bc+PATCHED:7a5435441416dc6f50dd93bb4d00d541132e999a)
Target: x86_64-unknown-linux-gnu
Thread model: posix
InstalledDir: /autofs/nccs-svm1_home1/wibking/.local/lib/python3.11/site-packages/_rocm_sdk_core/lib/llvm/bin
wibking@login11:~/quokka/build> env | grep HIP
wibking@login11:~/quokka/build> env | grep ROCM
wibking@login11:~/quokka/build>
```

These env vars are needed to be able to build ROCm apps with CMake, e.g.:
```

  asked CMake to find a package configuration file provided by "hip", but
  CMake did not find one.

  Could not find a package configuration file provided by "hip" with any of
  the following names:

    hipConfig.cmake
    hip-config.cmake

  Add the installation prefix of "hip" to CMAKE_PREFIX_PATH or set "hip_DIR"
  to a directory containing one of the above files.  If "hip" provides a

```
