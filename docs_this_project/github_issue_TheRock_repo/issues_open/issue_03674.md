## Background

Self-hosted GPU test runners have long queue times. While runner capacity expansion (on-prem + cloud), test filtering (smoketests vs full), and job filtering (change-based test selection) are being addressed separately, we can also reduce pressure by optimizing the test jobs themselves. Some jobs show 1-3 minutes of overhead from installing software, downloading artifacts, and unpacking artifacts.

For example, https://github.com/ROCm/TheRock/actions/runs/22457240043/job/65080326403 for hipsolver tests spent 18 seconds running (smoke) tests out of 1m37s total:
step | time
-- | --
fetch `build_tools` | 9s
checkout | 12s
setup test env | 39s
health checks | 4s
tests | 18s
post checkout | 7s

About 30s is spent there downloading and flattening artifacts. Another run for miopen https://github.com/ROCm/TheRock/actions/runs/22457240043/job/65080326481 spent almost 1 minute downloading and flattening artifacts.

## Ideas

Some design notes: https://github.com/ScottTodd/claude-rocm-workspace/blob/main/tasks/active/test-workflow-overhead.md

I think we should do a quantitative analysis of overhead like this and then considering implementing a cache for downloading, unzipped, and flattened artifacts.

Brainstorming potential building blocks / solutions:

* Set a `THEROCK_RUNNER_CACHE_ROOT` env var on test machines that jobs can write to and read from
* Use the cache for _downloads_ (`.tar.xz`)
* Use the cache for _extracted downloads_ (folders, one per artifact hash, possible to reuse across workflow runs)
* Use the cache for _flattened directories_ (merged folders from multiple artifacts, possible to reuse for multiple jobs within a workflow run)

Could build something into these scripts:
* [`build_tools/install_rocm_from_artifacts.py`](https://github.com/ROCm/TheRock/blob/main/build_tools/install_rocm_from_artifacts.py) (used by test workflows)
* [`build_tools/fetch_artifacts.py`](https://github.com/ROCm/TheRock/blob/main/build_tools/fetch_artifacts.py) (used by `build_tools/install_rocm_from_artifacts.py`)
* [`build_tools/artifact_manager.py`](https://github.com/ROCm/TheRock/blob/main/build_tools/artifact_manager.py) (used by build workflows, not as relevant here today)
