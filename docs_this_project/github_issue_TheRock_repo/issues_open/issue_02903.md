### Problem Description

rocprim test sort_keys_large_sizes tests fails pretty frequently on windows gfx1151 test runner slowing down the build testting a log because same test set needs to be ofren run 2-3 times to get it pass. That test set consist of from 40 other tests in it last about 30 min before the failure happen.

I believe we should either move this test to own test set so that it can be run separately or we should reduce the amount of memory this test uses on our gfx1151 windows runner. For now it uses 2^34 of memory which is close to 96gb and if the test machine has at the same time some other tasks also running (most likely) it is quite likely that this test will fail simply for out of resources type of error.

### Additional Information

Test that is failing is already warning about the issue on its comments, so this seems to be a kind of known issue:


```
    // Currently, CI enforces a hard limit of 96 GB on memory allocations.
    // Temporarily use sizes that will require less space than the limit.
    // On Windows, sizes above 2^34 (that are still under the 96 GB limit)
    // can hang due to issues that we can't currently catch by examining
    // the hipMalloc return value or querying available memory. Workaround
    // this for now by setting a different maximum size for that platform.
#if defined(_WIN32)
    const size_t max_pow2 = 34;
#else
    const size_t max_pow2 = 35;
#endif
```
