Environment

- GPU: AMD Radeon RX Vega (gfx900)

- PyTorch version: 2.10.0+rocm7.14.0a20260520
- Arch: gfx900
- Python: 3.12.3
- Package manager: uv

Steps to reproduce
1. Create a fresh Python 3.12 venv:
   python3 -m venv venv-linux-amd-3.12
   source venv-linux-amd-3.12/bin/activate

2. Install nightly PyTorch stack:
   TMPDIR=/media/HDD/tmp uv pip install --force-reinstall --no-cache-dir --pre torch torchvision torchaudio --index-url https://rocm.nightlies.amd.com/v2-staging/gfx900/ 

3. Import torch and torchvision:
   python -c "import torch; import torchvision; print(torch.__version__, torchvision.__version__)"

- The import of torchvision fails with:

Additional observations
- The nightly tarball/wheel for gfx900 appears to install successfully, but importing torchvision fails during meta registration of `torchvision::nms`.
- Using `uv pip install` with `--index-url https://rocm.nightlies.amd.com/v2-staging/gfx900/ ` works for installing torch, torchvision, torchaudio, but the resulting runtime environment is broken.
- When trying to reinstall only torchvision with `uv pip install --index-url https://rocm.nightlies.amd.com/v2-staging/gfx900/ /`, the solver fails with "torchvision was not found in the package registry".
- The same error occurs with ComfyUI as soon as it imports torchvision internally.
