### Problem Description

When doing a manual ASAN build by using a configure flags

```
cmake -B build -GNinja -DTHEROCK_VERBOSE=1 -DTHEROCK_SANITIZER=ASAN -DTHEROCK_AMDGPU_FAMILIES=gfx942 .

cmake --build build

```

rocrtst can be fixed by changing the code check and remove the ":xnack" variable from the gpu-name when parsing the gpu's hex-code that is generated from the gpuname.

Aqlprofile and roctracer can be fixed by enabling the ASAN_BUILD env-variable in their therock config files and then adding a check if that variable is enabled.
