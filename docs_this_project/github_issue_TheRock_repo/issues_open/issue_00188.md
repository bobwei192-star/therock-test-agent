### Suggestion Description

It would be ideal if it were possible to build all of ROCm as a reproducible build (e.g., https://reproducible-builds.org/).

This is particularly relevant for use-cases where we need to isolate a bug in a specific ROCm component that causes an application crash for our code when running in production on an HPC cluster that has ROCm modules that are provided by system integrators or HPC centers, in order to be sure that we are comparing against *exactly* the same version of the source for that component.
