### Summary
In CI Nightly run https://github.com/ROCm/TheRock/actions/runs/25299094784 multiple gfx1151 jobs failed with *two distinct signatures*:
1) **Infra / runner resource policy failures**: OOM kills (exit 137) and step timeout kills (15m/60m/90m watchdog).
2) **Product/runtime failures**: widespread SIGSEGV (-11), HSA/HIP runtime errors, accuracy mismatches, and missing/unreadable MIOpen DB files.

This issue aggregates the failures for triage and helps decide what should be addressed as runner/infra vs code/runtime.

### Context
- Workflow: `CI Nightly`
- Run: https://github.com/ROCm/TheRock/actions/runs/25299094784
- Affected arch: `gfx1151`
- Kernel: `6.17.0-1017-oem` (seen in multiple jobs)

---

## Failure signatures (grouped)

### (A) Infra / runner resource-limit failures
**A1. OOM / process killed**
Common patterns:
- `Process completed with exit code 137`
- `Subprocess killed`
- Memory monitor shows `Mem: ... (98–100%)`
- Followed by `Executing the custom container implementation failed. Please contact your self hosted runner administrator.`

**A2. Watchdog step timeout kills**
Common patterns:
- `[TIMEOUT DETECTED] Process ... killed at 15.1m / 60.1m / 90.2m (source: STEP_TIMEOUT_MINUTES)`
- Followed by `Executing the custom container implementation failed...`

### (B) Product/runtime failures (not runner infra)
**B1. Widespread segfaults (-11)**
- `***Exception: SegFault` in ctest output
- Test executables exit `-11`

**B2. Numerical correctness mismatch**
- rocFFT accuracy tests failing (`diff.l_inf` / `diff.l_2` thresholds exceeded)

**B3. HIP/HSA runtime failures**
- `hip create failure`
- `HSA_STATUS_ERROR_OUT_OF_RESOURCE`
These may be true runtime limitations/bugs or environment contamination between tests, but they are not generic GitHub Actions infra failures.

**B4. Packaging/config issues**
- MIOpen DB unreadable / missing, leading to “No engine configurations available”.
