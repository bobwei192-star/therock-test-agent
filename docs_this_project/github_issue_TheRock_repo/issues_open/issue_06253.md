### Problem Description

Hello! I am trying to use JAX on the AMD 6700S GPU (gfx1032). Usually I am able to spoof the gfx1030 card with `HSA_OVERRIDE_GFX_VERSION=10.3.0`. However, doing any sort of calculation or interaction with JAX at all seems to lead to a full system crash/reboot (Asus Zephyrus G14 2022) after the graphics pipe running at 100% for about 5 minutes (observed with radeontop). 

I set up my virtual environment like this: 

```
python -m pip install --index-url https://repo.amd.com/rocm/whl/gfx103X-all/ \
  "jaxlib==0.8.2" \
  "jax_rocm7_plugin==0.8.2+rocm7.13.0" \
  "jax_rocm7_pjrt==0.8.2+rocm7.13.0"
```
and 
```
 python -m pip install \
  "jax==0.8.2" \
  "jaxlib==0.8.2"
```
I'm able to run this: 

```
from jax import vmap
from jax import jit
from jax import random

key = random.key(1701)
key1, key2 = random.split(key)
mat = random.normal(key1, (150, 100))
batched_x = random.normal(key2, (10, 100))

def apply_matrix(x):
  return jnp.dot(mat, x)

def naively_batched_apply_matrix(v_batched):
  return jnp.stack([apply_matrix(v) for v in v_batched])

@jit
def vmap_batched_apply_matrix(batched_x):
  return vmap(apply_matrix)(batched_x)

np.testing.assert_allclose(naively_batched_apply_matrix(batched_x),
                           vmap_batched_apply_matrix(batched_x), atol=1E-4, rtol=1E-4)
print('Auto-vectorized with vmap')
%timeit vmap_batched_apply_matrix(batched_x).block_until_ready()
```
and it seems to complete successfully. I get: 
```
Auto-vectorized with vmap
140 μs ± 678 ns per loop (mean ± std. dev. of 7 runs, 10,000 loops each)
```
but I then have to quit the python environment to kill the GPU utilisation.

I narrowed down the function that could cause the crash to this: 
```
from jax._src import xla_bridge
backend = xla_bridge.get_backend()
```
which results in
```
W0630 15:30:43.500168   50704 rocm_pcie_bandwidth.cc:72] rsmi_dev_gpu_metrics_info_get failed for 0000:07:00.0: RSMI_STATUS_UNEXPECTED_DATA: Data read (usually from a file) or provided to function is not what was expected
```

When I set `ROCR_VISIBLE_DEVICES=0` and `HIP_VISIBLE_DEVICES=0`, those messages go away, but the same saturation of the graphics pipe occurs. 

I have experimented with setting these flags: 
```
--xla_gpu_force_compilation_parallelism=1
--xla_gpu_enable_triton_gemm=false
--xla_gpu_enable_latency_hiding_scheduler=false
--xla_gpu_disable_gpuasm_optimizations=true
```
and `XLA_PYTHON_CLIENT_MEM_FRACTION=0.5` and `XLA_PYTHON_CLIENT_PREALLOCATE=false`, to no avail.
