### Summary

The "Run build tests" step in the compiler-runtime stage currently runs with `continue-on-error: true`, so the `therock-build-tests` aggregate (currently the amd-comgr lit + ctest suite) only reports results without gating CI.

### Proposal

Remove `continue-on-error: true` from that step so the build-tests aggregate blocks CI on failure. Since the step is shared via `multi_arch_ci_linux.yml`, this gates both the primary Multi-Arch CI and the Multi-Arch CI ASAN runs.
