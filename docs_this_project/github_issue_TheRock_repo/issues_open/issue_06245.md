### Problem Description

After installing ROCm via multiarch pip on Ubuntu 22.04 (bare metal), running rocm-sdk test results in a failure in the testPreloadLibraries test with OSError: GLIBCXX_3.4.32 not found while loading librocdxg.so. The same issue is also observed on RHEL 9.8 and CentOS 9.

### Operating System

Ubuntu 22.04.5 LTS (Jammy Jellyfish)

### CPU

12th Gen Intel(R) Core(TM) i7-12700K

### GPU

Navi 31 Radeon RX 7900 XT

### ROCm Version

7.14

### Steps to Reproduce

#### Install Ubuntu 22.04 OS and AMD GPU drivers (ROCm)

```bash
sudo apt install -y libatomic1 libquadmath0
sudo apt install -y python3.11 python3.11-venv
sudo usermod -a -G render,video $LOGNAME
```

#### Logout/login or reboot required after this step

```bash
sudo reboot
```

#### Setup Python virtual environment

```bash
python3.11 -m venv .venv
source .venv/bin/activate
```

#### Install ROCm wheels packages

```bash
python -m pip install --index-url https://rocm.prereleases.amd.com/whl-multi-arch/ "rocm[libraries,devel,device-gfx1100]
```

#### Post-install test

```bash
rocm-sdk test
```
