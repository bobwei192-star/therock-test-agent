## Analysis

- `rocminfo` output passes all three sanity assertions (GPU Device Type, GFX Name, AMD Vendor) → the GPU is enumerable by the host
- `offload-arch` and `hipcc` compilation both succeed in ~1 second
- `./hipcc_check` produces zero output then hangs for ~4m59s → GPU kernel dispatch (`hipLaunchKernelGGL` + `hipDeviceSynchronize`) never returns
- The `pytest --timeout=300` per-test guard and the `timeout-minutes: 5` GHA step timeout are equal; the GHA watchdog terminates the step before pytest's signal fires

## Possible Cause

Runner-level GPU driver/hardware issue (wedged GPU, driver crash, or kernel module in bad state) on machine `xsjkrck2e09`. Not a code regression — no test source files or sanity scripts changed in commits landing around the run.

## Suggested Action

1. Re-run the job — if a different machine in the pool is picked up it will likely pass
2. If re-run fails on the same machine, reboot `xsjkrck2e09` and manually verify HIP kernel execution before re-queuing
3. If the hang recurs across multiple gfx1153 runners, compare against the previous passing nightly artifact for the same arch to rule out a build regression
