## Summary

3 MIOpenDriver-based tests fail on Windows gfx110X-all because `process.cpp:54` throws `miopen::Exception` with `"Capturing output not defined for Windows."` Process output capture is not implemented on Windows, so all MIOpenDriver test variants crash with an unhandled exception.
Observed during rocm-libraries bump PR #3807 (`7567d83` → `3557896`), CI run attempt 4.
