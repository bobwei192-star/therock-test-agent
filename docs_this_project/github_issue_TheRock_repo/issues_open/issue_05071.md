# #5071 - [Bug] Jax is building multiple packages for different manylinux versions

**State**: open  
**Author**: HereThereBeDragons  
**Created**: 2026-05-06T09:06:21Z  
**Updated**: 2026-06-09T17:25:10Z  
**URL**: https://github.com/ROCm/TheRock/issues/5071  

---

Jax is currently building multiple packages for multiple (many)linux versions, creating an unnecessarily large build matrix.

We should only support a _single manylinux version_ and provide one package per `rocm nightly x jax version x python version`. 
This would already create 1x3x4  = 12 packages per rocm nightly per jax package. Right now it creates nearly 30 packages for each jax package.

Following packages are affected ( using [gfx94X-dcgpu](https://rocm.nightlies.amd.com/v2-staging/gfx94X-dcgpu) as reference):

------------------------

**jax_rocm7_plugin**
version: 0.8.0 - not affected. stopped building since 20260421 - used manylinux_2_28

version: 0.8.2 - affected. building 3 versions
```
jax_rocm7_plugin-0.8.2+rocm7.13.0a20260504-cp314-cp314-linux_x86_64.whl
jax_rocm7_plugin-0.8.2+rocm7.13.0a20260504-cp314-cp314-manylinux2014_x86_64.whl
jax_rocm7_plugin-0.8.2+rocm7.13.0a20260504-cp314-cp314-manylinux_2_28_x86_64.whl
```

version: 0.9.0 - affected. building 3 versions
