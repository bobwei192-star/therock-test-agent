## Description

Building rocPRIM benchmarks in TheRock fails with a compile error in `shared/primbench/primbench.hpp` from the `rocm-libraries` submodule.

## Error
```
[rocPRIM_tests] In file included from /__w/TheRock/TheRock/rocm-libraries/projects/rocprim/benchmark/benchmark_block_radix_sort.hpp:23:
[rocPRIM_tests] /__w/TheRock/TheRock/rocm-libraries/projects/rocprim/benchmark/../../../shared/primbench/primbench.hpp:1506:5: error: use of undeclared identifier 'assert'
[rocPRIM_tests]  1506 |     assert(remaining_secs > 0.0);
[rocPRIM_tests]       |     ^~~~~~
[rocPRIM_tests] 1 error generated when compiling for gfx942.
```
https://github.com/ROCm/TheRock/actions/runs/24582835143/job/71892283369?pr=4334

## Root Cause

Upstream commit `7692a35da29` in rocm-libraries (https://github.com/ROCm/rocm-libraries/pull/5512, April 16 2026) added an `assert()` call to `shared/primbench/primbench.hpp` without adding `#include <cassert>`.

This was not caught by rocm-libraries CI because benchmarks are not built in their pre-checkin pipeline. The issue surfaces when TheRock enables rocPRIM benchmark builds.

## Fix

Add `#include <cassert>` to `shared/primbench/primbench.hpp`. The fix needs to land in [rocm-libraries](https://github.com/ROCm/rocm-libraries) and then the submodule pointer in TheRock needs to be updated.
