## Summary

The release package download script currently downloads every allow-listed package matching the requested ROCm version. For releases where we need to exclude a specific package variant, such as older PyTorch package versions, the filtering should happen at download time instead of during promotion or upload.

For ROCm 7.13, we stopped providing PyTorch 2.8 packages for the single-arch flow, while multi-arch still had PyTorch 2.8 packages available. To avoid promoting/uploading unwanted package variants, the download script should support explicit include/exclude filename filters.

The script already has separate handling for single-arch and multi-arch package discovery/downloads, so this filtering should be applied after package discovery and before download/listing.
## Proposed solution

Add CLI flags to `build_tools/packaging/download_python_packages.py`:

```bash
--include-package-glob "<pattern>"
--exclude-package-glob "<pattern>"
```
