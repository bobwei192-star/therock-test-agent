# #2708 - [Issue]: HIP compiler doesn't work from artifact install tree when ROCM_PATH is set

### Problem Description

There seems an issue with the ROCM_PATH environment variable lookup logic for the HIP compiler when it is built within TheRock.

Previously when the HIP compiler was installed as part of the ROCm packages, the ROCM_PATH environment variable was usually set to /opt/rocm and it worked fine.

With a HIP compiler built within TheRock setting ROCM_PATH to point at a artifact install tree makes the compiler run into an error not being able to find dependencies.

In particular this affects ROCgdb since it does set ROCM_PATH before invoking the HIP compiler, but will also affect other HIP compilation cases.

See reproduction steps.

It seems like TheRock's artifact install tree doesn't match the compiler's expected structure, so it fails to use the ROCM_PATH environment variable.

Since the use of ROCM_PATH is well document and even assumed by some tools, this has the potential to throw tools/users off when they see this error message from the HIP compiler.

We should correct TheRock's artifact install structure to match what the compiler expects when using ROCM_PATH, making things work both with and without setting the environment variable.

### Steps to Reproduce

Fetch a test executable. Here is one from ROCgdb (https://github.com/ROCm/ROCgdb/blob/amd-staging/gdb/testsuite/gdb.rocm/simple.cpp)

With a HIP compiler available from a TheRock build, in the artifact install tree (here available from /tmp/rock_artifacts/install), issue the following:

`/tmp/rock_artifacts/install/bin/hipcc  simple.cpp -w -lm -o simple`

The above command should work.

Now try setting ROCM_PATH to point at the artifact install tree:

`ROCM_PATH=/tmp/rock_artifacts/install /tmp/rock_artifacts/install/bin/hipcc  simple.cpp -w -lm -o simple`

The above yields the following error:

```
clang++: error: cannot find ROCm device library; provide its path via '--rocm-path' or '--rocm-device-lib-path', or pass '-nogpulib' to build without ROCm device library
failed to execute: (...)
```
