## Description
The "Upload Package repo to S3" step fails immediately after a successful package build because `upload_package_repo.py` looks for the package directory at the container path (`/__w/TheRock/TheRock/output/packages`) while the build step wrote packages to the host path (`/home/runner/_work/TheRock/TheRock/output/packages`).

## Context
- Workflow: Build Native Linux Packages (`build_native_linux_packages.yml`)
- Platform: Linux
- Impacted Arch: `all the suppported archs`
- Release type: `nightly`

## Sample run
- [View failing job](https://github.com/ROCm/TheRock/actions/runs/26019434706/job/76477070244)

## Suspected Root Cause
Commit https://github.com/ROCm/TheRock/commit/fc4e6be70f49f6de3d562f71899314fbe2709d7d added `container: image: ubuntu:24.04` to this workflow but did not update the `OUTPUT_DIR` / `ARTIFACTS_DIR` / `PACKAGE_DIST_DIR` environment variables from `${{ github.workspace }}/output*` (host-absolute path) to relative paths. The same fix was applied to `multi_arch_build_native_linux_packages.yml` in the same commit.

## Suggested Fix
In `.github/workflows/build_native_linux_packages.yml`, change the three env vars to relative paths (matching `multi_arch_build_native_linux_packages.yml`):

```yaml
      OUTPUT_DIR: output
      ARTIFACTS_DIR: output/artifacts
      PACKAGE_DIST_DIR: output/packages
```
