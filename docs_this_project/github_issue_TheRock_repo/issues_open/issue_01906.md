### Problem Description

When I install the rocm_sdk from python wheels in python application and then on same app run the "rocm_sdk root --path" command to find the ROCM_HOME directory. that will fail on first run for the exception because it can not find all of the files needed. Reason is that the windows version of python wheels are missing the files that are available on Linux version of wheels as a symlink. 

Id the "rocm_sdk path --root" command is executed for a second time in same virtual environment without reinstalling the pythoin wheels, then the command will work returning "./Lib/site-packages/_rocm_sdk_devel" as a path.
