## Summary:
In CI Nightly run https://github.com/ROCm/TheRock/actions/runs/24758063493 multiple jobs on Linux gfx110x failed across unrelated components. 

- Several failures look like environment/arch detection mismatches (rocBLAS searching gfx1030 libraries; rocprofiler-compute listing gfx1100/gfx1030 unsupported; counters returning 0).
- Some failures look like device visibility / runner provisioning issues (no ROCm-capable device is detected).
- hipCUB “invalid kernel file” could indicate incompatible/missing code objects for gfx110x or runtime mismatch.
- rocDecode segfaults likely need investigation in decode runtime path on gfx1103/780M + VAAPI/HIP init.

***Test Failures observed after enabling gpu runners:*** https://github.com/ROCm/TheRock/commit/0a12dbbfe47d3f933c75c6f6233aea01bfcc3479
