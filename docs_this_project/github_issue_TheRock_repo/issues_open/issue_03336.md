Parent issue tracking shipping ROCm multi-architecture packaging with Kpack based on this RFC: https://github.com/ROCm/TheRock/blob/main/docs/rfcs/RFC0008-Multi-Arch-Packaging.md.

## Overview

To fully support multi-arch the CI needs to partly be refactored and extended. This included to build artifacts, run all tests, uploads logs etc and to enable the CI on pre-submit and to fully transition to the multi-arch CI.

This issue tracks implementing the work for CI improvements required to fully support multi-arch.

Major milestones:

* Enable multi-arch CI on post-submit
* Enable multi-arch CI on pre-submit
* Enable `THEROCK_KPACK_SPLIT_ARTIFACTS` by default for developers
* Remove single stage (non-multi-arch) CI
