## Summary

All 20 test jobs for gfx1151 fail in the Run` rocm-sdk sanity tests` step. The `testConsoleScripts` test in `rocm_sdk.tests.core_test.ROCmCoreTest` calls the `rocminfo` console-script wrapper, which crashes with `SIGSEGV `(exit signal 11). The failure is consistent across multiple distinct runners (l`inux-strix-halo-gpu-rocm-3, -6, -7, -9, CS-RORDMZ-DT239, CS-RORDMZ-DT241`), ruling out a single-runner issue.

    output_text = subprocess.check_output(...)

**Counts**: 1 ERROR / 19 run — across 20 jobs
## Root Cause
The rocminfo binary from the rocm-core==7.14.0a20260611 package segfaults on gfx1151 (Strix Halo) hardware. The regression is in the nightly ROCm SDK package for this build date and needs investigation in the rocminfo component or its runtime dependencies (HSA-Runtime, hsa-rocr).
