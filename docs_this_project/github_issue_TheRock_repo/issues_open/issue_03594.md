# #3594 - Make workflow matrix combinations be dynamically created during setup

**State**: open

**Updated**: 2026-02-24T21:04:19Z

---

Right now we have in our workflow files matrix combinations hardcoded (e.g. pytorch vs python version vs arch ). There are multiple files (at least for each platform) that require the same combination, making it error prone to add changes.

```
  matrix:
    python_version: ["3.10", "3.11", "3.12", "3.13"]
    pytorch_git_ref: ["release/2.8", "release/2.9", "release/2.10", "nightly"]
    # redefine arch as inputs.x vars are not available in matrix context
    arch: ["${{ inputs.amdgpu_family }}"]
    exclude:
    # gfx1153 support is only available in pytorch 2.10+
    # see https://github.com/ROCm/TheRock/issues/2723
    - arch: "gfx1153"
      pytorch_git_ref: "release/2.8"
    - arch: "gfx1153"
      pytorch_git_ref: "release/2.9"
```

As such it would be good to have a script similar to `amdgpu_family_matrix.py` that creates the array lists for the matrix during the setup process (in `setup.yml`) which then are passed as input parameters to the specific workflow.

Comment for context:
> What you have now might work. Long term we should move the matrix into a file like https://github.com/ROCm/TheRock/blob/main/build_tools/github_actions/amdgpu_family_matrix.py that is read by a workflow like https://github.com/ROCm/TheRock/blob/main/.github/workflows/setup.yml to generate the complete set of jobs to run at the start of the release pipeline, including any "manifest" files. We can't keep kicking that can down the road for much longer, see https://github.com/ROCm/TheRock/issues/1236.
