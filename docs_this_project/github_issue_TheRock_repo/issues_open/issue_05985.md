### Problem Description
TheRock nightly therock-dist-linux-gfx1151-7.14.0a20260612.tar.gz installs successfully but rocminfo segfaults immediately after printing "ROCk module is loaded" on a Strix Halo system (gfx1151). The Ubuntu ROCm 7.1 packages (HSA runtime 1.18.0) work correctly on the same system.

### Hardware
CPU/APU: AMD Ryzen AI MAX+ 395 (Strix Halo)
GPU: Radeon 8060S (gfx1151, device ID 0x1586)
Unified memory: 128GB
Platform: Framework Desktop

### Software
OS: Ubuntu 26.04 LTS
Kernel: 7.0.0-1005-oem (#5-Ubuntu SMP PREEMPT_DYNAMIC Thu May 7 09:02:05 UTC 2026)
TheRock tarball: therock-dist-linux-gfx1151-7.14.0a20260612.tar.gz
TheRock HSA runtime: 1.21.0
Ubuntu ROCm HSA runtime (working): 1.18.0
Install path: /opt/rocm-therock

### Steps to Reproduce
bash# Download and extract
```
S3="https://therock-nightly-tarball.s3.amazonaws.com"
curl -L -o /tmp/therock-gfx1151.tar.gz \
  "${S3}/therock-dist-linux-gfx1151-7.14.0a20260612.tar.gz"
sudo mkdir -p /opt/rocm-therock
sudo tar -xzf /tmp/therock-gfx1151.tar.gz -C /opt/rocm-therock --strip-components=1

# Run rocminfo
/opt/rocm-therock/bin/rocminfo
```

**Actual Output**

ROCk module is loaded
Segmentation fault (core dumped)

Exit code: 139. No output with 2>&1 | head -40 — the segfault was swallowed by head, making initial diagnosis difficult.

**Additional context**
The Ubuntu ROCm 7.1 packages work correctly for inference (llama.cpp with -DGGML_HIP=ON -DAMDGPU_TARGETS=gfx1151). The motivation for TheRock was to access working hipBLASLt gfx1151 kernel libraries (TensileLibrary_*_gfx1151) and complete rocWMMA headers (Ubuntu's librocwmma-dev package is missing the internal/ subdirectory). TheRock's hipBLASLt library structure and rocWMMA headers are confirmed present and correctly structured in the tarball — the blocker is solely the HSA runtime segfault preventing any GPU access.
