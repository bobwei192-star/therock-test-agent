Following up on https://github.com/ROCm/TheRock/pull/120#discussion_r1977912063 and https://github.com/ROCm/TheRock/pull/179#discussion_r1991242245, we currently have many top level directories and the current alphabetical sorting does not make it clear how they relate to one another:

```
.github
base               # component
build_tools
cmake
comm-libs          # component
compiler           # component
core               # component
dockerfiles
docs
examples
external-builds
math-libs          # component
ml-libs            # component
patches
profiler           # component
tests
third-party
```

Here's one idea:

```
.github
build_tools
    ├── cmake
    └── dockerfiles
docs
examples
external-builds
patches
rocm
    ├── base
    ├── comm-libs
    ├── compiler
    ├── core
    ├── math-libs
    ├── ml-libs
    └── profiler
tests
third-party
```
