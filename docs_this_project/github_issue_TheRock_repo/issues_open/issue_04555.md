### Problem Description

When i run anima or zit model in comfyui, i found miopen does not generate user datebase, while sdxl model benchmark slovers and generate user datebase on first run.
I'm not sure this is a bug or a feature.
If i set `PYTORCH_TUNABLEOP_ENABLED=1`, it will get a bit faster after first run when using anima model. This is similar to run sdxl model on torch 2.10 which disabled hipblaslt for gfx1103, but TUNABLEOP force using some hipblaslt slovers.

### Steps to Reproduce

1. Install comfyui
2. pip install sageattention==1.0.6
3. replace sageattention\attn_qk_int8_per_block.py sageattention\attn_qk_int8_per_block_causal.py sageattention\quant_per_block.py by https://raw.githubusercontent.com/patientx/ComfyUI-Zluda/refs/heads/master/comfy/customzluda/sa/attn_qk_int8_per_block.py https://raw.githubusercontent.com/patientx/ComfyUI-Zluda/refs/heads/master/comfy/customzluda/sa/attn_qk_int8_per_block_causal.py https://raw.githubusercontent.com/patientx/ComfyUI-Zluda/refs/heads/master/comfy/customzluda/sa/quant_per_block.py
4. call "C:\Program Files (x86)\Microsoft Visual Studio\2022\BuildTools\VC\Auxiliary\Build\vcvars64.bat"
5. init rocm-sdk and set env:
```
rocm-sdk init
set ROCM_HOME=python_embeded\Lib\site-packages\_rocm_sdk_devel
set ROCM_PATH=%ROCM_HOME%
set HIP_PATH=%ROCM_HOME%
set PATH=%ROCM_HOME%\lib\llvm\bin;%ROCM_HOME%\bin;%PATH%
set CC=clang-cl
set CXX=clang-cl
set DISTUTILS_USE_SDK=1
set FLASH_ATTENTION_TRITON_AMD_ENABLE=TRUE
set TORCH_ROCM_AOTRITON_ENABLE_EXPERIMENTAL=1
set TORCH_BLAS_PREFER_HIPBLASLT=1
set COMFYUI_ENABLE_MIOPEN=1
set MIOPEN_SEARCH_CUTOFF=1
```
3. use following args to start comfyui:
```
python_embeded\python.exe ComfyUI\main.py --bf16-vae --use-sage-attention
```
