# #3208 - [Issue] missing binaries from rocprofiler-compute and rocprofiler-systems in python packages

**State**: open  
**Author**: jbonnell-amd  
**Created**: 2026-02-02T20:06:43Z  
**Updated**: 2026-03-04T17:06:21Z

---

It appears that when installing ROCm via pip install, we are missing binaries for rocprofiler-compute and rocprofiler-systems.

This was reproduced using the latest releases as of 02/02/2026.

When installing from tarball, we get the following rocprof* binaries.
```
./bin/rocprofv3
./bin/rocprof-sys-avail
./bin/rocprof-sys-causal
./bin/rocprof-sys-run
./bin/rocprof-attach
./bin/rocprof-compute
./bin/rocprof-sys-sample
./bin/rocprof-sys-instrument
./bin/rocprofv3-avail
```

However, when installing from pip, we only get the following rocprof* binaries.
```
./.venv/lib/python3.12/site-packages/_rocm_sdk_core/bin/rocprofv3
./.venv/lib/python3.12/site-packages/_rocm_sdk_core/bin/rocprof-attach
./.venv/lib/python3.12/site-packages/_rocm_sdk_core/bin/rocprofv3-avail
```

As you can see, all of the rocprof-sys* binaries as well as the rocprof-compute binary are missing from the pip install version. We need to ensure that the python packages contain the proper systems and compute binaries.
