Our generated files like `.venv\Lib\site-packages\_rocm_sdk_devel\lib\cmake\hip\hip-config.cmake` have CMake like this:
```cmake
if("OFF")   # <---- THIS IS THE PROBLEM
  if (WIN32)
    set_and_check(hip_HIPCC_EXECUTABLE "${hip_BIN_INSTALL_DIR}/hipcc.exe")
    set_and_check(hip_HIPCONFIG_EXECUTABLE "${hip_BIN_INSTALL_DIR}/hipconfig.exe")
  else()
    set_and_check(hip_HIPCC_EXECUTABLE "${hip_BIN_INSTALL_DIR}/hipcc")
    set_and_check(hip_HIPCONFIG_EXECUTABLE "${hip_BIN_INSTALL_DIR}/hipconfig")
  endif()
endif()
```

Those variables are needed to infer `HIP_PLATFORM=amd` (which we want to remove anyways... but such is the current state) using `hipconfig --platform`.

See https://github.com/ROCm/rocm-systems/blob/develop/projects/clr/hipamd/hip-config.cmake.in and https://github.com/ROCm/rocm-systems/blob/develop/projects/clr/hipamd/CMakeLists.txt

Without the automatic `HIP_PLATFORM=amd` discovery, we get errors like what we reported at https://github.com/pytorch/pytorch/pull/161754#discussion_r2323642163 where downstream projects see errors unless they explicitly add `-DHIP_PLATFORM=amd` to their CMake configure commands before calling `find_package(hip REQUIRED)`:
