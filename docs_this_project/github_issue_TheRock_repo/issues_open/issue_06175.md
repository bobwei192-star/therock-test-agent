### Problem Description

Trying to do some int8 matrix multiplication here, and this issue arise. gfx906 is not supported by rocblaslt but it tries to use it regardless.

### Operating System

Linux Mint 21.3 (Virginia)

### CPU

AMD Ryzen 5 5500

### GPU

AMD Radeon VII

### ROCm Version

7.15.0a20260626

### ROCm Component

hipBLAS, hipBLASLt

### Steps to Reproduce

Follow in instructions in [RELEASES.md](https://github.com/ROCm/TheRock/issues/RELEASES.md) for Multi-arch releases, install multi-arch ROCm and PyTorch Python packages.
Run the following Python script:
```
import torch
import os

M, K, N = 512, 512, 512

os.environ["TORCH_BLAS_PREFER_HIPBLASLT"] = "0"
os.environ["ROCBLAS_USE_HIPBLASLT"] = "1"
a = torch.randint(-128, 127, (M, K), dtype=torch.int8, device='cuda')
b = torch.randint(-128, 127, (K, N), dtype=torch.int8, device='cuda')
c = torch._int_mm(a, b)
```
