### Problem Description

`librocblas.so.5.4` in the 23.2.0 drop from https://repo.radeon.com/rocm/misc/flang/ is built with gfx942 (and other gfx9xx) targets, but `lib/rocblas/library/` is staged only for gfx1200/gfx1201. On gfx942 rocBLAS GEMM fails with:
```
rocBLAS error: Cannot read /shared/apps/ubuntu/opt/rocm-therock-23.2.0/lib/rocblas/library/TensileLibrary.dat: No such file or directory for GPU arch : gfx942
```


### Operating System

Ubuntu 22.04.5 LTS (Jammy Jellyfish)

### CPU

AMD Instinct MI300A Accelerator

### GPU

AMD Instinct MI300A Accelerator

### ROCm Version

the rock 23.2.0

### ROCm Component

rocBLAS

### Steps to Reproduce

Reproducer:

 ```
rocblas-bench -f gemm -r d --transposeA N --transposeB N -m 64 -n 64 -k 64 -i 1
```

 .so does contain gfx942 (and other gfx9xx) code
  ```
strings $ROCM_PATH/lib/librocblas.so.5.4 | grep -E '^gfx[0-9]+$' | sort -u
```
  but the kernel archive on disk does not
```
  ls $ROCM_PATH/lib/rocblas/library | wc -l           # 55, all *_fallback.dat
  awk -F_ '/gfx/{for(i=1;i<=NF;i++)if($i ~ /^gfx/)print $i}' \
     $ROCM_PATH/lib/rocblas/library/TensileManifest.txt | sed 's/\..*//' | sort -u
  ```

  For comparison: drop 23.1.0 in the same directory has 297 files, 242 containing gfx942, including
  Kernels.so-000-gfx942.hsaco and TensileLibrary_lazy_gfx942.dat.
