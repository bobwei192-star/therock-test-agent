This is a bottleneck for host-asan builds. hipSPARSELt is around 3x slower to build under host-asan while rocblas, rocfft, rocsparse, etc. are about equally fast. hipblaslt _is_ doing more work during its build, and MIOpen is also slow with and without.

So I looked into two details:
1. The host-asan build is including GPU xnack+ targets that should only be included for asan (not host-asan): https://github.com/ROCm/rocm-libraries/pull/8397#pullrequestreview-4581808138
2. Tensile is running python code under ASan with `LD_PRELOAD=libclang_rt.asan` during kernel generation at build time, which seems useful _for the kernel generator_ but not for _the resulting libraries at runtime_, so if this is the source of the slowdown I think we could restructure a bit. See:
    * https://github.com/ROCm/rocm-libraries/blob/b6f8346a54caa52acb9d85abfe422c628b56b3bd/shared/tensile/Tensile/cmake/TensileConfig.cmake#L283-L293
    * https://github.com/ROCm/rocm-libraries/blob/b6f8346a54caa52acb9d85abfe422c628b56b3bd/projects/hipblaslt/CMakeLists.txt#L178-L212
    * https://github.com/ROCm/rocm-libraries/blob/develop/projects/hipblaslt/cmake/hipblaslt_python.cmake
    * https://github.com/ROCm/rocm-libraries/blob/develop/projects/hipblaslt/device-library/CMakeLists.txt
