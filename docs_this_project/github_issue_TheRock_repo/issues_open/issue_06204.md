## Problem

ROCgdb's OpenMP GPU offload tests ([ROCm/ROCgdb#134](https://github.com/ROCm/ROCgdb/pull/134)) fail in CI with:

```
gdb compile failed, ld.lld: error: unable to find library -lompdevice
clang: error: ld.lld command failed with exit code 1 (use -v to see invocation)
/__w/ROCgdb/ROCgdb/build/lib/llvm/bin/clang-linker-wrapper: error: 'clang' failed
flang-23: error: linker command failed with exit code 1 (use -v to see invocation)
```

The linker cannot find `-lompdevice` because `amd-llvm_dev` is not included in the `--debug-tools` artifact set fetched by `install_rocm_from_artifacts.py`. The `ompdevice` static library ships inside `amd-llvm_dev` and is required at link time when building OpenMP GPU offload programs with `-fopenmp -fopenmp-targets=amdgcn-amd-amdhsa`.

## Fix

Add `amd-llvm_dev` to the `--debug-tools` artifact set in `build_tools/install_rocm_from_artifacts.py`, and deduplicate the scattered `amd-llvm_dev` appends across `--rocdecode`, `--rocjpeg`, and `--libhipcxx` into a single `needs_amd_llvm_dev` flag.
