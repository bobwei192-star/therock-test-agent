### Problem Description

In TheRock multi-arch packaging, the unversioned `libhsa-runtime64.so` alias only ships in `_rocm_sdk_devel/lib/`. The two runtime trees consumers actually ship against: _rocm_sdk_core/lib/ (where the SONAME libhsa-runtime64.so.1 lives) and _rocm_sdk_libraries/lib/, don't carry it.

This breaks librccl for any consumer that doesn't transit `_rocm_sdk_devel/lib/`. librccl ([rocm-systems/projects/rccl/src/misc/rocmwrap.cc:~177](https://github.com/ROCm/rccl/blob/main/src/misc/rocmwrap.cc)) calls `dlopen("libhsa-runtime64.so")` (unversioned). When librccl is loaded from a non-devel tree (or in a `/opt/rocm`-less deployment), the lookup falls through every search path and fails:

`[RCCL] Failed to find ROCm runtime library in (null) (RCCL_ROCR_PATH=(null))`
This bocks any RCCL collective in /opt/rocm-less deployments

The unversioned *.so alias is a Linux convention used as a developer-time link-time hook; the SONAME *.so.<n> is the runtime contract. RCCL's unversioned dlopen is its own bug, but the packaging asymmetry is independently fixable and unblocks /opt/rocm-less deployment without waiting on librccl.

### Steps to Reproduce

```
docker run --rm ubuntu:24.04 bash -lc '
  apt-get update -qq && apt-get install -y -qq python3-pip
  python3 -m pip install --break-system-packages --pre \
      --index-url https://rocm.nightlies.amd.com/whl-multi-arch/ \
      "rocm[libraries,devel,device-gfx950]"
  rocm-sdk init

  PURELIB=$(python3 -c "import sysconfig; print(sysconfig.get_paths()[\"purelib\"])")
  for tree in _rocm_sdk_core _rocm_sdk_libraries _rocm_sdk_devel; do
    echo "--- $tree/lib ---"
    out=$(ls "$PURELIB/$tree/lib"/libhsa-runtime64* 2>/dev/null | sed "s|$PURELIB/||")
    [ -z "$out" ] && echo "(empty)" || echo "$out"
  done
'
```

Output (7.14.0a20260601)
```
--- _rocm_sdk_core/lib ---
_rocm_sdk_core/lib/libhsa-runtime64.so.1
--- _rocm_sdk_libraries/lib ---
(empty)
--- _rocm_sdk_devel/lib ---
_rocm_sdk_devel/lib/libhsa-runtime64.so
_rocm_sdk_devel/lib/libhsa-runtime64.so.1
_rocm_sdk_devel/lib/libhsa-runtime64.so.1.21.0
```
