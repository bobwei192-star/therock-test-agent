Observations:

1.  The linker command for `hipsparse-bench.exe` includes both `-fopenmp=libomp` and a direct path to MSVC's OpenMP library: `"C:/Program Files/Microsoft Visual Studio/2022/Community/VC/Tools/MSVC/14.41.34120/lib/x64/libomp.lib"`.
2.  The linker command for `hipsparse-test.exe` involves a step to copy `C:/Windows/System32/libomp140.x86_64.dll` into the staging directory.
3.  A `dumpbin /exports` on `C:/Program Files/Microsoft Visual Studio/2022/Community/VC/Tools/MSVC/14.41.34120/lib/x64/libomp.lib` and `C:/Windows/System32/libomp140.x86_64.dll` confirmed that the symbol `__kmpc_dispatch_deinit` is not exported by these libraries.
4.  The symbol `__kmpc_dispatch_deinit` is defined (as an empty function) within TheRock's source tree at `compiler/amd-llvm/openmp/runtime/src/kmp_dispatch.cpp`.
    ```cpp
    void __kmpc_dispatch_deinit(ident_t *loc, kmp_int32 gtid) {}
    ```
5.  The build process for hipSPARSE appears to be using the OpenMP runtime from the Windows system, while the OpenMP runtime from amd-llvm does not seem to be built.
