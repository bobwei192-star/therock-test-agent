# #2266 - `libamdocl64.so` validation test fails due to wrong path and missing runtime dependencies

**State**: open  
**Author**: erman-gurses  
**Created**: 2025-11-21T23:47:03Z  
**Updated**: 2025-11-21T23:55:55Z  
**Assignees**: erman-gurses  
**URL**: https://github.com/ROCm/TheRock/issues/2266  

---

As a summary, there were two separate problems:

The test was looking in the wrong directory. It attempted to load `dist/lib/libamdocl64.so` but the actual file is generated under `dist/lib/opencl/libamdocl64.so`

This mismatch caused the CTest failure:

`OSError: /__w/TheRock/TheRock/build/core/ocl-clr/dist/lib/libamdocl64.so: cannot open shared object file: No such file or directory`

Even after fixing the path, the dynamic linker could not load the library’s dependencies. `libamdocl64.so` depends on several runtime libraries (e.g. `libamd_comgr_loader.so.1`), but those directories were not on the loader search path when CTest runs.
