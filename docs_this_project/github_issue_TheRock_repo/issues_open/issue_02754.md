### Summary
Currently, ROCm library benchmarks run on shared CI machines (using the same `test-runs-on` runners) in Nightly CI jobs. This shared infrastructure causes performance variability between runs, leading to false failures and unreliable benchmark results.

### Background
As part of the benchmark enablement effort, we've integrated microbenchmarks for the following ROCm libraries:
- 	rocFFT
- 	rocRAND
- 	rocSOLVER
- 	hipBLASLt
- 	rocBLAS (in progress)
- 	RCCL (in progress)

These benchmarks currently run on:
- 	Linux "gfx94x": Same runners as functional tests
- 	Windows "gfx1151": Same runners as functional tests

### Problem
Running benchmarks on shared CI servers causes Performance variability and false failures.

### Current Mitigation
All benchmark tests are currently marked as XFAIL to prevent blocking CI while running on shared infrastructure.

### Proposed Solution
Add dedicated performance servers for benchmark execution:

1. Create new benchmark-runs-on runner pools with dedicated GPU resources
2. Update amdgpu_family_matrix.py to use dedicated runners:
```
 "benchmark-runs-on": "linux-gfx94x-perf-server"  # Dedicated perf runner
 "benchmark-runs-on": "windows-gfx1151-perf-server"  # Dedicated perf runner
```
3. Remove XFAIL markers once dedicated servers are in place
4. Enable proper performance regression detection
