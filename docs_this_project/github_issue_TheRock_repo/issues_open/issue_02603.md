## Bug Summary: Versioned vs unversioned `librocprofv3-list-avail.so`

**Package:** `rocm-sdk-core==7.11.0a20251217` (from `rocm.nightlies.amd.com/v2-staging/gfx1151`)

**Steps to reproduce:**
```bash
python3 -m venv /tmp/test_venv
source /tmp/test_venv/bin/activate
uv pip install --extra-index-url=https://rocm.nightlies.amd.com/v2-staging/gfx1151 'rocm[devel]'
rocm-sdk init
$(rocm-sdk path --bin)/rocprofv3-avail list
```

**Error:**
```
OSError: .../site-packages/_rocm_sdk_core/lib/rocprofiler-sdk/librocprofv3-list-avail.so: cannot open shared object file: No such file or directory
```

**Root cause:**  
The `_rocm_sdk_core/lib/rocprofiler-sdk/` directory contains `librocprofv3-list-avail.so.1` but is missing the unversioned symlink `librocprofv3-list-avail.so`. The Python code in `rocprofv3/avail.py` attempts to load the unversioned `.so` file via `ctypes.CDLL()`.
