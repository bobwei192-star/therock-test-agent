## Summary

In a Test Artifacts run on Windows gfx120X-all runners, 27 out of 38 jobs failed due to what appears to be a GPU driver crash or hardware reset on the `labxsj-windows-11-gfx1201-gpu-rocm-runner-*` machines. The failure cascaded through the run in three distinct waves:

1. `hipInfo.exe` (launched by the "Driver / GPU sanity check" step) hung for 3+ minutes and timed out on multiple runners, leaving zombie process handles.
2. The pre-job `cleanup_processes.ps1` on subsequent jobs found those zombie handles, failed to terminate them (because `HasExited:True`), and exited with code 1 — causing 14 jobs to skip all test steps.
3. Jobs that did reach the test step hit `No WDDM adapters found` / `No devices found`, or received a wrong GPU arch (`gfx1036` instead of `gfx1201`) from the driver, causing rocBLAS to abort.

## Context

- **Workflow:** `Test Artifacts` (`.github/workflows/test_artifacts.yml` in `ROCm/rockrel`, dispatching to `ROCm/TheRock/.github/workflows/test_artifacts.yml`)
- **Run:** https://github.com/ROCm/rockrel/actions/runs/26505524009
- **Platform:** Windows
- **GPU arch:** `gfx120X-all` (`gfx1200`, `gfx1201`)
- **TheRock ref:** `eeb6ca76b8eed6e8ebdb783cb41e14e455baea09`
- **Affected runners:** `labxsj-windows-11-gfx1201-gpu-rocm-runner-01`, `-03`, `-04`, `-05`

## Failure signatures

### A — Pre-job cleanup exits 1 due to zombie `hipInfo.exe` (14 jobs)

`cleanup_processes.ps1` finds `hipInfo.exe` in the process table with `HasExited:True` and cannot remove the zombie handle:

```text
[*] Found 1 running build executable(s):
    > C:\runner\_work\rockrel\rockrel\build\bin\hipInfo.exe
[*] Attempting to stop executable(s) with WMI:
    > [pid:1800][HasExited:True] hipInfo.exe
...
[-] Failed to stop executable(s):
    > [pid:1800][HasExited:True] hipInfo.exe
##[error]Process completed with exit code 1.
```

All test steps are then skipped for these jobs.

### B — `hipInfo.exe` hangs in "Driver / GPU sanity check" (3 jobs)

```text
++ Exec [C:\runner\_work\rockrel\rockrel]$ 'build\bin\hipInfo.exe'
##[error]The action 'Driver / GPU sanity check' has timed out after 3 minutes.
```

### C — HIP runtime finds no GPU at test time (3 jobs)

```text
pid:1384 tid:0x4156 [topology_sysfs_get_system_props] No WDDM adapters found.
Error: No devices found
Query device error: cannot get device count
##[error]Process completed with exit code 1.
```

### D — rocBLAS reports wrong GPU arch `gfx1036`, aborts (2 jobs)

```text
rocBLAS error: Cannot read ...\build\bin\/rocblas/library/TensileLibrary.dat:
No such file or directory for GPU arch : gfx1036
Exception Code: 0x80000003
```

### E — Individual test-level failures on partially-recovered runners (5 jobs)

- `hipcub`: `HIP error 100: no ROCm-capable device is detected`
- `rocrand`: `test_rocrand_kernel_mrg: Process not started / unknown error` (1/52 tests)
- `hip-tests (PAL) shard 4`: `RTC.exe: Error listing tests: unknown error`
- `hip-tests (PAL) shard 3`: 120-minute timeout on `Unit_hipStreamValue_Wait_Blocking`
- `hipdnn_install`: `hipInfo.exe` still hanging when run completed

## Root cause

GPU driver crash or hardware instability on the gfx1201 test machines. The triggering commit (`eeb6ca76`) only modifies an AWS credential workflow — it is unrelated to this failure.

## Suggested fix

**Immediate:** Power-cycle or restart GPU drivers on `labxsj-windows-11-gfx1201-gpu-rocm-runner-01/03/04/05`, then re-run.

**Code hardening:** `cleanup_processes.ps1` should treat `HasExited:True` entries as already stopped, not as an error requiring `exit 1`. This avoids cascading pre-job failures when a zombie process handle lingers after a GPU driver reset.
