## Description

The `Run rocm-sdk sanity tests` step in the Windows PyTorch wheels test job times out after 5 minutes on every gfx110X run. The step invokes `rocm-sdk test`, which runs `rocm_sdk.tests.core_test.ROCmCoreTest.testConsoleScripts`. That test iterates `COMMON_CONSOLE_SCRIPT_TESTS` (shared across all platforms) — which includes `hipcc` — and calls `subprocess.check_output` with **no timeout**. On Windows/gfx110X runners, `hipcc --help` hangs because `hipcc` internally invokes `offload-arch.exe`, which is broken on these runners (see #4755).

## Context

- Platform: `Windows`
- Impacted Arch: `gfx110X-all`
- Workflow: `release_windows_pytorch_wheels.yml`
- All Python versions and PyTorch versions affected (25/25 Windows Test jobs fail in each run)
