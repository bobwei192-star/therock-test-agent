The repo ships two advisory PR-quality skills under `skills/` — `rocm-pr-quality` (the ROCm-wide base) and `therock-pr-quality` (the TheRock build-repo overlay) — plus the `therock_pr_bot` policy checker. Today these are only referenced from `CLAUDE.md`, so a contributor only encounters them if they use an AI assistant wired to that file. Someone browsing the repo or reading `CONTRIBUTING.md` has no pointer to them.

This is purely a discoverability gap; the skills themselves already exist and work.

Proposed fix: add a `skills/README.md` index and link to it from `CONTRIBUTING.md` (Pull requests + Style guides sections) so the skills are findable by humans, not just by AI assistants.
