## Root Cause (Suspected)
`PyType_FromMetaclass` is a CPython 3.12-only symbol (added in [bpo-45459](https://bugs.python.org/issue45459)). The `_rocisa.abi3.so` extension in the `rocm-sdk-libraries-gfx950-dcgpu==7.14.0a20260529` nightly wheel was compiled using this symbol, but the `.abi3.so` filename implies Stable ABI compatibility down to Python 3.2. When the test runner loads this `.so` under Python 3.11, the dynamic linker fails to resolve `PyType_FromMetaclass`.

The fix must be applied in the hipBLASLt TensileLite `rocisa` source (in `ROCm/rocm-libraries`): either lower the `Py_LIMITED_API` target and eliminate the 3.12+ symbol usage, or rename the extension to target only Python 3.12+.
