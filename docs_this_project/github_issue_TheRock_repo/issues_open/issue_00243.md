### Problem Description

Hi @stellaraccident , while iterating on Dockerfiles and patches like https://github.com/scottt/rocm-TheRock/blob/gfx1151/external-builds/pytorch/patches/v2.6.0/pytorch/hipified/0004-requirements.txt-setuptools-75.9-to-support-python_a.patch, I often find a rebuild of the whole ROCm stack is triggered on `docker build` due to the way the whole TheRock source tree is bind mounted in Dockerfiles e.g. in pytorch_dev_ubuntu_24.04.Dockerfile: 

```
RUN --mount=type=bind,target=/therock/src,rw bash /build_rocm.sh "$AMDGPU_TARGETS"

( .... snipped ...)

WORKDIR /therock/pytorch
RUN python3 setup.py build --cmake-only
```

See Docker build cache and bind mount discussion in https://github.com/moby/buildkit/issues/1903

Would you be amenable to reorganizing the source tree to avoid this? For example by moving git submodules in TheRock into a directory like `src/` and only mounting that in Dockerfiles? The general idea is to separate the Dockerfiles, patches and scripts that developers are frequently hacking on from the large source tree provided to `docker build` through the bind mount.
