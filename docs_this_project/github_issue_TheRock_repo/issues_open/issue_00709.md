### Problem Description

When building MIOpen as part of TheRock on Windows with the CMake option `-DBUILD_TESTING=OFF`, critical symbols (e.g., from `miopen::TensorDescriptor`) are not exported from `MIOpen.dll`. This leads to linker errors (undefined symbols) when trying to link against `MIOpen.lib` (e.g., when building `MIOpenDriver.exe`).

The root cause appears to be in `ml-libs/MIOpen/src/include/miopen/config.hpp`, where the internal symbol export macro `MIOPEN_INTERNALS_EXPORT` is effectively defined as empty when `MIOPEN_BUILD_TESTING` is not defined:

```cpp
#ifdef MIOPEN_BUILD_TESTING
#include <miopen/export_internals.h>
#else
#define MIOPEN_INTERNALS_EXPORT
#endif
```

This behavior means that the public API of the MIOpen library becomes unusable if tests are not being built, which is not the expected behavior for a library. Core library symbols should be exported regardless of whether tests are enabled.
