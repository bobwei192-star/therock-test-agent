## Summary

On **native Windows**, the `gfx110X-all` nightly PyTorch wheel installs cleanly and the ROCm runtime initializes and enumerates the GPU, but **any** device operation on the gfx1100 discrete GPU fails with `hipErrorInvalidImage` ("device kernel image is invalid"). No GPU compute runs.

## Environment

| | |
|---|---|
| dGPU (target) | AMD Radeon RX 7900 XT — **gfx1100** (Navi 31, RDNA3), 20 GB |
| iGPU (also present) | Ryzen 7 7800X3D integrated — gfx1036; enumerates as `cuda:0` (cf. #3392) |
| OS | Windows 11, build 10.0.26200.8524 |
| AMD driver | Adrenalin 32.0.31007.5012 (gaming) |
| torch | `2.10.0+rocm7.13.0a20260508` |
| ROCm | `rocm-sdk-libraries-gfx110X-all 7.13.0a20260508` |
| Python | 3.12 (venv) |
| Install | `pip install --index-url https://rocm.nightlies.amd.com/v2/gfx110X-all/ torch` |

## Repro

```python
import torch
print(torch.__version__, torch.cuda.is_available())
# 2.10.0+rocm7.13.0a20260508 True
print([torch.cuda.get_device_name(i) for i in range(torch.cuda.device_count())])
# ['AMD Radeon(TM) Graphics', 'AMD Radeon RX 7900 XT']

a = torch.randn(2048, 2048, device="cuda:1")   # the gfx1100 dGPU
b = torch.randn(2048, 2048, device="cuda:1")
(a @ b).sum()
```

```
torch.AcceleratorError: HIP error: device kernel image is invalid
Search for `hipErrorInvalidImage' in https://rocm.docs.amd.com/projects/HIP/en/latest/index.html ...
```

The runtime initializes and both GPUs enumerate, but the first GPU op fails. `cuda:0` (the gfx1036 iGPU) failing is expected — a gfx110X wheel has no kernels for it — but `cuda:1` (gfx1100) failing is unexpected, since gfx1100 is in the gfx110X family this wheel targets.
