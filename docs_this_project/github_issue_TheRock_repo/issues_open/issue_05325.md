Two large binary size increases slipped through undetected recently:

* https://github.com/ROCm/TheRock/pull/4881#issuecomment-4390767318 (+400MB)
* https://github.com/ROCm/TheRock/pull/4419#discussion_r3262592363 (+3GB)

We can't rely on just author/reviewer vigilance here. Need tooling support.

## Implementation ideas

A job like https://github.com/ROCm/TheRock/blob/main/.github/workflows/test_artifacts_structure.yml could diff sizes against a baseline.

How should we handle failures?
* Post a comment on the PR?
* Post to workflow summary?
* Fail a workflow job (unless an override is set?)
  * We don't want to block legitimate increases or necessitate excessive CI retriggering

We could also add a nightly/release job that compares against the previous nightly. That would at least catch issues within ~days instead of ~weeks. Could build with github actions workflows or a simple cron job / scheduled agent task.
