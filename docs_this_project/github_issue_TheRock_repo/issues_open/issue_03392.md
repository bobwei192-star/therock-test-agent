### Problem Description

After installing the ROCm .whl packages built by TheRock, I noticed that my discrete RX 9000 series GPU is enumerated as cuda:1, while the integrated GPU (iGPU) from my AMD 9000 series CPU is enumerated as cuda:0.

The problem is that even if I explicitly select cuda:1, the RX 9000 still does not work correctly in practice.
Some error appear like this:
```
torch.AcceleratorError: HIP error: device kernel image is invalid
Search for `hipErrorInvalidImage' in https://docs.nvidia.com/cuda/cuda-runtime-api/group__HIPRT__TYPES.html for more information.
HIP kernel errors might be asynchronously reported at some other API call, so the stacktrace below might be incorrect.
For debugging consider passing AMD_SERIALIZE_KERNEL=3
Compile with `TORCH_USE_HIP_DSA` to enable device-side assertions
```

So currently, the only workaround I found is setting:
```
CUDA_VISIBLE_DEVICES=1
```

Is there any other recommended way to control or fix the device ordering, so that the discrete GPU is properly recognized as cuda:0, or at least to completely hide the AMD iGPU, without relying on CUDA_VISIBLE_DEVICES?

However, when using the ROCm version bundled with the official AMD GPU windows driver 26.1.1 AI bundle, this issue does not occur. In that setup, the AMD iGPU does not appear as a CUDA device at all, and only the discrete GPU is visible.
