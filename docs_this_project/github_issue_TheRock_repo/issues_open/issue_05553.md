### Problem Description

I am trying to trigger a TunableOp search for GemmTunableOp_Half_NT(nt_65536_32768_384_ld_65536_65536_65536) very specifically, because this op seems to cause a ring comp_1.0.1 timeout for my PC (and hence desktop failure) after some time. However, when I try to replicate it with offline tuning, it couldn't create the matrices needed and instead searched for GemmTunableOp_Half_NT(nt_65536_32768_384_ld_65536_32768_65536).

### Operating System

Linux Mint 21.3 (Virginia)

### CPU

AMD Ryzen 5 5500

### GPU

AMD Radeon VII

### ROCm Version

ROCm 7.14.0a2026053

### ROCm Component

_No response_

### Steps to Reproduce

Follow in instructions in [RELEASES.md](https://github.com/ROCm/TheRock/issues/RELEASES.md) for Multi-arch releases, install multi-arch ROCm and PyTorch Python packages.
In this case, the specific one I installed is: rocm-7.14.0a20260531 and amd-torch-device-gfx906 2.11.0+rocm7.14.0a20260531

Create a csv file called "gemm_to_tune.csv"
Put this into it:
`GemmTunableOp_Half_NT,nt_65536_32768_384_ld_65536_65536_65536`

Then run the following Python script:
```
import torch
import os

os.environ["PYTORCH_TUNABLEOP_ENABLED"] = "1"
os.environ["PYTORCH_TUNABLEOP_TUNING"] = "1"
os.environ["PYTORCH_TUNABLEOP_VERBOSE"] = "3"
os.environ["PYTORCH_TUNABLEOP_HIPBLASLT_ENABLED"] = "0" # Have to be false for gfx906 or rocblaslt error
os.environ["PYTORCH_TUNABLEOP_FILENAME"] = "tuned_gemm.csv"

torch.cuda.tunable.tune_gemm_in_file('gemm_to_tune.csv')
```
