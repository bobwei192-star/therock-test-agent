## Description
The JAX and PyTorch test workflows currently use the Git ref in the job name to make GitHub Actions runs easier to identify. This works well when the ref is a version tag (e.g., rocm-jaxlib-v0.9.1), but may become less readable when workflows are triggered from custom branches or commit SHAs.

As support for building and testing from arbitrary branches and commits is expanded, using the full ref in the job name could clutter the GitHub Actions UI and make it more difficult to quickly identify jobs.

## Motivation
Keep workflow job names concise and easy to scan while still providing enough information to identify the build being tested.
