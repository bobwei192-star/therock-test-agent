### Problem Description

With the following test code (the official SD3 example), we can observe an issue on same gfx1151 device: it achieves better inference performance with 1 GB of dedicated memory for SD3 inference than with 4 GB of dedicated memory.


**4G Dedicate + TheRock 7.11.0a20260114**
````
pipeline round 1 time : 6.410919300000003
pipeline round 2 time : 6.393833900000004
pipeline round 3 time : 6.265024099999991
pipeline round 4 time : 6.267292600000005
````
 
**1G Dedicate + TheRock 7.11.0a20260114**
````
pipeline round 1 time : 5.874122200000002
pipeline round 2 time : 5.786160199999998
pipeline round 3 time : 5.83410649999999
pipeline round 4 time : 5.857788200000016
````

### Steps to Reproduce

Step 1. Use following command to setup test env:
```
python -m venv .venv
.venv\Scripts\activate

pip install --index-url https://rocm.nightlies.amd.com/v2/gfx1151/ torch==2.9.1+rocm7.11.0a20260114
pip install diffusers==0.35.1 accelerate==1.9.0  transformers==4.54.1 rich protobuf==6.33.4 sentencepiece
```

Step 2. Use following code for test
Check the pipeline time per round.
```
import os
os.environ["TORCH_ROCM_AOTRITON_ENABLE_EXPERIMENTAL"] = "1"
import torch
import time
from diffusers import StableDiffusion3Pipeline

pipe = StableDiffusion3Pipeline.from_pretrained("stabilityai/stable-diffusion-3-medium-diffusers", torch_dtype=torch.float16)
pipe = pipe.to("cuda")

for i in range(5):
    torch.cuda.synchronize()
    t0 = time.perf_counter()

    image = pipe(
        "A cat holding a sign that says hello world",
        negative_prompt="",
        num_inference_steps=28,
        guidance_scale=1.0,
        width=512,
        height=512,
    ).images[0]

    torch.cuda.synchronize()
    print(f"pipeline round {i} time : {time.perf_counter() - t0}")

    image.save("a.png")
```

Step 3. Adjust the dedicate memory with Adrenalin `Performance->Tuning` from 4G > 1G
