## Problem

The current structure of yaml and python files between `TheRock`, `rocm-libraries`, and `rocm-systems` results in  repetitive code for tasks like environment setup, build/test matrix configuration, and fanning out jobs for different OS. A change to CI workflows to one repo sometimes leads to the same change being missed in another repo.

CI workflows in the super repos also reference `TheRock` through a pinned commit. Changes made on `TheRock` repo do not test those changes against the CI workflows in the super repos, until a PR is raised on the super repo to update that commit SHA. Meanwhile, `TheRock` references the super repos through submodules — creating a circle.

## Tasks

This issue will track the work required across the different repos.
- [ ] Move the unique code from the super repo CI workflows and scripts to existing workflows and scripts in TheRock.
  - The behaviour of the code in TheRock should not change for commits and pull requests in TheRock, but exhibit the new behaviour when triggered by commits and pull requests from the super repos.
- [ ] CI yaml files on the super repos should have minimal logic, and call the yaml files on TheRock's develop branch.
  - No python scripts related to TheRock should exist on the super repos.
- [ ] commits and pull requests in TheRock should add checks to validate workflow sequence from the super repos.
