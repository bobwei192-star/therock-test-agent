- Tests take ~10-15 seconds to complete (when running concurrently); timeout is set to 120 sec so it's not a "timeout being too small" issue - increasing this doesn't help with the flake

## Bisection results

This was a tricky one. Bisecting in fusilli repo pointed to the commit where IREE  and TheRock were bumped from 11/12 -> 12/1 nightly. 

Ran bisection along IREE nightlies but it didn't repro the stall/segfault on `3.10.0rc20251201`. So I figured it had to do with TheRock bump from `7.10.0a20251112` to `7.11.0a20251201` which also happened in the same commit. To confirm I ran an ablation experiment separating the two:

Run with IREE@3.9.0rc20251112 TheRock@7.11.0a20251201 (i.e. bump only TheRock): GPU stalls / segfaults in 12/20 runs
https://github.com/iree-org/fusilli/actions/runs/19991537909

Run with IREE@3.10.0rc20251201 TheRock@7.10.0a20251112 (i.e. bump only IREE): NO stall/segfault - 20/20 pass
https://github.com/iree-org/fusilli/actions/runs/19991551345

So this is really boiling down to something something GPU driver / HIP issue in TheRock and needs further triaging.
