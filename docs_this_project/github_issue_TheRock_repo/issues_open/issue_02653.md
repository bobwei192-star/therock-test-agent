# #2653 - buildctl.py: expunging a disabled component leaves it disabled permanently

### Steps to reproduce

The state of your build directory is that a full build has occurred and all components are enabled.  `find . -name stage.prebuilt` finds no files.

1. `./build_tools/buildctl.py enable --build-dir=/therock/output/build rocSOLVER`

The output of buildctl.py shows that it has disabled all components other than rocSOLVER by creating stage.prebuilt files.  Check it for hipCUB specifically:

```
$ ls -l build/math-libs/hipCUB/stage.prebuilt
-rw-r--r-- 1 eble redacted 0 Dec 18 13:57 build/math-libs/hipCUB/stage.prebuilt
```

2. Imagine that you are new to TheRock, weeks have passed since step 1, you have forgotten that building hipCUB is disabled, and now you are trying to figure out why hipCUB is not rebuilding when you touch a file, so you decide to try expunging it.
3. `ninja -v -C /therock/output/build hipCUB+expunge`
4. `cmake -B /therock/output/build -GNinja .`
5. `ninja -v -C /therock/output/build hipCUB`

hipCUB is still disabled. (Find stage.prebuilt with same command as above.)

5. Imagine that someone reminds you of buildctl.py, so you decide to try re-enabling everything.
6. `./build_tools/buildctl.py enable --build-dir=/therock/output/build`

The output of buildctl.py does _not_ show hipCUB.  hipCUB is still disabled.  (Find stage.prebuilt with same command as above.)

### Workaround

Remove hipCUB's stage.prebuilt file by hand.
