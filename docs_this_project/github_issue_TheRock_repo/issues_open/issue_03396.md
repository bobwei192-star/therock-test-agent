# #3396 - Add knob to disable population of dist/rocm and enable this in CI

**State**: open  
**Author**: stellaraccident  
**Created**: 2026-02-12T18:21:52Z  
**Updated**: 2026-02-12T18:21:52Z  
**URL**: https://github.com/ROCm/TheRock/issues/3396  

---

We are starting to see accidental dependence on the dist/rocm directory from sub-project builds (as in #3358). This directory is populated lazily for humans consuming the build as a unified whole but it cannot be relied on intra-build. When it is needed, it is often due to legacy assumptions about product layout that need to be eliminated.

We should add a knob that either disables population of dist/rocm or move it to another CI-specific location and enable that knob on the CI only. This would cause accidental dependence on this direction to be a build error while still allowing the convenience for local use.
