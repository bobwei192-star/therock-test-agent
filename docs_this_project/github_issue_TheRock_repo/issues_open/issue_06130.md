## Description

The `github_job` field in the TheRock manifest currently contains a static value (`build_stage`) instead of the actual GitHub Actions job name.

### Current behavior

```json
{
  "the_rock_commit": "f1e3fc2311fde45bf3e17b0e3349200be9caa4b4",
  "github_job": "build_stage",
  "github_run_id": "28066115163",
  "rocm_package_version": "7.14.0a20260624",
  "rocm_version": "7.14.0"
}
```

### Expected behavior

Populate `github_job` with the actual GitHub Actions job name (or job identifier) associated with the workflow execution, rather than a static value such as `build_stage`.

This will ensure the manifest accurately represents the workflow execution details and improve traceability between generated artifacts and their corresponding GitHub Actions jobs.
