### Problem Description

I am trying to build ROCm and HIP from source using the TheRock repository, to experiment with and modify the source code. However, I consistently encounter out-of-memory (OOM) errors during the build process.
A typical error message is:
`Out of memory: Killed process 2442957 (clang-22) total-vm:2384408kB, anon-rss:1707148kB, file-rss:1104kB, shmem-rss:0kB, UID:1001 pgtables:4616kB oom_score_adj:0`
My system has 24 GB of RAM, but the build frequently triggers the OOM killer. In many cases, I am forced to build with -j1 to make progress, and even then, the issue still occurs intermittently.

- Is this level of memory usage expected when building TheRock/ROCm from source?
- Are there recommended build configurations or flags to reduce peak memory consumption?

As a fallback, I am considering building the project on a server with significantly more RAM. In that case:
- Is it feasible to build the binaries on another server, with much more RAM available, move them to my local machine, and use them reliably?

Any guidance or best practices would be greatly appreciated.
I really appreciate any help you can provide.
