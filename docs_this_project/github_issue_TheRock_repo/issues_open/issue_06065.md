For Linux, we currently run "component tests" (rocblas, rocfft, hipblaslt, miopen, etc.) only on Ubuntu. We should expand to other distros like RHEL 8.10 to ensure that there aren't any test failures unique to these other distros.

## Step 1: choose which containers to cover, create test images

We can create a prioritized list from the supported distros list: https://rocm.docs.amd.com/projects/install-on-linux/en/latest/reference/system-requirements.html#supported-distributions . As of 2026-06-23 that list is:

Ubuntu 22.04.5 | 5.15 [GA], 6.8 [HWE] | 2.35

In many cases, building on manylinux should hopefully be sufficient for distributing safely across these distros. If/when there are test failures we should root cause why that did not hold.

## Step 3: figure out how often to run these tests

We can start by manually triggering, then we should integrate into continuous CI/CD. For example, we could:

* Run on every PR (allowable via opt-in?), being careful to not 2x/3x/etc. our load on test runners
* Run as part of a new scheduled job (using Quartz?), e.g. "CI weekly" or "CI nightly"
* Run as part of nightly releases with another dimension to this matrix: https://github.com/ROCm/TheRock/blob/8484067b044134dc2a4b1f3fe45c415b2a631a17/.github/workflows/multi_arch_release_linux.yml#L172-L198
