### Suggestion Description

Currently, the ROCK docker image [[Ref](https://github.com/ROCm/TheRock/pkgs/container/therock_pytorch_dev_ubuntu_24_04_gfx942)] does not set USE_KINETO=1 when compiling PyTorch, which prevents torch profiler from correctly capturing GPU kernel information. Although the image supports the roc-profile tools, our framework (e.g., vllm) primarily relies on torch profiler.

Could you please set USE_KINETO=1 when building the image, or provide an additional docker image version with USE_KINETO=1 alongside the current USE_KINETO=0 version? This would greatly help us with kernel-level profiling in our workflow. Thanks
