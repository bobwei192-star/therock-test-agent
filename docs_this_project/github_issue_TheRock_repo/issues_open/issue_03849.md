# #3849 - Document that smrev needs to be deleted when building TheRock with a custom sub-repo commit

**State**: open  
**Author**: adelejjeh  
**Created**: 2026-03-09T17:19:05Z  
**Updated**: 2026-03-09T17:19:05Z  
**URL**: https://github.com/ROCm/TheRock/issues/3849  

---

Currently, if building TheRock with a custom llvm commit (i.e., use fetch_sources.py to first fetch the current state of the sub-repos, then manually update the amd-llvm repo commit/branch and then proceed to run the configure and compile steps), and if there is a .smrev file that got generated by the intitial fetch_sources.py run, then clang --version will return an **incorrect** commit hash for llvm due to the CMakeLists.txt setting the commit hash using the smrev. This should be either be fixed, or at the very least documented that users need to manually delete the smrev before re-configuring when they manually checkout a custom commit in the LLVM repository.
