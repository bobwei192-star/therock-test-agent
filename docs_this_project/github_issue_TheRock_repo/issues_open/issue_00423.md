# #423 - Split CI into multiple phases

With #386, artifacts from a previous phase can be reused. The CI doesn't profit from this yet, as the `build_linux_packages.yml` workflow builds in a single step

https://github.com/ROCm/TheRock/blob/8681b3e6a7ae47d35ddcfd52902dd56cd07d49fb/.github/workflows/build_linux_packages.yml#L82

and artifacts are uploaded in a subsequent step. The workflow needs to be refactored accordingly. This includes the following

- [ ] Instead of creating an index HTML file in a single step, the index must be created one (e.g. when building the target neutral artifacts). As there can be multiple steps to build target dependent artifacts, any subsequent steps need to append to the index.
- [ ] Split the build step to build the target neutral first and target dependent artifacts in subsequent step.
- [ ] Instead of uploading artifacts in a single step, this needs to be either moved to the corresponding step where targets get build _or_ should be even moved to the build system / script.
- [ ] Build logs should be uploaded after they where created, see #371.
- [ ] Move building target dependent artifacts to separate jobs thus that target neutral artifacts can build _once_ instead of multiple times.
