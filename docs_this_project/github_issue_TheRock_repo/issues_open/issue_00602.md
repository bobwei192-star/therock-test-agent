On Windows, there is special case code to put the Tensile library in bin/. This should not be special cased and should have the same layout between Windows and Posix (so that we can share kernel packages).

This was worked around in #597.

We should implement the library probing logic from hipblaslt and eliminate the weird hard-coding it has on the name of the shared library. See the implementation in hipBLASLt: https://github.com/ROCm/hipBLASLt/commit/50d9abdddc0fda4b00313dfbc19e435d3b3a3500

In the new rocBLAS build system, it should not be hard-coded to put the library in bin. If the legacy Windows build needs that, there should be an option.
