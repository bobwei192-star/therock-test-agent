Title: 
The current release packaging upload/download scripts in (TheRock/tree/main/build_tools/packaging) contain S3 handling and release layout logic that overlaps with functionality already implemented in:

build_tools/github_actions/publish_rocm_to_release_buckets.py

This script already provides reusable abstractions for:

* S3 storage backends
* Storage locations
* Release bucket configuration
* Release directory layout handling
* Artifact publishing flows for tarballs, Python packages, and native packages

As a follow-up improvement, investigate whether the packaging scripts can reuse:

* _therock_utils.storage_backend
* _therock_utils.storage_location
* Existing release bucket/layout helpers
* Shared copy/upload logic from publish_rocm_to_release_buckets.py

Goals:

* Reduce duplicated S3 handling logic
* Keep release layout behavior consistent across scripts
* Simplify future maintenance
* Improve testability and code reuse

This issue tracks evaluating and implementing that refactor where practical.
