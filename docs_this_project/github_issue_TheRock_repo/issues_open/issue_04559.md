#4045 and RFC0010 introduces source fetching to test scripts, as the goal is to have the test scripts reside with their code.

This adds a few minutes of cloning to test jobs, when these jobs really just need to clone the test scripts.

Modify `fetch_sources.py` to support new arguments that feed in what paths to sparse-checkout for specific submodules, something like `--submodule-sparse-checkout rocm-libraries:tests`. Investigations on how well `git` supports this functionality will be required.
