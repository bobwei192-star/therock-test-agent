# #968 - [pytorch] Use of CMAKE_INSTALL_RPATH_USE_LINK_PATH is not the right default for rocm wheels.

**State**: open  
**Author**: stellaraccident  
**Created**: 2025-07-03T00:17:35Z  
**Updated**: 2025-07-03T00:17:35Z  
**URL**: https://github.com/ROCm/TheRock/issues/968  

---

When using build_prod_wheels.py, the resulting pytorch shared libraries will include an absolute path to the venv/site-libs directory where the rocm libraries are installed. When running on the same host, this will mask library pre-loading issues, and when running on other hosts, it is sloppy/a nuisance.

Given that this is manipulated globally in the pytorch build in a way that is hard to guarantee, we should probably just add a correction to the list of post build steps to make wheels ready for distribution. From the `torch/lib` directory, this is being simulated in interactive builds by running `for i in *.so; do patchelf --set-rpath '$ORIGIN' $i; done`. A proper solution should only remove the absolute RPATHs. The result should look like this:
