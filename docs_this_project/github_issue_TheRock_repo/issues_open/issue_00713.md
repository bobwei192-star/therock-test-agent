rocBLAS links with hipBLASLt on Linux and can be directed to use either Tensile or hipBLASLt at runtime using environment variables like [`ROCBLAS_USE_HIPBLASLT`](https://rocm.docs.amd.com/projects/rocBLAS/en/latest/how-to/what-is-rocblas.html#use-of-tensile-and-hipblaslt) but hard codes `BUILD_WITH_HIPBLASLT=OFF` on Windows which can lead to surprising performance degradations.

See `BUILD_WITH_HIPBLASLT` in https://github.com/ROCm/rocBLAS/blob/develop/CMakeLists.txt:

```
if( WIN32 OR NOT BUILD_WITH_TENSILE OR NOT BUILD_SHARED_LIBS )
  set( BUILD_WITH_HIPBLASLT OFF )
endif()
```

I've verified that `librocblas.so` as built by TheRock on Linux links with hipblaslt but not on Windows:

```
$ eu-readelf --dynamic librocblas.so.5.0 | grep NEEDED.*hipblaslt
  NEEDED            Shared library: [libhipblaslt.so.0]
```
