### Problem Description

There are some items missing when doing a pip install of ROCm (`rocm[libraries,devel,device-gfx1100,profiler]==7.14.0a20260619"`) that are available when doing a package manager install (`amdrocm-core-sdk7.14-gfx1100`).

These missing items are as follows:

1. rdc (ROCm Data Center Tools)
2. libhipcxx
3. elfio

### Operating System

Ubuntu 24.04

### CPU

N/A

### GPU

Navi31XTX

### ROCm Version

7.14 June 18 nightly

### ROCm Component

rdc, rocprofiler

### Steps to Reproduce

Pip install repro
```
sudo apt install libatomic1 libquadmath0

# Install Python 3.12 or 3.13.
sudo apt install python3.12 python3.12-venv

# Setup Python virtual environment.
python3.12 -m venv .venv
source .venv/bin/activate

# Install ROCm wheels packages.
python -m pip install --pre --index-url https://rocm.nightlies.amd.com/whl-multi-arch/ "rocm[libraries,devel,device-gfx1100,profiler]==7.14.0a20260619"
rocm-sdk init
```

Missing items
```
(.venv) root@b718d8451a74:/mnt# find .venv -name rdc -o -name elfio -o -name libhipcxx
(.venv) root@b718d8451a74:/mnt#
```

Whereas, if you do a package manager install, you will find these 3 items installed

```
## Register ROCm packages ##
sudo tee /etc/apt/sources.list.d/rocm.list << EOF
deb [trusted=yes] https://rocm.nightlies.amd.com/packages-multi-arch/deb/20260619-27797822902 stable main
EOF

sudo apt update
sudo apt install amdrocm-core-sdk7.14-gfx1100
```

Items not missing

```
root@b1795aae83f8:/mnt# find /opt/rocm -name rdc -o -name elfio -o -name libhipcxx
/opt/rocm/core-7.14/share/rdc
/opt/rocm/core-7.14/share/doc/rdc
/opt/rocm/core-7.14/share/elfio
/opt/rocm/core-7.14/include/rdc
/opt/rocm/core-7.14/include/elfio
/opt/rocm/core-7.14/include/libhipcxx
/opt/rocm/core-7.14/libexec/rdc
/opt/rocm/core-7.14/lib/rdc
/opt/rocm/core-7.14/lib/cmake/rdc
/opt/rocm/core-7.14/lib/cmake/libhipcxx
```
