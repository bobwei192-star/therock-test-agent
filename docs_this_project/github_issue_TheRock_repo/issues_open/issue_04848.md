# #4848 - Embedded HIP offload information from target-specific `dist/rocm` binaries is not present

**Author**: kcossett-amd

**Updated**: 2026-06-09T23:01:23Z  
**Labels**: build issue

**Context**: 

I'm moving the `rocprofiler-systems` example suite to be built as a standalone that is target-specific.
The example binaries are used in our tests. Some of them are built with HIP offload information, and our test logic checks for that using `llvm-objdump --offloading`.

**Problem**: 

When `rocprofiler-systems-examples` is built through the target-specific portable distribution flow, the binaries under:
`output-linux-portable/build/dist/rocm`
no longer report offloading information with `llvm-objdump --offloading`.
For example:

```
(.venv) rocm@rocm:~/therock-branches/test-framework-update$ output-linux-portable/build/dist/rocm/lib/llvm/bin/llvm-objdump --offloading output-linux-portable/build/dist/rocm/share/rocprofiler-systems/examples/transpose

output-linux-portable/build/dist/rocm/share/rocprofiler-systems/examples/transpose:     file format elf64-x86-64
```

But when I check under `output-linux-portable/build/profiler/rocprofiler-systems-examples/...`
```
(.venv) rocm@rocm:~/therock-branches/test-framework-update$ sudo output-linux-portable/build/dist/rocm/lib/llvm/bin/llvm-objdump --offloading output-linux-portable/build/profiler/rocprofiler-systems-examples/build/transpose/transpose

output-linux-portable/build/profiler/rocprofiler-systems-examples/build/transpose/transpose:    file format elf64-x86-64
Extracting offload bundle: output-linux-portable/build/profiler/rocprofiler-systems-examples/build/transpose/transpose.0.host-x86_64-unknown-linux-gnu-
Extracting offload bundle: output-linux-portable/build/profiler/rocprofiler-systems-examples/build/transpose/transpose.0.hipv4-amdgcn-amd-amdhsa--gfx1100
Extracting offload bundle: output-linux-portable/build/profiler/rocprofiler-systems-examples/build/transpose/transpose.0.hipv4-amdgcn-amd-amdhsa--gfx1101
Extracting offload bundle: output-linux-portable/build/profiler/rocprofiler-systems-examples/build/transpose/transpose.0.hipv4-amdgcn-amd-amdhsa--gfx1102
Extracting offload bundle: output-linux-portable/build/profiler/rocprofiler-systems-examples/build/transpose/transpose.0.hipv4-amdgcn-amd-amdhsa--gfx1103
```

The expected information is present. Our testing suite depends on knowing this information for certain binaries, else the test is skipped.

**Note**:
 - Discovered while working on: https://github.com/ROCm/TheRock/pull/4758 
   - For now, I have built the standalone examples as `TARGET_NEUTRAL` and things work fine with that. 
 - This appears specific to HIP-built binaries such as `transpose`; OpenMP-built binaries using `amdclang/amdflang` do not show the same behavior in the final `dist/rocm` tree.
