With ROCm/rocm-libraries@f5a0867cd7bbf62b2f0166b61b69e03224502aa3, hipBLASLt will error if trying to build for not fully supported architectures:
```
[hipBLASLt configure] CMake Error at cmake/TensileliteSupportedArchitectures.cmake:79 (message):
[hipBLASLt configure]   Unsupported GPU target: gfx1102
[hipBLASLt configure] 
[hipBLASLt configure]   Supported targets are:
[hipBLASLt configure]   gfx908;gfx90a;gfx942;gfx950;gfx1100;gfx1101;gfx1103;gfx1150;gfx1151;gfx1200;gfx1201;gfx908:xnack+;gfx908:xnack-;gfx90a:xnack+;gfx90a:xnack-
[hipBLASLt configure] Call Stack (most recent call first):
[hipBLASLt configure]   CMakeLists.txt:116 (tensilelite_validate_gpu_targets)
```

[Note](https://github.com/ROCm/rocm-libraries/blob/78d9061bdb904f6f33dc03c6f90e0afc2a1cfe60/projects/hipblaslt/next-cmake/cmake/TensileliteSupportedArchitectures.cmake#L31-L34) from CMake file:
```
# gfx10XX architectures (e.g., gfx1010, gfx1011, gfx1030, etc...) are technically supported by tensilelite,
# but are NOT included in the default "all" build in hipBLASLt. This is because "extops" builds are not supported 
# for legacy devices. Including these architectures would result in build failures or incomplete feature support.
```
