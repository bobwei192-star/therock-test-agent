PyTorch has a dynamic list of disabled tests that can be updated by filing github issues and _not just by modifying code_. See the dashboard at https://hud.pytorch.org/disabled.

This file: https://github.com/pytorch/pytorch/blob/main/tools/stats/import_test_stats.py loads the underlying list of disabled tests from https://ossci-metrics.s3.amazonaws.com/disabled-tests-condensed.json.

Note that the list is for "latest pytorch" and I don't think historical versions for prior releases are maintained.

We should

- [ ] Diff our https://github.com/ROCm/TheRock/tree/main/external-builds/pytorch/skip_tests against that list
- [ ] Explore how we can combine the skip_tests file with the upstream disabled-tests-condensed.json file to generate a consolidated skip list for the PR run
- [ ] Reformat skip_tests files to match the upstream file?
- [ ] If there are test cases in both then maybe we can use the upstream list [in addition to our own extra filtering]
