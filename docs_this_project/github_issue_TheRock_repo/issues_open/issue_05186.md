# #5186 - Evaluate reusing existing Lambda S3 event parsing utilities for PEP503 index generation

The current PEP503 Lambda wrapper implements its own S3 event parsing and record extraction logic.

There is already similar functionality available in:

`TheRock-Infra/tools/artifact_lambda_handler/s3_index_handler.py`

including reusable helpers such as:

* `_extract_s3_records()`
* event normalization for S3/SQS/EventBridge payloads
* directory/prefix extraction patterns

This issue tracks evaluating whether the existing logic can be shared or reused by the PEP503 Lambda handler to reduce duplication and keep event handling behavior consistent across Lambda index generation flows.

Potential areas to evaluate:

* Reusing `_extract_s3_records()`
* Supporting additional event wrapper formats (SQS/EventBridge)
* Consolidating common Lambda S3 event utilities into shared helpers/modules
* Reducing duplicated event parsing logic across handlers

This is raised as a follow up to actions as part of https://github.com/ROCm/TheRock-Infra/pull/129
Not required for the current PR since the current implementation only targets direct S3 event payloads.
