I'd like to optimize which jobs run for a few specific CI usage scenarios with some generic tooling:

1. PRs in https://github.com/ROCm/rocm-libraries should be able to use prebuilt artifacts for the "foundation" and "compiler-runtime" stages
2. PRs in TheRock that only update [`build_tools/packaging/`](https://github.com/ROCm/TheRock/tree/main/build_tools/packaging/) should be able to use prebuilt artifacts for _all_ stages
3. PRs in TheRock that only update [`external-builds/`](https://github.com/ROCm/TheRock/tree/main/external-builds) should be able to use prebuilt rocm python packages

More generally for the CMake build system, the build topology tracks "source sets", which we can use to compute the list of affected and unaffected artifact groups for a given set of modified paths: https://github.com/ROCm/TheRock/blob/fb66bc4e17443e0b3fd28d357fa6773eaa0f247b/BUILD_TOPOLOGY.toml#L74-L124 https://github.com/ROCm/TheRock/blob/fb66bc4e17443e0b3fd28d357fa6773eaa0f247b/BUILD_TOPOLOGY.toml#L248-L259

Building stage-aware prebuilt artifacts CI configuration would involve changes across these files:

* https://github.com/ROCm/TheRock/blob/main/build_tools/github_actions/configure_ci.py
  * Compute which stages to use prebuilt artifacts for
* https://github.com/ROCm/TheRock/blob/main/build_tools/find_artifacts_for_commit.py and https://github.com/ROCm/TheRock/blob/main/build_tools/find_latest_artifacts.py
  * Find a workflow run to pull baseline artifacts from
* https://github.com/ROCm/TheRock/blob/main/.github/workflows/setup.yml
  * Propagate the selection of which stages should use prebuilt artifacts (and from which workflow run id)
* https://github.com/ROCm/TheRock/blob/main/.github/workflows/multi_arch_ci.yml
  * Propagate the selection of which stages should use prebuilt artifacts (and from which workflow run id)
* https://github.com/ROCm/TheRock/blob/main/.github/workflows/multi_arch_build_portable_linux.yml
  * Skip or no-op workflow jobs if using prebuilt artifacts for that stage
