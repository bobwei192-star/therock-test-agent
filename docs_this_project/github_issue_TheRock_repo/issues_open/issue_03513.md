### Environment

* OS: Ubuntu 25.04
* Compiler: GCC 15 (defaults to -std=gnu23)
* Python: 3.13
* GPU target: gfx1151 (AMD Strix Halo iGPU / Radeon 8060S)
* Terminal: Kitty
* Build method: standard cmake + ninja with .venv

### Instructions to reproduce:

On a fresh install of Ubuntu 25.10:
```
cmake -B build -GNinja . -DTHEROCK_AMDGPU_FAMILIES=gfx1151 -DCMAKE_C_COMPILER_LAUNCHER=ccache -DCMAKE_CXX_COMPILER_LAUNCHER=ccache
cmake --build build
```

File: `math-libs/BLAS/CMakeLists.txt`

Error:
```
ImportError: cannot import name 'setup' from 'setuptools' (unknown location)
```
rocBLAS creates a virtualenv with --system-site-packages. On Python 3.13 Ubuntu, the system setuptools stub doesn't expose setup. The venv needs pip/setuptools upgraded first.

Fix: Pass `-DTENSILE_VENV_UPGRADE_PIP=ON` in `CMAKE_ARGS` for rocBLAS. This triggers `pip install --upgrade pip packaging setuptools` in the new venv before Tensile is installed.
