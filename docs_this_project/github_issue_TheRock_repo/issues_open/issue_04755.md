When running `offload-arch.exe` with no existing `amdgpu64_*.dll` on the system and with `amdgpu64_7.dll` existing under `TheRock/therock-build/bin`, it still cannot find the amdhip64.dll with error: `Failed to load amdhip64.dll: Can't open: The specified module could not be found`.

When amdgpu64_7.dll did exist, offload-arch.exe worked fine! but it did produce the wrong architecture of gfx1150

This occurs on a Windows strix-halo machine (gfx1151)

Repro steps (on powershell admin):
```
# remove existing amdgpu64_*.dll files
$ takeown /f C:\Windows\System32\amdhip64_7.dll; icacls C:\Windows\System32\amdhip64_7.dll /grant Administrators:F; Remove-Item C:\Windows\System32\amdhip64_7.dll -Force
$ takeown /f C:\Windows\System32\amdhip64_6.dll; icacls C:\Windows\System32\amdhip64_6.dll /grant Administrators:F; Remove-Item C:\Windows\System32\amdhip64_6.dll -Force
$ cd ~; git clone https://github.com/ROCm/TheRock.git; cd TheRock; pip install boto3; python build_tools/install_rocm_from_artifacts.py --run-id 24434318178 --amdgpu-family gfx1151 --base-only
$ cd therock-build/lib/llvm/bin
$ ./offload-arch.exe --verbose
```
