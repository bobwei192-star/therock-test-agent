# #56 - Rework how rccl pulls in msccl and enable

**State**: open  
**Author**: stellaraccident  
**Created**: 2025-02-04T02:32:30Z  
**Updated**: 2025-02-04T02:32:30Z  
**URL**: https://github.com/ROCm/TheRock/issues/56  

---

Currently, we set:

```
      -DENABLE_MSCCLPP=OFF
      -DENABLE_MSCCL_KERNEL=OFF
```

in the rccl build. This is because the MSCCL download itself is flaky, failing on CI sometimes, there shouldn't be a sub-build like this, and when it does work, it is not propagating find_package flags needed to locate the correct ROCM-under-construction (so it fails based on machine state).
