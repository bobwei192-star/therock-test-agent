## Parent
ROCm ASAN Packaging for TheRock #5914 

## Goal
Extend the ASAN release pipeline to produce ASAN-instrumented RPM and DEB packages, placed in the folder structure defined in [RFC0012](https://github.com/ROCm/TheRock/blob/main/docs/rfcs/RFC0012-Repo-Structure.md).

## Tasks
- [ ] Trigger the ASAN release pipeline on a weekly guidance
- [ ] The packages install location is /opt/rocm/core-7.14
- [ ] Upload ASAN tarball to the nightly artifact store.
- [ ] Place ASAN packages in the dedicated folder defined in RFC0012 (confirm exact path with RFC)
- [ ] Confirm artifact appears in the nightly index and is downloadable
- [ ] Define a minimal smoke test: extract tarball, set `LD_PRELOAD` to the ASAN runtime, run a simple ROCm compute program. And confirm test detects memory voilation
- [ ] Document how to install ASAN packages with expected `LD_PRELOAD` environment setup

## Acceptance Criteria
- Every successful ASAN release build produces an ASAN native linux packages with the standard name
- The ASAN tarball is accessible from the nightly artifact store
- Standard (non-ASAN) nightly packages is unaffected
- Proper documentation with install instructions separately for asan packages.
