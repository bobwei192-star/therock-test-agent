### Problem Description

Can't build hipBLASLt. 
1. The "Target VGA" Problem (Architecture Mismatch) - Even though I have an RX 6800 (gfx1030), the build system's auto-detection is failing because I'm building in a Windows environment where the ROCm tools can't "see" the hardware directly through the compiler. It defaults to its internal "fail-safe" architecture, which is gfx1100 (RDNA 3 / 7000 series). It creates a mismatch that crashes the configuration.
2. The OpenMP - This is the "Fatal Error" that actually stops the build. I'm using gfortran (from w64devkit) to compile the Fortran parts of the math library, but the build script is trying to link it against Microsoft Visual Studio’s OpenMP library (libomp.lib). These two are incompatible. Gfortran needs libgomp, while Visual Studio provides libomp. Because the script finds the "wrong" one first, it fails the sanity check and quits.

Now I will try again with Visual Studio 2022 Community Edition(with Intel oneAPI Base & HPC Toolkits).
