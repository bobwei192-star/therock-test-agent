# #2720 - Long build time of rccl for gfx1151 is due to gfx906 fallback

Since rccl is disabled for gfx1151, this means that it is being built with an empty GPU_TARGETS. This is one of those projects which applies defaults (rocm-cmake seems to do one and then if it didn't the clang frontend does as well) to gfx906. For some reason, gfx906 build times regressed badly with recent compilers and/or rccl, and that is the cause of the build time slowness.

It is doubly unfortunate because this is completely wrong and is basically wasted energy on something that will never get used.
