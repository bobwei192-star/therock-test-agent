### Problem Description

### Summary
On native Windows (PyTorch ROCm / TheRock stack), Qwen3-TTS inference works but the **decoder is extremely slow** (overall ~10–12x real-time). MIOpen logs show the **caller provides workspace=0**, which triggers repeated `IsEnoughWorkspace` warnings (e.g., `GemmFwdRest` requires workspace but receives ptr/size 0) and leads to a **naive convolution solver path**.

Additionally, MIOpen environment variables appear **not reflected in logs/behavior**:
- `MIOPEN_FIND_MODE=2`
- `MIOPEN_DEBUG_DISABLE_FIND_DB=1`
but the log still shows `findMode: NORMAL(1)` and `Find-db regenerating`.

### Observed
Performance:
- Overall RTF ~12.6x (see `qwen_stdout_cmd.log`).

MIOpen evidence (see `miopen_one_run_cmd.log`):
- (line 85) `FindConvFwdAlgorithm ... workspace = 0` → MIOpen receives workspace=0 from the caller.
- (lines 2708–2709) `GemmFwdRest ... workspace required ... provided ptr: 0 size: 0`
- (line 39 and lines 2704–2707) `ConvDirectNaiveConvFwd` selected / registered; `workspace_sz = 0` (naive path).
- (lines 98–99) `Find-db regenerating` and `findMode: NORMAL(1)` even with `MIOPEN_FIND_MODE=2` and `MIOPEN_DEBUG_DISABLE_FIND_DB=1`.
- (line 83) MIOpenDriver command lines logged (from `MIOPEN_ENABLE_LOGGING_CMD=1`), but `MIOpenDriver.exe` is not present in the pip-based environment (see `repro_miopen.bat` + `miopen_driver_stderr.log`).

### Expected
- When workspace is available, the caller should pass a non-zero workspace so MIOpen can select faster convolution solvers.
- `MIOPEN_DEBUG_DISABLE_FIND_DB=1` should disable FindDb per MIOpen documentation, but logs still show `Find-db regenerating`.

### Steps to Reproduce

1. Native Windows 11 system with AMD GPU (RX 9060 XT).
2. Create venv (Python 3.12) and install ROCm Windows PyTorch wheels (torch/torchaudio/torchvision) matching ROCm 7.2.
3. Set environment variables:
   - MIOPEN_ENABLE_LOGGING=1
   - MIOPEN_LOG_LEVEL=6
   - MIOPEN_ENABLE_LOGGING_CMD=1
   - MIOPEN_FIND_MODE=2
   - MIOPEN_DEBUG_DISABLE_FIND_DB=1
   - (optional) QWEN_RUNS=1, QWEN_WARMUP_RUNS=0, short test text via QWEN_TEST_TEXT
4. Run:
   - `python test_qwen_rocm.py`
5. Observe:
   - Very slow decoder / high RTF in stdout
   - MIOpen log shows workspace=0, GemmFwdRest workspace warnings, naive solver selection
