### Problem Description

Setting up a new linux ubuntu 24.04 system 
Installed AMD base driver from ub distro
Installed Rock nightly  7.13.0a20260430
Tried to execute hip-test - bit extract and observed kernel hangs.

To overcome the issue - we installed the legacy mainline rocm build 17098
Kernel test passed. 
then reinstalled rock nightly 7.13.0a20260430 and the kernel test passed as well 

so there are some missing base driver pieces in the Rock nightly that used to get installed in the legacy rocm.
Rock needs to test this scenario

### Steps to Reproduce

1. setup fresh ub 24.04
2. Install amd base drivers from distro
3. Install Rock nightly build
4. build a simple kernel test for the system gfxId - bit_extract from https://github.com/ROCm/rocm-systems/tree/develop/projects/hip-tests/samples/0_Intro/bit_extract
5. Execute the test. observe the kernel hang.
6. Install the legacy rocm mainline build
7. Execute the test. observe the test pass correctly
8. Reinstall the same Rock nightly
9. Execute the test. observe the test pass correctly


this shows that Rock nightly is missing some base driver items that the legacy rocm used to install
