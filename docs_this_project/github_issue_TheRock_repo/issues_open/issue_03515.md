### Environment

* OS: Ubuntu 25.04
* Compiler: GCC 15 (defaults to -std=gnu23)
* Python: 3.13
* GPU target: gfx1151 (AMD Strix Halo iGPU / Radeon 8060S)
* Terminal: Kitty
* Build method: standard cmake + ninja with .venv

### Instructions to reproduce:

On a fresh install of Ubuntu 25.10:
```
cmake -B build -GNinja . -DTHEROCK_AMDGPU_FAMILIES=gfx1151 -DCMAKE_C_COMPILER_LAUNCHER=ccache -DCMAKE_CXX_COMPILER_LAUNCHER=ccache
cmake --build build
```

File: `rocm-systems/projects/rocprofiler-systems/cmake/PAPI.cmake`
Also: `rocm-systems/projects/rocprofiler-systems/external/papi/src/high-level/papi_hl.c`

GCC 15 defaults to C23, where () in a function declaration means (void) (no parameters). This causes two failures in PAPI:

Error 1 — conflicting declaration:
```
papi_hl.c:409:12: error: conflicting types for '_internal_hl_read_user_events';
have 'int(const char *)'
```
Line 170 declares static int _internal_hl_read_user_events(); (no params) but line 409 defines it with const char *user_events.

Fix: Update the forward declaration at line 170 to static int _internal_hl_read_user_events(const char *user_events);

Error 2 — incompatible pointer types:
```
papi_vector.c:221:36: error: assignment to 'int (*)(papi_mdi_t *)' from
incompatible pointer type 'int (*)(void)'
```
vec_int_dummy() uses old-style C "any params" declaration, which GCC 15 C23 treats as incompatible when assigned to typed function pointers.

Fix: Add -std=gnu17 -Wno-incompatible-pointer-types to all CFLAGS= entries in the PAPI ExternalProject in PAPI.cmake.
