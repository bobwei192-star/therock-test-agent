Split from #3477 

### Environment

* OS: Ubuntu 25.04
* Compiler: GCC 15 (defaults to -std=gnu23)
* Python: 3.13
* GPU target: gfx1151 (AMD Strix Halo iGPU / Radeon 8060S)
* Terminal: Kitty
* Build method: standard cmake + ninja with .venv

### Instructions to reproduce:

On a fresh install of Ubuntu 25.10:
```
cmake -B build -GNinja . -DTHEROCK_AMDGPU_FAMILIES=gfx1151 -DCMAKE_C_COMPILER_LAUNCHER=ccache -DCMAKE_CXX_COMPILER_LAUNCHER=ccache
cmake --build build
```

GCC 15 no longer provides headers like `<cstdint>` or `<algorithm>` transitively. This caused errors in several bundled libraries:

| File | Error | Fix |
|------|-------|-----|
| `rocm-systems/projects/rocprofiler-sdk/external/yaml-cpp/src/emitterutils.cpp` | `unknown type name 'uint16_t'` | Add `#include <cstdint>` |
| `rocm-systems/projects/rocprofiler-sdk/external/elfio/elfio/elf_types.hpp` | `unknown type name 'uint16_t'` | Add `#include <cstdint>` |
| `rocm-systems/projects/rocprofiler-systems/source/lib/logger/logger.hpp` | `'any_of' is not a member of 'std'` | Add `#include <algorithm>` |
| `rocm-systems/projects/rocprofiler-systems/external/dyninst/common/h/dyntypes.h` | `'uint32_t' does not name a type` | Add `#include <stdint.h>` |
| `rocm-libraries/projects/hipdnn/plugins/miopen_legacy_plugin/MiopenContainer.hpp` | `unknown type name 'uint32_t'` | Add `#include <cstdint>` |
