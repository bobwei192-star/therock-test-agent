### Description of errors

The page to install releases: https://github.com/ROCm/TheRock/blob/main/RELEASES.md only has commands for unix and are not all compatible with windows.

Windows users would have to translate the commands to be compatible with their terminal/cmd.

For example:

```
python -m venv .venv 
source .venv/bin/activate
```
works for unix, but the second line will generate errors for windows users as 'source' is not a known command and venvs on windows are activated under '.venv\scripts\activate' 

Not sure how to approach this since the install for torch/rocm are fine, maybe another box for windows commands on venv creation? A dedicated windows install page?

This is just a small issue but from a larger perspective, the docs should be easier to follow for users that aren't familiar with Linux in order to appear more inviting to a wider audience of Windows users. For example, https://github.com/ROCm/TheRock/pull/2494 is a good step in making the process easier for Windows users.
