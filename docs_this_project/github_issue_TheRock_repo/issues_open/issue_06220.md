# #6220 - [Build]: amd-llvm doesn't build with a recent clang

### Problem Description

TheRock's `amd-llvm` fails to build when the C/C++ compiler is a recent Clang (22 in my case) due to mismatches around precompiled headers. The same clang builds upstream LLVM just fine.


### Operating System

Ubuntu 24.04 LTS (Noble Numbat)

### CPU

AMD EPYC 9655 96-Core Processor

### GPU

AMD Instinct MI350X

### ROCm Version

HEAD

### ROCm Component

llvm-project

### Steps to Reproduce

## Steps to reproduce

```
cmake -G Ninja -B build-no-ccache -DTHEROCK_AMDGPU_FAMILIES=gfx950-dcgpu -DCMAKE_C_COMPILER=$HOME/.local/bin/clang-22 -DCMAKE_CXX_COMPILER=$HOME/.local/bin/clang++-22 -DTHEROCK_ENABLE_ALL=OFF -DTHEROCK_ENABLE_COMPILER=ON -DTHEROCK_ENABLE_CORE=ON -DTHEROCK_ENABLE_MATH_LIBS=OFF -DTHEROCK_ENABLE_COMM_LIBS=OFF -DTHEROCK_ENABLE_PROFILER=OFF -DTHEROCK_BUILD_TESTING=OFF
```

Replace `$HOME/.local/bin/clang-22` and `clang++-22` with any released Clang 22 you'd like, and this'll *probably* work with the `clang` you can get in `/opt/rocm`.

Do this in an environment that doesn't have an /opt/rocm or other TheRock install to avoid confounding issues.

Then, `ninja -C build-no-ccache`

TheRock compiles
