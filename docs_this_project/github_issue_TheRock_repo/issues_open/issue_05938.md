## Parent
ROCm ASAN Packaging for TheRock #5914 

## Goal
Extend the ROCm release pipeline to produce and publish an ASAN-instrumented tarball alongside the standard tarball.

## Tasks
- [ ] Trigger the ASAN release pipeline on a weekly guidance
- [ ] Apply the `-asan` naming convention: tarball must be named `therock-dist-linux-asan-<arch>-<version>.tar.gz`
- [ ] Upload ASAN tarball to the nightly artifact store.
- [ ] Confirm artifact appears in the nightly index and is downloadable
- [ ] Define a minimal smoke test: extract tarball, set `LD_PRELOAD` to the ASAN runtime, run a simple ROCm compute program. And confirm test detects memory voilation
- [ ] Document how to install ASAN tar ball with expected `LD_PRELOAD` environment setup

## Acceptance Criteria
- Every successful ASAN release build produces an ASAN tarball with the correct name
- The ASAN tarball is accessible from the nightly artifact store
- Standard (non-ASAN) nightly tarball is unaffected
- Proper documentation with install instructions separately for asan tar ball.
