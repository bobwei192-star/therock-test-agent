# #4658 - Add a CI configuration option to skip building comm-libs (RCCL)

**State**: open  
**Author**: ScottTodd  
**Created**: 2026-04-17T20:13:36Z  
**Updated**: 2026-05-18T17:28:38Z  
**Labels**: CICD

---

https://github.com/ROCm/TheRock/pull/4501 switched to building RCCL as a single target-neutral stage in multi-arch CI/CD to work around RCCL host code being target-specific when compiled separately for different GPU targets. This increased the CI critical path time by ~2h20m:

* Before: https://github.com/ROCm/TheRock/actions/runs/24549881155/job/71776796930 (1h03m single gpu family)
* After: https://github.com/ROCm/TheRock/actions/runs/24564363941/job/71826804417 (3h22m for `gfx906;gfx908;gfx90a;gfx942;gfx950`)
  * note that we also disabled RCCL for all Radeon targets in the same PR, now it only builds for Instinct targets

This time increase is going to substantially slow down development, so we can add a configuration option to the CI pipeline that allows skipping the comm-libs build stage and any tests depending on it.

This would be similar to the "use prebuilt artifacts" feature discussed on https://github.com/ROCm/TheRock/issues/3399, but rather than prune the _start_ of the workflow graph this would prune the _end_ of the workflow graph.

Implementation ideas:
* https://github.com/ROCm/TheRock/blob/main/.github/workflows/multi_arch_build_portable_linux.yml skip the build stage with a condition like `!contains(inputs.prebuilt_stages, 'comm-libs')` for "skip_stages"
* https://github.com/ROCm/TheRock/blob/main/build_tools/github_actions/configure_multi_arch_ci.py check for a new label and generate that "skip_stages" list
* * https://github.com/ROCm/TheRock/blob/main/build_tools/github_actions/fetch_test_configurations.py skip tests by filtering here somehow
* https://github.com/ROCm/TheRock/blob/main/docs/development/ci_behavior_manipulation.md document here
