# #181 - rocBLAS depends on a floating pin of Tensile that does not track any branch

This is a tracking issue to provide context on what we expect will be a situation that will persist for some time: rocBLAS includes a `tensile_tag.txt` file which contains a manual, project specific pin of Tensile. Further, it frequently includes interlocked changes that require this exact version. There is not any branch or tag correspondence other than this text file. By default, the rocBLAS project dynamically fetches sources from GitHub for Tensile and uses those. We override to use a local repository (which lets us edit/dev and also apply patches) and need to synchronize to this exact commit.

When doing submodule bumps, we need to manually reset the math-libs/BLAS/Tensile submodule pin to the same commit hash as is in `tensile_tag.txt`. Failure to do so results in an untested/unsynchronized build that may fail.
