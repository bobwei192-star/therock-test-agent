### Environment

* OS: Ubuntu 25.04
* Compiler: GCC 15 (defaults to -std=gnu23)
* Python: 3.13
* GPU target: gfx1151 (AMD Strix Halo iGPU / Radeon 8060S)
* Terminal: Kitty
* Build method: standard cmake + ninja with .venv

### Instructions to reproduce:

On a fresh install of Ubuntu 25.10:
```
cmake -B build -GNinja . -DTHEROCK_AMDGPU_FAMILIES=gfx1151 -DCMAKE_C_COMPILER_LAUNCHER=ccache -DCMAKE_CXX_COMPILER_LAUNCHER=ccache
cmake --build build
```

File: `math-libs/BLAS/CMakeLists.txt`

Error:
```
stl_algobase.h:1191:22: error: invalid operands to binary expression
('const std::pair<const int, std::shared_ptr<std::variant<...>>>' and same type)
```
GCC 15's std::equal requires operator== for the element type. The large std::variant in rocRoller's KernelGraphTest.cpp doesn't provide one. The library itself builds fine.

Fix: Pass `-DROCROLLER_BUILD_TESTING=OFF` in CMAKE_ARGS for rocRoller until upstream adds the missing operator.
