Samples like https://github.com/ROCm/hip-tests/tree/amd-staging/samples/0_Intro/square use hipify like so:
```bash
$ Building without cmake
/opt/rocm/hip/bin/hipify-perl square.cu > square.cpp
/opt/rocm/hip/bin/hipcc  square.cpp -o square.out
/opt/rocm/hip/bin/hipcc -use-staticlib  square.cpp -o square.out.static
```

I wanted to follow those steps using our wheels, but I'm noticing that `hipify` (`hipify-clang` and `hipify-perl`) is missing from the python packages, at least on Windows but I think on Linux too. I see those executables/symlinks in tarballs under `bin/` but they are missing from `_rocm_sdk_core/bin`.

I'm looking through commits like https://github.com/ROCm/TheRock/commit/56729146acda37c1a3176ba9b64a4f192286bfb3 to see where we might be missing tool registration / copying. We at least don't have console script / entry point wrappers in
* [`build_tools/packaging/python/templates/rocm-sdk-core/src/rocm_sdk_core/_cli.py`](https://github.com/ROCm/TheRock/blob/main/build_tools/packaging/python/templates/rocm-sdk-core/src/rocm_sdk_core/_cli.py)
* [`build_tools/packaging/python/templates/rocm-sdk-core/setup.py`](https://github.com/ROCm/TheRock/blob/main/build_tools/packaging/python/templates/rocm-sdk-core/setup.py)

In the case of hipify, I think PyTorch is not affected because it provides its own similarly named https://github.com/pytorch/pytorch/tree/main/torch/utils/hipify, which we invoke here: https://github.com/ROCm/TheRock/blob/7ea979da3603b8d30ccbfd2e92fe24e7d64dbf41/external-builds/pytorch/repo_management.py#L195-L200
