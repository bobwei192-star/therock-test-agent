### Summary
When PyTorch is installed through TheRock, `torch.cuda.get_device_properties(0).name` returns the generic string `"AMD Radeon Graphics"` instead of the SKU-specific marketing name (e.g., `"AMD Instinct MI355X"`). The same PyTorch installed from the upstream ROCm index (`--index-url https://download.pytorch.org/whl/rocm7.1`) correctly reports `"AMD Instinct MI355X"` on the same hardware, so this appears to be specific to the TheRock-shipped wheels.
Downstream tools that key off this string fall back to a generic arch path instead of the SKU-specific one.

### Reproducer
​```bash
python3 -c "import torch; props = torch.cuda.get_device_properties(0); print(props.name)"
​```
On an MI355X machine:
- Expected (and what `--index-url https://download.pytorch.org/whl/rocm7.1` gives): `AMD Instinct MI355X`
- Actual (PyTorch from TheRock): `AMD Radeon Graphics`

### Environment
- Hardware: AMD Instinct MI355X (gfx950) — likely reproduces on other Instinct SKUs as well
- PyTorch (broken): installed through TheRock (hit on the nightly benchmark runner)
- PyTorch (working baseline): `pip install torch --index-url https://download.pytorch.org/whl/rocm7.1`
