# PyTorch Docker image source rebuild leaves split device wheels incompatible

## Issue

Rebuilding PyTorch from the source tree included in the ROCm/UFB PyTorch 2.12 development image succeeds for `gfx942`, but the resulting Python environment is internally inconsistent. The source rebuild installs an editable torch package with version `2.12.0+git4212f7d`, while the image still has preinstalled split device wheels such as `amd-torch-device-*` and `amd-torchvision-device-*` that require the original torch wheel version, `torch==2.12.0+rocm7.14.0a20260602`. As a result, pip reports dependency conflicts immediately after the rebuild, and users can end up mixing a locally rebuilt editable torch with stale prebuilt device packages from the original image.

The source checkout in the image was:

```text
Remote: https://github.com/ROCm/pytorch.git
Branch: release/2.12
Commit: 4212f7d42f625a4eb6af2c1ca457b70e2ed8d154
```

The original installed torch wheel was:

```text
torch 2.12.0+rocm7.14.0a20260602
torch.version.git_version f51350b83ff0ee94e29d5559ee4d8a2aab04fbb7
```

After the source rebuild, pip installs:

```text
torch 2.12.0+git4212f7d
```

The rebuild itself reports success:

```text
Created wheel for torch: filename=torch-2.12.0+git4212f7d-0.editable-cp312-cp312-linux_x86_64.whl size=23247
Successfully built torch
Successfully installed torch-2.12.0+git4212f7d
```

Expected behavior: after rebuilding PyTorch from the source tree included in the development image, the Python environment should remain coherent. Either the matching `amd-torch-device-*` packages should also be rebuilt/replaced, the image should document that those split device packages must be removed before local source rebuilds, or there should be a supported rebuild path that does not leave stale split device packages pinned to the previous torch wheel version.

## Steps To Reproduce

Start from the UFB PyTorch image:

```bash
IMAGE=rocm/ufb-private:pytorch-2.12.0-ubuntu24.04-py3.12-nightlies-device-all-rocm7.14.0a20260602-f51350b83f
docker run --name pytorch-rebuild-gfx942-repro \
  --device=/dev/kfd \
  --device=/dev/dri \
  --ipc=host \
  --cap-add=SYS_PTRACE \
  --security-opt seccomp=unconfined \
  -it "${IMAGE}" /bin/bash
```

Inside the container:

```bash
cd /workspace/pytorch

git remote -v
git rev-parse HEAD

git submodule init
git submodule update --init --recursive

python tools/amd_build/build_amd.py

rm -rf build

/opt/venv/bin/python -m pip install -r requirements.txt

PYTORCH_ROCM_ARCH=gfx942 USE_ROCM=1 /opt/venv/bin/python setup.py develop 2>&1 | tee build_gfx942.log
```
