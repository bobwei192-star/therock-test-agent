  ## AI Analysis

  - `amd-smi` is the first entry in `LINUX_CONSOLE_SCRIPT_TESTS` in [`core_test.py`](https://github.com/ROCm/TheRock/blob/677eaab47f1de85516d932e6bb7b8728a6eea053/build_tools/packaging/python/templates/rocm/src/rocm_sdk/tests/core_test.py): `("amd-smi", [], "AMD-SMI", True)`.
  - `subprocess.check_output` is called with no `timeout=` argument, so a hung `amd-smi` process stalls the entire test run for the full 5-minute GHA watchdog window.
  - This is an infra-level issue: `amd-smi` hangs at the driver/GPU level on the gfx1153 runner. The analogous hang (`amd-smi static`) was seen on a different gfx1153 runner in [#4887](https://github.com/ROCm/TheRock/issues/4887) (now closed).
