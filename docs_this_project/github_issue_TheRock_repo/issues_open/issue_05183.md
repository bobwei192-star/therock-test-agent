### AI-Generated Customer Impact:

Developers and end-users utilizing hipBLASLt on specific RDNA GPU architectures (e.g., gfx1100, gfx1101, gfx1150, gfx1201) across Linux and Windows operating systems are affected. Due to insufficient automated testing coverage for these platforms, critical regressions and defects can be introduced into released software versions without being detected during the development cycle. This significantly increases the risk of customers encountering unexpected failures, incorrect computational results, or performance degradation when running their applications on these RDNA systems. Ultimately, this can lead to a degraded user experience, wasted development time, and potential blocking issues for their workloads, undermining confidence in the hipBLASLt library on these platforms.

## Summary

hipBLASLt (including TensileLite) is exercised in **Math-CI** (Jenkins) and **TheRock** (GitHub Actions). Several **RDNA** `{OS, gfx`} combinations are **claimed for release** but are **not covered per PR**, so regressions can merge before those platforms are tested.

This issue asks TheRock / CI owners to **close the gap**---especially on TheRock---so shipped RDNA targets are better aligned with automated PR validation.

**Internal tracking:** [AIHPBLAS-3426](https://amd-hub.atlassian.net/browse/AIHPBLAS-3426) --- *Additional RDNA Hipblaslt CI {platform, arch} machine requirements* (reporter: Alumbaugh, T.J.).

## Background

* TheRock ships hipBLASLt on platforms that may not run hipBLASLt/TensileLite tests on every PR, which increases the risk of defects reaching consumers of those builds.
* Math-CI and TheRock already cover a subset of CDNA/RDNA targets; the table below summarizes **current vs needed** coverage from the AMD-internal write-up.

## Current coverage (from AIHPBLAS-3426)

**TheRock CI:** Linux `gfx942`; Windows `gfx1151`. *(Linux `gfx950` is listed in TheRock but currently skipped.)*
