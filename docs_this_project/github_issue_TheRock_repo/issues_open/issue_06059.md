## Summary

The RCCL device kernel compilation for `gfx1012` fails with "unterminated conditional directive" errors in the hipified copy of `nccl_device/utility.h` placed at `build/comm-libs/rccl/build/include/nccl_device/utility.h`. Only one ninja step fails (`specialized_reducescatter_pat_ll_prod_u64_0_0_2.o` for `gfx1012`); all other gfx1012 device kernel steps complete, suggesting a stale or corrupt build artifact for that specific object rather than a systematic source defect.

## AI Analysis

- The rocm-systems bump (#6020) is the only change between the passing and failing run
- The bump adds new RCCL fault-injection transport files (`net_ib_ops_fault.cc/.h`) gated on `-DENABLE_FAULT_INJECTION` (which is set in the build)
- The source `utility.h` blob SHA is identical at both rocm-systems refs — the error is in the build-generated hipified copy
- A forced clean rebuild of the gfx1012 device kernels should confirm whether this is a transient stale-cache artifact or a deterministic bug
