# #2006 - [Issue]: Triton fails to locate HIP linker unless TRITON_HIP_LLD_PATH is explicitly set

### Problem Description

When running PyTorch 2.7.1 with Triton ([AOTInductor test](https://docs.pytorch.org/docs/stable/torch.compiler_aot_inductor.html#model-compilation)), Triton fails to compile kernels unless the environment variable `TRITON_HIP_LLD_PATH` is manually exported:

```bash
export TRITON_HIP_LLD_PATH="$(rocm-sdk path --root)/llvm/bin/ld.lld"  
```

Without this variable, runtime compilation fails with errors indicating that `ld.lld` cannot be found even though the ROCm toolchain is correctly installed and available in `$PATH`.
