# #5379 - Support WSL ROCDXG artifact uploads from fork PRs

**State**: open  
**Author**: jayhawk-commits  
**Created**: 2026-05-21T00:08:03Z  
**Updated**: 2026-05-21T00:08:03Z  
**URL**: https://github.com/ROCm/TheRock/issues/5379  

---

WSL ROCDXG jobs currently build successfully for fork PRs, but S3 artifact and log uploads fail because the WSL Windows runner path does not have credentials for `therock-ci-artifacts-external`.

PR #5378 changes the WSL ROCDXG workflow to keep fork PR build validation enabled while skipping S3 uploads from forks.

The existing Linux fork-upload path is documented in [`docs/development/s3_buckets.md`](https://github.com/ROCm/TheRock/blob/main/docs/development/s3_buckets.md): Linux containers mount runner baseline credentials at `/home/awsconfig/credentials.ini` and expose them through `AWS_SHARED_CREDENTIALS_FILE`.

The planned migration of Windows build runners, along with control of the Windows build image and corresponding WSL image, should give us a cleaner way to provide an equivalent fork-safe credential model for WSL ROCDXG uploads.
