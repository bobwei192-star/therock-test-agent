### Problem Description

At runtime MIOpen depends on having _dev artifacts (specifically, rocrand_dev) for its runtime kernel compilation. When specifying any subprojects using "fetch_artifact_args" only the _lib (and optionally _test) arguments are downloaded. Without these artifacts, a number of MIOpen tests can't be run in TheRock.
