We're considering this approach as one way to make multi-arch packages easier to migrate to. See https://gist.github.com/stellaraccident/0455bd97aff0813515aeb6981fc0f2ab#option-9-three-prong-strategy-proposed for background on this proposal and other options that were considered.

## Background

Packages in the current https://rocm.nightlies.amd.com/whl-multi-arch/ and https://rocm.nightlies.amd.com/whl-staging-multi-arch/ indices currently have no "default" device extra (see https://github.com/ROCm/TheRock/blob/main/RELEASES.md#installing-multi-arch-pytorch-python-packages). This results in `pip install torch` including zero device packages and not producing a functional install.

```
pip install torch              → host only (431 MB, NO device code)
pip install torch[gfx1100]     → host + gfx1100 + gfx11 (1.1 GB)
pip install torch[all]         → host + all 16 device wheels
```

Users of the standard `pip install` are already accustomed to instructions from https://pytorch.org/get-started/locally/ like `pip3 install torch torchvision --index-url https://download.pytorch.org/whl/rocm7.2` that works for all supported GPUs.

## Solution 1 - wheel variants

The long term fix that gets the best of all worlds will be https://wheelnext.dev/ with `uv pip install` and a wheel variant provider that leverages https://pypi.org/project/rocm-bootstrap/. This will result in `pip install torch` (no index needed?) automatically detecting a specific ROCm-capable GPU and installing device packages for just that GPU.

```
# Wheelnext-aware resolver (future uv): automatic
uv install torch                → detects gfx1100, installs host + gfx1100 + gfx11

# Standard pip with extras: manual selection
pip install torch[gfx1100] --index-url https://amd.../wheelnext/
```

## Solution 2 (this issue) - index with all extras by default

This would be **in addition to solution 1**

We can publish a separate index where `pip install --index-url={URL} torch` adds an automatic requirement of the `device-all` extra (e.g. `device-gfx942,device-gfx1100,device-gfx1151,...`).

```
pip install torch   →  torch (host, 431 MB)
                        + amd-torch-device-gfx900 (54 MB)
                        + amd-torch-device-gfx1100 (241 MB)
                        + ... (all 16 device wheels)
                        = ~5.5 GB total
```

Producing such an index could involve:
- [ ] Rewrite the `rocm`, `torch`, `torchvision`, etc. entry point packages to include the automatic requirement on `device-all`
- [ ] Push the rewritten wheels to a separate subfolder in the release buckets
- [ ] Generate an index for that subfolder using the rewritten wheels and links to the original wheels for all other packages (avoiding duplicating storage for those packages that don't need any edits)
- [ ] Updating documentation to users can install from either index. Recommend the "it just works but uses more data" path first, then the "this takes more effort but has much smaller download sizes" path second.
