# #1205 - [llvm-project] hipcc links against `libamdhip64.so`

**State**: open  
**Author**: marbre  
**Created**: 2025-08-05T23:03:22Z  
**Updated**: 2025-08-06T13:25:11Z  
**URL**: https://github.com/ROCm/TheRock/issues/1205  

---

hipcc adds `-lamdhip64` to the linker flags in [clang/lib/Driver/ToolChains/Linux.cpp#L986](https://github.com/ROCm/llvm-project/blob/7a4ef367ec9840dea1e2e6c9a35135d061f8e108/clang/lib/Driver/ToolChains/Linux.cpp#L986) when building for a GPU target and hence `libamdhip64.so` is linked. When installing ROCm via TheRock Python packages, symlinks cannot be created and therefore only `libamdhip64.so.7` is present.

A local fix is applied with #1204 but a proper fix is required upstream.
