### Environment

* OS: Ubuntu 25.04
* Compiler: GCC 15 (defaults to -std=gnu23)
* Python: 3.13
* GPU target: gfx1151 (AMD Strix Halo iGPU / Radeon 8060S)
* Terminal: Kitty
* Build method: standard cmake + ninja with .venv

### Instructions to reproduce:

On a fresh install of Ubuntu 25.10:
```
cmake -B build -GNinja . -DTHEROCK_AMDGPU_FAMILIES=gfx1151 -DCMAKE_C_COMPILER_LAUNCHER=ccache -DCMAKE_CXX_COMPILER_LAUNCHER=ccache
cmake --build build
```


Files:
```
math-libs/CMakeLists.txt
math-libs/BLAS/CMakeLists.txt
```

When building inside a .venv, tools like lit and pytest are in .venv/bin/ which is not on the system PATH used by subproject cmake processes. Two separate failures:

Error 1 (libhipcxx):
```
CMake Error at test/CMakeLists.txt:188 (find_program):
  Could not find libcudacxx_LIT using the following names: lit
```
Fix in math-libs/CMakeLists.txt: compute bin dir from Python3_EXECUTABLE and pass explicitly:

```
cmake_path(GET Python3_EXECUTABLE PARENT_PATH _libhipcxx_python_bin_dir)
# In CMAKE_ARGS:
"-DLLVM_EXTERNAL_LIT=${_libhipcxx_python_bin_dir}/lit"
"-Dlibcudacxx_LIT=${_libhipcxx_python_bin_dir}/lit"
```
Error 2 (rocRoller):
```
CMake Error at test/CMakeLists.txt:76 (find_package):
  Could not find a package configuration file provided by "Pytest"
```
Fix in math-libs/BLAS/CMakeLists.txt:
```
cmake_path(GET Python3_EXECUTABLE PARENT_PATH _rocroller_python_bin_dir)
cmake_path(GET _rocroller_python_bin_dir PARENT_PATH _rocroller_venv_dir)
# In CMAKE_ARGS:
"-DPytest_DIR=${_rocroller_venv_dir}/share/Pytest/cmake"
"-DPYTEST_EXECUTABLE=${_rocroller_python_bin_dir}/pytest"
```
