### Background

We currently have https://github.com/ROCm/TheRock/blob/main/external-builds/pytorch/setup_venv.sh that is used here: https://github.com/ROCm/TheRock/blob/f3077c1406d2d44a91ada8dc5dcffc9ec7e3193f/.github/workflows/test_pytorch_wheels.yml#L66-L81

I'd like to convert this to Python and add a few features from similar scripts used on other projects like https://github.com/iree-org/iree/blob/main/build_tools/pkgci/setup_venv.py.

There is some overlap with https://github.com/ROCm/TheRock/blob/main/build_tools/install_rocm_from_artifacts.py, https://github.com/ROCm/TheRock/blob/main/build_tools/fetch_artifacts.py, and https://github.com/ROCm/TheRock/blob/main/.github/actions/setup_test_environment/action.yml. We can build out more of a library of Python utility functions shared across these files as part of this work.

### Desired features

* Take a directory path to create a venv at
* Install requested packages (rocm and/or torch)
  * From a previous CI run
  * From the latest nightly
  * From the "latest" dev
  * From a specific version
  * From a specific commit
  * From a specific date

The tricky detail across all these scripts that this sort of tooling can help with is linking between git history, workflow run history, and release artifacts. We have some information available in each location that can build an association but with a solid model in a scripting language we can build more advanced tooling like https://github.com/iree-org/iree/tree/main/build_tools/pkgci/bisect
