## Problem

The full PyTorch test dispatch added in #4499 (`dispatch_pytorch_wheels_full_test` in `.github/workflows/multi_arch_release_linux_pytorch_wheels.yml`) is hardcoded to a single configuration:

- `amdgpu_family: gfx94X-dcgpu`
- `test_runs_on: linux-gfx942-1gpu-ossci-rocm` / `test_runs_on_multi_gpu: linux-gfx942-8gpu-ossci-rocm`
- `python_version: 3.12`
- Linux only

We should be able to run "full" tests for any supported GPU family and platform, not just Linux gfx942.

## Proposed approach

Move the matrix construction out of the workflow YAML into a Python script that generates the full-test matrix, similar to:

- what @geomin12 started in #5326, and
- the `configure_pytorch_test_matrix.py` pattern introduced for the standard suite in #5661.

This would let the matrix be derived from the supported family/platform policy rather than hardcoded rows, and keep the standard and full test paths consistent.
