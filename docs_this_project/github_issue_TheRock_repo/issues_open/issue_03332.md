PyTorch wheels will be optimized for WheelNext using split packages:

```
# Base wheel (host code)
torch_rocm-2.12.0-cp311-linux_x86_64.whl

# Architecture-specific wheels (device code)
torch_rocm_gfx110X-2.12.0-cp311-linux_x86_64.whl
torch_rocm_gfx94X-2.12.0-cp311-linux_x86_64.whl

# WheelNext will dynamically resolve dependencies based on detected GPU or explicit hints
```

Implementation plan:

1. Build PyTorch packages for individual GPU targets or families (i.e. using existing `PYTORCH_ROCM_ARCH` env var setting), either with the existing set of 2+4+4+1+... list or by fanning out to 1+1+1+1+1+...
2. Run a (not yet implemented) tool to repackage those packages into a base wheel (containing host code) and architecture-specific wheels (containing device code)
3. Construct pip/uv install commands to install the right wheels for a given configuration manually
4. Set up a wheelnext variant provider that dynamically installs the right wheels for a given configuration (with override env vars for usage on non-GPU machines and for testing)
