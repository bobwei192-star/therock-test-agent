It is currently very difficult to locate build errors when something goes wrong in the pytorch builds because the logs are flooded with warnings of the following:

```
warning: 'dllexport' attribute only applies to functions, variables, classes, and Objective-C interfaces [-Wignored-attributes]
```

This is due to the use of a macro that automatically applies the `dllexport` attribute and can't be easily fixed. Using `-Wno-ignored-attributes` to silence them is the simplest solution at the cost of potentially silencing one of these warnings when a different attribute is ignored that shouldn't be.

Another possibility would be to add some code to llvm to handle this differently so that this specific attribute warning may be silenced without ignoring other ignored attribute warnings.
