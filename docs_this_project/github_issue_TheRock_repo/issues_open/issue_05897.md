### Problem Description

A normal very small 400K parameters attention layer network training, at last epoch caused a well-known GPU reset in `gfx1103` ADM 780M iGPU.
Laptop RAM 64GB.

### Operating System

Ubuntu 24.04.2 LTS (Noble Numbat)

### CPU

AMD Ryzen 7 PRO 7840U w/ Radeon 780M Graphics

### GPU

AMD Ryzen 7 PRO 7840U w/ Radeon 780M Graphics

### ROCm Version

rocm7.14.0a20260519

### ROCm Component

_No response_

### Steps to Reproduce

Install and activate venv.
```
pip install --index-url https://rocm.nightlies.amd.com/whl-staging-multi-arch/ \
  torch[device-gfx1100] torchvision[device-gfx1100] torchaudio
```

Checks:
```
python -c "import torch; print(torch.__version__); print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0))"
2.12.0+rocm7.14.0a20260519
True
AMD Radeon 780M Graphics
```

```
python -c "import torch; print(torch.__version__); print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0))"
2.12.0+rocm7.14.0a20260519
True
AMD Radeon 780M Graphics
```

For test, just start training of very small GRU network:
`python train_gru.py `
First error happens here:
```
mynet.py", line 25, in __init__
n = torch.arange(N, device=device)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
torch.AcceleratorError: CUDA error: device kernel image is invalid Search for `hipErrorInvalidImage' in https://rocm.docs.amd.com/projects/HIP/en/latest/index.html for more information. CUDA kernel errors might be asynchronously reported at some other API call, so the stacktrace below might be incorrect.
For debugging consider passing AMD_SERIALIZE_KERNEL=3
Device-side assertion tracking was not enabled by user.
```
Solution to that is:
`export HSA_OVERRIDE_GFX_VERSION=11.0.0`

Then start training of a real small 2 layer regular attention layer (<400k params):
`python train.py --d-model 64 --num-heads 4 --window-size 20   --batch-size 512 --sequence-length 15 --epochs 20`

At epoch 19, I've clicked on already open Microsoft Edge Web App, then GPU reset came in, Gnome session logged out, and after logging in, it was a new session. Everything lost.
