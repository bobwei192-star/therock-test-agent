The issue seems to stem from a C++ type clash when HIPRTC attempts to compile code.

- **The Linux system told the compiler:** uint64_t is an unsigned long.
- **The MIOpen Header** (miopen_cstdint.hpp) explicitly hardcoded: uint64_t is an unsigned long long.
```
MIOpen(HIP): Warning [ParseAndLoadDb] File is unreadable: "/home/r/Applio/.venv/lib/python3.12/site-packages/_rocm_sdk_libraries/share/miopen/db/gfx1032_16.HIP.fdb.txt"
MIOpen(HIP): Error [Compile] 'hiprtcCompileProgram(prog.get(), c_options.size(), c_options.data())' MIOpenNeuron.cpp: HIPRTC_ERROR_COMPILATION (6)
MIOpen(HIP): Error [BuildHip] HIPRTC status = HIPRTC_ERROR_COMPILATION (6), source file: MIOpenNeuron.cpp
MIOpen(HIP): Warning [BuildHip] In file included from /tmp/comgr-6517-15-07b813/input/MIOpenNeuron.cpp:35:
/tmp/comgr-6517-15-07b813/include/miopen_cstdint.hpp:36:34: error: typedef redefinition with different types ('__hip_internal::uint64_t' (aka 'unsigned long long') vs '__uint64_t' (aka 'unsigned long'))

```
As a workaround, I was able to resolve this by disabling MIOpen and adding these lines to the startup script of the application.
```
# Disable MIOpen
import torch
torch.backends.cudnn.enabled = False
```
