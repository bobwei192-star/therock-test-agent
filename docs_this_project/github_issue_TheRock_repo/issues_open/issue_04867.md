Another follow-up task: the summary step in `upload_python_packages.py` should be updated to handle non-rocm-python packages. See on https://github.com/ROCm/TheRock/actions/runs/24915005026 how this wrote:

build rocm:
> [ROCm Python packages (kpack-split)](https://therock-ci-artifacts.s3.amazonaws.com/24915005026-windows/python/index.html)
> Replace `<YOUR_TARGET>` with your GPU target (e.g. `gfx942`, `gfx1201`):
> ```bash
> pip install rocm[libraries,devel,device-<YOUR_TARGET>] --pre ^
>     --find-links=https://therock-ci-artifacts.s3.amazonaws.com/24915005026-windows/python/index.html
> ```

build torch:
> ROCm Python packages (multi-arch build)
> 
> Per-family indexes:
> - [.kpack_staging](https://therock-ci-artifacts.s3.amazonaws.com/24915005026-windows/python/.kpack_staging/index.html)
> 
> ```bash
> pip install rocm[libraries,devel] --pre ^
>     --find-links=https://therock-ci-artifacts.s3.amazonaws.com/24915005026-windows/python/.kpack_staging/index.html
> ```

(then we'll have a separate job for testing torch that will write its own instructions :P)
