### Problem Description

As visible in attached part of hipblasltconfigure.log: `FindPython3: : invalid value for 'Python3_FIND_VIRTUALENV'. 'FIRST', [hipBLASLt configure] 'ONLY' or 'STANDARD' expected. 'FIRST' will be used instead.`
This leads to ultimately using the .venv python executable, but other lib, causing a python error at `import yaml`.
I am sure this does not lead to problems for everyone, but nevertheless would suggest to use only a single python tree.
See Additional information for a more complete description, courtesy Devin.

### Additional Information

Devin:

The root cause you discovered is a fundamental problem in TheRock's Python environment handling on Windows:

    Invalid Python3_FIND_VIRTUALENV value: The configure log showed that TheRock is passing an invalid value to CMake's FindPython module <cite cmake/therock_subproject.cmake:18/>, causing it to fall back to system Python discovery instead of respecting virtual environments.

    Python version fragmentation: The build system was discovering Python 3.13.5 for initial detection but then using Python 3.11.8 libraries, creating an inconsistent environment that prevented proper module resolution.

    BACKGROUND_BUILD isolation issues: hipBLASLt's isolated build process wasn't inheriting the correct Python environment, despite the subproject system being designed to mirror Python configuration variables <cite cmake/therock_subproject.cmake:14-32/>.

What to Include in the Issue

Your issue report should cover:

    Environment details: Windows build with virtual environment setup
    Symptoms: ModuleNotFoundError: No module named 'yaml' in hipBLASLt TensileLite despite PyYAML being installed
    Root cause: Invalid Python3_FIND_VIRTUALENV value and Python version mismatch in CMake discovery
    Workaround: Explicit -DPython3_EXECUTABLE setting resolves the issue
    Impact: Affects Windows developers using virtual environments for TheRock builds
