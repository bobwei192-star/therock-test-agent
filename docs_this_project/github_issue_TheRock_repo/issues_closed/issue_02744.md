On my recently refreshed Ubuntu 25.04 box, I'm getting include errors in amdsmi trying to find DRM headers.

amdsmi has a complicated relationship to drm and tries to delay load it, etc. We really should rip all of this out for TheRock builds where it is always available as it just causes complexity.

I'm not sure how this was ever working because it doesn't appear to ever add the include directories properly.

As a quick hack to keep going, I added this to the amdsmi CMakeLists:

```
# HACK: DO NOT SUBMIT
if(THEROCK_BINARY_DIR)
    include_directories(${THEROCK_BINARY_DIR}/third-party/sysdeps/linux/libdrm/build/dist/lib/rocm_sysdeps/include)
endif()
```
