### Problem Description

When allocating ~16 GB in total I receive an error complaining about running out of memory.
```
torch.OutOfMemoryError: HIP out of memory. Tried to allocate 16.00 GiB. GPU 0 has a total capacity of 15.24 GiB of which 15.09 GiB is free. Of the allocated memory 0 bytes is allocated by PyTorch, and 0 bytes is reserved by PyTorch but unallocated. If reserved but unallocated memory is large try setting PYTORCH_HIP_ALLOC_CONF=expandable_segments:True to avoid fragmentation.  See documentation for Memory Management  (https://pytorch.org/docs/stable/notes/cuda.html#environment-variables)
```

The error message indicates that there are only 15.24 GB of memory available, but the machine should have 96 GB of memory available to the GPU.

dmesg shows this output, so I think I have the settings right (?):
```
[    4.972801] amdgpu 0000:c5:00.0: amdgpu: VRAM: 98304M 0x0000008000000000 - 0x00000097FFFFFFFF (98304M used)
[    4.972828] [drm] Detected VRAM RAM=98304M, BAR=98304M
[    4.973019] [drm] amdgpu: 98304M of VRAM memory ready
```

The machine is an GMKTek Evo-X2 with 128GB unified RAM. I.e. gfx1151.

Please let me know if there is more information that I can provide.
