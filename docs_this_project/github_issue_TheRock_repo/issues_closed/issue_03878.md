Similar to https://github.com/ROCm/TheRock/issues/3291, we should build and later also test JAX as part of our CI workflows, so issues like https://github.com/ROCm/TheRock/issues/3876 are caught earlier.

We should mostly be able to follow the patterns used for building pytorch (see https://github.com/ROCm/TheRock/pull/3303), though note that currently JAX builds from ROCm tarballs and NOT python packages, and we do not build tarballs outside of the narrow set of release workflows where JAX is currently integrated.
