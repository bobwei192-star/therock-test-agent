## Background

Our existing CI/CD workflows are a first pass at what is needed to build, test, and release ROCm. Many of these workflows currently hardcode the configurations that they will run:

* https://github.com/ROCm/TheRock/blob/36ff686e6467dd43db040f8f158bb109969582e6/.github/workflows/build_linux_packages.yml#L72
* https://github.com/ROCm/TheRock/blob/36ff686e6467dd43db040f8f158bb109969582e6/.github/workflows/build_windows_packages.yml#L95
* https://github.com/ROCm/TheRock/blob/36ff686e6467dd43db040f8f158bb109969582e6/.github/workflows/portable_linux_package_matrix.yml#L42-L50
* https://github.com/ROCm/TheRock/blob/36ff686e6467dd43db040f8f158bb109969582e6/.github/workflows/publish_pytorch_dev_docker.yml#L5-L12

These all need to be expanded to cover the full range of device targets (e.g. gfx942, gfx1201) and other configurations (e.g. Linux, Windows, Python 3.11, Python 3.12, etc.) that we support.

## Implementation ideas

We'll need to define a matrix of jobs _somehow_. Some available options are:

1. One workflow file per configuration, possibly with these workflows being generated from a template.
2. Each workflow manages its own matrix
3. Each workflow takes inputs and we have root workflows (like the current https://github.com/ROCm/TheRock/blob/main/.github/workflows/ci.yml) generate a matrix that calls those individual workflows with specific inputs

I option 3, since that lets us use workflow_dispatch and all sorts of other triggering to run a build/test/release of a specific configuration without needing code changes.

We could also have the matrix that is generated in root workflows be sourced from a configuration script/job (see https://github.com/ROCm/TheRock/issues/199), rather than have it be defined directly in a workflow yaml file.

> [!NOTE]
> For test workflows, we will need to pick a `runs-on` with a compatible GPU for each target that we test (use gfx942 builds on mi300 runners, etc.) 

## Concrete examples

https://github.com/ROCm/TheRock/pull/193 started to restructure one workflow like this:

```yml
name: Build Linux Packages
on:
  workflow_dispatch:
    inputs:
      package_version:
        type: string
        default: ADHOCBUILD
      amdgpu_families:
        type: string
        default: gfx94X-dcgpu
```

```diff
-           -DTHEROCK_AMDGPU_FAMILIES=gfx110X-dgpu \
+           -DTHEROCK_AMDGPU_FAMILIES=${amdgpu_families} \
```
