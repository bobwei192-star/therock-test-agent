# Summary

PyTorch has many [linalg operators](https://docs.pytorch.org/docs/stable/linalg.html) which use [Magma and/or cusolver/hipsolver](https://docs.pytorch.org/docs/stable/backends.html#torch.backends.cuda.preferred_linalg_library) as a backend. For ROCm, there are some operators which are not yet supported by hipsolver because of [missing APIs](https://github.com/ROCm/frameworks-internal/issues/15409#issuecomment-3921977606). Meanwhile, there may also be performance gaps for other linalg operators between Magma and hipsolver, which make the former preferable as a backend, until the latter catches up.

# PyTorch operators requiring Magma

As of [Feb 2026](https://github.com/ROCm/frameworks-internal/issues/15409#issuecomment-3931472306), the following PyTorch operators require Magma for ROCm due to missing hipsolver support:
* `torch.linalg.eig`: [AISOLVE-69](https://amd-hub.atlassian.net/browse/AISOLVE-69)
* `torch.linalg.eigh`: [AISOLVE-71](https://amd-hub.atlassian.net/browse/AISOLVE-71), [AISOLVE-70](https://amd-hub.atlassian.net/browse/AISOLVE-70)
* `torch.linalg.ldl_solve`: [AISOLVE-72](https://amd-hub.atlassian.net/browse/AISOLVE-72)

# Magma build considerations

* [Dependencies on ROCm packages](https://github.com/pytorch/pytorch/blob/8cc5b51e5a4b46af36f44f1ee5660c5780eba478/.ci/magma-rocm/package_files/build.sh#L19):

1. HIP runtime
2. hipblas
3. hipsparse

This means Magma package needs to be rebuilt anytime the underlying dependency libraries change


* Build time: [~2h](https://github.com/pytorch/pytorch/actions/runs/21270846391) for 14 gfx archs on a `linux.2xlarge` (maps to an AWS [`m5.2x.large`](https://aws.amazon.com/ec2/instance-types/general-purpose/?refid=6f46081e-bd62-4fb3-9702-51a445ef2b67) instance)

# Options to integrate Magma builds in theRock

1. Integrate Magma builds into theRock build flow to build with other ROCm packages - use manifest to determine when to rebuild Magma
2. Build Magma before building PyTorch
3. Integrate Magma build into the PyTorch build flow
