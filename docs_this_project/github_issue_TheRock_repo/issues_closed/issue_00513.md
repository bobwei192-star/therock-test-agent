For hipSOLVER tests, and likely other BLAS library tests the tests need various .dll files. Running tests without all requisite .dll files will throw errors like:

![Image](https://github.com/user-attachments/assets/60be1cef-2255-425a-828d-e574b4ad7a39)

> hipsolver-test.exe - System Error
>
> The code execution cannot proceed because openblas.dll was not found. Reinstalling the program may fix this problem.

Each project has some code tries during the build phase to copy .dll files into place from assorted sources. For hipSOLVER that is https://github.com/ROCm/hipSOLVER/blob/bb8408f52a166c5ce587b204c61a51ae44bc1daa/clients/gtest/CMakeLists.txt#L156-L180:

```cmake
if(WIN32)
  file(GLOB third_party_dlls
    LIST_DIRECTORIES OFF
    CONFIGURE_DEPENDS
    ${cblas_DIR}/bin/*.dll
    ${GTest_DIR}/bin/*.dll
    $ENV{rocblas_DIR}/bin/*.dll
    $ENV{rocsolver_DIR}/bin/*.dll
    $ENV{HIP_DIR}/bin/*.dll
    $ENV{HIP_DIR}/bin/hipinfo.exe
    ${CMAKE_SOURCE_DIR}/rtest.*
  )
  foreach(file_i ${third_party_dlls})
    add_custom_command(TARGET hipsolver-test
      POST_BUILD
      COMMAND ${CMAKE_COMMAND}
      ARGS -E copy ${file_i} ${PROJECT_BINARY_DIR}/staging/
    )
  endforeach()
  add_custom_command(TARGET hipsolver-test
    POST_BUILD
    COMMAND ${CMAKE_COMMAND}
    ARGS -E copy_directory $ENV{rocblas_DIR}/bin/rocblas/library ${PROJECT_BINARY_DIR}/staging/library
  )
endif()
```

Right now those sources are a mix of `_DIR` environment variables and CMake variables (https://github.com/ROCm/hipSOLVER/blob/bb8408f52a166c5ce587b204c61a51ae44bc1daa/clients/gtest/CMakeLists.txt#L156-L180), populated in various ways:

* `rmake.py` (which can be used for convenience but should not be a hard requirement): https://github.com/ROCm/hipSOLVER/blob/bb8408f52a166c5ce587b204c61a51ae44bc1daa/rmake.py#L182-L188

    ```python
        if args.rocblas_path:
            cmake_options.append("-DCUSTOM_ROCBLAS=" + cmake_path(args.rocblas_path))
            os.environ['rocblas_DIR'] = cmake_path(args.rocblas_path)
    ```

* `CMakeLists.txt` (deep in the project, not at the root): https://github.com/ROCm/hipSOLVER/blob/bb8408f52a166c5ce587b204c61a51ae44bc1daa/library/src/CMakeLists.txt#L113-L135

    ```cmake
      # Find rocBLAS
      if(NOT TARGET rocblas)
        if(CUSTOM_ROCBLAS)
          set(ENV{rocblas_DIR} ${CUSTOM_ROCBLAS})
          find_package(rocblas REQUIRED CONFIG NO_CMAKE_PATH)
        else()
          find_package(rocblas REQUIRED CONFIG PATHS /opt/rocm /opt/rocm/rocblas)
        endif()
        list(APPEND static_depends PACKAGE rocblas)
      endif()
    ```

* `toolchain-windows.cmake` (fine to initialize compiler options but should not be used for finding dependencies): https://github.com/ROCm/hipBLAS/blob/1094548b083642192be8de47767c6ab9daf45ad0/toolchain-windows.cmake#L45-L92

    ```cmake
    # paths required for client testing on windows
    if (NOT ROCBLAS_PATH)
      if (ROCBLAS_CUSTOM_PATH)
        file(TO_CMAKE_PATH "${ROCBLAS_CUSTOM_PATH}" ROCBLAS_PATH )
      else()
        set(ROCBLAS_PATH "C:/hipSDK")
      endif()
    endif()
    ```

When built by TheRock, toolchain files and wrapper python scripts are not used, so some of these variables are undefined. In hipSOLVER this leads to an error at build time: `Error copying directory from "/bin/rocblas/library" to "D:/projects/TheRock/build/math-libs/BLAS/hipSOLVER/build/clients/staging/library".` (that was expecting a `$ENV{rocblas_DIR}` prefix ahead of `/bin`). In other projects, the build succeeds but tests fail because some DLLs were not copied (silent failures, or missing some extra configuration?).
