## Summary

`amdhip64_7.dll` has a direct dependency on `rocm_kpack.dll` (the kpack runtime), but `rocm_kpack.dll` is not staged or distributed to the output artifact directories. This causes all HIP executables and tests to fail to load on Windows with:

```
error while loading shared libraries: ?: cannot open shared object file: No such file or directory
```

The DLL is built at `core/rocm-kpack/build/runtime/rocm_kpack.dll` but never reaches `dist/rocm/bin/` or any component dist directories.

## Reproduction

Build TheRock on Windows with kpack enabled (`THEROCK_KPACK_SPLIT_ARTIFACTS=ON`), then attempt to run any HIP test:

```
> dumpbin /dependents dist\rocm\bin\amdhip64_7.dll
  Image has the following dependencies:
    rocm_kpack.dll    <-- missing from dist
    SETUPAPI.dll
    KERNEL32.dll
    ...
```

## Workaround

Manually copy the DLL:
```powershell
copy build\core\rocm-kpack\build\runtime\rocm_kpack.dll build\dist\rocm\bin\
```

## Fix

The rocm-kpack runtime library needs to be included in the stage/dist packaging for the core-kpack component so it ships alongside `amdhip64_7.dll`.
