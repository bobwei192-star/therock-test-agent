### Problem Description

When running ASAN CI tests, LeakSanitizer (LSAN) reports false positive memory leaks from test environment components that do not fully cooperate with LSAN's
  expectations:

  - Bash shell internal allocations (make_if_command, make_simple_command, command_connect, yyparse, /usr/bin/bash) - Bash intentionally does not free all memory
  before exit since the OS reclaims it anyway. This is standard shell behavior, not a real leak.
  - Python's internal memory allocator (pymalloc) (_PyMem_RawMalloc, PyObject_Malloc, PyMem_Malloc, _PyObject_Malloc, _PyBytes_Resize, PyMem_RawMalloc,
  PyMem_Realloc, _PyObject_Realloc) - Python uses memory pools that are not fully freed before process exit.
  - rpds library (rpds::HashTrieMapPy, rpds::HashTrieSetPy) - A Rust-based persistent data structures library used as a pytest dependency.

  These false positives generate over 500KB of noise in CI output, making it difficult to identify actual memory leaks in ROCm components.
