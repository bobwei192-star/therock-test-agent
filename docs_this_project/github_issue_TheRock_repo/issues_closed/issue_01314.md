Using the latest Windows nightly build of PyTorch in a [Wan2GP](https://github.com/deepbeepmeep/Wan2GP) workflow, I hit this error in the console before the UI starts up:

> ModuleNotFoundError: No module named 'torch.distributed'

Looking closer, there’s a script at `Wan2GP\models\wan\distributed\fsdp.py` that contains:

`from torch.distributed.fsdp import FullyShardedDataParallel as FSDP`

This is not the same case as https://github.com/ROCm/TheRock/issues/1202, so:

`pip install transformers==4.41.2`

or

`pip install git+https://github.com/huggingface/transformers`

didn’t help, since this isn’t a `transformers` issue - the script explicitly imports `torch.distributed`, which is missing from the current Windows wheels.

Could https://github.com/pytorch/gloo/pull/461 help resolve this?
https://github.com/pytorch/pytorch/pull/160776#discussion_r2292094385 seems promising too.

I hope we don’t need RCCL as well, which is currently unsupported on Windows: https://github.com/ROCm/TheRock/blob/main/docs/development/windows_support.md#supported-subprojects
