### Problem Description

Program directory C:\Program Files\AMD\ROCm exists, but has no subdirectories.
Building rocBlas fails with an exception from theRock\build\math-libs\BLAS\rocBLAS\build\virtualenv\Lib\site-packages\Tensile\Utilities\toolchain.py.
The below text is a summary from Devin, which apparently has no access to tensile py library, but helped me get to the point where I could finally resolve what was happening after first pursuing all sorts of other trails.

    The specific error: max() called on empty sequence in Toolchain.py line 55
    Root cause: Empty C:\Program Files\AMD\ROCm directory from previous installation
    Expected behavior: Should either ignore empty directories or provide a clear error message
    Actual behavior: Cryptic max() error that's difficult to debug

In my opinion, a similar scenario is possible on Linux.
Fix, before a new release of tensile, where I presume this will be fixed: Remove subdirectory ROCm.

### Steps to Reproduce

Create an empty directory C:\Program Files\AMD\ROCm (windows, obviously, but similar likely possible on Linux) and run cmake --build build --target therock-dist (after configuration).
If you have a build tree with at least core/clr dist filled out, you can run rocBlas alone from H:\theRock\build\math-libs\BLAS\rocBLAS\build using the command "cmake --build build".
