## PyTorch/JAX/native package dispatch rejected (10 GPU families)

**Affected arches:** `gfx950-dcgpu, gfx94X-dcgpu, gfx110X-all, gfx120X-all, gfx1151, gfx1150, gfx1153, gfx90a, gfx908, gfx1152`

**Failing step:** `Trigger building PyTorch wheels`

**Exact error:**
```text
🔎 Found workflow, id: 174743032, name: Release portable Linux PyTorch Wheels, path: .github/workflows/release_portable_linux_pytorch_wheels.yml
🚀 Calling GitHub API to dispatch workflow...
##[error]Provided value 'nightly' for input 'release_type' not in the list of allowed values - https://docs.github.com/rest/actions/workflows#create-a-workflow-dispatch-event
```

**Root cause:** Commit [a7415d438b3c](https://github.com/ROCm/TheRock/commit/a7415d438b3c5925f15e287eed468b3129812f29) changed `release_type` in the `workflow_dispatch` trigger of `release_portable_linux_pytorch_wheels.yml`, `release_portable_linux_jax_wheels.yml`, and `build_native_linux_packages.yml` from `type: string` to `type: choice` with only `- dev`. The nightly scheduler passes `"nightly"`, which the GitHub API now rejects against the choice allowlist.

**Fix:** Add `nightly` and `prerelease` to the `options` list for `release_type` in the `workflow_dispatch` trigger of all three affected workflow files.
