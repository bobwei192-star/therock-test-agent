## Context

- **Workflow:** Multi-Arch Release (`.github/workflows/multi_arch_release.yml` in ROCm/rockrel)
- **Run:** https://github.com/ROCm/rockrel/actions/runs/26993223184
- **Platform:** Linux
- **Affected distros:** `rhel10`, `ubuntu2404`, `sles16`

## Error

All 3 `Trigger Native Install Test` jobs fail immediately with:

```
##[error]Unable to find workflow 'test_native_linux_packages_install.yml' in ROCm/rockrel 😥
```

## Failing Jobs

- [Trigger Native Install Test - rhel10](https://github.com/ROCm/rockrel/actions/runs/26993223184/job/79698255492)
- [Trigger Native Install Test - ubuntu2404](https://github.com/ROCm/rockrel/actions/runs/26993223184/job/79698255520)
- [Trigger Native Install Test - sles16](https://github.com/ROCm/rockrel/actions/runs/26993223184/job/79698255611)

## Root Cause

Regression introduced by TheRock PR [#5398](https://github.com/ROCm/TheRock/pull/5398) — *"Install test invocation added to multi arch Linux release/build workflow"*, merged 2026-06-05.

PR #5398 added the `trigger_test_native_install` job to `multi_arch_release_linux.yml` and added `test_native_linux_packages_install.yml` to **ROCm/TheRock**, but did not add a corresponding wrapper workflow to **ROCm/rockrel**. The dispatch action defaults to the executing repo context (rockrel), where the file does not exist.

## Fix

Add `test_native_linux_packages_install.yml` wrapper workflow to ROCm/rockrel — tracked in rockrel issue [#48](https://github.com/ROCm/rockrel/issues/48).
