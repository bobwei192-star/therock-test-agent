### Description

The `wsl-rocdxg` stage in Multi-Arch CI ASAN failed during **Fetch inbound artifacts** because the WSL Windows runner (`ROCR-Runtime-libhsakmt-WSL` runner group) ran out of disk space while bootstrapping `amd-llvm_dev_generic.tar.zst`.

Platform: `Windows (WSL)`
Stage: `wsl-rocdxg`
workflow: `.github/workflows/multi_arch_build_wsl_rocdxg_artifacts.yml`

### Failing run
- Workflow run:  https://github.com/ROCm/TheRock/actions/runs/25979696409/job/76367479402

### Suspected Root Cause

The runner started with only 25.18 GB free (83.15% used). Bootstrapping the 28 inbound artifacts from the `compiler-runtime` stage — especially the large `amd-llvm_dev_generic.tar.zst` — exhausted the remaining space. The `health_status.py` disk check emitted a warning but does not gate execution, so the job proceeded to a hard `[Errno 5]` I/O failure.

### Suggested Fix

- **Immediate:** Clean the WSL runner disk and re-run the PR.
- **Long-term:** Add a hard disk-space pre-check step to `.github/workflows/multi_arch_build_wsl_rocdxg_artifacts.yml` that fails fast (before downloading any artifacts) when free space is below ~30 GB.
