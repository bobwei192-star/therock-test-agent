### Problem Description

ROCm fails to build because automake-1.16 is missing. I have 1.17.

I was able to bypass that with some symlink shenanigans but then cmake complains:

```
[amd-llvm configure] CMake Error at /home/rena/projects/ai/ComfyUI/lol/TheRock/compiler/amd-llvm/amd/device-libs/cmake/OCL.cmake:12 (cmake_policy):
[amd-llvm configure]   Policy CMP0053 may not be set to OLD behavior because this version of CMake
[amd-llvm configure]   no longer supports it.  The policy was introduced in CMake version 3.1.0,
[amd-llvm configure]   and use of NEW behavior is now required.
```


### Operating System

Artix Linux AMD64

### CPU

AMD Ryzen 5 5600 6-Core Processor

### GPU

AMD Radeon RX 7800 XT

### ROCm Version

whichever one it's supposed to be building?

### Steps to Reproduce

1. Follow the setup steps for Ubuntu, skipping the apt-get step since apt-get isn't on Artix (the relevant packages should already be installed)
2. cmake -B build -GNinja . -DTHEROCK_AMDGPU_FAMILIES=gfx110X-dgpu
3. cmake --build build
