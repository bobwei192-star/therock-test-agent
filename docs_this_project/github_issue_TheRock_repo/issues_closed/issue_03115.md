## Background

We noticed these failures after adding a new test workflow for ROCm Python packages in https://github.com/ROCm/TheRock/pull/3099. Existing automated test coverage for `rocm-sdk test` was only here, after installing torch (which does not install ROCm devel packages: https://github.com/ROCm/TheRock/blob/8e5be537b56753343afa88fc2ef4f405905d8b8a/.github/workflows/test_pytorch_wheels.yml#L143-L145

I can't reproduce this on my Ubuntu host machine, but I can reproduce it in our test dockerfile, so this is definitely a system dependency that is leaking out.
