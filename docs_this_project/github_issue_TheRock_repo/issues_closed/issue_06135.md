## Summary

The math-libs CMake configure step fails for the gfx103X-all shard because the new `THEROCK_ENABLE_HIPTENSOR` block in `math-libs/CMakeLists.txt` declares `composable_kernel` as a `BUILD_DEPS` target, but for gfx103X all seven targets have `composable_kernel` in `EXCLUDE_TARGET_PROJECTS` (per [`cmake/therock_amdgpu_targets.cmake`](https://github.com/ROCm/TheRock/blob/main/cmake/therock_amdgpu_targets.cmake#L133)). After filtering, `_ck_enabled` is set to `OFF` and the `composable_kernel` CMake target is never created, so `therock_cmake_subproject_declare(hipTensor)` errors when it tries to resolve the dep.
