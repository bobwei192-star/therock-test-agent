## Problem

**Severity:** P1 — blocks making hip-tests/aqlprofile-tests target-neutral; affects PyTorch compatibility

**Related:** Reverted commit [8940752](https://github.com/ROCm/TheRock/commit/894075268645a7eb6fcfdbbf06d5261229d2752c), issues #3432, #3428

PR #3348 attempted to:
1. Change `core-hiptests` and `aqlprofile-tests` from `target-specific` to `target-neutral` in `BUILD_TOPOLOGY.toml`
2. Introduce `THEROCK_DIST_AMDGPU_FAMILIES` as a separate cache variable for distribution targets
3. Have target-neutral artifacts built with all dist targets

This was reverted because in **mono-builds** (normal single-stage builds without multi-arch CI), when `THEROCK_DIST_AMDGPU_FAMILIES` was not explicitly set, the dist targets defaulted to ALL available architectures (~30+). This caused downstream consumers (PyTorch) to see the full arch list and try to compile for all of them.

## Root Cause

In `therock_amdgpu_targets.cmake`, the validation function defaults empty `THEROCK_DIST_AMDGPU_FAMILIES` to `THEROCK_AMDGPU_FAMILIES`. The reverted version of the code had a different default that expanded to all available targets instead. The current code (post-revert) at line 262-264 correctly defaults to `THEROCK_AMDGPU_FAMILIES`:
```cmake
set(_dist_families "${THEROCK_DIST_AMDGPU_FAMILIES}")
if(NOT _dist_families)
  set(_dist_families "${THEROCK_AMDGPU_FAMILIES}")
endif()
```

But making hip-tests target-neutral still requires that the dist targets concept works correctly in both mono-build and multi-arch contexts.

## Fix Direction

The replacement needs to ensure:
- In **mono-builds**: dist targets = build targets (same as today)
- In **multi-arch CI**: dist targets = all families in the distribution (set explicitly)
- target-neutral artifacts are built with the dist targets, not all known targets
- The advertised arch list in installed CMake configs reflects only the dist targets, not all possible targets
