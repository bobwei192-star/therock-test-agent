### Problem Description

I've installed "gfx120-all" nightly torch on my local Windows pc but for me it behaves like this torch wasn't even built for gfx1200, because it gives `False` for `torch.cuda.is_available`, so it cant even detect a usable gpu. 

Ive built a torch locally too using the instructions here:
https://github.com/ROCm/TheRock/tree/main/external-builds/pytorch
and it's closer to a working state, because at least it gives `True` for `torch.cuda.is_available`. I've MIOpen issues with it but that's a whole another topic.

So Idk what's with the automatic nightly releases for gfx1200, but for now, they aren't usable at all.
I think this issue will not be solved until the devs do debugging on their local gfx1200 systems.


### Operating System

Windows 11

### CPU

Intel Core i5 14th gen

### GPU

AMD Radeon RX 9060 XT gfx1200

### ROCm Version

7.0.0

### Steps to Reproduce

On a Windows 11 system with the AMD 25.6.3 driver, gfx1200 gpu, follow the scripts (I'm using python 3.12 here):

`py -m venv venv`
`venv\scripts\activate`
`python -m pip install --index-url https://d2awnip2yjpvqn.cloudfront.net/v2/gfx120X-all/ torch`
`pip show torch` (this says: 2.7.0a0+rocm7.0.0rc20250717 for now)
`python -c "import torch; print(torch.cuda.is_available())"`
-And this command's output for me is: False

So like nothing can be done that's related to torch in any way.

### Additional Information

Theoretically gfx120X-all should at least give `True` for `torch.cuda.is_available`, because
https://github.com/ROCm/TheRock/blob/142fcbc288251e34585535eab941191a1837dd73/cmake/therock_amdgpu_targets.cmake#L111
clearly includes gfx1200, so it actually builds for this card too. So what could be the problem?
