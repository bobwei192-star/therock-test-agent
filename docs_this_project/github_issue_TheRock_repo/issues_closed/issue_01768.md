### Problem Description

 I'm hitting a strange issue with rocprofv3 (packaged in TheRock dist) between two nightly versions of TheRock and would appreciate some pointers on triaging this further (or tagging the right team/folks).

For the same application being profiled, version A (good) has the profiler finish gracefully and generate the pftrace file etc, version B (bad) has the profiler run for a bit and then just hang indefinitely. During this time rocm-smi shows the GPU being on 100% util and SCLK at the max (~2100MHz). The only way to recover from this is with a Ctrl+C abort. I also wonder how regressions to tools packaged in TheRock nightlies are surfaced?
