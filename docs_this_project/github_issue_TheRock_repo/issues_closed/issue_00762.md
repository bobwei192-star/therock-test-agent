I'm imagining a script that checks for installed software, disk space, environment variables, etc. and reports what it found along with recommendations to fix common setup issues. When users experience build issues, we can ask them to run the script and include the output in their bug report.

For example:

* On Linux and Windows
  * Check Python version, warn if not in a venv
  * Check installed Python packages (from requirements.txt)
  * Check free disk space
  * Check processor core count
  * Check submodule status
  * Check git
  * Check cmake
  * Check ninja
  * Check ccache
* On Linux
  * Check for gcc
* On Windows
  * Check for MSVC (installed, activated)
  * Check for long path support
  * Check for symlink support
  * Check for existing HIP SDK install (files, env vars)

We can also (instead?) put more checks in the CMake project itself, but some environment issues are easier to check for in a standalone script. If we instead extended the CMake project configure logs, maybe the CMake configure output would be sufficient for bug reporting? I'm not sure... I like the option of having a nice "diagnose me" script.

See also the subproject `rdeps.py` files for some ideas that existing ROCm developers may be accustomed to. I'm hesitant to add any scripts that take action to install dependencies system-wide though.

Related: https://github.com/ROCm/TheRock/issues/710#issuecomment-2907427841
