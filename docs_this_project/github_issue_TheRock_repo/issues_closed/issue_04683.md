## Summary
The "Release Windows PyTorch Wheels" workflow fails during HIP device compilation of ATen's int4mm kernel. The failure is caused by calling `std::memcpy` inside code compiled for the GPU device, which clang/hipcc treats as a host-only function on this ROCm-on-Windows toolchain.

### Context
- Platform: `windows`
- Archs Impacted: `gfx90a`
- Pytorch versions: `2.9, 2.10, 2.11`
- run: https://github.com/ROCm/TheRock/actions/runs/24656187795
