### Problem Description

After installation of the either the nightly builds by TheRock or [official preview builds](https://repo.radeon.com/rocm/windows/) of Windows PyTorch, I found the *.exe files in the Scripts are always hardcoded with an absolute path in the venv, if I rename the venv folder or move to another drive, it will show as follows when run a python scirpt (like running ComfyUI main.py, [the related issue](https://github.com/Comfy-Org/ComfyUI/issues/11546):  
```bash
Fatal error in launcher: Unable to create process using '"D:\a\python_embeded\python.exe" "C:\python_embeded\Scripts\offload-arch.exe" ': The system cannot find the file specified.
[WARNING] amdgpu-arch failed with return code 1
=> reason: .\python_embeded\Scripts\amdgpu-arch.exe contains a hardcoded path for python.exe (D:\a\python_embeded) which does not exist.
```  

`hipinfo.exe` also has this problem. THE NVIDIA CUDA PyTorch doesn't have this kind of problem. Hope it can be fixed as it is important for distibuting standalone embedded version of Python software based on Windows PyTorch ROCm7.
