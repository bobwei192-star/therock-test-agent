# #405 - ROCPRIM_HAS_INT128_SUPPORT is not supported on Windows gtest builds

**State**: closed  
**Author**: stellaraccident  
**Created**: 2025-04-12T00:17:23Z  
**Updated**: 2025-04-14T22:39:26Z  
**URL**: https://github.com/ROCm/TheRock/issues/405  

---

rocPRIM defaults INT128 support to 1, which works when compiling on clang/gcc. However, if linking against system-built libraries (like gtest) on Windows, they do not have native INT128 support and will typically exclude support functions, resulting in undefined symbols like `?PrintTo@internal@testing@@YAX_MPEAV?$basic_ostream@DU?$char_traits@D@std@@@std@@@Z`.

We have to disable INT128 support manually on Windows builds.
