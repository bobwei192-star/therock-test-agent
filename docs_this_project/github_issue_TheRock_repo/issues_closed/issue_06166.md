## Summary

After the rocm-systems bump, `split_artifacts.py` now raises `RuntimeError` instead of silently emitting an untransformed generic artifact when fat binaries contain no device code matching `--gpu-targets`. For the gfx101X-dgpu build shard, rocWMMA is excluded from gfx1010/1011/1012 (all RDNA1, no MFMA support, per [#1944](https://github.com/ROCm/TheRock/issues/1944)), so its GPU_TARGETS list is empty after filtering. rocWMMA falls back to its own default targets (gfx1100, gfx90a, etc.) and compiles 37 fat binaries — none of which contain gfx101X device code. The new strict guard then refuses to split the `rocwmma_test` artifact.

## Error

```
FAILED: artifacts/rocwmma_test_generic/artifact_manifest.txt B:/build/artifacts/rocwmma_test_generic/artifact_manifest.txt
Error: Found 37 fat binaries, but no device code objects matched --gpu-targets (gfx1010, gfx1011, gfx1012). Refusing to emit an untransformed generic artifact.
ninja: build stopped: cannot make progress due to previous errors.
```

## Fix

PR [#6042](https://github.com/ROCm/TheRock/pull/6042) ("Disable rocWMMA samples and tests when targets are excluded") contains the required `math-libs/CMakeLists.txt` fix: detect all-filtered rocWMMA shards before subproject activation, disable tests/samples/benchmarks for those shards, and remove the `test` component from the rocwmma artifact so `rocwmma_test_generic` is never produced for unsupported arch families. PR #6042 must merge alongside or before PR #6163.
