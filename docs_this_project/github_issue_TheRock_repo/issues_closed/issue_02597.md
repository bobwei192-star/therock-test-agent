# #2597 - Break sysdeps artifact apart, specifically with respect to amd-mesa

The sysdeps artifact has gotten too monolithic, especially with the addition of amd-mesa.

It currently has these contents:

```
    therock-bzip2
    therock-elfutils
    therock-libbacktrace
    therock-libcap
    therock-libdrm
    therock-liblzma
    therock-numactl
    therock-sqlite3
    therock-zlib
    therock-zstd
    therock-amd-mesa
```

I'd like to strike a balance between kitchen-sink and so finely sliced that it is an administrative burden. I propose that we keep the current sysdeps artifact for everything except mesa and then add a new one for `sysdeps-amd-mesa` just for that. This will set a good precedent for the future: very common sysdeps that are typically considered base system libraries (and not very big) can go in `sysdeps` and we have separate artifacts for big/distinct/sparsely-used ones. This will end up getting reflected in our packaging namespace as well.

We may move the host-blas library to sysdeps like the rest of the distributed libraries (it predated this distinction) at some point, and we would definitely want that in its own artifact as well.

Note that this will also require some corresponding changes to BUILD_TOPOLOGY.toml.
