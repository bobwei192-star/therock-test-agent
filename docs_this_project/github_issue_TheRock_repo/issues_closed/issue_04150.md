### 🐛 Describe the bug
When installing the latest PyTorch nightly wheel for ROCm 7.2 on Linux (Python 3.12), importing `torch` fails immediately. The wheel seems to be missing the `librocprofiler-sdk.so.1` library.

**Error Log:**
```python -c "import torch"
Traceback (most recent call last):
  File "<string>", line 1, in <module>
  File "/media/HDD/programs/AI/ComfyUI/venv-linux/lib/python3.12/site-packages/torch/__init__.py", line 443, in <module>
    from torch._C import *  # noqa: F403
    ^^^^^^^^^^^^^^^^^^^^^^
ImportError: librocprofiler-sdk.so.1: cannot open shared object file: No such file or directory
```

### Versions
* **OS:** Linux (Mint / Ubuntu base)
* **GPU:** AMD Radeon RX Vega 56 (gfx900)
* **Python version:** 3.12.3
* **PyTorch version:** `2.12.0.dev20260324+rocm7.2-cp312-cp312-manylinux_2_28_x86_64.whl`

### Steps to reproduce
1. Create a fresh Python 3.12 virtual environment.
2. Install the nightly wheel:
   `pip install --pre torch torchvision torchaudio --index-url https://download.pytorch.org/whl/nightly/rocm7.2`
3. Run `python -c "import torch"`

### Expected behavior
PyTorch should import successfully. The missing shared library `librocprofiler-sdk.so.1` should either be bundled inside the wheel or not be a hard dependency for a basic import.
