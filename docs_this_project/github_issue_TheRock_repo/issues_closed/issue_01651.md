### Problem Description

ROCm build configuration step uses this function to set a `PROJECT_VERSION_TWEAK` variable based on the git monorepo state: https://github.com/ROCm/rocm-cmake/blob/bb18c61579e46741ad8c6d663fedfcd76aa621be/share/rocmcmakebuildtools/cmake/ROCMSetupVersion.cmake#L87-L117 . In the typical case installing via TheRock this equals running `git describe --dirty --long --match [0-9]* --always` on the `rocm-libraries` monorepo.

Consider the following fetching of sources in preparation for building ROCm stack with TheRock:
```
# download identical ROCm source to two directories using TheRock
WORKDIR /app_1
RUN git clone https://github.com/ROCm/TheRock.git && \
    cd TheRock && \
    git reset --hard 7afbe45 && \
    git rev-parse HEAD > /app_1/therock.txt && \
    git submodule status > /app_1/therock_submodules.txt && \
    pip install -r requirements.txt && \
    python build_tools/fetch_sources.py

WORKDIR /app_2
RUN git clone https://github.com/ROCm/TheRock.git && \
    cd TheRock && \
    git reset --hard 7afbe45 && \
    git rev-parse HEAD > /app_2/therock.txt && \
    git submodule status > /app_2/therock_submodules.txt && \
    pip install -r requirements.txt && \
    python build_tools/fetch_sources.py
```

The contents of `therock.txt` and `therock_submodules.txt` are identical in both `app_1` and `app_2`.

Expectation: `PROJECT_VERSION_TWEAK` is identical for both source versions.

Actual: `PROJECT_VERSION_TWEAK` differs for each fetched source, as observed in the following:
```
root@0a6ff2c33672:/app_1/TheRock/rocm-libraries# git describe --dirty --long --match [0-9]* --always
5a5ca7a740

root@0a6ff2c33672:/app_2/TheRock/rocm-libraries# git describe --dirty --long --match [0-9]* --always
ed3e44e41b
```

This impacts all ROCm libraries, especially MIOpen which uses `MIOPEN_VERSION_TWEAK` as part of the user tuning database filenames. It is probably related to applying the [patches here](https://github.com/ROCm/TheRock/tree/main/patches/amd-mainline/rocm-libraries) which change the monorepo state and output of `git describe`.
