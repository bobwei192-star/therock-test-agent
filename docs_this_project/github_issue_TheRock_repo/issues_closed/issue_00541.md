See https://github.com/ROCm/TheRock/blob/main/.github/workflows/build_linux_packages.yml and https://github.com/ROCm/TheRock/blob/main/.github/workflows/build_windows_packages.yml.

The Windows workflow should have the same "report" and "upload" steps. This might involve creating a [python?] script and sharing that script between both workflows.

Artifact uploading will enable test workflows to pull the artifacts from the build jobs and run tests. Users/developers will also be able to use these artifacts - see https://github.com/ROCm/TheRock/blob/main/RELEASES.md#from-per-commit-ci-builds.

We'll need to decide on naming / namespacing. Should Windows artifacts go in the existing `s3://therock-artifacts` bucket with Windows file names, or should they go in a new bucket like "therock-windows-artifacts"?
