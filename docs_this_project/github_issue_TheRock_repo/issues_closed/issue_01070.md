We believe that the hurdles are cleared to migrate the existing Linux PyTorch builds to:

1. ROCm 2.7 backport release branch (from upstream 2.7 release branch): our hosted release branch has many more post release fixes for post cut
2. Stand up PyTorch at head nightlies

For 1, we need to update the existing jobs to use the checkout instructions from https://github.com/ROCm/TheRock/tree/main/external-builds/pytorch#v27x, also making sure that triton, audio, and vision are enabled.

For 2, we should have a policy to always do a "head on head" (rocm head on pytorch head) build every night on the most recent Python version supported by PyTorch. The checkout instructions for this scenario are here: https://github.com/ROCm/TheRock/tree/main/external-builds/pytorch#pytorch-nightly . This will involve creating a new build/test pipeline.
