Current support for rocRoller requires a hard coded path to amdclang++ for assembling and linking. This makes resulting library fragile and it only runs in very specific contexts. The rocRoller team is replacing this functionality with direct calls to the comgr c api. When this feature lands we need to update the rocRoller commit and update patches.
