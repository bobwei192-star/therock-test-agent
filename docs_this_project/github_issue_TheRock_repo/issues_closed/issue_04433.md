## Issue in packaging jobs

On https://github.com/ROCm/TheRock/pull/4421, the "Build Python" job https://github.com/ROCm/TheRock/actions/runs/24187929660/job/70618360191?pr=4421 ran
```diff
    python ./build_tools/artifact_manager.py fetch \
      --run-id="24187929660" \
      --stage=all \
+     --amdgpu-families="gfx94X-dcgpu;gfx110X-all;gfx1151;gfx120X-all" \
      --output-dir="/home/runner/_work/TheRock/TheRock"
```

this only fetched generic and gfx1151 artifacts, NOT gfx942, gfx1100, etc. artifacts:
```diff
   ++ Extracting aqlprofile_test_generic.tar.zst
+  ++ Downloading blas_lib_gfx1151.tar.zst
-  ++ Downloading blas_lib_gfx1100.tar.zst  # NOT IN THE LOGS
   ++ Extracting base_dev_generic.tar.zst
   ++ Extracting base_test_generic.tar.zst
   ++ Extracting base_lib_generic.tar.zst
   ++ Downloading blas_test_gfx1151.tar.zst
   ++ Extracting blas_run_generic.tar.zst
```

## Why test jobs work

The test jobs like https://github.com/ROCm/TheRock/actions/runs/24187929660/job/70629251373?pr=4421 were passed underlying target names in addition to families:
```diff
Run ./.github/actions/setup_test_environment
  with:
    ARTIFACT_RUN_ID: 24187929660
    ARTIFACT_GROUP: gfx94X-dcgpu
+   AMDGPU_TARGETS: gfx942

...

  s3://therock-ci-artifacts/24187929660-linux/blas_lib_generic.tar.zst
+ s3://therock-ci-artifacts/24187929660-linux/blas_lib_gfx942.tar.zst
  s3://therock-ci-artifacts/24187929660-linux/blas_run_generic.tar.zst
  s3://therock-ci-artifacts/24187929660-linux/blas_test_generic.tar.zst
+ s3://therock-ci-artifacts/24187929660-linux/blas_test_gfx942.tar.zst
```

This comes through via:
* `fetch-gfx-targets`: https://github.com/ROCm/TheRock/blob/5250110dbb4e9fbe3e2062f6145734785ff0ea83/build_tools/github_actions/amdgpu_family_matrix.py#L71-L89
* `amdgpu_targets` in family_info https://github.com/ROCm/TheRock/blob/5250110dbb4e9fbe3e2062f6145734785ff0ea83/build_tools/github_actions/configure_multi_arch_ci.py#L832-L841
* plumbing to the test job: https://github.com/ROCm/TheRock/blob/5250110dbb4e9fbe3e2062f6145734785ff0ea83/.github/workflows/multi_arch_ci_linux.yml#L90-L104

## Potential solutions

Ideas:

* Expand dist_amdgpu_families with an extra list, keep existing fetch-gfx-targets
* Expand fetch-gfx-targets to the complete lists (though this might be overly connected to which GPU test runners we actually have right now)
* Expand during `artifact_manager.py fetch`
