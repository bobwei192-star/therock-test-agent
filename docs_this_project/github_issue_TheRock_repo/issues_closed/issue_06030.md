## Description

The `Build PyTorch (fat + split)` CI job (torch release/2.10, py3.12) fails to build because gfx1250 is included in the arch list but composable_kernel in PyTorch release/2.10 does not define `CK_BUFFER_RESOURCE_3RD_DWORD` for gfx1250. The same root cause as the release/2.11 issue (#5910), but in the CI fat build path (`multi_arch_ci_linux.yml`) which does not apply the per-ref gfx125x exclusion that `configure_pytorch_release_matrix.py` applies for release workflows.

## Context

- **Platform:** Linux
- **Arch:** gfx1250 (gfx125X-dcgpu)
- **Workflow:** Multi-Arch CI (PR bump run)
- **Repo:** ROCm/TheRock (PR #6020, Bump rocm-systems from a0952b2 to 971dc69)
- **Failing run:** https://github.com/ROCm/TheRock/actions/runs/27923247524

## Root Cause

Commit `1cc8ec5` ("Add gfx1250 (CDNA5) build support", 2026-06-17, PR #5744) added `gfx125X-dcgpu` to the build matrix and correctly added per-ref `exclude_amdgpu_families: {"gfx125x"}` exclusions in `configure_pytorch_release_matrix.py` for `release/2.9`, `release/2.10`, `release/2.11`, and `nightly`. However, it did not apply the same exclusion to `multi_arch_ci_linux.yml`'s `build_pytorch_wheel_fat` job, which passes `dist_amdgpu_families` (the full set including `gfx125X-dcgpu`) directly to `build_portable_linux_pytorch_wheels_ci.yml` without filtering:

```yaml
build_pytorch_wheel_fat:
  uses: ./.github/workflows/build_portable_linux_pytorch_wheels_ci.yml
  with:
    pytorch_git_ref: "release/2.10"
    amdgpu_families: ${{ fromJSON(inputs.build_config).dist_amdgpu_families }}  # includes gfx125X-dcgpu
```

The release workflows call `configure_pytorch_release_matrix.py` which applies the exclusion; the CI fat build does not.

## Expectation

The `build_pytorch_wheel_fat` CI job must exclude `gfx125X-dcgpu` from the build arch list for pytorch refs where gfx1250 is not yet supported (currently all refs except potentially release/2.12+).

## Impact

- Affects any PR with the `ci:run-all-archs` label (which expands `linux_names` to all families including gfx125x). Standard bump PRs without that label only get the presubmit families (gfx94x, gfx110x, gfx1151, gfx120x) and are not affected.
- PR #6020 has the `ci:run-all-archs` label; PR #5971 (previous bump) did not — which is why the failure appeared now and not before.
- Affects torch release/2.10 (confirmed); the same issue would appear for any other pytorch ref in `PYTORCH_REFS_LINUX` with `exclude_amdgpu_families: {"gfx125x"}` when triggered with `ci:run-all-archs` or on a nightly/push run.

## Suggested Fix Options

1. **Filter gfx125x from `amdgpu_families` in `multi_arch_ci_linux.yml`** before passing to the fat build job — call `configure_pytorch_release_matrix.py` or apply the exclusion inline per `pytorch_git_ref`.

2. **Move the exclusion logic into `build_portable_linux_pytorch_wheels_ci.yml`** — accept a `pytorch_git_ref` input and call `configure_pytorch_release_matrix.py` internally to filter families before expanding to gfx targets.
