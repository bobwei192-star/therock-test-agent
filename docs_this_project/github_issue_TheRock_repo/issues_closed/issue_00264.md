## Summary

When building the test and benchmark clients and not using BLIS it is assumed that the NETLINB reference implementation should be used. Library names are hardcoded but it is never checkec if the libs are present. The patch refactors to use `find_package()` instead which also allows to use other LAPACK and BLAS providers like OpenBLAS.
