Following on https://github.com/ROCm/TheRock/issues/542, we can fork these files for Windows to publish nightly releases of PyTorch + ROCm on Windows:

* https://github.com/ROCm/TheRock/blob/main/.github/workflows/build_linux_pytorch_wheels.yml
* https://github.com/ROCm/TheRock/blob/main/.github/workflows/release_linux_pytorch_wheels.yml

Ideally we'd build off of the `rocm-sdk` wheels that are already in use on Linux (https://github.com/ROCm/TheRock/issues/703), though we may need some changes to get that ready on Windows.

See also https://github.com/ROCm/TheRock/issues/804 for workflow triggering.

Some details:

* Linux builds under a manylinux docker container. Windows does not need to do that.
* Linux currently uses https://github.com/ROCm/TheRock/blob/main/external-builds/pytorch/build_prod_wheels.py. Windows may be able to use that too.
