## Problem

**Severity:** P0 Blocker — causes ALL library test failures in multi-arch CI

When `THEROCK_KPACK_SPLIT_ARTIFACTS=ON`, the build produces per-individual-target archives:
```
blas_lib_generic.tar.zst     ← host-only stripped binaries
blas_lib_gfx942.tar.zst      ← device code for gfx942
```

But the test fetch logic in two places constructs filenames using the **family name** (e.g., `gfx94X-dcgpu`), not the individual target name (`gfx942`):

**Path A: `fetch_artifacts.py` (used by test setup via `install_rocm_from_artifacts.py`)**
- `list_artifacts_for_group()` at `build_tools/fetch_artifacts.py:69-72` filters: `if artifact_group in filename or "generic" in filename`
- `artifact_group` = `"gfx94X-dcgpu"` (from workflow input)
- `"gfx94X-dcgpu"` is NOT a substring of `"blas_lib_gfx942"` → arch-specific artifacts are excluded
- Generic artifacts pass (via the `"generic"` check)

**Path B: `artifact_manager.py` (used for inter-stage fetching)**
- `build_tools/artifact_manager.py:314`: `filename = f"{artifact_name}_{comp}_{tf}{ext}"` where `tf` = `"gfx94X-dcgpu"`
- Constructs `blas_lib_gfx94X-dcgpu.tar.zst` which doesn't exist as an uploaded artifact

**Result:** Tests get stripped host binaries but no `.kpack` device code → `hipErrorInvalidImage` / `hipErrorInvalidKernelFile` (errors 200/201) on every library test.

## Evidence

From [run 21854651990](https://github.com/ROCm/TheRock/actions/runs/21854651990) on the `multi_arch/integration-kpack` branch (which has `THEROCK_KPACK_SPLIT_ARTIFACTS=ON`):

- All 14 build jobs succeeded
- 19 out of ~20 component test jobs **failed** on gfx94X-dcgpu

Tests that **pass** don't use split library kernels:
- `hip-tests` — compiles its own kernels inline
- `rocroller` — code generation tool, no GPU kernels
- `aqlprofile` — profiling, not kernel execution
- `hipblas` / `hipdnn` — thin wrappers, tests don't exercise device code paths

Tests that **fail** all try to load device code from split libraries:
- rocfft, hiprand, rocblas, rocsolver, rocprim, rocthrust, hipfft, hipsolver, hipcub, hipsparse, rocwmma, rccl, miopen, miopen_plugin, rocr-debug-agent, hipdnn-samples, rocrand, hipblaslt

Error categories:
1. `hipErrorInvalidImage`/`hipErrorInvalidKernelFile` (200/201) — most tests
2. `rocBLAS error: Cannot read TensileLibrary.dat: Illegal seek for GPU arch: gfx942` — rocsolver/rocblas (database files also in arch-specific artifacts)
3. RCCL 15-min timeout — cascading from kernel failures

## Key Files

- `build_tools/fetch_artifacts.py:62-77` — `list_artifacts_for_group()` filter
- `build_tools/artifact_manager.py:283-323` — inter-stage fetch filename construction
- `build_tools/install_rocm_from_artifacts.py` — test setup entry point
- `.github/workflows/test_component.yml:92` — `ARTIFACT_GROUP` set to `amdgpu_families`

## Fix Direction

The fetch logic needs to be aware that kpack split produces per-individual-target artifacts. Options:
1. **Backend listing + regex match**: Instead of substring matching on family name, list all available artifacts and match against known targets within the family
2. **Push-side metadata**: When pushing, write a manifest of artifact names that maps family → individual targets
3. **Naming convention**: Change the split naming to include the family name (but this breaks the architectural goal of per-target packages)

For `artifact_manager.py` (inter-stage), the TODO comment at `build_tools/artifact_manager.py:14-17` already notes this should switch to `--generic-only` for build stages since they only need host artifacts. Test stages need a different approach that fetches all per-target archives.
