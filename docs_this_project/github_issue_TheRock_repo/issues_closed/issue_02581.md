### Problem Description

The patchelf edited libraries are not providing the expected symbol version tag

Current library provides the following symbol version
/usr/lib/rpm/elfdeps --provides librocm_sysdeps_va.so.2
libva.so.2(AMDROCM_SYSDEPS_1.0)(64bit)
libva.so.2(VA_API_0.32.0)(64bit)
libva.so.2(VA_API_0.33.0)(64bit)
libva.so.2(VA_API_0.33.0)(64bit)
librocm_sysdeps_va.so.2()(64bit)

The required symbol version is **librocm_sysdeps_va.so.2(AMDROCM_SYSDEPS_1.0)(64bit)** 

Observed the following error while trying installation of ROCk packages
- **nothing provides librocm_sysdeps_va.so.2(AMDROCM_SYSDEPS_1.0)(64bit)** needed by amdrocm-sysdeps0.0.3-0.0.3-20220406645.x86_64 from rocm

The following patch creates the library by using patchelf
https://github.com/ROCm/TheRock/pull/2267/

The issue applies to  librocm_sysdeps_va-drm.so.2  and librocm_sysdeps_gallium_drv_video.so as well
