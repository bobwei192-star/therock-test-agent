## Description

rocThrust ASAN build fails in TheRock CI during CMake configure when `BUILD_BENCHMARK=ON`.

Google Benchmark's `add_cxx_compiler_flag()` modifies `CMAKE_CXX_FLAGS`, which triggers a `variable_watch()` callback from ROCMChecks.

https://therock-ci-artifacts.s3.amazonaws.com/24405879122-linux/logs/gfx94X-dcgpu-asan/rocThrust_configure.log

## Root Cause

Two issues when fetching Google Benchmark via `FetchContent` under ASAN:
1. **ROCMChecks warning**: Google Benchmark's `add_cxx_compiler_flag()` modifies `CMAKE_CXX_FLAGS` directly, triggering the `variable_watch()` callback registered by ROCMChecks.
2. **Regex detection failure**: Google Benchmark's `try_run` tests compile but can't execute under ASAN (sanitizer runtime not preloaded during configure).

Same class of issue as hipCUB (#4455), fixed in ROCm/rocm-libraries#6338.

## Upstream Fix

Apply the same pattern as hipCUB (efd316edd19d13bd7eaf91160c2960055ed6fd42):
- Override `rocm_check_toolchain_var` with a flag-gated no-op macro around `FetchContent_MakeAvailable`
- Pre-set `HAVE_STD_REGEX` and `RUN_HAVE_STD_REGEX` before fetching Google Benchmark
