### Steps to Reproduce

Pip installing the nightly wheels for pytorch grabs an out of date version.

This command:
```
python -m pip install --index-url https://rocm.nightlies.amd.com/v2-staging/gfx94X-dcgpu --pre torch torchaudio torchvision
```

Installs these versions:
```
pip freeze | grep rocm
pytorch-triton-rocm==3.5.1+git63e58716.rocmsdk20251122
rocm==7.10.0a20251113
rocm-sdk-core==7.10.0a20251113
rocm-sdk-libraries-gfx94X-dcgpu==7.10.0a20251113
torch==2.10.0a0+rocmsdk20251122
torchaudio==2.10.0a0+rocmsdk20251122
torchvision==0.25.0a0+rocmsdk20251122
```

Appears to be using 7.10.0a20251113 instead of recent versions from the workflow runs:
- https://github.com/ROCm/TheRock/actions/workflows/release_portable_linux_pytorch_wheels.yml?query=branch%3Amain
