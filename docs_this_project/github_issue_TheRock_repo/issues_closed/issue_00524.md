Tensile (https://github.com/ROCm/Tensile) is ...
> a tool for creating benchmark-driven backend libraries for GEMMs, GEMM-like problems (such as batched GEMM), and general N-dimensional tensor contractions on a GPU. The Tensile library is mainly used as a backend library for rocBLAS.

We currently disable Tensile in rocBLAS on Windows, which also disables it indirectly in MIOpen: https://github.com/ROCm/TheRock/blob/8be68cd6a5e9325f2e07fa083547e5a3b0f61179/math-libs/BLAS/CMakeLists.txt#L91-L105

We should enable it to ensure that we get the full set of kernels in rocBLAS, both for performance as build correctness (I see errors building MIOpen without it).

Here are my configure+build logs of the `rocBLAS` target with it enabled: https://gist.github.com/ScottTodd/cd90cf656ee34b7759a3c72ce816b96a.

@jammm has patches to get it building here: https://github.com/ROCm/TheRock/compare/main...jammm:TheRock:jam/gfx1151-win?expand=1
