# #1011 - Target environment had an LD_LIBRARY_PATH set with a prior version of ROCm in it. ROCm wheels use RUNPATH, which can be overridden by LD_LIBRARY_PATH. Either we should force these to use RPATH or harden library resolution to hard error if a conflicting LD_LIBRARY_PATH.

**State**: closed  
**Author**: stellaraccident  
**Created**: 2025-07-11T22:44:09Z  
**Updated**: 2025-07-14T14:02:08Z

---

On some clusters, various LD_LIBRARY_PATH based approaches have been used to create isolated ROCm installations. Since the default RUNPATH based approach allows LD_LIBRARY_PATH to override, this can cause resolution of libraries from alternate locations without intending to. Switching to the older "RPATH" allows us to create the hermetic Python wheels as intended.
