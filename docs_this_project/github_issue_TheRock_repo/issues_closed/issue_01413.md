### Problem Description

There are many reports of GPU hangs with gfx1151 under ROCm, which I also experience running llamacpp-rocm from the lemonade repo (https://github.com/lemonade-sdk/llamacpp-rocm/releases), eg:

https://gitlab.freedesktop.org/drm/amd/-/issues/4321
https://github.com/ROCm/ROCm/issues/5151
https://github.com/ROCm/rocm-libraries/issues/1132

In that first link, Mario Limonciello investigated the issue internally and found that a fix has already been created back in July. His comment stating that can be found here: https://gitlab.freedesktop.org/drm/amd/-/issues/4321#note_3045126

Mario also mentioned in the following comment: https://gitlab.freedesktop.org/drm/amd/-/issues/4321#note_3048205
That this fix is not yet backported to any release branch, including ROCm 7. Furthermore, it will not be included in the initial ROCm 7 release, but will actually hit ROCm in a later point release.

In my opinion leaving such a critical bug unfixed for so long is unacceptable. Since this llamacpp build specifically targets gfx1151, I was wondering if you could bring this issue up internally and push for an earlier fix than a ROCm 7 point release. Gfx1151 has been marketed and sold with a big focus on AI, but the AI experience right now is terrible due to this bug.

### Steps to Reproduce

Please build the following Dockerfile:

```
FROM ubuntu

RUN apt update && apt install -y wget sudo unzip libatomic1

RUN useradd -m docker
RUN echo "docker ALL=(ALL) NOPASSWD:ALL" >> /etc/sudoers
USER docker
WORKDIR /home/docker

RUN wget https://github.com/lemonade-sdk/llamacpp-rocm/releases/download/b1053/llama-b1053-ubuntu-rocm-gfx1151-x64.zip
RUN unzip ./*.zip -d llamacpp
RUN chmod +x llamacpp/llama-bench

WORKDIR /home/docker/llamacpp

ENTRYPOINT ["./llama-bench", "-m", "/home/docker/models/unsloth_GLM-4.5-Air-GGUF_UD-Q4_K_XL_GLM-4.5-Air-UD-Q4_K_XL-00001-of-00002.gguf", "-fa", "1", "-r", "5", "--mmap", "0"]
```

With the following command: `docker build . -t rocm-hang`

Then make sure you have a `models` folder in whatever folder you are going to launch the container from. Make sure this folder contains the following two files:

https://huggingface.co/unsloth/GLM-4.5-Air-GGUF/resolve/main/UD-Q4_K_XL/GLM-4.5-Air-UD-Q4_K_XL-00001-of-00002.gguf
https://huggingface.co/unsloth/GLM-4.5-Air-GGUF/resolve/main/UD-Q4_K_XL/GLM-4.5-Air-UD-Q4_K_XL-00002-of-00002.gguf

(My apologies for the big download, but I find it only triggers reliably with big models)

Then run the example with the following command: `docker run --device=/dev/dri --device=/dev/kfd -v=./models:/home/docker/models rocm-hang`

Mind you, this does not crash it 100% of the time. I would say it's about 50:50. When it does crash, journalctl shows a GPU hang. In some cases it recovers successfully, in other cases it brings down the entire DE with it, so be mindful if you have important stuff open :)
