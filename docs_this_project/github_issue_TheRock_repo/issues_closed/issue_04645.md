### Problem Description

### HIP runtime reports wrong `totalGlobalMem` on Strix Halo (gfx1151) — ~64 GiB instead of full UMA

On a Framework Desktop (AMD Ryzen AI Max+ 395, Radeon 8060S / gfx1151) with 128 GiB unified memory, a recent TheRock `develop` build regressed the value reported by `hipGetDeviceProperties().totalGlobalMem` and `hipMemGetInfo(&free, &total)` for `total`. The `free` value is still correct. `rocminfo` memory pools are unchanged between the two builds.

This breaks downstream tools that size allocations against `total` (e.g. llama.cpp / `llama-bench` fails to load large models that actually fit in the pool).

### Impact
llama-bench with a ~80 GiB model fails to load on the bad build (VRAM detected as 64077 MiB) but loads fine on the good build (VRAM detected as 126976 MiB), even though both sit on the same kernel/firmware/BIOS and rocminfo is unchanged.

### Steps to Reproduce

### Versions

- **Good:** TheRock `develop` @ `5250110dbb4e` (build 2228) — reports full 126976 MiB
- **Bad:**  TheRock `develop` @ `9f3522b2973d` (build 2314) — reports only 64077 MiB

### Reproducer

```cpp
// hipmem.cpp
#include <hip/hip_runtime.h>
#include <cstdio>
int main() {
  int n = 0; hipGetDeviceCount(&n);
  for (int i = 0; i < n; i++) {
    hipDeviceProp_t p{}; hipGetDeviceProperties(&p, i);
    size_t free = 0, total = 0;
    hipSetDevice(i); hipMemGetInfo(&free, &total);
    printf("dev %d: name=%s totalGlobalMem=%zu MiB  memGetInfo.total=%zu MiB  free=%zu MiB\n",
           i, p.name, p.totalGlobalMem >> 20, total >> 20, free >> 20);
  }
}
```
Build and run against each ROCm tree:
```
$ROCM/llvm/bin/clang++ -x hip hipmem.cpp --rocm-path=$ROCM \
  -L$ROCM/lib -lamdhip64 -Wl,-rpath,$ROCM/lib -o hipmem
./hipmem
```

Output
Good build — develop-2228-5250110dbb4e:
`dev 0: name=Radeon 8060S Graphics totalGlobalMem=126976 MiB  memGetInfo.total=126976 MiB  free=124609 MiB`

Bad build — develop-2314-9f3522b2973d:
`dev 0: name=Radeon 8060S Graphics totalGlobalMem=64077 MiB   memGetInfo.total=64077 MiB   free=124609 MiB`
