### Problem Description

Doing a basic configure & build failed on a rebuild due to some changelog file:
```
[rocm-core configure] CMake Error at utils.cmake:213 (message):
[rocm-core configure]   Failed to compress: gzip:
[rocm-core configure]   /home/ben/src/TheRock/build/base/rocm-core/build/DEBIAN/changelog.Debian.gz
[rocm-core configure]   already exists; not overwritten
```

The workaround is deleting the file manually and then trying to build again. I got to this point by building once with the default cmake build type and then reconfiguring as Debug before building again. If there are special ways to build a second time it'd be nice to mention them in https://github.com/ROCm/TheRock/tree/main?tab=readme-ov-file#cmake-build-usage, but I suspect this is a bug (missing a `-f` to the gzip command?).
