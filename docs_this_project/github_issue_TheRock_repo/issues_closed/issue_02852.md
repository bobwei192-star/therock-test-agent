Reproduce steps:
1) Install latest Comfyui
2) Install latest rocm (for 2026-01-11)
`pip install --upgrade --index-url https://d2awnip2yjpvqn.cloudfront.net/v2/gfx120X-all/ torch torchvision torchaudio numpy
`
3) Run any SD workflow like "galaxy bottle"

Expected behavior:
Clip Text Encode work almost instantly

Actual  behavior:
Clip Text Encode takes a few seconds. GPU load is very low during clip encoding.

- 2.5s with default v1-5-pruned-emaonly-fp16.safetensors
- 10s with any SDXL checkpoint, for example Juggernaut XL

A few moths ago, as I remember, Clip Text Encode worked much faster. I've tried downgrading rocm in my installation to confirm it, but failed.  It would be great if someone with old version of rocm measure Clip Text Encode timings.

Disclaimer:
I'm not sure if it's really issue, and it caused by rocm (not comfyui), but need advice.
