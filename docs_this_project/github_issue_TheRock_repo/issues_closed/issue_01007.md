With #983 merged the job to build PyTorch wheels is triggered _after_ TheRock was successfully build. when building the PyTorch wheels is triggered the specific TheRock version to use is passed as an input parameter. `publish_pytorch_dev_docker.yml` needs to be revised in a similar way. Right now the install script automatically picks to install TheRock build one day before by default:
https://github.com/ROCm/TheRock/blob/5dbd3375bfd815e0d72f1a18d8ff00e14bd93b4b/dockerfiles/pytorch-dev/install_rocm_from_release.sh#L53

To align this with our other release workflows a Docker image build at YYYYMMDD should pick up TheRock build at the same date!
