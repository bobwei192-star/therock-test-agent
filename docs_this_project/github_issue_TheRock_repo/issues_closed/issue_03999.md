### Problem Description

`rocm-sdk test` fails on **ROCmDevelTest.testSharedLibrariesLoad** when testing the devel wheel (Linux gfx1151, `rocm[libraries,devel]`). The test loads every path under the devel package with `ctypes.CDLL(path)`; one of those paths is a **directory**: `.../site-packages/_rocm_sdk_devel/etc/ld.so.conf.d`, which causes:

  OSError: .../etc/ld.so.conf.d: cannot read file data: Is a directory

**Context:** TheRock PR #3984, Run 23159857624, Job 67317639808  (Linux [gfx1151](https://github.com/ROCm/TheRock/actions/runs/23159857624/job/67317639808?pr=3984), Test rocm[libraries,devel] wheels). Code: `rocm_sdk/tests/devel_test.py` → `testSharedLibrariesLoad`; it passes each path to `ctypes.CDLL()` without checking if the path is a file. 

**Suggested fix:** In `testSharedLibrariesLoad` (or the logic that collects paths to load), only call `ctypes.CDLL(path)` for **files** (e.g. `path.is_file()` or filter by extension `.so`); skip directories.
