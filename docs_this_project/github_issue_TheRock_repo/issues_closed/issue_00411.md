Currently, we are getting TheRock artifacts through s3 buckets locally + in CI workflows. 

We want to create a standard way to install TheRock, so users can install TheRock with runner_id or release_id and get all the artifacts

ex: `python provision.py --version 1.0.5 --output-dir ./rocm/` -> installs TheRock artifacts (no flatten) with version 1.0.5 to `./rocm` directory

<br/><br/>

from the doc:

Provision Phase · Given a provision_id of TheRock as defined by Runner_id or a Release_id
- Removes previous binaries and libraries under ~/common_work_dir/therock
- Installs binaries and libraries under ~/common_work_dir/therock based on the version defined above
- Special note: If therockpath is provided from the assignment phase rather than a provision_id, only remove logs. Similarly, we can speed this up if the machine detects the image is already installed and only wipes sub_dirs other than “therock” before a test run.
