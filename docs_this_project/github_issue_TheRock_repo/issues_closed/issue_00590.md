Splitting this from the discussion thread at https://github.com/ROCm/TheRock/discussions/409#discussioncomment-13096228. Over there, we were choosing which compiler to use for the PyTorch source build. The usual default would be MSVC (cl.exe), but clang-cl seems to work more easily. PyTorch sets `HIP_CXX_FLAGS` here: https://github.com/pytorch/pytorch/blob/37f92bbe0a21f4c98a7e8283171bbbc42962ca4f/cmake/Dependencies.cmake#L1037-L1132.

While I can build successfully with clang-cl, I get 700MB+ of warning logs, most looking like this:

```
D:/projects/TheRock/external-builds/pytorch/src/aten/src\ATen/core/Dimname.h:38:1: warning: __declspec attribute 'dllimport' is not supported [-Wignored-attributes]
   38 | TORCH_API std::ostream& operator<<(std::ostream& out, const Dimname& dimname);
      | ^~~~~~~~~
D:/projects/TheRock/external-builds/pytorch/src\c10/macros/Export.h:98:19: note: expanded from macro 'TORCH_API'
   98 | #define TORCH_API C10_IMPORT
      |                   ^~~~~~~~~~

   51 | #define C10_IMPORT __declspec(dllimport)
      |                               ^~~~~~~~~
```

Places to look:
* https://clang.llvm.org/docs/UsersManual.html#microsoft-extensions
* https://github.com/pytorch/pytorch/blob/main/c10/macros/Export.h
* https://github.com/pytorch/pytorch/blob/main/CMakeLists.txt
* https://github.com/pytorch/pytorch/blob/main/setup.py

The ROCm projects generate lots of warnings too (https://github.com/ROCm/TheRock/issues/47), but this is an order of magnitude worse (~60MB of build logs across all of TheRock, 700MB here for PyTorch), and I can't run with interactive console output with that much logging.
