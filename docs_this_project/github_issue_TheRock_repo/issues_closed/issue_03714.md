### Problem Description

1. Duplicate Installation Locations
  - Symlinks are installed both at the root level (stage/amdclang) and in the bin directory (stage/bin/amdclang)
  - Only the bin/ location is correct; root-level symlinks should not exist

2. Broken Symlink Paths
  - Symlinks in bin/ use incorrect relative paths (lib/llvm/bin/amdclang)
  - From the bin/ directory context, this path fails to resolve
  - Correct path should be ../lib/llvm/bin/amdclang to navigate up one directory level
