## Problem

On the `multi_arch/integration-kpack` CI run ([#22654730725](https://github.com/ROCm/TheRock/actions/runs/22654730725/)), the gfx110X-all rocblas test on Windows fails with:

```
rocBLAS error: Cannot read C:\runner\_work\TheRock\TheRock\build\bin\rocblas/library/TensileLibrary.dat: No such file or directory for GPU arch : gfx1101
 List of available TensileLibrary Files :
```

The list of available files is empty and rocblas spins until the 15-minute timeout.

## Root Cause

On Windows, most of the library database files (including TensileLibrary.dat) are placed in the `bin/` directory rather than `lib/`. The kpack plugin likely only searches `lib/` for these files.

## Suggested Fix

Update the kpack plugin to search both `bin/` and `lib/` directories on Windows for library data files like TensileLibrary.dat.
