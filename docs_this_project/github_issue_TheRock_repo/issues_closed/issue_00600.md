Per https://rocm.docs.amd.com/projects/install-on-windows/en/latest/reference/component-support.html, Linux has `rocminfo` for system management while Windows has `hipInfo`. Ideally there would be a single tool name shared across both operating systems, but that's what we have today.

The source for hipInfo is here: https://github.com/ROCm/hip-tests/tree/amd-staging/samples/1_Utils/hipInfo. We can add that repository via a submodule or some other mechanism, configure it as a subproject, then add that binary to our dist as an artifact. We might want to build other tests or binaries from that repository or we might not.

Beyond general alignment between TheRock's "community builds" and the current ROCm/HIP releases, the `hipInfo.exe` tool is useful for getting information about the current system, and we use `rocminfo` as an initial sanity check during CI/CD testing (see also https://github.com/ROCm/TheRock/issues/544).
