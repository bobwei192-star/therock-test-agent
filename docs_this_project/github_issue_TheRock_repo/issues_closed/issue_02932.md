## Description

CMake produces a dev warning about mismatched `if`/`endif` arguments in `comm-libs/CMakeLists.txt`:

```
CMake Warning (dev) in comm-libs/CMakeLists.txt:
A logical block opening on the line

  /develop/therock/comm-libs/CMakeLists.txt:2 (if)

closes on the line

  /develop/therock/comm-libs/CMakeLists.txt:92 (endif)

with mis-matching arguments.
```

## Root Cause

Line 2 has:
```cmake
if(THEROCK_ENABLE_RCCL AND THEROCK_SANITIZER STREQUAL "")
```

But line 92 has:
```cmake
endif(THEROCK_ENABLE_RCCL)
```

The `endif()` argument doesn't match the `if()` condition.

Introduced in #2785 (commit f7add7732).

## Suggested Fix

Change line 92 to use modern CMake style with no argument:
```cmake
endif()
```

This is the recommended approach as repeating the condition in `endif()` is deprecated style.
