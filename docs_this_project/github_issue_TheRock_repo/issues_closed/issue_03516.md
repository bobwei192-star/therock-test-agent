### Environment

* OS: Ubuntu 25.04
* Compiler: GCC 15 (defaults to -std=gnu23)
* Python: 3.13
* GPU target: gfx1151 (AMD Strix Halo iGPU / Radeon 8060S)
* Terminal: Kitty
* Build method: standard cmake + ninja with .venv

### Instructions to reproduce:

On a fresh install of Ubuntu 25.10:
```
cmake -B build -GNinja . -DTHEROCK_AMDGPU_FAMILIES=gfx1151 -DCMAKE_C_COMPILER_LAUNCHER=ccache -DCMAKE_CXX_COMPILER_LAUNCHER=ccache
cmake --build build
```


File: `rocm-systems/projects/rocprofiler-systems/cmake/DyninstElfUtils.cmake`

Error:
```
i386_regs.c:88:11: error: initializer-string for array of 'char' truncates NUL terminator
but destination lacks 'nonstring' attribute [-Werror=unterminated-string-initialization]
```
elfutils has static const char baseregs[][2] initialized with two-character string literals like "ax" — these are 3 bytes including NUL, which truncates under the new GCC 15 warning.

Fix: Add -Wno-unterminated-string-initialization to the elfutils CFLAGS in the CONFIGURE_COMMAND of the DyninstElfUtils ExternalProject.
