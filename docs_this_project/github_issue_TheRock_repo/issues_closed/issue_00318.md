CMake released a major version 4.0.0 on March 28, 2025:

* Announcement: https://www.kitware.com/cmake-4-0-0-available-for-download/
* Release notes: https://cmake.org/cmake/help/v4.0/release/4.0.html

We should support this version across the full build of ROCm.

## Known issues

### Windows amd-llvm

Our Windows CI caught issues with amd-llvm using this new version: https://github.com/ROCm/TheRock/actions/runs/14182869714/job/39732671543.

```
[amd-llvm configure] CMake Error at C:/home/runner/_work/TheRock/TheRock/compiler/amd-llvm/amd/device-libs/cmake/OCL.cmake:12 (cmake_policy):
[amd-llvm configure]   Policy CMP0053 may not be set to OLD behavior because this version of CMake
[amd-llvm configure]   no longer supports it.  The policy was introduced in CMake version 3.1.0,
[amd-llvm configure]   and use of NEW behavior is now required.
[amd-llvm configure] 
[amd-llvm configure]   Please either update your CMakeLists.txt files to conform to the new
[amd-llvm configure]   behavior or use an older version of CMake that still supports the old
[amd-llvm configure]   behavior.  Run cmake --help-policy CMP0053 for more information.
```

That stems from https://cmake.org/cmake/help/latest/policy/CMP0053.html

> **Warning:** The `OLD` behavior of this policy was removed in CMake version 4.0. This policy must be set to `NEW` by a call to [`cmake_minimum_required()`](https://cmake.org/cmake/help/latest/command/cmake_minimum_required.html#command:cmake_minimum_required) or [`cmake_policy()`](https://cmake.org/cmake/help/latest/command/cmake_policy.html#command:cmake_policy).

and this code: https://github.com/ROCm/llvm-project/blob/82b836e284b8e24104789bdc1b35a710d65dfaea/amd/device-libs/cmake/OCL.cmake#L12
