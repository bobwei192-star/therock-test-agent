### Problem Description

Due to another bug, I am having a rocm sdk and torch wheels with mixed versions installed.

```
pip list installed | grep rocm
rocm                            7.0.0rc20250708
rocm-sdk-core                   7.0.0rc20250708
rocm-sdk-devel                  7.0.0rc20250716
rocm-sdk-libraries-gfx94X-dcgpu 7.0.0rc20250708
torch                           2.7.0a0+rocmsdk20250709
torchaudio                      2.7.0a0+rocmsdk20250709
torchvision                     0.22.0+rocmsdk20250709
```

Library soname's in 20250708 version and 20250716 versions are different for some libraries like libamdhip64.so where 20250708 libamdhip64.so.6 and 20250716 version has libamdhip64.so.7
and somehow the command "python -m rocm_sdk path --root" will create a broken symlinks. For example for libamdhi64.so it will try to create a symlink to non-existing libamdhip64.so.7 file.

```
find -name libamdhi*
./.venv/lib/python3.12/site-packages/_rocm_sdk_devel/lib/libamdhip64.so
./.venv/lib/python3.12/site-packages/_rocm_sdk_devel/lib/libamdhip64.so.7.0.51830-0df79267f
./.venv/lib/python3.12/site-packages/_rocm_sdk_devel/lib/libamdhip64.so.7
./.venv/lib/python3.12/site-packages/_rocm_sdk_core/lib/libamdhip64.so.6
((.venv) ) milaitio@SharkMi300X-2:~/1$ 
((.venv) ) milaitio@SharkMi300X-2:~/1$ 
((.venv) ) milaitio@SharkMi300X-2:~/1$ ls -la ./.venv/lib/python3.12/site-packages/_rocm_sdk_devel/lib/libamdhip64.so.7
lrwxrwxrwx 1 milaitio milaitio 41 Jul 16 18:29 ./.venv/lib/python3.12/site-packages/_rocm_sdk_devel/lib/libamdhip64.so.7 -> ../../_rocm_sdk_core/lib/libamdhip64.so.7

```

library-soname hardcoding occurs at least here:

https://github.com/ROCm/TheRock/blob/edc11168cc4aeba804ab0576ce409de29ea81102/build_tools/packaging/python/templates/rocm/src/rocm_sdk/_dist_info.py#L163-L177
