### Suggestion Description

`_ck_enabled` is used to keep the information if CK is valid for a target architecture, and later is used for CK and MIOpen to decide if CK should be built or used as a dependency.

CK should rely only on its own THEROCK_ENABLE_COMPOSABLE_KERNEL flag, which should be set as ON only for the platforms CK supports, as it's done for every other component. MIOpen could then rely on the same flag THEROCK_ENABLE_COMPOSABLE_KERNEL and THEROCK_MIOPEN_USE_COMPOSABLE_KERNEL to decide whether it should build with the CK dependency or not.
