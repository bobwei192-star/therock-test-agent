This is a follow-up from the discussion in https://github.com/ROCm/TheRock/issues/1262

The problem was introduced in #1269, and breaks the most common way to run something out of a virtualenv, i.e. `/path/to/virtualenv/bin/python /path/to/script.py`

The issue stems from the implementation in https://github.com/ROCm/TheRock/blob/main/build_tools/packaging/python/templates/rocm/src/rocm_sdk/_dist_info.py#L99 - it calls and expects to have available a command called `amdgpu-arch`, which ultimately resolves to https://github.com/ROCm/TheRock/blob/main/build_tools/packaging/python/templates/rocm-sdk-core/src/rocm_sdk_core/_cli.py#L42, where it just runs a binary. 

I suspect the solution with the least amount of change is something like

```python
os.environ["PATH"] = f"{sys.prefix}/bin{os.pathsep}{os.environ["PATH"]}"
```

at the beginning of `discover_current_target_family`, which will add the virtualenv bin dir to the search path; I've tested it on linux, I know it should work on OS X, and I suspect it will work in Windows as well.
