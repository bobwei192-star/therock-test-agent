At the moment, build logs get uploaded in a dedicated step of the workflow:

https://github.com/ROCm/TheRock/blob/463c0763fc9185c83252ab1e723510048493085d/.github/workflows/build_linux_packages.yml#L137-L145

As a result, logs only get uploaded if the build passes, but not if a target failed to build as this step is then skipped. Build logs should always be uploaded however. Options are

* Creating a step that is not skipped even if the build failed
* Integrating a the upload logic into the build system (e.g. upload the script as part of the build process)
  * Needs to be optional and configurable
 
In both cases, the upload could be moved to a script.
