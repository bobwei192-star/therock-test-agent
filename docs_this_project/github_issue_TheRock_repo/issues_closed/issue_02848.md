### Problem Description

Rocm 7.1.1 makes gfx1010 cards about 2-3x slower than rocm 6. When running ComfyUI, the tiled operations such as tiled VAE encode/decode take over 10 minutes, and are slower than even the CPU based VAE decode and encode.
I checked against both the system Rocm 7.1.1 SLES packages, as well as the Rocm python wheels. In both cases PyTorch needs to be compiled with Flash attention and Memory efficient attention disabled (gfx1010 cannot use it anyway), but when pytorch is compiled against Rocm 6.4 with the same flags it works as expected with tiled VAE encode/decode taking a few seconds depending on resolution size.

### Operating System

openSUSE Leap 15.6

### CPU

AMD Ryzen 5 4600H with Radeon Graphics

### GPU

AMD Radeon RX 5600M   

### ROCm Version

ROCm 7.1.1

### Steps to Reproduce

- Install the rocm7.1.1 wheels from `https://rocm.nightlies.amd.com/v2/gfx101X-dgpu/`
- Compile Pytorch, torchvision and torchaudio using the instructions in `https://github.com/ROCm/TheRock/blob/main/external-builds/pytorch/README.md`, pass the flags for disabling flash attention
- Run ComfyUI with a workflow that uses tiled vae encode/decode (that is the most obvious example of the regression, sampling speed seems to be the same, and scales accordingly with resolution
