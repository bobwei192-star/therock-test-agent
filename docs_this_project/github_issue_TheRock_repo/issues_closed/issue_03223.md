## Problem Description

The bundled ncurses build fails on modern Linux systems (tested on Ubuntu 25.04 with GCC 14.2) due to a conflict between ncurses's autoconf `pid_t` detection and modern glibc headers.

### Root Cause

The ncurses configure script's `pid_t` detection test fails on modern systems. When the test fails, configure adds `#define pid_t int` to `confdefs.h`. This conflicts with glibc's `typedef __pid_t pid_t` in `<unistd.h>`, causing subsequent configure tests to fail:

```
configure:28109: checking for wchar_t
configure:28130: /usr/bin/g++ -c -O2 ... conftest.cc >&5
/usr/include/unistd.h:260:9: error: multiple types in one declaration
  260 | typedef __pid_t pid_t;
confdefs.h:1:15: error: declaration does not declare anything [-fpermissive]
    1 | #define pid_t int
```

This cascades to break `wchar_t` detection, which causes `--enable-widec` to silently fail:
- `USE_WIDEC_SUPPORT` is set to 0
- `NCURSES_WIDECHAR` is set to 0
- `SIZEOF_WCHAR_T` is set to 0

When the library is then built, `NCURSES_CH_T` is defined as `cchar_t` (unconditionally), but `cchar_t` is only typedef'd inside `#if NCURSES_WIDECHAR` blocks, leading to compile errors:

```
../include/curses.h:233:22: error: unknown type name 'cchar_t'; did you mean 'wchar_t'?
  233 | #define NCURSES_CH_T cchar_t
```

### Build Output

```
[therock-ncurses] ../include/curses.h:233:22: error: unknown type name 'cchar_t'; did you mean 'wchar_t'?
[therock-ncurses]   233 | #define NCURSES_CH_T cchar_t
[therock-ncurses]       |                      ^~~~~~~
[therock-ncurses] /path/to/ncurses/source/ncurses/curses.priv.h:2200:59: note: in expansion of macro 'NCURSES_CH_T'
```

### Solution

Pass `ac_cv_type_pid_t=yes` to the ncurses configure script. This overrides the broken detection and tells configure that `pid_t` is available (which it is on all POSIX/Linux systems).

With this fix:
- `wchar_t` detection succeeds
- `USE_WIDEC_SUPPORT=1` and `NCURSES_WIDECHAR=1` are correctly set
- `cchar_t` is properly typedef'd
- The build succeeds

### Environment

- Ubuntu 25.04 (Plucky Puffin)
- GCC 14.2.0
- ncurses 6.5

This likely affects any modern Linux distribution with recent glibc/GCC versions.
