# #5177 - [Issue] Windows PyTorch `nightly` wheel build fails in `caffe2_update_option()` due to invalid `set(... FORCE) in cmake/public/utils.cmake`

The Windows PyTorch wheel release job is failing during CMake configure because `caffe2_update_option()` eventually calls `set()` with `FORCE` but without `CACHE`, which is invalid CMake syntax.

## Environment:
- Workflow: `Release Windows PyTorch Wheels`
- platform: `windows`
- impacted Archs: `gfx90a, gfx1152,gfx1150,gfx1153`
- Pytorch Version: `nightly`

## Failing job:
- https://github.com/ROCm/TheRock/actions/runs/25655980101/job/75304433235
- https://github.com/ROCm/TheRock/actions/runs/25655003673/job/75301250488
- https://github.com/ROCm/TheRock/actions/runs/25654851859/job/75300758353
- https://github.com/ROCm/TheRock/actions/runs/25654791337/job/75300570199

```text

ROCm is enabled.
CMake Error at cmake/public/utils.cmake:123 (set):
  set given invalid arguments: FORCE specified without CACHE
```
