### Summary:
The Multi-Arch CI ASAN workflow fails while building rocshmem because the build passes `-fsanitize=address` to AMDGPU offload compilation for a broad set of GPU architectures (e.g. gfx1100, gfx1102, gfx1200, gfx90a, etc.). Clang reports that ASan is not supported for those offload arch targets unless the arch includes `xnack+`. This diagnostic is emitted as -Woption-ignored, but because the build uses `-Werror`, it becomes a hard error and stops the build.

### Full Logs:

Job: https://github.com/ROCm/TheRock/actions/runs/24545377594/job/71762059329
Workflow: .github/workflows/multi_arch_ci_asan.yml (Multi-Arch CI ASAN)
