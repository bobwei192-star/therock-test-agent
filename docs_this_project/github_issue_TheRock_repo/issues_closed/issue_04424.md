## Summary

When ROCm is installed via pip (`uv pip install rocm`), the `rocpd2pftrace` tool (and other `rocpd2*` tools) are not accessible on `PATH`. They are shipped inside the `rocm-sdk-core` package at `_rocm_sdk_core/bin/` but are missing from the `console_scripts` entry points in `setup.py`.

These tools are documented as user-facing in the [rocprofiler-sdk docs](https://rocm.docs.amd.com/projects/rocprofiler-sdk/en/develop/how-to/using-rocpd-output-format.html#dedicated-conversion-tools).

## Tools affected

The following tools are present in `_rocm_sdk_core/bin/` but have no `console_scripts` entry:
- `rocpd`
- `rocpd2csv`
- `rocpd2otf2`
- `rocpd2pftrace`
- `rocpd2summary`

By contrast, `rocprofv3`, `rocprofv3-attach`, and `rocprofv3-avail` are properly registered and work fine.

## Where to fix

`build_tools/packaging/python/templates/rocm-sdk-core/setup.py` — the Linux `console_scripts` list needs entries for the `rocpd*` tools, similar to how `rocprofv3` is registered.

## Additional issue: wrong RPATH on libpyrocpd.so

Even when invoking `rocpd2pftrace` directly from its installed location, it fails with:
```
ImportError: librocm_sysdeps_sqlite3.so: cannot open shared object file: No such file or directory
```

`libpyrocpd.cpython-312-x86_64-linux-gnu.so` has RPATH:
```
$ORIGIN:$ORIGIN/../../..:$ORIGIN/../../../rocprofiler-sdk:$ORIGIN/../../../../../_rocm_sdk_core/lib/llvm/lib
```

`$ORIGIN/../../..` resolves to `_rocm_sdk_core/lib/`, but `librocm_sysdeps_sqlite3.so` (and `librocm_sysdeps_elf.so.1`, `librocm_sysdeps_dw.so.1`, `librocm_sysdeps_drm.so.2`, `librocm_sysdeps_drm_amdgpu.so.1`) are in `_rocm_sdk_core/lib/rocm_sysdeps/lib/`. The RPATH is missing `$ORIGIN/../../../rocm_sysdeps/lib`.

## Environment

- `rocm 7.13.0a20260407` (from `--extra-index-url https://rocm.nightlies.amd.com/v2-staging/gfx1151`)
- Ubuntu 24.04, pip-installed ROCm in a venv
- `rocprofv3 1.2.2 (478a7a43)`
