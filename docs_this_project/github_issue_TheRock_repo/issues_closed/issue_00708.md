Hello,

I would suggest porting all shell scripts to python or powershell equivalent. Currently it's suggested to run on Windows:

https://github.com/ROCm/TheRock/blob/a742718b33993332131863885a621fceef6f9b48/external-builds/pytorch/README.md?plain=1#L103

There are two problems with this:

1. `bash` command is aliased to run default WSL distribution when WSL is enabled. Which is definitely not what we want to do. This is easy to workaround by changing PATH or using sh.exe instead. Depending on environment.
2. `link` has a conflict, if you run bash from MSYS2 (or any derivative of it like git for windows), it will prepend `/usr/bin/` to path, and `link` is part of GNU Core Utilities. So during linking instead of using MSVC's link.exe it will try (and fail) to use /usr/bin/link.exe

On top of that removing shell dependency from Windows build makes it easier to run. Of course likely everyone has already bash.exe/sh.exe because of git for windows install, but either way it simplifies things to not use it.
