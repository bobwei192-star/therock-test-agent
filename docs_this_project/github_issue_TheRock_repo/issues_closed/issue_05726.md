### Problem Description

Hello,

I noticed that when I install the nightly pip package

```
python -m pip install --pre \
  --index-url https://rocm.nightlies.amd.com/whl-multi-arch/ \
  "rocm[devel]"
```

the file system structure is different from the ubuntu package

```
sudo apt-get install rocm
```

in that the pip package pointed by `rocm-sdk path --bin` does not contain amdllvm but it is present in the ubuntu package under the rocm/bin directory. If the amdclang binary in `rocm-sdk path --bin` is used (or `rocm-sdk path --root` such as when ROCM_PATH is set on CMake scripts), then it will look for the sibling amdllvm symlink which is not found and will land in the following error:

https://github.com/ROCm/TheRock/blob/5045d587d3e2a525f717a027bb1f5a6fd4b81e1e/build_tools/_therock_utils/exe_stub_gen.py#L59-L68

Please note that the amdclang binary is also symlinked to the virtual environment's bin folder, which is different from the `rocm-sdk path --bin` directory. When used from ${VIRTUAL_ENV}/bin, it works correctly.
