# #1686 - [Issue]: Compiler flags used in Caffe2 CMakefile cause MSVC to fail

### Problem Description

Windows builds against torch.lib will fail due to compiler flags in "torch/share/cmake/Caffe2/Caffe2Targets.cmake" that are not compatible with MSVC.  In particular, there are several "-Wno-*" flags that MSVC does not handle.


### Operating System

Windows 10 22H2

### CPU

AMD Ryzen Threadripper PRO 5955WX 16-Cores

### GPU

AMD Radeon RX 9060 XT

### ROCm Version

Multiple (e.g. 7.0.0rc20250908, 7.9.0rc20251001)

### ROCm Component

_No response_

### Steps to Reproduce

After installing ROCm and PyTorch packages create a simple torch.lib example such as described here: https://docs.pytorch.org/cppdocs/installing.html#minimal-example

Then build the example with CMake...

set ROCM_PATH
`for /f "tokens=*" %A in ('python -m rocm_sdk path --root') do set ROCM_PATH=%A`
set PYTORCH_ROCM_ARCH
`for /f "tokens=*" %A in ('python -m rocm_sdk targets') do set PYTORCH_ROCM_ARCH=%A`

get torch cmake path
`for /f "tokens=*" %A in ('python -c "import torch; print(torch.utils.cmake_prefix_path)"') do set TORCH_CMAKE=%A`
configure build
`cmake -B build -DCMAKE_PREFIX_PATH=%TORCH_CMAKE%`

build step will fail
`cmake --build build --config Release --verbose`
