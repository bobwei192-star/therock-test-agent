## Summary
The PyTorch wheel build in CI is failing at link time when building multiple C++ test executables. The linker reports an unresolved symbol `rsmi_init` coming from `libtorch_hip.so`, which indicates ROCm SMI (librocm_smi64 / rocm-smi) is not being linked into the final link step.

**Pytorch Version:** `2.8, 2.9, 2.10, 2.11`

## Error log:

```
/opt/rh/gcc-toolset-13/root/usr/libexec/gcc/x86_64-redhat-linux/13/ld: /__w/TheRock/TheRock/external-builds/pytorch/pytorch/build/lib/libtorch_hip.so: undefined reference to `rsmi_init'
collect2: error: ld returned 1 exit status
ninja: build stopped: subcommand failed.
```
