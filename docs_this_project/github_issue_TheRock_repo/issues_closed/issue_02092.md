Currently, TheRock disables all LLVM tools to save on build time/size/etc. This makes sense for TheRock CI, builds, releases, etc.

We would also like to transition to using TheRock build infrastructure for our compiler CI and local testing, and most or all of the LLVM tools are required to adequately test, particularly by llvm-lit.

To fix this, we should add optional (opt-in) support to build all of the LLVM tools. This option can used by the compiler CI, and ignored by TheRock CI/releases.
