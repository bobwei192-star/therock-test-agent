### Problem Description

TheRock expects libdir to be "lib" and hardcodes it in the main CMakeLists.txt:
`set(CMAKE_INSTALL_LIBDIR "lib")`
with distiributions setting libdir to sometthing else e.g. "lib64" for 64bit libs, the all automake libraries end in the, for TheRock, wrong path. Set
`--libdir "${CMAKE_INSTALL_PREFIX}/${CMAKE_INSTALL_LIBDIR}"`
for all `configure` calls to fix this issue
