### Problem Description

Since the MIOpen devs obviously don't care, I'll duplicate this here. Maybe someone here will actually care about anyone using Windows or the consumer grade peasants. https://github.com/ROCm/rocm-libraries/issues/4790

It is configured to fallback to the configuration for `gfx908` if the current gpu is not `gfx950`, `gfx942`, `gfx90a`. Don't believe this stupidity? Here is the proof. https://github.com/ROCm/rocm-libraries/blob/e726564afa34c660b766c64fd64d1ceb13acdf1a/projects/miopen/src/conv/heuristics/ai_heuristics.cpp#L482-L493

Additionally at the start of this json file "gfx908_metadata.tn.model", you can see it is configured with `"num_cu": "120"`, which seems excessive especially for the gfx115X series.

### Operating System

Win 11

### CPU

Any

### GPU

Any

### ROCm Version

7.12

### ROCm Component

MIOpen
