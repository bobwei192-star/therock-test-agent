## Problem

Extended test gating is currently spread across two separate inputs with different sources and scopes:

- `run_functional_tests` (boolean) — a global flag generated by `configure_ci.py`, threaded through `setup.yml` -> `ci_nightly.yml` -> `ci_linux.yml` / `ci_windows.yml` -> `test_artifacts.yml`
- `benchmark_runs_on` (string) — a per-architecture runner label defined in `amdgpu_family_matrix.py`, embedded in the variant matrix and passed via `ci_nightly.yml` -> `ci_linux.yml` / `ci_windows.yml` -> `test_benchmarks.yml`

There is no single place to globally enable/disable benchmarks — benchmark gating is implicit (runner label is non-empty). Adding `run_functional_tests` as a standalone workflow_call input in every workflow adds more inputs.

## Proposal

Introduce a single `run_extended_tests` JSON object as a global flag in `configure_ci.py`, containing enablement booleans for both extended test types:

```yaml
# Global object from configure_ci.py (passed through the workflow chain)
run_extended_tests: '{"functional": true, "benchmarks": true}'
```

Per-architecture `benchmark-runs-on` runner label stays in the variant matrix (in `amdgpu_family_matrix.py`) since it varies per GPU family. The global `benchmarks` flag controls whether benchmarks run at all.

Benchmark gating in `ci_linux.yml` / `ci_windows.yml` would check both:

```yaml
# Benchmarks run only if globally enabled AND a runner is available for this variant
if: >-
  ${{
    fromJSON(inputs.run_extended_tests).benchmarks == true &&
    matrix.variant.benchmark-runs-on != ''
  }}
```
