### Problem Description

The wheel splitter / publish pipeline does not emit the amd-torch-device-gfx11 family bundle for torch 2.11.0 on the 7.14.0rc0 prerelease index, while per-arch gfx11xx wheels and the 2.12.0 gfx11 bundle are published correctly.

This blocks 7.14.0rc0 PyTorch 2.11 device-all docker image builds.

Jira: [ROCM-27293](https://amd-hub.atlassian.net/browse/ROCM-27293?xpis=eyJicmlkZ2UiOiJyb3ZvLWNoYXQuc2lkZWJhciIsImlkIjoiMTc4Mjc2MjM4MDQxOCIsInNvdXJjZSI6ImppcmEtSlNXIn0%3D)

Error
`Could not find a version that satisfies the requirement amd-torch-device-gfx11==2.11.0+rocm7.14.0rc0
       (from versions: 2.9.1+rocm7.14.0rc0, 2.10.0+rocm7.14.0rc0, 2.12.0+rocm7.14.0rc0)
ERROR: No matching distribution found for amd-torch-device-gfx11==2.11.0+rocm7.14.0rc0`

Observed behavior
- torch-2.11.0+rocm7.14.0rc0 downloads successfully
- Per-arch wheels (amd-torch-device-gfx1100 … gfx1153, gfx1200, gfx1201) for 2.11.0 download successfully
- The gfx11 family bundle does exist for 2.12.0+rocm7.14.0rc0 on the same index
- The Jun 24 nightly (7.14.0a20260624) did publish the gfx11 family bundle for torch 2.11.0, indicating a regression after Jun 24

Root cause analysis
The https://github.com/ROCm/rockrel/actions/runs/28210872254/job/83571800737 produces amd_torch_device_gfx11-2.12.0+rocm7.14.0rc0, but the https://github.com/ROCm/rockrel/actions/runs/28210872254/job/83571800749 does not produce the equivalent gfx11 family bundle for 2.11.0.
