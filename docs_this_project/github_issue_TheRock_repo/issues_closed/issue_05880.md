## Context

- Failing run: [Run #27591696691](https://github.com/ROCm/rockrel/actions/runs/27591696691)
- Platform: Linux , windows
- Failing job: [release / Build | py 3.10 | torch release/2.10 / Build | py 3.10 | torch release/2.10](https://github.com/ROCm/rockrel/actions/runs/27591696691/job/81574008561)
- PyTorch version: `release/2.10` (and other non-2.9 refs)
- Python version: 3.10 (and others)

## Error

```
ValueError: Manifest PyTorch ref 'release/2.9' does not match 'release/2.10'
```

## Suspected Root Cause

The `prepare_pytorch_manifest` job introduced in [#5633](https://github.com/ROCm/TheRock/pull/5633) passes `--output pytorch_manifest.json` (a fixed filename) to `generate_pytorch_source_manifest.py`. Because the release matrix spawns one `prepare_pytorch_manifest` job per `pytorch_git_ref` in parallel, all jobs resolve to the same S3 key:

```
<run_id>-linux/manifests/pytorch/pytorch_manifest.json
```

The last generate-manifest job to finish overwrites the S3 object. In run 27591696691 the `release/2.9` job (finished 03:22:00 Z) overwrote the `release/2.10` manifest (finished 03:20:38 Z). The build job for `release/2.10` then downloaded the `release/2.9` content and failed validation.

The generator already has a per-ref filename function (`manifest_filename()` → `therock-manifest_torch_linux_release_2_10.json`), but it is bypassed when `--output` is provided.
