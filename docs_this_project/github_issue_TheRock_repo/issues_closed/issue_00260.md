## Summary

When building the hipBLAS clients, Fortran modules are compiled. When using a machine with enough parallelism due to a high core count, compiling the `hipblas_fortran_client` target fails with
> hipblas.mod’ for reading at (1): No such file or directory"

It succeeds in subsequent runs as `hipblas.mod` is created in the first run but not timely enough.

One possibility to resolve this is to replace `add_dependency()` with `target_link_libraries()` thus that `hipblas.mod` is present when needed.
