### Problem Description

ROCm device bitcode libraries are not installed in the Clang resource directory, causing Clang to fail to locate device libraries when compiling OpenMP/HIP GPU offload code.

**This is the root cause issue.** The symlink workaround in #3198 / #3203 is a temporary fix and should be removed once this issue is resolved.

### Background

Clang searches for device libraries in two locations:

1. **Clang resource directory**: `<LLVM_DIR>/lib/clang/<ver>/lib/amdgcn/bitcode`
2. **Legacy ROCm layout**: `${ROCM_ROOT}/amdgcn/bitcode`

TheRock currently installs device libraries at: lib/llvm/amdgcn/bitcode/

Because this path falls outside Clang’s default search directories, AOMP unit tests fail.
