# #2019 - `offload-arch` on Windows loads lower versioned amdhip64_n.dll

When running offload-arch.exe on Windows with `--verbose`, it finds and selects older runtime dlls in `/system32/`. 
```
(myenv) C:\Users\harkgill\arch>offload-arch --verbose
Found HIP runtime: C:/WINDOWS/system32/amdhip64_6.dll
gfx1103
```
This occurs even when newer amdhip64_7.dll files are present in `/system32/` and in `\.venv\Lib\site-packages\_rocm_sdk_core\bin` which is on PATH and should be checked.
