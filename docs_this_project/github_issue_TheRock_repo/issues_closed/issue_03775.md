Some consumer GPUs have lower memory, thus breaking tests and causing delays

For example, `windows-strix-halo-gpu-rocm-7` has 16 GB of memory, and when running tests, it often causes crashes or times out

- rocprim smoke tests would timeout at 30 mins, when it usually takes ~ 6 mins

As consumer GPUs will have varying memory, we need to do logic where we determine how much memory, then pick the correct test filter to match memory usage.

This will need to apply to the CI as well as local tests
