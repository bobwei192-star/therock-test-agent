# #1418 - rocm-sdk test shouldn't require Fortran

**State**: closed  
**Author**: marbre

**Updated**: 2025-09-10T14:32:41Z

---

Currently `rocm-sdk test` is failing if gfortran isn't installed. This should not be the case.

---

If fortran is still optional (I believe it should be), users should be able to run `rocm-sdk test` and other commands that don't use fortran and see at _most_ warnings, not errors.

The errors observed are `ERROR: testSharedLibrariesLoad (rocm_sdk.tests.libraries_test.ROCmLibrariesTest.testSharedLibrariesLoad) [Check shared library loads] (so_path=PosixPath('/__w/TheRock/TheRock/.venv/lib/python3.11/site-packages/_rocm_sdk_libraries_gfx94X_dcgpu/lib/libhipsolver_fortran.so.1'))`

The requirement comes from `*_fortran.so` files:
```
λ ls C:\Users\Nod-Shark16\Downloads\therock-dist-linux-gfx94X-dcgpu-7.0.0rc20250905\lib | grep solver

libhipsolver.so.1@

libhipsolver_fortran.so.1@
libhipsolver_fortran.so.1.0
librocsolver.so@
librocsolver.so.0@
librocsolver.so.0.7
```

Here is the test code: https://github.com/ROCm/TheRock/blob/main/build_tools/packaging/python/templates/rocm/src/rocm_sdk/tests/libraries_test.py . We could make those fortran tests optional (perhaps conditional on a check for the dependent libraries). See how certain tests are skipped in https://github.com/ROCm/TheRock/blob/main/build_tools/packaging/python/templates/rocm/src/rocm_sdk/tests/devel_test.py and https://github.com/ROCm/TheRock/blob/main/build_tools/packaging/python/templates/rocm/src/rocm_sdk/tests/core_test.py
