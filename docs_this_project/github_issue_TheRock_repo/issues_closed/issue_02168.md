### Problem Description

I've been testing/building llama.cpp with rocWMMA and noticed that the rocWMMA header isn’t included when building Python packages via `build_python_packages.py`. I double checked that there is built rocWMMA in `build` directory of ROCm and that there is no rocWMMA in `_rocm_sdk_devel`. There is a [discussion on Discord](https://discord.com/channels/1239631572886491286/1438824822531424276) regarding this issue.

### Steps to Reproduce

To reproduce the error, build ROCm with TheRock and then build llama.cpp with WMMA support enabled. You can use scripts from here #1652, and run:
1. `python llamacpp_repo.py`
2. `python llamacpp_build.py`

It works perfectly fine when you specify direct path to the built ROCm via TheRock:
```bash
python llamacpp_build.py --rocm-targets gfx1100 --rocm-dir /therock/output/build/dist/rocm/
```
