### Problem Description

When running the official demo code `stabilityai/stable-diffusion-3-medium-diffusers` with the `TheRock package (2.10.0a0+rocm7.11.0a20251124)`, the FlowMatchEulerDiscrete scheduler crashes with an "IndexError: index 0 is out of bounds" when generating 1024x1024 SD3 images under heavy memory pressure.

We can easily observe the issue on the AMD Radeon RX 9070 GRE (12G), but there is no issue if we use the AMD Radeon™ AI PRO R9700 (32G) to infer the same code with the same Python virtual environment and TheRock package.

It seems that the issue only occurs under heavy memory pressure; for 512×512 images, everything works fine."

1. Copy example from :  `https://huggingface.co/stabilityai/stable-diffusion-3-medium`
2. Modfiy the code  `pipe = pipe.to("cuda")` to `pipe.enable_model_cpu_offload(device=torch.device("cuda"))`
```
import torch
from diffusers import StableDiffusion3Pipeline

pipe.enable_model_cpu_offload()
# pipe = pipe.to("cuda")

image = pipe(
    "A cat holding a sign that says hello world",
    negative_prompt="",
    num_inference_steps=28,
    guidance_scale=7.0,
).images[0]
image
```
