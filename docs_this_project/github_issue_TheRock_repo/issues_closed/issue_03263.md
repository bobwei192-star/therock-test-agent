Past several days we've been seeing long queue times for CPU builders, due to a recent Azure and Github outage on Monday. There seems to also be capacity pressure for the US East region where our runners are being hosted. The AKS node pools have been experiencing limited scaling due to limited quota and it's difficult to request additional quota.  Additional node pools of different SKUs have been used to add more capacity on the order of 30-60 Nodes, in order to restore normal amount of runners for both windows and linux CPU runners. There is some small impact from needing to scale down the node pools for planned k8s upgrades.
