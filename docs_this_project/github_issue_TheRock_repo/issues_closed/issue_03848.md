### Problem Description

When building TheRock on linux using the steps that are described in the main README, on a system that already has a version of rocm installed in /opt/rocm, I encountered failures in the hip-tests step. It seems like some tests were pulling amdclang from /opt/rocm instead of the one that got built by TheRock. The workaround I had to use is to set `-DBUILD_TESTING=OFF` when configuring.

It would be ideal if this can be fixed, but if not then at the very least it should be mentioned in the docs somewhere along with a suitable workaround.
