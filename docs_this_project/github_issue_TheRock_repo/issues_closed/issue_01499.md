Currently, these are wedged into the CMake build, which means that they run as part of the final-final ctest step. Let's take this out and make them do a `pytest` invocation at the head of the pipeline to ensure that the build tools are compatible with the build environment. These take negligible time to run but getting the signal early that they are broken saves hours of iteration.
