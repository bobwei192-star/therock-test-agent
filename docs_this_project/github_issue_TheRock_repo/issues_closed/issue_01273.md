---

Looking through https://github.com/ROCm/TheRock/blob/main/cmake/therock_artifacts.cmake and https://github.com/ROCm/TheRock/blob/main/cmake/therock_subproject.cmake, my theory is that the expunge targets delete only files in the `build/dist/rocm/` directory and custom commands like this only tell CMake that the manifest files in `build/artifacts` are outputs of the artifact populate command: https://github.com/ROCm/TheRock/blob/7fa20cdc80c0740a3bef596619598c185f97b236/cmake/therock_artifacts.cmake#L130-L138

Since the output manifests still exist in build/artifacts, CMake sees no reason to re-run the artifact -> dist flatten steps.
