## Suspected Root Cause

[_get_rocm_sdk_target_family()](https://github.com/ROCm/rocm-jax/blob/rocm-jaxlib-v0.9.1/build/ci_build#L144) was added in ROCm/rocm-jax commit [6ac05b35ed](https://github.com/ROCm/rocm-jax/commit/6ac05b35ed7196164b88d12b27c400e8520b5ebd). It uses a regex that requires the GPU family name to end with a -letter… segment:

```gfx[0-9A-Za-z]+(?:[-_][0-9A-Za-z]+)*?[-_][A-Za-z][0-9A-Za-z]* ```

This works for gfx950-dcgpu, gfx94X-dcgpu, gfx103X-all, etc. but fails for single-segment names like gfx900, gfx906, gfx908, gfx90a, gfx1150–gfx1153 — all of which produce tarball URLs of the form therock-dist-linux-gfx900-7.14.0a20260519.tar.gz where no letter-prefixed suffix segment exists.

## Suggested Fix
 (fix in ROCm/rocm-jax — preferred long-term): Update the regex in build/ci_build to anchor on the version-number boundary instead of requiring a letter suffix:

```
pythonmatch = re.search(
    r"(?<![A-Za-z0-9])"
    r"(gfx[0-9A-Za-z]+(?:[-_][0-9A-Za-z]*[A-Za-z][0-9A-Za-z]*)*)"
    r"(?=[-_][0-9]|\.tar|$|(?![A-Za-z0-9\-_]))",
    therock_path or "",
)
```
