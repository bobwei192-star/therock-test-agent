### Problem Description

Recently, while using the Windows PyTorch nightly builds, I found that YOLO inference consistently triggers a GPU driver timeout, and although the program eventually finishes, the inference results are always incorrect. This behavior started occurring specifically from the `rocm7.10.0a20251106` nightly build.

I initially discovered this while trying to diagnose the unusually slow performance of my image-processing application on the RX 9070. I also previously reported a convolution performance [issue.](https://github.com/ROCm/rocm-libraries/issues/2567#issue-3608511347) To isolate the issue, I tested YOLO inference independently and also ran several convolution benchmarks.
In the `rocm7.10.0a20251106`  nightly build, some Winograd kernels for gfx12 appear to be completed, and certain FP16 convolution kernels now achieve ideal performance; however, most convolution kernels remain much slower than expected.
I am not certain whether the YOLO inference errors are related to the new Winograd kernels on gfx12, but the problem only began appearing starting with the `rocm7.10.0a20251106` build


Use this example:
```
import time
import urllib.request
from ultralytics import YOLO
import torch
import os

# --------------------------------------------------------
# 0. Prepare Image
# --------------------------------------------------------
img_url = "https://ultralytics.com/images/bus.jpg"
img_path = "sample.jpg"

if not os.path.exists(img_path):
    print("Downloading sample image...")
    urllib.request.urlretrieve(img_url, img_path)
    print("Saved as", img_path)

# --------------------------------------------------------
# 1. Load Model
# --------------------------------------------------------
model = YOLO("yolov8s.pt")

device = "cuda" if torch.cuda.is_available() else "cpu"
model.to(device)

print(f"Using device: {device}")

# --------------------------------------------------------
# 2. Inference
# --------------------------------------------------------
print("Running first inference...")
result = model(img_path, imgsz=640)
result[0].show()
```
