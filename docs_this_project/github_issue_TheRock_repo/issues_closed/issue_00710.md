### Problem Description

Not sure if I missed something or we are still is lacking gfx1201 support.

At least `test_conv_transpose2d` test says `Unsupported PAL device with ASIC revision #62` which is something. But the other `test_conv_cudnn_nhwc_support`, just gives generic compilation error without much more info. (See results in additional info section)

I did default build with `cmake -GNinja .. -DCMAKE_BUILD_TYPE=RelWithDebInfo -DTHEROCK_AMDGPU_FAMILIES=gfx1201 -DTHEROCK_AMDGPU_WINDOWS_INTEROP_DIR="E:\dev\amdgpu-windows-interop"`

Also interesting is that when `MIOpenDriver.exe` is run as part of the test, it gives different result. I don't know how it works, maybe you pass things in environment variable, but 

Running `MIOpenDriver.exe conv -n 1 -c 8 -H 5 -W 5 -k 4 -y 3 -x 3 -p 1 -q 1 -u 1 -v 1 -l 1 -j 1 -m conv -g 1 -F 2 -t 1` standalone crashes after it fails to find device. I would suggest just aborting/exiting earlier if such condition happens, instead of crashing later.
