Ninja log files can be converted into tracing formats with tools like https://github.com/nico/ninjatracing. Having this data more accessible will let TheRock and subproject maintainers to more effectively analyze build times.

Log uploading is currently performed by https://github.com/ROCm/TheRock/blob/main/build_tools/github_actions/upload_build_logs_to_s3.py. We may need to move and/or rename the per-project files so they have unique names in the s3 bucket. We may also need to update https://github.com/ROCm/TheRock/blob/main/build_tools/github_actions/create_log_index.py which generates log index pages like https://therock-artifacts.s3.amazonaws.com/15743405144-linux/logs/gfx94X-dcgpu/index.html.

Follow up tasks:

- [ ] Convert per-project log files into per-project trace.json files automatically
- [ ] Merge per-project log files or trace files into combined files to see the entire build process in one trace
