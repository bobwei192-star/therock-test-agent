### Problem Description

ROCr had incorrect values for VGPR count for gfx1151 which was causing crashes for various workloads.

This has been addressed with:
	- [hsakmt: Expose and use CWSR and Control stack sizes by kentrussell · Pull Request #2200 · ROCm/rocm-systems](https://github.com/ROCm/rocm-systems/pull/2200)
	- [[PATCH] drm/amdkfd: Export the cwsr_size and ctl_stack_size to userspace - Mario Limonciello](https://lore.kernel.org/amd-gfx/20251205184158.2261400-1-mario.limonciello@amd.com/) (cc: @superm1)

User-mode fix is available in the latest nightly builds from TheRock (since Dec 19: [Bump `rocm-systems` 20251218 by jayhawk-commits · Pull Request #2643 · ROCm/TheRock](https://github.com/ROCm/TheRock/pull/2643)) and will be available in **ROCm 7.2** and **ROCm 7.11**.

Kernel mode driver fix is available in:
	- Ubuntu **OEM** kernel 1018 and newer
	- see https://github.com/ROCm/TheRock/issues/2991#issuecomment-3768808325
	- see also: https://rocm.docs.amd.com/en/latest/how-to/system-optimization/strixhalo.html#required-kernel-version

It is not yet available in Ubuntu 24.04. generic kernel.
