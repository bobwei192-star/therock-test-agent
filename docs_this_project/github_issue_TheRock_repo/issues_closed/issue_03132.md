### Problem Description

When I try to build, the bundled ncurses build fails. It tries to write to /usr just to build. This is really brutal and unavoidable, and I can't bring myself to change permissions for an ncurses install.

I also have ncurses available and don't want to use the bundled install. I've tried quite an array of different flags to try to use my systems ncurses, but the build flags don't prevent building.

In `third-party/sysdeps/linux/CMakeLists.txt:9`, this line is unconditional - ncurses gets added regardless of any flags.

`add_subdirectory("${CMAKE_CURRENT_SOURCE_DIR}/../common/ncurses" "${CMAKE_CURRENT_BINARY_DIR}/ncurses")`

These flags have no effect:

- `-DTHEROCK_BUNDLED_NCURSES=false` - Only affects RUNTIME_DEPS (for linking), not building
- `-DTHEROCK_ENABLE_SYSDEPS_NCURSES=false` - Only controls artifact output, not building
- `-DTHEROCK_BUNDLE_SYSDEPS=OFF` - A dozen+ cmake errors, `non-existent target`, I think under `therock-grpc`

Desired results:

There should be a meaningful way to keep bundled libraries from trying to build, & especially to keep them from trying to install.
