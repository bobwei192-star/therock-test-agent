### Problem Description

This could be complicated to explain, will keep it simple:

My own compiled ROCm was working fine to me few days ago, until few days ago, not sure if some system package broke it, so I'm looking for ideas...

First, I was doing some operations with Torch (self-compiled with ROCm 7.1.0 + Torch 2.9.1) and I started getting segfaults from libhsa-runtim64.so, no matter what app uses, even a simple pytorch test script crashed.

Then, decided to test the (stable) wheels from pytorch (ROCm 7.1.0 + Torch 2.10.0), and got the same problem, just crashes against libhsa-runtime64.so

And tried the TheRock nightlies version (Torch 2.9.1+rocm7.11.0a20260118), and worked fantastic.

If works in the nighties, then probably you guys fixed something, or realized something causes the crashes.
