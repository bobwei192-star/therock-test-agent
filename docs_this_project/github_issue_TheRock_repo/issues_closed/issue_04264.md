**Summary:**
The CI job fails during the GDB testsuite run. Failures are concentrated in ROCm-specific GDB tests and appear driven by:
- inability to generate core files in the CI environment (Could not generate a core file), and
- test timeouts in interrupt / deep-stack tests.
The test harness reports multiple LLVM failures (some may be in an “ignored failures” list), and a smaller set of unexpected ROCm failures that cause the overall job to fail.


**GCC unexpected failures (1)**
- `gdb.rocm/corefile.exp`
- `gdb.rocm/device-interrupt.exp`
- `gdb.rocm/load-core-remote-system.exp`

**LLVM failures:**
- `gdb.rocm/runtime-core.exp: fault=pagefault: Could not generate a core file`
- `gdb.rocm/runtime-core.exp: fault=abort: Could not generate a core file`
- `gdb.rocm/runtime-core.exp: fault=assert: Could not generate a core file`
- `gdb.rocm/device-interrupt.exp: ... interrupt (timeout)`
- `gdb.rocm/deep-stack.exp: backtrace (timeout)`
