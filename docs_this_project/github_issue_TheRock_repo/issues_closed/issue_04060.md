### Summary: 
The CI job for "Release Windows PyTorch Wheels" is failing to install the Ninja build tool using Chocolatey due to a "429 Too Many Requests" error from the Chocolatey package source.

**Effected Archs:** `gfx110x, gx103x, gfx120x, gfx1151, gfx1152, gfx101x`
### Runs:
- https://github.com/ROCm/TheRock/actions/runs/23286656754/job/67711681519
- https://github.com/ROCm/TheRock/actions/runs/23286556982/job/67711354926
- https://github.com/ROCm/TheRock/actions/runs/23286061722/job/67709720688
- https://github.com/ROCm/TheRock/actions/runs/23284531312/job/67704899486
- https://github.com/ROCm/TheRock/actions/runs/23284511475/job/67704841513

### Observed behavior:
- Chocolatey attempts to download ninja.1.13.1 from https://community.chocolatey.org/api/v2/package/ninja/1.13.1
- The request receives a 429 (Too Many Requests) error from the server.
- Repeated attempts in the same run also fail. 
- As a result, ninja is not installed and the workflow fails with exit code 

### Error log:
```

```

### Suggested resolution:

- Add retry logic for Chocolatey install steps.
- Consider using alternative installation methods for ninja (e.g., pip install ninja or downloading the binary directly).
