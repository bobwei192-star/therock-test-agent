## Summary

When not linking BLIS, it is assumed that the reference NETLIB BLAS should be used. This 
switches to `find_package()` instead of assuming that the reference BLAS is available without any check. This also allows to use other providers than the reference NETLIB implementation like OpenBLAS.

Instead of setting `BLAS_LIBRARY` to `BLAS_LIBRARIES` (provided by `find_package(BLAS)`) a more comprehensive refactoring could drop using `BLAS_LIBRARY` and use `BLAS_LIBRARIES` directly.
