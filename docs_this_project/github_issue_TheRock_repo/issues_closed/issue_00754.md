### Problem Description

I compiled ROCm and PyTorch using TheRock on my local machine (Windows 11, AMD Radeon RX 7800XT).
PyTorch appears to work normally — torch.cuda.is_available() returns True.
However, when I run a torch.nn.Linear layer, I get the following error:
```bash
rocblaslt error: Cannot read "TensileLibrary_lazy_gfx1101.dat": No error
rocblaslt error: Could not load "TensileLibrary_lazy_gfx1101.dat"
Error in Linear: CUDA error: HIPBLAS_STATUS_INVALID_VALUE when calling `hipblasLtMatmulAlgoGetHeuristic( ltHandle, computeDesc.descriptor(), Adesc.descriptor(), Bdesc.descriptor(), Cdesc.descriptor(), Cdesc.descriptor(), preference.descriptor(), 1, &heuristicResult, &returnedResult)`
```
A same issue occurs when I run transformer models.

**If you want to reproduce the error, run the following code:**
```python
import torch
import torch.nn as nn

linear = nn.Linear(128, 128).cuda()
x = torch.randn(1, 128).cuda()
y = linear(x)  # Error occurs here
```
After testing, other commonly used networks (e.g., CNN, LSTM) do not produce this error.
