### Suggestion Description


Current Behavior
When installing torch with a specific ROCm version from the ROCm wheel index (nightlies or prereleases), pip does not automatically install compatible versions of torchaudio and torchvision. Instead, if these packages are installed without explicit version pinning, pip may select the latest available versions from the index, which can be incompatible with the installed torch version.

Example of problematic workflow:

`export ROCM_VERSION="7.10.0rc1"
export GFX_FAMILY="gfx94X-dcgpu"
export INDEX_URL="https://rocm.prereleases.amd.com/whl-staging"

# Installs torch correctly
python3.11 -m pip install --index-url $INDEX_URL/${GFX_FAMILY}/ --pre "torch==2.9.1+rocm$ROCM_VERSION"'

Later, installing torchaudio/torchvision without explicit versions may pull incompatible versions
python3.11 -m pip install --index-url $INDEX_URL/${GFX_FAMILY}/ --pre torchaudio torchvision
May install mismatched versions causing runtime errors


Expected behavior:
When only torch is specified in pip install, pip should automatically select compatible versions of torchaudio and torchvision that match the torch version (by default, or provide clear version constraints in the index/wheel metadata), so users do not have to manually specify all three versions.
(based on the related commits  https://github.com/ROCm/pytorch/blob/release/2.9/related_commits) 

```
python3.11 -m pip install --index-url $INDEX_URL/${GFX_FAMILY}/ --pre "torch==2.9.1+rocm$ROCM_VERSION"
# Should install torchaudio==2.9.0+rocm$ROCM_VERSION and torchvision==0.24.0+rocm$ROCM_VERSION (or other compatible versions)
```

Benefit:
Improves usability and reduces installation errors from mismatched versions.
