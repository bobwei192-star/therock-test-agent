On Linux, one can use `hipcc` from TheRock's build directory like `hipcc hello.cpp -o hello`  as expected. On Windows, `hipcc` fails looking for `clang++` and `lld`:

```
D:\therock-output-gfx1036\build\dist\rocm\bin\hipcc.exe -Og .\hello.cpp -c -o .\hello.o
'D:\therock-output-gfx1036\build\dist\rocm\bin\clang++' is not recognized as an internal or external command,
operable program or batch file.
The system cannot find the path specified.
'nvcc' is not recognized as an internal or external command,
operable program or batch file.
Device not supported - Defaulting to AMD
```

To successfully build hip test program, I have to:

1. Make a symlink to `clang++` and place it next to `hipcc.exe` in `dist\rocm\bin` (**Tools**) 
2. Explicitly pass `-I` to `dist\rocm\include`, which should be in the default header search path (**Headers**) 
3. Explicitly pass `-L` to the directory containing `amdhip64.lib`, which is not placed under `dist\rocm\lib` (**Libraries**) 
   * All `.lib` files should probably be installed in `dist\rocm\lib`

```
\therock-output\build\dist\rocm\bin\hipcc -c -Og --driver-mode=g++ -D__HIP_PLATFORM_AMD__ --offload-arch=$AMD_GPU_FAMILY -I D:\therock-output-gfx1036\build\dist\rocm\include .\hello.cpp -o hello.o      
\therock-output\build\dist\rocm\lib\llvm\bin\clang.exe -Og --driver-mode=g++ --offload-arch=$AMD_GPU_FAMILY -D__HIP_PLATFORM_AMD__ -L D:\therock-output\build\core\clr\dist\lib .\hello.o -lamdhip64 -o hello.exe
```
