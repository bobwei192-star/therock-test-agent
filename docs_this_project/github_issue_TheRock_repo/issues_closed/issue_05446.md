### Problem Description

The `rocm_sdk_device_gfx*` wheels ship no `rccl_lib_*.kpack` files for any architecture.
  `librccl.so` searches for its GPU kernels via a relative kpack path
  (`../.kpack/rccl_lib_@GFXARCH@.kpack`) but those files were never packaged. As a result,
  all tests that exercise the RCCL device API fail with "invalid kernel file".

  Other libraries (rocBLAS, rocRAND, rocFFT) do ship their kpack files under
  `_rocm_sdk_libraries/.kpack/`, but RCCL is entirely absent:

  ```bash
  $ find .venv/lib/python3.12/site-packages -name 'rccl_lib_*.kpack'
  # (no output)
```
```bash
  $ ls .venv/lib/python3.12/site-packages/_rocm_sdk_libraries/.kpack/
  blas_lib_gfx1100.kpack  fft_lib_gfx1100.kpack  rand_lib_gfx1100.kpack
```

### Steps to Reproduce

```bash
python3 -m venv .venv
  source .venv/bin/activate
  pip install --index-url https://rocm.nightlies.amd.com/whl-multi-arch/ "rocm[libraries,device-all]"
  rocm-sdk init
```
  Confirm the kpack files are absent
```bash
  find .venv/lib/python3.12/site-packages -name 'rccl_lib_*.kpack'
  # returns nothing
```
  Build rocm-examples and run the affected tests
```bash
  SITE=.venv/lib/python3.12/site-packages
  export LD_LIBRARY_PATH=${SITE}/_rocm_sdk_core/lib:${SITE}/_rocm_sdk_libraries/lib:${SITE}/_rocm_sdk_devel/lib

  git clone https://github.com/ROCm/rocm-examples.git
  cd rocm-examples
  cmake -B build \
    -DROCM_PATH=${SITE}/_rocm_sdk_devel \
    -DROCM_EXAMPLES_ENABLE_ROCDECODE=OFF \
    -Wno-dev
  cmake --build build -j$(nproc)
  cd build
  ctest -j1 --output-on-failure -R 'rccl_device_api|rccl_send_recv'
```
  Both tests fail with "invalid kernel file" when RCCL attempts to load its device kernels.
