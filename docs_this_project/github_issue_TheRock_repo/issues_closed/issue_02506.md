### Problem Description

On a clean install of Ubuntu 22.04 or RHEL 9.7 and installing ROCm 7.10 using tarball method, running test_hip_api results in a segmentation fault. This likely affects other distros, but was first observed on Ubuntu 22.04 and RHEL 9.7.

Here is a log of the failure:

rocm@rocm-MS-7C73:~/rocm-install$ test_hip_api
Running main() from /therock/output/build/third-party/googletest/source/googletest/src/gtest_main.cc
[==========] Running 1 test from 1 test suite.
[----------] Global test environment set-up.
[----------] 1 test from HIPTests
[ RUN      ] HIPTests.Saxpy
Segmentation fault (core dumped)

### Operating System

Ubuntu 22.04.5 / RHEL 9.7

### CPU

Intel Core i7-10700

### GPU

AMD Radeon RX 7900 XT

### ROCm Version

ROCm 7.10 Prerelease

### ROCm Component

N/A

### Steps to Reproduce

1. Clean OS install of Ubuntu 22.04 or RHEL 9.7 from distro ISO
2. Follow the steps to install tarball
sudo usermod -a -G render,video $LOGNAME

mkdir therock-tarball && cd therock-tarball

wget https://repo.amd.com/tarball/therock-dist-linux-gfx950-dcgpu-7.10.0rc2.tar.gz

mkdir install
tar -xf *.tar.gz -C install

cd therock-tarball
ROCM_INSTALL_PATH=$(pwd)/install
sudo tee /etc/profile.d/set-rocm-env.sh << EOF
export ROCM_PATH=$ROCM_INSTALL_PATH
export PATH=\$PATH:\$ROCM_PATH/bin
export LD_LIBRARY_PATH=\$ROCM_PATH/lib
EOF
sudo chmod +x /etc/profile.d/set-rocm-env.sh


3. Reboot system for all changes to take effect
4. Verify rocminfo can be run
5. Run test_hip_api
