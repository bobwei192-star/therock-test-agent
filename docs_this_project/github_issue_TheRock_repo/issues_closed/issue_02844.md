### Problem Description

When running a WAN 2.2 workflow in ComfyUI, the process consistently fails during VAE decoding with a HIP out-of-memory error, despite sufficient total VRAM.
```
!!! Exception during processing !!! HIP out of memory. Tried to allocate 8.54 GiB. GPU 0 has a total capacity of 15.92 GiB of which 0 bytes is free. Of the allocated memory 3.97 GiB is allocated by PyTorch, and 9.27 GiB is reserved by PyTorch but unallocated. If reserved but unallocated memory is large try setting PYTORCH_ALLOC_CONF=expandable_segments:True to avoid fragmentation.  See documentation for Memory Management  (https://pytorch.org/docs/stable/notes/cuda.html#environment-variables)
```

Attempted Workaround:
Setting the environment variable as suggested.
Resulting Warning:
```
E:\AI\ComfyUI\venv\Lib\site-packages\torch\nn\modules\sparse.py:166: UserWarning: expandable_segments not supported on this platform (Triggered internally at B:\src\torch\c10/hip/HIPAllocatorConfig.h:40.)
  torch.empty((num_embeddings, embedding_dim), **factory_kwargs),
```

Notes:
PyTorch reports large amounts of reserved but unallocated memory, suggesting fragmentation.
The recommended expandable_segments allocator option appears unsupported on ROCm.
GPU has 16 GB VRAM, but a single ~8.5 GB contiguous allocation fails.
