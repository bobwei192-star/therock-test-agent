## Summary

The `storage-libs / hipfile` stage fails during the link phase for all three test executables (`hipfile_tests`, `hipfile_system_tests`, `test/amd_detail/internal_tests`) with undefined symbol errors from `libgtest.a` / `libgmock.a`:

```text
[hipfile] FAILED: test/hipfile_tests
[hipfile] ld.lld: error: undefined symbol: pthread_getspecific
[hipfile] >>> referenced by gtest-port.h:1774
[hipfile] >>>               gtest-all.cc.o:(testing::internal::UnitTestImpl::~UnitTestImpl()) in archive .../libgtest.a
[hipfile] ld.lld: error: undefined symbol: pthread_key_delete
[hipfile] ld.lld: error: undefined symbol: pthread_key_create
[hipfile] ld.lld: error: undefined symbol: pthread_setspecific
[hipfile] clang++: error: linker command failed with exit code 1
```

## Root Cause

The hipfile test CMakeLists files in `rocm-systems` (pinned at `7ced1a6`) do not link `Threads::Threads` (i.e., `-lpthread`) for the test executables:

- `projects/hipfile/test/CMakeLists.txt` — `hipfile_tests` and `hipfile_system_tests` link only against `GTest::gtest` (and `GTest::gtest_main`), with no `Threads::Threads`
- `projects/hipfile/test/amd_detail/CMakeLists.txt` — calls `find_package(Threads REQUIRED)` but does not add `target_link_libraries(internal_tests PRIVATE Threads::Threads)`

`ld.lld` (LLVM's linker, used in the ASAN toolchain) is strict about undefined symbols in static archives. GTest's `ThreadLocal<T>` implementation uses POSIX thread-key APIs (`pthread_key_create`, etc.) from `gtest-port.h:1774–1819`, which require explicit `-lpthread` linkage under lld. GNU ld resolves these transitively from glibc, masking the bug in non-ASAN builds.
