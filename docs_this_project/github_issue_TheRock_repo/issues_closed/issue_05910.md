## Summary

The `torch release/2.11` job in `multi_arch_release_linux_pytorch_wheels.yml` fails to build PyTorch wheels because `gfx125X-dcgpu` (gfx1250) was included in the build matrix without being excluded for PyTorch `release/2.11`.

## Context
- Workflow: `multi_arch_release_linux_pytorch_wheels.yml`
- Platform: Linux
- GPU arch: gfx1250
- PyTorch ref: `release/2.11`
- Python version: `all python versions affected`
- ROCm version: `7.14.0a20260617`
- Failing run: https://github.com/ROCm/rockrel/actions/runs/27674732763/job/81847294318

## Error

```
/__w/rockrel/rockrel/external-builds/pytorch/pytorch/aten/src/ATen/../../../third_party/composable_kernel/include/ck/utility/amd_buffer_addressing_builtins.hpp:33:48: error: use of undeclared identifier 'CK_BUFFER_RESOURCE_3RD_DWORD'
   33 |     wave_buffer_resource.config(Number<3>{}) = CK_BUFFER_RESOURCE_3RD_DWORD;
```

## Root Cause

TheRock commit [`1cc8ec5`](https://github.com/ROCm/TheRock/commit/1cc8ec570e9f5c720de9ce52dc485228c2df0274) ("Add gfx1250 (CDNA5) build support") added `gfx125X-dcgpu` to the amdgpu families and correctly excluded it from `release/2.9`, `release/2.10`, `release/2.12`, and `nightly` in `configure_pytorch_release_matrix.py` — but omitted the exclusion for `release/2.11`. PyTorch `release/2.11`'s vendored Composable Kernel does not support gfx1250 (`CK_BUFFER_RESOURCE_3RD_DWORD` is undefined for that arch). See also https://github.com/ROCm/TheRock/issues/5833.
