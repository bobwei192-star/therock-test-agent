### Problem Description

On Strix Halo (gfx1151), AllTalk and other PyTorch workloads report:
MIOpen(HIP): Warning [OpenRuntimeLibraryForDevice] CK grouped conv library not found for device gfx1151.

The file ships as libMIOpenCKGroupedConv_gfx1151.so.1.0 in _rocm_sdk_libraries_gfx1151/lib/ but MIOpen's dlopen() call looks for the unversioned name libMIOpenCKGroupedConv_gfx1151.so. No symlink is created during installation.

### Operating System

Ubuntu 26.04 LTS (Resolute Raccoon), Linux kernel 7.0

### CPU

AMD Ryzen AI Max+ 395 (Strix Halo, 16-core Zen 5)

### GPU

AMD Radeon 8060S — gfx1151 (integrated, Strix Halo), 128GB unified memory (shared between CPU and GPU), GTT size unlocked via GRUB: amdgpu.gttsize=122880

### ROCm Version

torch-2.11.0+rocm7.13.0a20260425

### ROCm Component

MIOpen

_No response_

### Additional Information

Fix: Create symlinks:

bash
ln -s libMIOpenCKGroupedConv_gfx1151.so.1.0 libMIOpenCKGroupedConv_gfx1151.so.1
ln -s libMIOpenCKGroupedConv_gfx1151.so.1.0 libMIOpenCKGroupedConv_gfx1151.so

After creating these symlinks the warning disappears and MIOpen uses the CK grouped convolution solver correctly.
