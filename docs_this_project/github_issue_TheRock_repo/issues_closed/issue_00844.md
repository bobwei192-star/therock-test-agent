### Suggestion Description

It would be great to have builds not only for gfx1151, but also for gfx1150. The entry in the amdgpu family matrix [here](https://github.com/ROCm/TheRock/blob/63a61412ede895704301838ad159bf6624ec60af/build_tools/github_actions/amdgpu_family_matrix.py#L35) seems to suggest that it might have been the original intention to support all gfx115x, but the family is only specified as gfx1151.

I don't know how much the features differ between the two, would it be leaving too much on the table to build for generic gfx115x as is the case for gfx110x? In that case it might make more sense to have separate builds, is that something you would consider? Figure 2 in https://www.tomshardware.com/pc-components/gpus/amd-unveils-rocm-7-new-platform-boosts-ai-performance-up-to-3-5x-adds-radeon-gpu-support makes me hopeful that better support for gfx1150 is something that is on the roadmap for ROCm 7, would be awesome if true!
