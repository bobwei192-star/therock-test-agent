### Issue 1: Latest nightly build causes `std::bad_alloc` crash

When I install the latest ROCm nightly build for PyTorch, ComfyUI crashes immediately upon launch with a `std::bad_alloc` error.

**Steps to Reproduce (Installation):**
```bash
python -m pip install \
  --index-url https://rocm.nightlies.amd.com/v2/gfx120X-all/ \
  torch torchaudio torchvision
```

**Error Log:**
```log
venv) ubuntu@ubuntu:/media/ubuntu/7C8284B882847882/linux/comfyui-workspace/test/ComfyUI$ python3 main.py 
Checkpoint files will always be loaded safely.
Total VRAM 16304 MB, total RAM 47543 MB
pytorch version: 2.7.1+rocm7.0.0rc20250901
AMD arch: gfx1201
ROCm version: (7, 1)
Set vram state to: NORMAL_VRAM
Device: cuda:0 AMD Radeon Graphics : native
Using sub quadratic optimization for attention, if you have memory or speed issues try using: --use-split-cross-attention
terminate called after throwing an instance of 'std::bad_alloc'
  what():  std::bad_alloc
Aborted (core dumped)
```
This error appears related to memory allocation, as discussed in other contexts [here](https://github.com/comfyanonymous/ComfyUI/issues/9604#issuecomment-3248840320).

**Working Solution (Downgrade):**
Downgrading `torch` and `torchaudio` to the **`20250826`** build resolves the crash. This suggests a potential regression in the builds released after this date.

**Functional Downgrade Command:**
```bash
python -m pip install \
  --force-reinstall \
  torch-2.9.0a0+rocm7.0.0rc20250826-cp312-cp312-linux_x86_64.whl \
  torchaudio-2.8.0a0+rocm7.0.0rc20250826-cp312-cp312-linux_x86_64.whl \
  torchvision-0.24.0a0+rocm7.0.0rc20250901-cp312-cp312-linux_x86_64.whl \
  --extra-index-url https://rocm.nightlies.amd.com/v2/gfx120X-all/
```

### Issue 2: Discrepancy in reported Attention mechanism status

When using the functional older build, the ComfyUI startup log reports the attention mechanism as `attention_sub_quad`, which typically implies that optimized attention backends (like flash-attention) are not active.

**Log Snippet:**
```log
(venv) kite@kite:/media/kite/7C8284B882847882/linux/test2/ComfyUI$ python3 main.py 
...
Using sub quadratic optimization for attention, if you have memory or speed issues try using: --use-split-cross-attention
...
[ComfyUI Attention日志] Attention机制选择为: attention_sub_quad```
**Question:** This is contrary to the expectation that recent ROCm releases should support optimized attention. Could you confirm if a more efficient attention implementation is expected to be enabled by default on the Linux + ROCm platform?

---

### Issue 3: More frequent Out-of-Memory errors on Linux vs. Windows

I have observed that identical workflows in ComfyUI are more prone to VRAM-related errors on Linux than on Windows.

**Typical Error Message:**
```log
HIP error: an illegal memory access was encountered
Search for hipErrorIllegalAddress' in https://docs.nvidia.com/cuda/cuda-runtime-api/group__HIPRT__TYPES.html for more information. HIP kernel errors might be asynchronously reported at some other API call, so the stacktrace below might be incorrect. For debugging consider passing AMD_SERIALIZE_KERNEL=3 Compile with TORCH_USE_HIP_DSA` to enable device-side assertions.
```
On Windows, this issue can often be mitigated with launch arguments like `--disable-smart-memory --reserve-vram 8`. These arguments seem less effective on Linux, suggesting that the root cause may be different or that memory management is under more pressure on this platform.
