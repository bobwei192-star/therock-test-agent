## Summary

The `Multi-Arch CI ASAN` workflow is failing while configuring `hipBLASLt` because the build injects `-shared-libsan` into global linker flags, and that flag is then used during CMake's Fortran compiler test. `gfortran` does not recognize `-shared-libsan`, so configure fails before the project actually builds.

## Failing job

- Workflow: `Multi-Arch CI ASAN`
- Run: https://github.com/ROCm/TheRock/actions/runs/25534114320/job/74948992292
- Failing ref: `f40a854896f2f5d2d6e88dcf7f7952479402ef44`

## Failure 

From the job log, `hipBLASLt` fails during configure while CMake is checking the Fortran compiler:

```text
[hipBLASLt configure] -- Detecting Fortran compiler ABI info - failed
[hipBLASLt configure] -- Check for working Fortran compiler: /opt/rh/gcc-toolset-13/root/usr/bin/gfortran - broken
...
[hipBLASLt configure] : && /opt/rh/gcc-toolset-13/root/usr/bin/gfortran -fsanitize=address -shared-libsan -fsanitize=address -shared-libsan ...
[hipBLASLt configure] gfortran: error: unrecognized command-line option ‘-shared-libsan’; did you mean ‘-shared-libgcc’?
```
