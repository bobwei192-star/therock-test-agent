This will be needed for some optimized kernels, notably we disable flash attention on Windows: https://github.com/ROCm/TheRock/blob/446bd9837474ff722a4f93cff51e5a1ce3661854/external-builds/pytorch/build_prod_wheels.py#L539-L543

@scottt and @jammm have made progress on this:

* https://github.com/ROCm/TheRock/discussions/409
* https://github.com/ROCm/TheRock/discussions/244

They have also distributed some unofficial wheels that are already in use by the community. Now that we have our "official" Windows release train running we can look to including these optional features there too.

The discussions above have lots of history, with a mentions of different repositories at varying levels of working / supported:

* https://github.com/ROCm/aotriton
* https://github.com/lshqqytiger/triton
* https://github.com/woct0rdho/triton-windows
* https://github.com/scottt/aotriton

The latest working approaches seem to involve building a branch of aotriton with some settings _on Linux_, downloading those prebuilt kernels onto Windows, building some other Windows deps, then connecting all the pieces together in a PyTorch source build. In the short term we can try to do all of that in CI/CD jobs and then in the long term search for an architectural setup that will be easier to maintain.
