## Issue 1: bitsandbytes Compilation Fails

**Blocker:** Cannot compile bitsandbytes for ROCm/HIP on TheRock 7.11.

```
bitsandbytes library load error: Configured ROCm binary not found at 
/home/user/venv/lib/python3.11/site-packages/bitsandbytes/libbitsandbytes_rocm72.so
```

cmake fails entirely - the build system doesn't recognize the HIP backend properly for gfx1151.

**Impact:** Blocks 4-bit quantization for LLM-based nodes (VibeVoice TTS, etc.), forcing full-precision inference and higher VRAM usage.

**Request:** Prebuilt `bitsandbytes` wheel for TheRock 7.11 / gfx1151, or upstream fixes to enable HIP compilation.

## Issue 2: torchaudio Resampler bf16 Crash

**Error:**
```
RuntimeError: "reflection_pad1d" not implemented for 'BFloat16'
```

**Root Cause:** VibeVoice TTS outputs bf16 audio tensors. torchaudio's resampler kernel is hardcoded fp32-only.

**Workaround Implemented:** Created `HaloCodecShim` class to force `.float()` conversion at audio boundaries:

```python
class HaloCodecShim:
    @staticmethod
    def load_audio(path, target_sr=16000):
        waveform, sr = torchaudio.load(path, backend="soundfile")
        if sr != target_sr:
            resampler = torchaudio.transforms.Resample(sr, target_sr)
            waveform = resampler(waveform.float())  # Force fp32
        return waveform.to(dtype=torch.float32)
```

## Issue 3: LatentSync VAE Decode "Joker Face" Artifact

**Symptom:** Rainbow/static noise artifacts in lip-sync masked region when running VAE decode in fp16 or with mixed precision enabled. The face detection works (finds the mouth area), but the diffusion/decode produces corrupted output.

**Root Cause:** hipBLASLt kernels for gfx1151 don't fully support certain mixed-precision matrix operations:

```
UserWarning: gemm_and_bias error: HIPBLAS_STATUS_NOT_SUPPORTED when calling hipblasLtMatmul
with transpose_mat1 1 transpose_mat2 0 m 1280 n 8192 k 5120 mat1_ld 5120 mat2_ld 5120 
result_ld 1280 abType 0 cType 0 computeType 6 scaleType 0
Will attempt to recover by calling unfused cublas path.
```

**Workaround Attempted:** Force fp32 in inference.py and disable mixed precision:

```python
# inference.py line 22
dtype = torch.float32  # FORCE FP32 FOR STRIX HALO

# nodes.py - disable mixed precision detection
use_mixed_precision = False  # FORCE OFF FOR AMD
```

This reduces but does not fully eliminate the artifact. Still investigating VAE decode precision path.
