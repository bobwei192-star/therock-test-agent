## Summary

All four `Build JAX Wheels` jobs fail immediately when `build/ci_build` tries to run `docker build` to construct the manylinux builder image. The Docker daemon is not available on `aws-linux-scale-rocm-prod` runners (`/var/run/docker.sock` is absent).

## Exact Error

```text
ERROR: failed to connect to the docker API at unix:///var/run/docker.sock; check if the path is correct and if the daemon is running: dial unix /var/run/docker.sock: connect: no such file or directory
No manylinux builder image passed. Building a new builder image...
subprocess.CalledProcessError: Command '['docker', 'build', '-t=ghcr.io/rocm/jax-manylinux_2_28-therock-7.14.0a20260623', '--file=docker/manylinux/Dockerfile.jax-manylinux_2_28-therock', ...]' returned non-zero exit status 1.
##[error]Process completed with exit code 1.
```

## Suspected Root Cause

PR [TheRock#6037](https://github.com/ROCm/TheRock/pull/6037) (commit [405a3df](https://github.com/ROCm/TheRock/commit/405a3df73dd5905a8cdb47b9f2c63610a23cfcb4), merged 2026-06-22 18:40 UTC) migrated all Linux workflow `runs-on` labels from `azure-linux-scale-rocm` to `aws-linux-scale-rocm-prod`, including `multi_arch_build_linux_jax_wheels.yml`. The June 22 nightly ran on Azure (Docker available, build succeeded); the June 23 nightly ran on AWS (Docker absent, build fails).

The JAX build requires `docker build` to construct a manylinux image — see [`rocm-jax/build/ci_build` line 175](https://github.com/ROCm/rocm-jax/blob/rocm-jaxlib-v0.9.1/build/ci_build#L175). This is not currently supported without Docker.
