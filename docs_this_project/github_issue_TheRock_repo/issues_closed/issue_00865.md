### Suggestion Description

At the moment RockBuilder printouts all the logs to console without separating them to own build system like TheRock core build does.

- e should produce similar log files, so there would be separate config, build and install phase logs for each application build
    - this would be default functionality
    - it should be exist an option to change the default behavior and disable the generation of log files and just printout everything to screen.

- there should be also be possible to generate debug/info/warning/error level of logs from the rockbuilder messages itself and by default not to show the debug level messages. (change current print-commands to print_dbg, print_info, print_wrn, print_err-type of function calls)
