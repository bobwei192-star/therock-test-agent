### Problem Description

https://github.com/ggml-org/llama.cpp/blob/master/docs/build.md#hip
with https://therock-nightly-tarball.s3.amazonaws.com/therock-dist-windows-gfx1151-7.10.0a20251028.tar.gz
Tried windows 11 with strix halo (gfx1151)
If just set HIP_PATH=therock-dist-windows-gfx1151-7.10.0a20251028 folder
When running 
set HIP_PATH=therock-dist-windows-gfx1151-7.10.0a20251028_test
set PATH=%HIP_PATH%\bin;%PATH%
cmake -S . -B build -G Ninja -DGPU_TARGETS=gfx1151 -DGGML_HIP=ON -DCMAKE_C_COMPILER=clang -DCMAKE_CXX_COMPILER=clang++ -DCMAKE_BUILD_TYPE=Release -DLLAMA_CURL=OFF
cmake --build build

it will show many issue like missing files or compilation error.
So far it can be workaround by adding relative files like clang, ld,. llvm as well as amdgcn etc from general hip sdk windows package.

But it is better for support these files in native therock windows package for llama.cpp hip windows backend build

Thanks!
