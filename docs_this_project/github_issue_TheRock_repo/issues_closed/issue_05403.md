### Problem Description

Customer reported while trying to install amdrocm-core-sdk , the following issue is observed

zypper install -y amdgpu-dkms amdrocm-core-sdk-gfx90a

Problem: 1: **nothing provides 'libdw.so.1(ELFUTILS_0.186)(64bit)'** needed by the to be installed amdrocm-profiler7.13-7.13.0-2.x86_64

OS:
cat /etc/os-release 
NAME="SLES"
VERSION="15-SP7"
VERSION_ID="15.7"
PRETTY_NAME="SUSE Linux Enterprise Server 15 SP7"
ID="sles"
ID_LIKE="suse"
ANSI_COLOR="0;32"
CPE_NAME="cpe:/o:suse:sles:15:sp7"
DOCUMENTATION_URL="https://documentation.suse.com/"

Easily reproducible in nightly build as well

Root cause:
The package is not providing libdw.so . Earlier version like 7.11 and 7.12 were having the same
**rpm -qlp amdrocm-profiler7.14-7.14.0~20260521-26203389112.x86_64.rpm | grep dw**

 rpm -qlp amdrocm-profiler7.11-7.11.0-2.x86_64.rpm | grep dw
/opt/rocm/core-7.11/lib/rocprofiler-systems/libdw-0.188.so
/opt/rocm/core-7.11/lib/rocprofiler-systems/libdw.so
/opt/rocm/core-7.11/lib/rocprofiler-systems/libdw.so.1

Rather than using libdw.so from the profiler packages, should use the ones provided by amdrocm-sysdeps 

readelf -d librocprof-sys.so.1
