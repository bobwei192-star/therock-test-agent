### Problem Description


Per the discussion in https://teams.microsoft.com/l/message/19:HKqqVUgpG31aOnEzxVscKSKYEk_nbAh8cfxYFShj4GE1@thread.tacv2/19:HKqqVUgpG31aOnEzxVscKSKYEk_nbAh8cfxYFShj4GE1@thread.tacv2::1769103640520?parentMessageId=19:HKqqVUgpG31aOnEzxVscKSKYEk_nbAh8cfxYFShj4GE1@thread.tacv2::1769103640520

vLLM build process depends on the output of running scripts with torch import. Polluting the output with messages breaks the build process.
Repro instructions:

```
docker run -it --rm ubuntu:24.04 # Without GPUs
1  apt-get update -y     && apt-get install -y software-properties-common build-essential git curl sudo vim less libopenmpi-dev libpci-dev python3-venv python3-dev
2  python3 -m venv /opt/python --system-site-packages
3  export PATH=/opt/python/bin:$PATH
4  source /opt/python/bin/activate
5  export PIP_EXTRA_INDEX_URL=https://rocm.nightlies.amd.com/v2-staging/gfx94X-dcgpu/
6  pip install --index-url ${PIP_EXTRA_INDEX_URL} --pre "rocm[libraries,devel]>=7.11"
7  rocm-sdk init
8  pip install --index-url ${PIP_EXTRA_INDEX_URL} 'torch==2.9.1' torchaudio torchvision triton
9  python -c 'import torch' 2> /dev/null
```
Observe the warning message
```
[WARNING] offload-arch failed with return code 1
[stderr]
```
printed into the standard output
