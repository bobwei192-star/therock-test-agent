## Description

hipCUB fails to configure with ASAN enabled and `BUILD_BENCHMARK=ON`. Two issues:

1. Google Benchmark's `add_cxx_compiler_flag()` modifies `CMAKE_CXX_FLAGS`, triggering a ROCMChecks `variable_watch()` callback that emits a CMake WARNING.

2. Google Benchmark's `try_run` regex detection tests compile but fail to execute under ASAN (sanitizer runtime not available during configure), causing `Failed to determine the source files for the regular expression backend`.

## Root Cause

hipCUB's `CMakeLists.txt` loads ROCMChecks early (line 122, `include(ROCmCMakeBuildToolsDependency)`) before Dependencies.cmake runs. By then, `variable_watch(CMAKE_CXX_FLAGS rocm_check_toolchain_var)` is already installed and `ROCM_WARN_TOOLCHAIN_VAR` is already `ON` in the cache. Dependencies.cmake does `set(ROCM_WARN_TOOLCHAIN_VAR OFF CACHE BOOL "")` but without `FORCE` this is a no-op.

rocRAND avoids both issues: it sets `ROCM_WARN_TOOLCHAIN_VAR OFF` before loading ROCMChecks (both happen inside its `Dependencies.cmake`), and pre-sets `HAVE_STD_REGEX`/`RUN_HAVE_STD_REGEX` to skip `try_run`.

## Upstream Fix
1. Override `rocm_check_toolchain_var` with a flag-gated no-op macro around `FetchContent_MakeAvailable` (same pattern as rocwmma and composablekernel).
2. Pre-set `HAVE_STD_REGEX ON` and `RUN_HAVE_STD_REGEX 1` before fetching Google Benchmark (same as rocRAND).
