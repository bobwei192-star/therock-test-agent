## Problem

**Severity:** P1 — blocks `THEROCK_KPACK_SPLIT_ARTIFACTS` for developers

When `THEROCK_KPACK_SPLIT_ARTIFACTS=ON`, the `build/dist/rocm` directory is populated from **unsplit** (fat binary) artifacts, not from split artifacts. The kpack `.kpack` files and `.kpm` manifests are never flattened into the dist directory.

## Root Cause

In `therock_artifacts.cmake`, the flatten command is part of the initial (unsplit) custom command at lines 191-196:
```cmake
if(ARG_DISTRIBUTION)
  list(APPEND _flatten_command_list
    COMMAND "${Python3_EXECUTABLE}" "${_fileset_tool}" artifact-flatten
      -o "${_dist_dir}" ${_component_dirs}
  )
endif()
```

`_component_dirs` at this point refers to `artifacts-unsplit/` directories. The flatten runs as part of the unsplit populate command. After the split commands run (lines 237-245), the generic + per-arch artifacts exist in `build/artifacts/`, but **nothing flattens them to dist/rocm**.

Additionally, line 260 replaces `_component_dirs` with the split generic dirs, and line 280 makes `dist-${DISTRIBUTION}` depend on the artifact target. But the artifact target only depends on the split manifests — the flatten already happened (from unsplit) before the split commands ran.

## Fix Direction

For local dev, `dist/rocm` should contain:
1. Generic (host) artifacts → stripped binaries
2. Per-arch artifacts for the configured `THEROCK_AMDGPU_TARGETS` → `.kpack` files, database files

The challenge: at CMake configure time, we know `THEROCK_AMDGPU_TARGETS`, but we don't know the exact directory names the splitter will produce (they depend on the individual targets in the fat binary). However, since we control the compilation, the targets in the fat binary should match `THEROCK_AMDGPU_TARGETS`.

Options:
1. **Add a post-split flatten step**: After the split command, add another custom command that flattens both generic and per-target artifacts to dist/rocm
2. **Move the flatten to after split**: Instead of flattening from unsplit, flatten from split outputs
3. **Deferred flatten**: A final "dist-populate" step that scans `build/artifacts/` for all matching artifacts and flattens them

## Key Files

- `cmake/therock_artifacts.cmake:171-206` — unsplit populate + flatten
- `cmake/therock_artifacts.cmake:208-261` — split without flatten
- `build_tools/fileset_tool.py:150-159` — `do_artifact_flatten()`
