## Root Cause

The test matrix generation (`Configure test matrix` step) likely doesn't produce entries for certain families, possibly because:
1. No test runners available for those GPU families
2. The `sanity_check_only_for_family` flag is set for non-dcgpu families
3. Template expansion failure in the matrix

Looking at `.github/workflows/multi_arch_ci_linux.yml:87`: `sanity_check_only_for_family: ${{ matrix.family_info.sanity_check_only_for_family }}` — this flag controls whether only sanity checks run. The gfx1151 and gfx120X-all families likely have this set to true.

For gfx110X-all, the complete skip (including sanity) suggests no test runners are available for that family.
