[Spack](https://spack.io/) is an open source project from the US Department of Energy that provides a framework for installing software from source code. With Spack, open source software developers can define and contribute build recipes to make it easier for end-users to install their code from source. 

AMD has worked with DOE over the past few years to add ROCm packages to Spack. With each release of ROCm, AMD diligently updates their maintained Spack packages with each ROCm release. The purpose of this is to make it easier for end users to install select packages from the ROCm ecosystem using the compiler of their choice while targeting specific GPU platforms relevant to their use-case.

Installing packages from source with Spack only takes a few steps, e.g.

```
# Clone spack
git clone https://github.com/spack/spack ~/spack/

# Make spack binaries available in your environment; perhaps add this to your ~/.bashrc
source ~/spack/share/spack/setup-env.sh

# Find available compilers on your system. Make sure you have a working C, C++, and Fortran compiler
spack compiler find

# For example, install hipblas for gfx1100
spack install hipblas amdgpu_target=gfx1100

# To make packages visible to your environment, load them. This loads the package and all of its dependencies to your environment.
spack load hipblas
```

AMD even has documentation for this at https://rocm.docs.amd.com/projects/install-on-linux/en/latest/how-to/spack.html

Does it feel like this project is duplicating efforts at AMD ?
