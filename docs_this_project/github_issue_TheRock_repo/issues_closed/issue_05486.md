## Summary
The `Build Linux native Packages::deb::gfx908` job fails during the `Build packages` step because the packaging script cannot locate `composable-kernel_*_generic` artifact directories for the `amdrocm-ck` package.

- **Platform:** Linux
- **GPU arch:** `All gfx for which CK was enabled`

## Root Cause
PR #5462 (commit `d15ad25526`) changed `"Gfxarch": "True"` → `"Gfxarch": "False"` for `amdrocm-ck` in `build_tools/packaging/linux/package.json`. In a non-kpack build, `packaging_utils.py:673` uses `"generic"` as the artifact directory suffix when `Gfxarch=False`, but only arch-specific archives (`composable-kernel_*_gfx908`) are downloaded. No `*_generic` directories exist, so `sourcedir_list` is empty and the build aborts.

## Suggested Fix
Revert `"Gfxarch"` to `"True"` for `amdrocm-ck` in `build_tools/packaging/linux/package.json` (line 2056). If arch-neutral packaging is desired in kpack mode, use a per-artifact `"Artifact_Gfxarch": "False"` key instead, conditioned on the kpack path.
