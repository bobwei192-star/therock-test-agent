### Steps to Reproduce

I have a docker file [here](https://github.com/ROCm/rocm-libraries/blob/develop/projects/hipdnn/dockerfiles/Dockerfile.ubuntu24) that can be used to replicate the environment.

Example run command to replicate from build artifacts:
`docker build -f Dockerfile.ubuntu24 --build-arg THEROCK_ASIC=gfx90X --build-arg THEROCK_GIT_TAG=7.10.0a20251117 -t rocm/miopen-private:prebuilt_release_20251117_gfx90x .`

Can also be replicated by installing artifacts from a nightly gfx90X in other ways.

---

You can check the config file in /opt/rocm/include/miopen/config.h (or wherever the include files are installed in your environment), to see if CK is enabled. 

On:
`#define MIOPEN_USE_COMPOSABLEKERNEL 1`

Off:
`#define MIOPEN_USE_COMPOSABLEKERNEL 0`

---

Once you have MIOpenDriver from the install you can replicate by running a command that requires it.

Here is a command that fails without CK present:
```
./bin/MIOpenDriver CBAInferbfp16 --spatial_dim 2 --batchsize 4 --in_channels 8 --out_channels 16 --group_count 4 --in_h 20 --in_w 20 --fil_h 3 --fil_w 3 --conv_stride_h 2 --conv_stride_w 2 --pad_mode same --dilation_h 1 --dilation_w 1 --in_layout NHWC --fil_layout NHWC --out_layout NHWC --alpha 1 --beta 1 --activMode 7 -J 4 -t 1
```
