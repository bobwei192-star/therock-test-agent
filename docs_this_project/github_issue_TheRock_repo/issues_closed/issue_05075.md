Inside `jax_rocm7_plugin` package, `Requires-Dist` for `jax-rocm7-pjrt` is taking a wildcard for version 0.9.1 and not requiring a specific rocm version. This is different to all previous versions of `jax_rocm7_plugin`.
 
Is that on purpose or a regression?
 
jax_rocm7_plugin-**0.9.0**+rocm7.13.0a20260504
 ```
Requires-Dist: jax-rocm7-pjrt==0.9.0+rocm7.13.0a20260504
 ```



jax_rocm7_plugin-**0.9.1**+rocm7.13.0a20260505
``` 
Requires-Dist: jax-rocm7-pjrt==0.9.1.*
 ```

Maybe `rocm-tag` is missing during package creation?
