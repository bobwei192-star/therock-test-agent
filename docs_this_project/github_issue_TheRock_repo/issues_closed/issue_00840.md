There have been intermittent occurrences where builds would stall with no apparent progress being made, as shown per the [runner logs](https://github.com/ROCm/TheRock/actions/runs/15642188842/job/44071923090) on github:

![Image](https://github.com/user-attachments/assets/081d8aa8-c46f-45a1-9041-574f64d7955c)
As the problem happens randomly, it would be easier to troubleshoot in real time as it occurs. To gain remote access to the pod, `kubectl` has a facility to launch a remote shell interactively as detailed [here](https://support.sitecore.com/kb?id=kb_article_view&sysparm_article=KB1003270).

In this case, the name of the runner was found from the `Set up job` step of the logs:
```haskell
Current runner version: '2.324.0'
Runner name: 'azure-windows-scale-rocm-2jjjw-runner-7htbh'
Machine name: 'AZURE-WINDOWS-S'
```
Running 
```bash
kubectl exec -it azure-windows-scale-rocm-2jjjw-runner-7htbh  -n arc-runners -- powershell
```
Would give a remote powershell session through which a task manager, such as btop, could be used to monitor CPU usage, what build processes are running, and what their command line arguments are.

This can be installed and ran with these commands:
```powershell
$progresspreference="SilentlyContinue"; Invoke-WebRequest https://github.com/aristocratos/btop4win/releases/download/v1.0.4/btop4win-x64.zip -OutFile btop4win-x64.zip; Expand-Archive btop4win-x64.zip -Force; $env:PATH="$env:PATH;$pwd\btop4win-x64\btop4win\"; btop4win.exe
```

After pressing `e` to enable a process tree view, scrolling down to a leaf process, and pressing enter to view the full task arguments, it can be seen that it is, in fact, stalling on `b2` as the whole system and the process has nearly 0% cpu usage.

![Image](https://github.com/user-attachments/assets/bd4f340b-2ee1-45fe-b007-332645c39e4d)

In case the command line argument gets truncated, and resizing the terminal window isn't enough, this powershell command can be used to save all processes arguments.
```powershell
Get-CimInstance Win32_Process | Select-Object Name, ProcessId, CommandLine | Export-Csv -Path "process_info.csv" -NoTypeInformation
Get-Content .\process_info.csv
```
