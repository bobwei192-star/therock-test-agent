Looking at recent wheels from https://d2awnip2yjpvqn.cloudfront.net/v2/gfx94X-dcgpu/, the `rocm-*/PKG-INFO` files contain buggy metadata:

For example with https://d2awnip2yjpvqn.cloudfront.net/v2/gfx94X-dcgpu/rocm-7.0.0rc20250627.tar.gz
```
Metadata-Version: 2.1
Name: rocm
Version: 7.0.0rc20250627
Requires-Dist: rocm==7.0.0rc20250627
Requires-Dist: rocm-sdk-core==7.0.0rc20250627
Provides-Extra: libraries
Requires-Dist: rocm-sdk-libraries-dev==7.0.0rc20250627; extra == "libraries"
Provides-Extra: devel
Requires-Dist: rocm-sdk-devel==7.0.0rc20250627; extra == "devel"
```

Note `Requires-Dist: rocm-sdk-libraries-dev==7.0.0rc20250627; extra == "libraries"`. The `rocm-sdk-libraries-dev` package does not exist, it should be `rocm-sdk-libraries-gfx94X-dcgpu`.

This results in package installation issues (need to confirm the exact symptoms, but there is a bug regardless).
