### Problem Description

After installing ROCm via multiarch tarball on RHEL 8.10 (baremetals), the post installation command `test_hip_api` gives `Segmentation fault (core dumped)`.

### Operating System

RHEL 8.10

### CPU

Intel(R) Core(TM) i7-10700 CPU @ 2.90GHz

### GPU

Navi 31 Radeon RX 7900 XT

### ROCm Version

7.14.0

### ROCm Component

_No response_

### Steps to Reproduce
```
# install RHEL 8.10 OS, then install drivers
sudo dnf install "kernel-headers-$(uname -r)" "kernel-devel-$(uname -r)"

sudo tee /etc/yum.repos.d/amdgpu.repo <<EOF
[amdgpu]
name=amdgpu
baseurl=https://repo.radeon.com/amdgpu/31.30/el/8/main/x86_64/
enabled=1
priority=50
gpgcheck=1
gpgkey=https://repo.radeon.com/rocm/rocm.gpg.key
EOF
sudo dnf clean all

sudo dnf install amdgpu-dkms

# sudo reboot

sudo dnf install libatomic libquadmath
sudo usermod -a -G render,video $LOGNAME # Add the current user to the render and video groups

# Install ROCm
mkdir therock-tarball && cd therock-tarball
wget https://rocm.nightlies.amd.com/tarball-multi-arch/therock-dist-linux-gfx110X-all-7.14.0a20260615.tar.gz

mkdir install
tar -xf *.tar.gz -C install

ROCM_INSTALL_PATH=$(pwd)/install
sudo tee /etc/profile.d/set-rocm-env.sh << EOF
export ROCM_PATH=$ROCM_INSTALL_PATH
export PATH=\$PATH:\$ROCM_PATH/bin
export LD_LIBRARY_PATH=\$ROCM_PATH/lib
EOF
sudo chmod +x /etc/profile.d/set-rocm-env.sh
source /etc/profile.d/set-rocm-env.sh

test_hip_api
```
