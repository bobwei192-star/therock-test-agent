## Summary

The bundled elfutils sysdep (third-party/sysdeps/linux/elfutils, elfutils 0.192) fails to compile under the amd-llvm/clang toolchain that #4032 routed third-party builds through. The failure is a `-Werror`-promoted warning in `libcpu/riscv_disasm.c:1259` that gcc was lenient about but clang treats as a hard error.

This is **adjacent to** but distinct from #3477's catalog item 6 (which is the rocprofiler-systems-vendored elfutils with a different GCC-15 warning). The failure described here is in the TheRock-bundled elfutils, under clang.

## Reproduction

A clean build with the amd-llvm/clang toolchain (post-#4032) on Ubuntu 26.04 + clang from the amd-llvm tree, configuring elfutils via `third-party/sysdeps/linux/elfutils/CMakeLists.txt`, hits:

```
clang -D_GNU_SOURCE -DHAVE_CONFIG_H ... -std=gnu99 -Wall -Wshadow -Wformat=2 -Wold-style-definition
      -Wstrict-prototypes -Wnull-dereference -Wimplicit-fallthrough -Werror -Wunused -Wextra ...
      -fdollars-in-identifiers -D_FORTIFY_SOURCE=3 -g -O2 -fno-addrsig
      -c .../libcpu/riscv_disasm.c

riscv_disasm.c:1259:24: error: initializing 'struct known_csrs *' with an expression of type 'const void *'
                       discards qualifiers [-Werror,-Wincompatible-pointer-types-discards-qualifiers]
 1259 |                   struct known_csrs *found = bsearch (&key, known,
      |                                      ^       ~~~~~~~~~~~~~~~~~~~~~
```

Cause: `bsearch` returns `const void *` (C11), and `known` is `static const struct known_csrs known[]`. Assigning the return to non-const `struct known_csrs *` discards the const qualifier. Clang's `-Werror -Wincompatible-pointer-types-discards-qualifiers` rejects it; gcc was lenient.

## Workaround

Adding a `CFLAGS` override to the configure invocation in `third-party/sysdeps/linux/elfutils/CMakeLists.txt` is sufficient:

```cmake
"CFLAGS=-g -O2 -Wno-error=incompatible-pointer-types-discards-qualifiers"
```

(The explicit `-g -O2` preserves configure's defaults, which would otherwise be dropped when `CFLAGS` is set explicitly. The `-Wno-error=...` demotes the specific clang error to a warning while keeping `-Werror` for everything else.)

This unblocks the build; verified on a local TheRock build.

## Suggested resolution

Two paths, both viable:

1. **Upstream fix**: file an elfutils bug at `libcpu/riscv_disasm.c:1259` and patch the assignment to `const struct known_csrs *found = bsearch(...)` (or add an explicit cast if the const propagation is invasive).
2. **TheRock-side workaround**: land the `CFLAGS` patch above in `third-party/sysdeps/linux/elfutils/CMakeLists.txt` with a comment pointing at the upstream issue.

Doing both in parallel (file upstream, land workaround as a bridge) seems reasonable. Happy to file a PR for the workaround if that's the preferred path.
