## Summary

The `amd-smi` console script fails with exit code 1 when running `rocm-sdk test` after installing from nightlies.

## Reproduction

```bash
python -m pip install -i https://rocm.nightlies.amd.com/v2/gfx94X-dcgpu/ \
  "rocm[libraries,devel]==7.11.0a20260112"
rocm-sdk test
```

## Error

```
ERROR: testConsoleScripts (rocm_sdk.tests.core_test.ROCmCoreTest.testConsoleScripts) [Check console-script amd-smi]
----------------------------------------------------------------------
Traceback (most recent call last):
  File ".../rocm_sdk/tests/core_test.py", line 135, in testConsoleScripts
    output_text = subprocess.check_output(
subprocess.CalledProcessError: Command '[PosixPath('.../bin/amd-smi')]' returned non-zero exit status 1.
```

## Root Cause Analysis

This regression was introduced when amdsmi was moved from `base/` to `core/` in #2188 (merged 2026-01-10).

The artifact restructuring did not update/test the Python wheel packaging. The console script entry points for `amd-smi` were not updated to reflect the new artifact location, causing the script to fail at runtime.

**Timeline:**
- **2026-01-10**: #2188 merged (amdsmi base/ → core/)
- **2026-01-12**: Affected nightly built
- **2026-01-13**: Reverted (commit dbad34bf)

## Related PRs

- #2188 - Original move (merged, then reverted)
- #2877 - Reland attempt (currently draft, WIP)

## Recommendation

Before relanding #2877, ensure:
1. `rocm-sdk test` passes with all console scripts
2. The wheel packaging correctly reflects the new artifact structure
3. CI includes rocm-sdk self-tests as a gate
