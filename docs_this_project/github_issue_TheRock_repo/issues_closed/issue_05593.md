## Summary

The `profiler-apps` stage fails to link all `causal` example targets in `rocprofiler-systems-examples` with `ld.lld: error: undefined symbol: pthread_barrier_init` and `pthread_barrier_wait`.

Before PR #5196, `profiler/CMakeLists.txt` skipped the entire `rocprofiler-systems` block (including `rocprofiler-systems-examples`) whenever `THEROCK_SANITIZER` was non-empty. PR #5196 removed that guard to enable ASAN builds of rocprofiler-systems, but the causal example targets — which call `pthread_barrier_init`/`pthread_barrier_wait` directly — do not resolve `-lpthread` when linked with the ROCm `ld.lld` linker under ASAN. CMake's `FindThreads` sets `Threads::Threads` to an empty library when it detects implicit pthread support, but lld is strict: the linker invocation only carries `-ldl`, not `-lpthread`, so the barrier symbols remain unresolved.

## Suggested Fix

**Upstream fix (preferred):** In `ROCm/rocm-systems` — [`projects/rocprofiler-systems/examples/causal-helpers.cmake`](https://github.com/ROCm/rocm-systems/blob/b8c378ef2c7220a68122d13b81dab5addd61e016/projects/rocprofiler-systems/examples/causal-helpers.cmake), force `-lpthread` when `CMAKE_THREAD_LIBS_INIT` is empty, then bump the submodule in TheRock.

**TheRock workaround:** Add `-DCMAKE_THREAD_LIBS_INIT=-lpthread` to the `CMAKE_ARGS` of the `rocprofiler-systems-examples` subproject declaration in [`profiler/CMakeLists.txt`](https://github.com/ROCm/TheRock/blob/7b7b238e3e0c2e98436ea230b5114a5b6b946abf/profiler/CMakeLists.txt#L382-L384).
