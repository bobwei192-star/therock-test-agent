### Summary
The workflow job "Release portable Linux PyTorch Wheels" fails during the ROCm Python wheel install step because the pinned ROCm version requested by the workflow is not available on the configured package index.

### Context:
- Effected Arch: gfx94x
- Run: https://github.com/ROCm/TheRock/actions/runs/24765980043
- Rocm Version: 7.13.0a20260422

### Failing job:
- https://github.com/ROCm/TheRock/actions/runs/24765980043/job/72518069795
- https://github.com/ROCm/TheRock/actions/runs/24765980043/job/72518069537

###  What failed
The build script attempts to install a pinned ROCm meta-package version:
```
pip install --force-reinstall --pre \
  --index-url https://rocm.nightlies.amd.com/v2/gfx94X-dcgpu/ \
  'rocm[libraries,devel]==7.13.0a20260422'
```

pip then errors:

- Could not find a version that satisfies the requirement rocm-sdk-core==7.13.0a20260422 (from rocm)
- No matching distribution found for rocm-sdk-core==7.13.0a20260422

the requested 7.13.0a20260422 is not published there.

## Root cause
The workflow/run is pinning rocm_version to a build that is not present in the target index URL path (gfx94X-dcgpu).
