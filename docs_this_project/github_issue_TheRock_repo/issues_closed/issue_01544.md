### Problem Description

The build environment was attempted to be set up for the specified machine in order for ROCm. However, during the build process, the system's RAM becomes completely filled (the system is equipped with 256 GB of RAM), and the swap space is also exhausted, resulting in an out-of-memory (OOM) error. Below are the steps to reproduce the issue. 

### Operating System

Ubuntu 24.04.3 LTS (Noble Numbat)

### CPU

2 x AMD EPYC 7453 28-Core Processor

### GPU

AMDRadeon RX 7900 XTX, AMD Radeon RX 9070 XT, AMD Radeon RX 6700 XT

### ROCm Version

7.0.0

### ROCm Component

flang

### Steps to Reproduce

```
git clone https://github.com/ROCm/TheRock.git
cd TheRock
python ./build_tools/fetch_sources.py

./build_tools/linux_portable_build.py --interactive
python3 -m venv .venv && source .venv/bin/activate
pip install -r /therock/src/requirements.txt

cmake -S /therock/src -B /therock/output/build -GNinja . -DTHEROCK_AMDGPU_FAMILIES=gfx1100
cmake --build /therock/output/build
```

### (Optional for Linux users) Output of /opt/rocm/bin/rocminfo --support

_No response_

### Additional Information

This issue has been consistently observed starting from the following commit:
**Commit** 267c4d9
No OOM issues were encountered before this commit when using the same hardware and build configuration.
