# #1522 - Release pipelines should save artifacts and not just monolithic build results

The monolithic release pipelines aren't really how the system was designed to function (e.g. https://github.com/ROCm/TheRock/blob/main/.github/workflows/release_portable_linux_packages.yml).

In these, we are doing the build and then directly constructing packages (tarballs, wheels). This needs to be broken into multi-phase builds so that the first phase builds and stores the artifacts, and the subsequent phases build various packages.

As-is, we are also not storing the artifacts anywhere, which makes it impossible to do anything with the release assets after the fact other than what was hard-coded in the pipeline.
