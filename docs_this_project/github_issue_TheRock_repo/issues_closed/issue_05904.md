## Summary

The nightly `multi_arch_release_linux_jax_wheels.yml` workflow is failing for all `jax rocm-jaxlib-v0.9.1` matrix jobs (py 3.11 / 3.12 / 3.13 / 3.14) with a Bazel patch conflict.

**Sample failing job:** https://github.com/ROCm/rockrel/actions/runs/27675153165/job/81848545380

```
Apply anyway? [n]
Skipping patch.
2 out of 2 hunks ignored -- saving rejects to file jaxlib/rocm/BUILD.rej
```

## Suspected Root Cause

[ROCm/rocm-jax PR #466](https://github.com/ROCm/rocm-jax/pull/466) (merged 2026-06-16) bumped `JAX_COMMIT` in `jax_rocm_plugin/third_party/jax/workspace.bzl` from `7f75782a` to `0dd06959`. The new pin already contains the visibility changes from [ROCm/jax commit 594e74d15b — "Fix visibility in BUILD files"](https://github.com/ROCm/jax/commit/594e74d15b2f8ece824eb4699194c3eda71635ad) that `0008-Expose-rocm-plugin-targets.patch` was meant to apply. Bazel rejects the patch as already present, aborting the build.

## Suggested Fix

In `ROCm/rocm-jax` on branch `rocm-jaxlib-v0.9.1`:
1. Remove `patches` and `patch_tool`/`patch_args` entries from `jax_rocm_plugin/third_party/jax/workspace.bzl`.
2. Delete `jax_rocm_plugin/third_party/jax/0008-Expose-rocm-plugin-targets.patch`.
