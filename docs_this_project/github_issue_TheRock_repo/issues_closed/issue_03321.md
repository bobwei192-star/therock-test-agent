The `build_tools/third_party/s3_management/manage.py` script should be refactored to move it server side in a subsequent step. Currently `manage.py` is called within workflow like
https://github.com/ROCm/TheRock/blob/e267c57b9a21c28fc75872793418c32703e2821d/.github/workflows/build_portable_linux_pytorch_wheels.yml#L222
after a build was pushed. Instead the script should run server side as AWS Lambda.

As part of this issue it must be evaluated how to refactor this script to allow moving it server side. It is not desired to re-generate _all_ index files in the prefixes:
https://github.com/ROCm/TheRock/blob/e267c57b9a21c28fc75872793418c32703e2821d/build_tools/third_party/s3_management/manage.py#L38-L51

Instead only the index files for a specific prefix (`CUSTOM_PREFIX`?) should be regenerated.
