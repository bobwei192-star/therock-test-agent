**Summary**
TheRock CI builds across  mono-repos rocm-libraries, rocm-systems are failing intermittently due to GitHub returning HTTP 500 errors during git clone / git fetch operations. I have  been observing  this issues daily from the past week, approximately IST morning between 2AM to 9AM.

Builds fail during source checkout with errors like:

unable to access 'https://github.com/ROCm/TheRock/': The requested URL returned error: 500
RPC failed; HTTP 500 curl 22 The requested URL returned error: 500
expected 'packfile'

Both Linux and Windows jobs are affected. Retries of the same workflow typically succeed once the window passes.

**Affected repos:**
ROCm/rocm-libraries
ROCm/rocm-systems

**Example failures:**

https://github.com/ROCm/rocm-libraries/actions/runs/22639002078/attempts/1
https://github.com/ROCm/rocm-libraries/actions/runs/22639095394/attempts/1
https://github.com/ROCm/rocm-systems/actions/runs/22623196123/attempts/1

**Possible causes:**
GitHub API / Git server-side instability during the affected time window
Rate limiting or throttling on large mono-repo fetches.
