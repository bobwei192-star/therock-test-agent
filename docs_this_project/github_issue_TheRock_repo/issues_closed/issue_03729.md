## Summary

Remove the `amd-comgr-stub` delay-load/namespace-isolation mechanism and instead have comgr statically link LLVM/Clang/LLD component libraries. This eliminates the dependency on `libLLVM.so` at runtime while keeping LLVM symbols hidden behind comgr's existing version script.

## Motivation

The delay-load stub uses `dlmopen` for linker namespace isolation to prevent LLVM symbol interposition when user code also loads LLVM. However, glibc namespace support is insufficient for reliable isolation across supported distros, so this approach is being abandoned.

## Approach

TheRock already builds both `libLLVM.so` (monolithic shared) and individual static `.a` component libraries (`BUILD_SHARED_LIBS=OFF` + `LLVM_BUILD_LLVM_DYLIB=ON`). Comgr's `CMakeLists.txt` already has working static-link code paths, but they're gated on `LLVM_LINK_LLVM_DYLIB` — a variable set by `find_package(LLVM)` as a normal CMake variable that can't be overridden by TheRock's pre_hook.

### Upstream change (amd-llvm repo)

Add a `COMGR_STATIC_LLVM` option to `amd/comgr/CMakeLists.txt` (4 lines) that overrides `LLVM_LINK_LLVM_DYLIB` and `CLANG_LINK_CLANG_DYLIB` after `find_package`:

```cmake
option(COMGR_STATIC_LLVM "Statically link LLVM/Clang into comgr (hides LLVM symbols)" OFF)
if(COMGR_STATIC_LLVM)
  set(LLVM_LINK_LLVM_DYLIB OFF)
  set(CLANG_LINK_CLANG_DYLIB OFF)
endif()
```

### TheRock changes

1. Pass `-DCOMGR_STATIC_LLVM=ON` in comgr's CMAKE_ARGS
2. Remove `THEROCK_FLAG_COMGR_DELAY_LOAD` option from top-level CMakeLists.txt
3. Remove target name indirection and stub subproject from `compiler/CMakeLists.txt`
4. Delete `compiler/amd-comgr-stub/` directory
5. Clean up `compiler/artifact-amd-llvm.toml` (remove stub sections)
6. Clean up `CMakePresets.json` (remove delay-load and impl-specific sanitizer entries)
7. Clean up `build_tools/packaging/linux/package.json` (remove stub entry)
8. Clean up `compiler/pre_hook_amd-comgr.cmake` (remove now-irrelevant `LLVM_LINK_LLVM_DYLIB` setting)

### Why symbol hiding works

Comgr already uses:
- `-fvisibility=hidden` (comgr `CMakeLists.txt` line 219)
- A version script (`exportmap.in`) with `local: *;` that only exports `amd_comgr_*` symbols

When LLVM is statically linked into `libamd_comgr.so`, all LLVM symbols are automatically hidden.

### Verification

```bash
# LLVM symbols hidden
nm -D libamd_comgr.so | grep -c "LLVM"  # Expected: 0
# No runtime dependency on libLLVM.so
ldd libamd_comgr.so | grep -c libLLVM   # Expected: 0
# comgr API symbols exported
nm -D libamd_comgr.so | grep "amd_comgr_"  # Expected: all API symbols
```
