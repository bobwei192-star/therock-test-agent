## Problem

**Severity:** P2 — workaround exists (keep hip-tests target-specific)

Without the target-neutral fix from the reverted PR #3348 (see sibling issue for dist target advertising), hip-tests remain `target-specific`. When `THEROCK_KPACK_SPLIT_ARTIFACTS=ON`, the kpack splitter attempts to split hip-test binaries. These test binaries may not contain device code in the expected fat binary format, or may have device code that doesn't match expected patterns. The splitter hits an assertion or error.

## Fix Direction

Two paths:
1. **Fix dist target advertising** so hip-tests become target-neutral (splitter skips target-neutral artifacts)
2. **Make the splitter resilient** to binaries without device code — it should produce a generic-only artifact rather than crashing

Both should be done — the splitter should be robust regardless.
