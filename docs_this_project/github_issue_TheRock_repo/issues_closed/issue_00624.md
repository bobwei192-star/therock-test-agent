## Problem

When publishing a nightly release, a small test runs on every release. A test summary is posted on the release page https://github.com/ROCm/TheRock/releases/tag/nightly-tarball. However, this is incomplete and only provides results for a single GPU-family.

## Background

The workflow `release_portable_linux_packages.yml` triggers `test_release_packages.yml` with

https://github.com/ROCm/TheRock/blob/5e6b5da2074313b2e1ae1331c14dad97cc734b65/.github/workflows/release_portable_linux_packages.yml#L175-L180

for nightly releases. The latter tests the release tarball and posts a summary via

https://github.com/ROCm/TheRock/blob/5e6b5da2074313b2e1ae1331c14dad97cc734b65/.github/workflows/test_release_packages.yml#L109-L114

While it expected that only the _last version_ is tested, it also overwrites the summary for tarballs with the same version but build for a different GPU-family. The root cause is that a matrix is used at
https://github.com/ROCm/TheRock/blob/5e6b5da2074313b2e1ae1331c14dad97cc734b65/.github/workflows/release_portable_linux_packages.yml#L95-L96
and thus `test_release_packages.yml` is triggered for each GPU-family and the latest _wins_.

## Expected behavior

The release page should contain test results for **all** GPU-families of the latest release.
