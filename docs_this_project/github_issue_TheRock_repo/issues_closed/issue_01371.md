### Problem Description

## Problem

Audio transformer models (wav2vec2) crash with segfault in libLLVM.so during model inference on AMD Strix Halo. Other PyTorch workloads (LLMs, vision models, basic tensor ops) work correctly.

## Hardware/Software Environment

- **Hardware**: AMD RYZEN AI MAX+ 395 w/ Radeon 8060S (Strix Halo)
- **OS**: Linux 6.16.3-200.fc42.x86_64 (Fedora 42)  
- **Memory**: 128GB unified memory
- **Python**: 3.13.7
- **PyTorch**: 2.7.1+rocm7.0.0rc20250828 (nightly)
- **transformers**: 4.55.4
- **librosa**: 0.11.0
- **pytorch-triton-rocm**: 3.3.1+rocm7.0.0rc20250828

## Reproduction Steps

1. Create test audio:
   ```bash
   ffmpeg -f lavfi -i "sine=frequency=1000:duration=5" -ar 16000 test_audio.wav
   ```

2. Run minimal script:
   ```python
   import torch, librosa
   from transformers import Wav2Vec2ForCTC, Wav2Vec2Processor

   processor = Wav2Vec2Processor.from_pretrained("facebook/wav2vec2-base-960h")
   model = Wav2Vec2ForCTC.from_pretrained("facebook/wav2vec2-base-960h") 
   audio_input, _ = librosa.load("test_audio.wav", sr=16000)
   input_values = processor(audio_input, sampling_rate=16000, return_tensors="pt").input_values
   result = model(input_values, output_hidden_states=True)  # <-- CRASHES HERE
   ```

**Expected**: Model runs inference successfully  
**Actual**: Segmentation fault in libLLVM.so

## Evidence this is ROCm audio-specific

- ✓ Basic tensor ops work: `torch.randn(1000,1000).cuda() @ torch.randn(1000,1000).cuda()`  
- ✓ LLM inference works normally
- ✓ Computer vision models work normally  
- ✗ Only crashes during audio transformer forward pass
- **dmesg**: `python[8626]: segfault at 0 ip 00007fab5aa3bc0a sp 00007ffeb4f4a2f0 error 4 in libLLVM.so.20.0git`

## Impact  

Blocks all audio AI workloads on Strix Halo systems with ROCm nightly builds.
