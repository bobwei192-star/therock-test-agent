### Problem Description

On my system, I’ve noticed a discrepancy in the versions installed depending on the use of the `--pre` flag.

Running:

`python -m pip install --pre --index-url https://rocm.nightlies.amd.com/v2/gfx120X-all/ torch torchvision torchaudio`

installs the `20251218` builds of torch, torchvision, and torchaudio, while:

`python -m pip install --index-url https://rocm.nightlies.amd.com/v2/gfx120X-all/ torch torchvision torchaudio`

installs the `20260104` builds.

Note the absence of the `--pre` flag in the second command.

I’ve been observing this behavior for some time, and it seems counterintuitive that omitting `--pre` results in a newer build. Many users assume that TheRock wheels require this flag in order to install the latest versions.

Is there a specific reason for this versioning logic, or is it simply a side effect of how the wheels are tagged?
