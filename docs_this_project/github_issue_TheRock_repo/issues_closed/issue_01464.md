### Problem Description

When using nightly python sdks from https://github.com/ROCm/TheRock/blob/main/RELEASES.md#installing-rocm-python-packages there is no cmake files present in `/usr/local/lib/python3.12/dist-packages/_rocm_sdk_core/lib/`

Possibly due to : https://github.com/ROCm/TheRock/blob/main/build_tools/build_python_packages.py#L45

The cmake files are present in the nightly tar-balls.

### Operating System

Ubuntu 24.04

### CPU

AMD Strix Halo 

### GPU

AMD Strix Halo

### ROCm Version

ROCm 7.0.0 nightly

### ROCm Component

rocm-cmake

### Steps to Reproduce

1. Install nightly python packages from https://github.com/ROCm/TheRock/blob/main/RELEASES.md#installing-rocm-python-packages 
2. Observe no cmake files present in `/usr/local/lib/python3.12/dist-packages/_rocm_sdk_core/lib/`
3. Install nightly tarball to /opt/rocm
4. Observe cmake files present in `/opt/rocm/lib/`
