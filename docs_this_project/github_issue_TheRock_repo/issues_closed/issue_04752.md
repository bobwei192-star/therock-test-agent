As part of the DCGPU team we are seeing a large uptick in issues with installing our packages as TheROCK becomes more popular.  Legacy ROCm installation via amdgpu-repo properly registers package provides with RPM but TheROCK does not, which causes installation issues with packages that require amd libraries.  I know this is a public-facing repository so I will censor internal names.  

**We would like to know if any plans are in place to remedy this?**

One potential solution is a "shim" RPM file which checks for the presence of these libraries and creates a provides statement for them.

**Installation Steps:**
`wget https://rocm.nightlies.amd.com/tarball/therock-dist-linux-gfx110X-all-7.12.0a20260202.tar.gz`
`mkdir -p /opt/rocm-7.12.0 && tar -xf *.tar.gz -C /opt/rocm-7.12.0`
`sudo ln -sf /opt/rocm-$BUILDNUM /etc/alternatives/rocm`
`sudo ln -sf /etc/alternatives/rocm /opt/rocm`
