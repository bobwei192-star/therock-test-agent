## Error
```
usage: build_prod_wheels.py [-h] {install-rocm,build} ...
build_prod_wheels.py: error: unrecognized arguments: --root-checkout-dir B:/src
##[error]Process completed with exit code 2.
```

## Root Cause
[PR #5531](https://github.com/ROCm/TheRock/pull/5531) (merged 2026-06-19T01:52:57Z, commit `fd17662f`) updated `multi_arch_build_windows_pytorch_wheels.yml` to pass `--root-checkout-dir` to `build_prod_wheels.py`, and simultaneously added that argument to the script. However, the rockrel nightly workflow pinned TheRock at SHA `677eaab47f` — the direct parent of `fd17662f` — so the reusable YAML executes at HEAD (which has `--root-checkout-dir` in the call) while the Python script in the checked-out source is the old version that does not accept it.
