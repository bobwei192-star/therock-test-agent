# #6152 - PR-quality skill should defer to the repo's machine policy gate (therock_pr_bot)

The `rocm-pr-quality` base skill and the `therock_pr_bot` enforcement gate can disagree. The skill allows waivers and self-evident-docs exemptions for work-tracking (M4/M5), but the bot hard-requires a `JIRA ID`/`ISSUE ID` reference with no waiver — so a PR the skill considers complete can still be blocked by the bot (e.g. a docs-only change). The skill also says nothing about the bot's branch-name and Conventional-Commits title rules, so drafted PRs can trip the gate on the first push.

Rather than copy the bot's rules into the skill (which would drift out of sync with `policy.yml`), the skill should discover any machine-enforced PR policy at run time, conform to it, and treat it as authoritative over the skill's own assumptions.

Proposed fix: add a base operating principle plus an author-assist step that detect and conform to a machine policy gate, and point the TheRock overlay at the live `skills/therock_pr_bot/policy.yml` and `FAQ.md` as the source of truth.
