## Problem

Python 3.10 was added to the PyTorch wheel build matrices, but the builds and tests are failing due to compatibility issues in rocm_sdk packages and dependency resolution problems.

## Current Errors

### 1. rocm-sdk test failure
```
Unknown option: -P
```
The `-P` flag used in rocm_sdk tests and CLI is not available in Python 3.10 (introduced in Python 3.11).

### 2. networkx dependency resolution
```
ERROR: Package 'networkx' requires a different Python: 3.10.11 not in '>=3.11'
```
networkx 3.5+ requires Python 3.11+, but pip tries to install it for Python 3.10.

### 3. MarkupSafe missing wheels  
```
ERROR: Could not find a version that satisfies the requirement MarkupSafe>=2.0
```
MarkupSafe 3.0.3 doesn't provide Python 3.10 wheels, and our S3 bucket doesn't have compatible versions.

## Root Causes

1. **rocm_sdk packages** - Hard-coded use of Python 3.11+ features
2. **S3 package index** - Missing Python version metadata and compatible package versions
3. **update_dependencies.py** - Explicitly filtering out cp310 wheels

## Solution Required

1. Make rocm_sdk tests compatible with Python 3.10
2. Update S3 package management to handle multi-version dependencies
3. Add proper Python version metadata to package indices
4. Remove cp310 wheel filtering

## Impact

Without these fixes, Python 3.10 cannot be used for PyTorch wheel builds despite being added to the build matrix.
