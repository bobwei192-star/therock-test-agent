### Problem Description

[hipBLASLt/tensilelite/Tensile/Toolchain/Validators.py](https://github.com/ROCm/hipBLASLt/blob/d2f9dd521c7e7abc6306bd5698fd3fcfd2b67175/tensilelite/Tensile/Toolchain/Validators.py#L35C39-L35C64) tries to find and "validate" the ROCm toolchain in a series of directories and would fail the build even if those toolchains are not used by TheRock.

E.g. one community member had an empty `C:/Program Files/AMD/ROCm` on their build machine, possibly from uninstalling a previous HIP WIndows SDK, and the code would fail the build and raise`ValueError` from https://github.com/ROCm/hipBLASLt/blob/d2f9dd521c7e7abc6306bd5698fd3fcfd2b67175/tensilelite/Tensile/Toolchain/Validators.py#L5

(`max(versions, key=lambda d: tuple(map(int, d.name.split("."))))` raises `ValueError` due to `versions` being an empty list)
