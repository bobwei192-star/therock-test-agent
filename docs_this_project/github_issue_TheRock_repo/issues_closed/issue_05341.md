Forking discussion from https://github.com/ROCm/TheRock/pull/4419#discussion_r3262592363

Tarballs at https://rocm.nightlies.amd.com/tarball-multi-arch/ saw a 10GB uncompressed size increase:

| | Tarball compressed size | Tarball uncompressed size |
| -- | -- | -- |
`therock-dist-windows-gfx1151-7.13.0a20260425.tar.gz` | 2.80GB | 13.5GB
`therock-dist-windows-gfx1151-7.14.0a20260519.tar.gz` | 4.59GB | 23.4GB

The likely culprit is https://github.com/ROCm/TheRock/pull/4419 - we were previously setting the debug format only in `Debug` and `RelWithDebInfo`:
```cmake
if(WIN32 AND NOT DEFINED CMAKE_MSVC_DEBUG_INFORMATION_FORMAT)
  set(CMAKE_MSVC_DEBUG_INFORMATION_FORMAT "$<$<CONFIG:Debug,RelWithDebInfo>:Embedded>")
```
that PR changed it to be set unconditionally:
```cmake
if(WIN32 AND NOT DEFINED CMAKE_MSVC_DEBUG_INFORMATION_FORMAT)
  set(CMAKE_MSVC_DEBUG_INFORMATION_FORMAT "Embedded")
```

Docs: https://cmake.org/cmake/help/latest/variable/CMAKE_MSVC_DEBUG_INFORMATION_FORMAT.html

This enabled debug information in all subproject builds, notably LLVM, which added 1-2GB compressed / 10GB of uncompressed data to:
* the `amd-llvm_dev_generic` artifact
* `rocm[devel]` python packages
* rocm tarballs
