The `build_tools/third_party/s3_management/update_dependencies.py` script should be refactored to run server side. Currently the script is manually executed. Once a retention policy is in place, dependencies will be tidied up and will need to be re-uploaded. Furthermore, we want to occasional check for updated dependencies instead of shipping old ones via the self-contained repository APIs. The script needs to either run when dependencies get clean up / get deleted or on a fixed interval.
