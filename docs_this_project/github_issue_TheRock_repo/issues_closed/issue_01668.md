### Problem Description

Hi TheRock Team,
There is `librocm_smi64.so.7` symlink missing in rocm libs directory when installed from wheels.

I wanted to get HF transformers working, which I evetualy managed and get several LLM models interference working, however tensorflow-rocm pip module is looking for `librocm_smi64.so.7` once I created symlink all worked as expected
`ln -s ../../_rocm_sdk_core/lib/librocm_smi64.so.1 librocm_smi64.so.7`

the environment is installed as follows in python:3.12-slim docker image
```
apt install libquadmath0
apt-get install libgfortran5
apt install libgomp1
apt install libnuma1
apt install libatomic1

python -m pip install \
  --index-url https://rocm.nightlies.amd.com/v2/gfx1151/ \
  "rocm[libraries,devel]"

python -m pip install \
  --index-url https://rocm.nightlies.amd.com/v2/gfx1151/ \
  --pre torch torchaudio torchvision

python -m pip install   tensorflow-rocm -f https://repo.radeon.com/rocm/manylinux/rocm-rel-7.0/

python -m pip install transformers accelerate kernels
```

when `librocm_smi64.so.7` symlink is present and following env variables are set
```
export ROCM_PATH=$(rocm-sdk path --root)
export HIP_PLATFORM=amd
export HIP_PATH=$ROCM_PATH
export HIP_CLANG_PATH=$ROCM_PATH/llvm/bin
export HIP_INCLUDE_PATH=$ROCM_PATH/include
export HIP_LIB_PATH=$ROCM_PATH/lib
export HIP_DEVICE_LIB_PATH=$ROCM_PATH/lib/llvm/amdgcn/bitcode
export PATH="$ROCM_PATH/bin:$HIP_CLANG_PATH:$PATH"
export LD_LIBRARY_PATH="$HIP_LIB_PATH:$ROCM_PATH/lib:$ROCM_PATH/lib64:$ROCM_PATH/llvm/lib:${LD_LIBRARY_PATH:-}"
export LIBRARY_PATH="$HIP_LIB_PATH:$ROCM_PATH/lib:$ROCM_PATH/lib64:${LIBRARY_PATH:-}"
export CPATH="$HIP_INCLUDE_PATH:${CPATH:-}"
export PKG_CONFIG_PATH="$ROCM_PATH/lib/pkgconfig:${PKG_CONFIG_PATH:-}"

```
simple HF transformers inference works

```
from transformers import AutoTokenizer, AutoModelForCausalLM
import torch

model_name="google/gemma-3-270m-it"

device = torch.device("cuda") if torch.cuda.is_available() else torch.device("cpu")
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForCausalLM.from_pretrained(model_name)
model.to(device)
messages = [
    {"role": "user", "content": "Who are you?"},
]
inputs = tokenizer.apply_chat_template(
	messages,
	add_generation_prompt=True,
	tokenize=True,
	return_dict=True,
	return_tensors="pt",
).to(model.device)

outputs = model.generate(**inputs, max_new_tokens=40)
print(tokenizer.decode(outputs[0][inputs["input_ids"].shape[-1]:]))
```

the only strange observation is, that it looks like that rocm pytorch is ignoring VRAM and utilize primarily GTT. I am also observing the same thing in rocm build of llama.cpp is this expected behaviour?
