Docs at https://github.com/ROCm/TheRock/blob/main/docs/development/ci_behavior_manipulation.md#pull-request say that `test:...` means "Run full tests only for the specified projects (e.g. `test:rocthrust`, `test:hipblaslt`)"

On https://github.com/ROCm/TheRock/pull/4935, https://github.com/ROCm/TheRock/actions/runs/25134138451?pr=4935 ran _all_ subproject tests with test type "full" though, despite having the `test:rccl` label.

Looks like some issue parsing `LINUX_TEST_LABELS` and threading through the top level configure job down to "configure test matrix"?
