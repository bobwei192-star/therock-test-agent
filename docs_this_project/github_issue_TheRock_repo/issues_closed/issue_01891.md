Package promotion from  pre-release area to release area needs to be automated.
We need to follow the convention of dropping certain keywords to make the packages be standard

## Details

When we build release candidates they have a suffix like `rc0`, `rc1`, etc. in their package versions. Once we have validated a release candidate we want to "promote" it to stable by dropping those suffixes.

## Alternatives

1. Rebuild packages without suffixes in their versions
    * This risks changing the binaries, if we do not fully pin all commits and dependencies
2. Build without version suffixes, e.g. into one subfolder per release candidate
    * This makes it harder to differentiate between installed release candidate packages
