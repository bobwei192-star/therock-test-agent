# #3950 - [Issue] `llvm-strip` causes `libunwind.so.99` to have offset ELF load command

**State**: closed  
**Author**: kcossett-amd  
**Created**: 2026-03-12T23:15:00Z  
**Updated**: 2026-03-17T18:21:55Z  
**URL**: https://github.com/ROCm/TheRock/issues/3950  

---

When I pulled an `amdrocm-profiler7.12_7.12` artifact (for ex: [amdrocm-profiler7.12_7.12.0~20260311-22937000989_amd64.deb](https://rocm.nightlies.amd.com/deb/20260311-22937000989/pool/main/amdrocm-profiler7.12_7.12.0~20260311-22937000989_amd64.deb)) , and I ran the following:

`ldd <path-to-extract>/opt/rocm/core-7.12/bin/rocprof-sys-instrument`, it fails with the following:
```
error while loading shared libraries: libunwind.so.99: ELF load command address/offset not page-aligned
```

Following my investigation from #3932 , the problem lies with `llvm-strip` being used on the library. If in a fresh build, you package the binary without having `llvm-strip` being used, the `ldd` command above does not fail. However, after using `llvm-strip`, it fails with the exact error above. 

**To reproduce the problem locally**:

```
# Build the profiler in the manylinux container
cmake -S /therock/src -B /therock/output/build -GNinja \
  -DTHEROCK_AMDGPU_FAMILIES=gfx94X-all \
  -DTHEROCK_ENABLE_ALL=OFF \
  -DTHEROCK_ENABLE_PROFILER=ON \
  -DTHEROCK_ENABLE_CORE_RUNTIME=ON \
  -DTHEROCK_ENABLE_HIP_RUNTIME=ON \
  -DTHEROCK_ENABLE_COMPILER=ON
cmake --build /therock/output/build -- -l 16
```
Then using the `build_package.py` script, build the deb package:
```
python3 build_tools/packaging/linux/build_package.py \
  --artifacts-dir output-linux-portable/build/artifacts \
  --dest-dir build/packages \
  --target gfx94X-all \
  --pkg-type deb \
  --rocm-version 7.12.0 \
  --pkg-names amdrocm-profiler
```
Extract it:
```
dpkg-deb -x build/packages/amdrocm-profiler7.12_7.12.0_amd64.deb build/packages/extracted
```
Then run the `ldd` command and observe the failure
```
ldd build/packages/extracted/opt/rocm/core-7.12/bin/rocprof-sys-instrument
build/packages/extracted/opt/rocm/core-7.12/bin/rocprof-sys-instrument: error while loading shared libraries: libunwind.so.99: ELF load command address/offset not page-aligned
```

**NOTE**
My system has `llvm-strip --version`:
```
llvm-strip, compatible with GNU strip
Ubuntu LLVM version 18.1.3
  Optimized build.
```

Rerunning these steps with the fix in #3932 prevents this error as it stops `llvm-strip` from being ran.

On the other hand, if I now install `llvm-20` via `sudo apt install llvm-20` and change https://github.com/ROCm/TheRock/blob/main/build_tools/packaging/linux/template/debian_rules.j2#L27 to instead force it to use the v20:
```
ln -sf "$(shell command -v llvm-strip-20 || command -v llvm-strip)"   debian/.llvm-tools/strip
ln -sf "$(shell command -v llvm-objcopy-20 || command -v llvm-objcopy)" debian/.llvm-tools/objcopy
```
Then when I redo the steps above, **no error is observed** when using `ldd`. I am thus led to believe that
https://github.com/ROCm/TheRock/blob/main/.github/workflows/build_native_linux_packages.yml#L113
should be updated to install `llvm-20`.
