### Problem Description

When running the `hipfft_callback` or `rocfft_callback` tests from rocm-examples on a
  multi-arch pip install, HIP aborts when loading the rocFFT callback kernel from
  `fft_lib_gfx<arch>.kpack`:

  Cannot create GlobalVar Obj for symbol:
  _ZL31store_cb_default_complex_double.static.701465ed1bdc0ef

  All `fft_lib_*.kpack` files in `_rocm_sdk_libraries/.kpack/` are identical at 2,780 bytes
  regardless of architecture, suggesting they are the same placeholder rather than
  per-arch compiled objects:

  ```bash
  $ wc -c .venv/lib/python3.12/site-packages/_rocm_sdk_libraries/.kpack/fft_lib_gfx*.kpack
  2780 fft_lib_gfx1100.kpack
  2780 fft_lib_gfx90a.kpack
  2780 fft_lib_gfx942.kpack
```
  The embedded HSACO (10,864 bytes) is missing the static variable
  _ZL31store_cb_default_complex_double.static.*, which the callback kernel references
  but is absent from the code object.

### Steps to Reproduce

**Install ROCm (multi-arch pip)**
```bash
  python3 -m venv .venv
  source .venv/bin/activate
  pip install --index-url https://rocm.nightlies.amd.com/whl-multi-arch/ "rocm[libraries,device-all]"
  rocm-sdk init
```
  **Build rocm-examples**
```bash
  SITE=.venv/lib/python3.12/site-packages
  export LD_LIBRARY_PATH=${SITE}/_rocm_sdk_core/lib:${SITE}/_rocm_sdk_libraries/lib:${SITE}/_rocm_sdk_devel/lib

  git clone https://github.com/ROCm/rocm-examples.git
  cd rocm-examples
  cmake -B build \
    -DROCM_PATH=${SITE}/_rocm_sdk_devel \
    -DROCM_EXAMPLES_ENABLE_ROCDECODE=OFF \
    -Wno-dev
  cmake --build build -j$(nproc)
```
  **Run the affected tests**
```bash
  cd build
  ctest -j1 --output-on-failure -R 'hipfft_callback|rocfft_callback'
```
  HIP aborts with the symbol error shown above.
