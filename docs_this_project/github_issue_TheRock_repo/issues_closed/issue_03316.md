During investigating multi-arch builds I noticed that both the rocrtst lib and test artifact bundle `libhwloc.so.5`: 
```
./artifacts/rocrtst_lib_generic/core/rocrtst/stage/lib/rocrtst/lib/libhwloc.so.5
./artifacts/rocrtst_test_generic/core/rocrtst/stage/lib/rocrtst/lib/libhwloc.so.5
```

rocrtst should not bundle `libhwloc.so` and should depend on the lib bundled with sysdeps-hwloc (`libhwloc.so` or rather `librocm_sysdeps_hwloc.so.5`).
