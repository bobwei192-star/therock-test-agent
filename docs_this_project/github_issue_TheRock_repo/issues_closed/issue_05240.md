Summary

Add a reusable post-failure log processing step for CI workflows that:

Detects failed logs
Renames them using the 0.error.*.log convention
Writes a short failure summary to GITHUB_STEP_SUMMARY

This makes failures easier to triage by ensuring the most relevant logs sort first in artifacts and are visible in the GitHub Actions summary.

Motivation

When CI jobs fail, the key log files are often buried among many other outputs. That makes root-cause analysis slower than it needs to be.

Automatically identifying failed logs and renaming them helps by:

making error logs sort first in directory listings and archives
making them easier to spot in uploaded artifacts
surfacing them directly in the GitHub Actions job summary
