### Problem Description

OS: 10.0.26100
CPU: AMD RYZEN AI MAX+ 395 w/ Radeon 8060S
GPU: AMD Radeon(TM) 8060S Graphics

When using the lemonade nightly builds of llamacpp with ROCm7.9 or 7.10 I am unable to load larger models(gpt-oss120b glm-air-4.5) as I receive a failed to load model error. It states that it failed to allocate ROCm0 buffers.

My Strix Halo has 96GB of available vRAM and 32GB of system ram. 

I run the following llamacpp args: `./llama-server.exe -m "C:\Users\kpoin\.cache\gguf\gpt-oss-120b-mxfp4-00001-of-00003.gguf" --alias "gpt-oss-120b" --host 0.0.0.0 --port 9000 -c 0 -ngl 999 --no-mmap --mlock --metrics --jinja -fa on --reasoning-format auto -b 2048 -ub 2048`

I have no issues running this on ROCm 7.0 or Vulkan builds with llamacpp.
