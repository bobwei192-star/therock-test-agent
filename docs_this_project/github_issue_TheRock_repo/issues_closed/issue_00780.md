Overview: 

Create a test result database for all CI workflows encompassing all projects. The database will be used to create an easy to read and analyze dashboard that highlights the last passed build and frequency of failures for various test cases.   

Purpose: 

Currently our CI workflows provide overall build results for each workflow; finding the last working build for a particular testcase and obtaining the failure percentage per testcase requires manual analysis of the logs. 

Purpose of this document is to explain how we could overcome the above with this design, create a Redshift database cluster in AWS, use query editor to parse s3 logs for each of the CI workflows. End goal is to create a quick sight dashboard from Redshift database queries to make useful conclusions from the results data.
