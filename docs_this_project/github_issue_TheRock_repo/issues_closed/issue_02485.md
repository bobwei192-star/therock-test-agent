### Problem Description

A little while ago I noted for later versions than rocm6.4 I saw extreme regressions in 4xultrasharp model upscale performance. See https://github.com/comfyanonymous/ComfyUI/pull/10302#issuecomment-3410741865

With cudnn enabled  I saw
* 2.9.0+rocm6.4 ImageUpscaleWithModel **1.84s/it**
* 2.10.0a0+rocm7.10.0a20251015 ImageUpscaleWithModel **81.14s/it**

With cudnn disabled they both perform similarly ~10s/it (so also significantly slower than rocm6.4 with cudnn enabled).

I recently tried the latest pytorch rocm7.1 too and this regression appears to still exist, while other operations seem to generally be better. Any ideas why this has regressed so much, or what I can try to resolve this?
