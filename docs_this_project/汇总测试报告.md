### 「闸门」含义

```
              ┌──────────────────────┐
              │  Sanity 闸门           │
              │  耗时 ~30 秒           │
              └─────────┬────────────┘
                        │ 通过?
               ┌───────┴───────┐
               ▼               ▼
             是               否
        ┌────────────┐   ┌──────────────┐
        │ 放行        │   │ 拦截          │
        │ 所有组件并行  │   │ 跳过后续测试    │
        │ 开始测试     │   │ 报告基础设施问题 │
        └────────────┘   └──────────────┘
```

| 特性 | 说明 |
|------|------|
| **前置依赖** | 工作流依赖关系 |
| **快速失败** | 耗时约 30 秒 |
| **区分故障类型** | Sanity 失败 → 基础设施问题；组件失败 → 组件自身问题 |
| **节省成本** | 避免在 bad build 上跑大量无用测试 |
