OpenCode + TheRock 循环测试工程（Loop Engineering）完整落地方案
项目定位
基于 anomalyco/opencode 的 therock-test-agent 二次改造，封装CLI 全自动串行分级测试引擎，专门针对 ROCm/TheRock 多组件分级测试（Quick → Standard → Comprehensive → Full），带全局状态持久化、断点续跑、分层报告输出。
核心依赖：
基础框架：https://github.com/anomalyco/opencode.git（复用其终端交互、命令执行、日志、子进程封装、agent 调度能力）
被测工程：https://github.com/ROCm/TheRock.git
扩展目标：Loop Engineering 循环调度层、全局状态机、分级任务排序、结构化报告导出

二、核心需求拆解与实现方案
输入参数（CLI 交互入口）
启动 agent 后交互式输入 3 项全局参数，持久存入 global_state.json：
rocm_artifacts_path：ROCm 编译产物根目录
gpu_model：目标 GPU（如 gfx1151）
component_list：待测试 TheRock 组件数组（rocblas/rocsparse/rocalution/rocprofiler_sdk...）
Step1：全局状态文件 global_state.json 规范
统一状态存储，每次操作原子写入，支持断点续跑、中途退出恢复。
json
{
  "meta": {
    "start_timestamp": "2026-07-08Txx:xx:xx",
    "gpu_model": "gfx1151",
    "rocm_artifacts_path": "/opt/rocm-build",
    "all_components": ["rocblas", "rocsparse", "rocalution", "rocprofiler_sdk"],
    "test_level_order": ["quick", "standard", "comprehensive", "full"],
    "result_root_dir": "./therock_test_results_xxxxxx"
  },
  "schedule": {
    "total_task_count": 0,
    "finished_task_count": 0,
    "current_running_task": null,
    "pending_tasks": [],
    "completed_tasks": [],
    "failed_tasks": [],
    "skipped_tasks": []
  },
  "global_result": {
    "total_pass": 0,
    "total_fail": 0,
    "total_skip": 0,
    "all_failure_summary": []
  }
}
状态模块能力：
加锁读写，避免多进程覆盖；
每次任务前后自动更新进度；
重启 CLI 自动加载历史进度，跳过已跑完组件。
Step2：任务排序规则（双重递增优先级）
两层排序，保证耗时短、压力小任务优先执行，高负载 Full 放最后：
第一优先级：测试分级权重
quick(1) < standard(2) < comprehensive(3) < full(4)
第二优先级：内置组件耗时权重（预定义常量）
稀疏库 / Profiler/full 负载权重更高，后置执行：
rocprim < rocblas < rocsolver < hipsparselt < rocsparse < rocalution < rocprofiler_sdk
排序输出：生成一维有序任务列表存入 global_state["schedule"]["pending_tasks"]。
Step3：结果目录自动创建规范
自动生成带时间戳根目录，目录结构固定，写入状态文件：

therock_test_results_20260708_153000/

├── summary_total.md           # 汇总测试报告
├── full_logs/                 # 全量原始日志
│   ├── rocblas_quick.log

└── failure_details/           # 失败组件明细报告
    ├── rocsparse_full_fail_report.md
    └── rocalution_comprehensive_fail_report.md
Step4：主循环引擎 Loop Core
串行阻塞执行，单任务生命周期：
plaintext
for task in pending_tasks:
    1. 更新 global_state → current_running_task = task
    2. 调用 therock-test-agent 底层接口执行对应组件+分级测试
    3. 实时捕获 stdout/stderr、返回码、超时、GPU 报错（GPU reset ring timeout 等）
    4. 导出本次测试完整 log 到 full_logs/
    5. 判断结果：
       - Pass：移入 completed_tasks，pass 计数+1
       - Fail：移入 failed_tasks，生成独立失败明细报告存入 failure_details/
       - Timeout/Hang：标记特殊失败，记录 GPU 复位日志
    6. 更新全局汇总指标、刷新 global_state.json
    7. 清空 current_running_task

复用 agent 中调用 TheRock ctest/build 命令的进程封装，自带超时控制、流捕获、信号处理（解决 GPU hang 卡死无返回问题）。
日志工具链
复用 opencode 日志分级、文件落盘、颜色输出，统一全局日志格式。

异常捕获层

四、三大输出报告定义（满足需求）
1. 全局汇总报告 summary_total.md
基础环境：GPU 型号、ROCm 产物路径、全部组件、测试分级范围
全局统计：总任务数 / 完成 / 失败 / 跳过
失败总览：所有失败组件 + 测试级别 + 根因摘要（匹配你之前 gfx1151 7 大类根因）
稳定性告警：标记触发 GPU Ring Timeout / MES 队列异常的 Full 负载任务
附件链接：每个失败明细报告、对应完整日志路径
2. 全量原始测试日志 full_logs/
每个任务独立日志文件，包含：
CTest 完整输出
dmesg 实时截取（GPU 复位、MES 报错）
amd-smi 定时采样记录（功耗、温度、VRAM/GTT）
执行耗时、进程退出码、超时标记
3. 单组件失败明细 failure_details/*.md
针对 FAIL 任务专项拆解：
组件名、测试等级、GPU、执行起止时间
报错关键字匹配：GPU ring timeout、FP32 NaN、unified memory permission fault、HIP Graph deadlock、Python3.14 兼容报错
关联 ROCm GitHub Issue 映射（#5151/#6034/#6372 自动填入）
复现条件、硬件影响评估、临时规避建议
原始日志片段截取（报错上下文）
五、关键增强功能（Loop Engineering 核心价值）
断点续跑
进程意外终止、GPU 卡死重启后，重新启动 CLI 自动读取 global_state，跳过已完成任务，从当前中断任务继续。
GPU 异常特殊标记
自动识别 GPU reset ring timeout，在报告中单独归类硬件稳定性故障，区分代码精度失败与硬件队列卡死。
分级批量调度
支持单跑 quick、全跑四级、只跑 full 压测三种模式。
批量失败根因聚合
自动统计 7 大类根因分布（gfx1151 硬件限制、Python3.14 兼容、Comprehensive 独有失败、统一内存、Profiler、HIP Graph、GPU Ring 超时），输出和你测试报告一致的汇总统计。
超时防护
为 full 级稀疏任务配置超长超时，检测 GPU 无响应时主动 kill 测试进程，记录 hang 状态并写入失败报告，避免整个 loop 卡死。

# cli/entry.py 入口示例
from opencode.terminal import prompt_input
from opencode.process import SubprocessRunner
from opencode.fileutil import atomic_write_json
from therock_loop_engine.state.global_state import GlobalStateManager
from therock_loop_engine.scheduler.test_loop import TestLoopScheduler

def main():
    # 交互式输入三参数
    artifacts = prompt_input("Input ROCm artifacts path: ")
    gpu = prompt_input("Input GPU model (e.g. gfx1151): ")
    components = prompt_input("Input component list split by comma: ").split(",")

    # 初始化全局状态
    state_mgr = GlobalStateManager()
    state_mgr.init_meta(artifacts, gpu, components)
    state_mgr.create_result_dir()

    # 任务排序
    scheduler = TestLoopScheduler(state_mgr)
    scheduler.sort_all_tasks()

    # 启动循环测试
    scheduler.run_loop()

if __name__ == "__main__":
    main()
