## 评论 (1 条)

### 评论 #1 — jlgreathouse (2019-03-18T21:07:05Z)

You can pass in `-cl-std=CL1.2` at compile time, but I see in your bug report that you have already tried that. For anyone finding this issue in the future: you can force this flag into your compile path with the environment variable `AMD_OCL_BUILD_OPTIONS_APPEND` or you can fully replace your OpenCL kernel build options with `AMD_OCL_BUILD_OPTIONS`

Actually, the error you are seeing [was also an error in OpenCL 1.2](https://www.khronos.org/registry/OpenCL/sdk/2.1/docs/man/xhtml/local.html). To quote, "Variables allocated in the __local address space inside a kernel function must occur at kernel function scope." As such, even if you are building in CL1.2 mode, this error is still valid.
