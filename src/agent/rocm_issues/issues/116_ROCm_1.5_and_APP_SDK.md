### 评论 #1 — jedwards-AMD (2017-05-04T14:56:06Z)

An OpenCL application compiled for the appropriate target platform should work with the OpenCL implementation running on the ROCm stack. This means that APP SDK applications should run on a OpenCL supported by ROCm drivers. However, the APP SDK is no longer supported, and any issues found in the APP SDK will not be fixed.

### 评论 #5 — gstoner (2017-07-02T01:42:06Z)

if you are just looking for the header which SDK used to supply.  Use this now 

Install the ROCm OpenCL implementation (assuming you already have the ‘rocm’ package installed):

sudo apt-get install rocm-opencl-dev

Set an environment variable that points to the installation directory for OpenCL:

export OPENCL_ROOT=/opt/rocm/opencl

that is all you need to start developing with OpenCL
