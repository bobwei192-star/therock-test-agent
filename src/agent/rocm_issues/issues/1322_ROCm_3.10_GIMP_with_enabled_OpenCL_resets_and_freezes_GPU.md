### 评论 #4 — baryluk (2020-12-09T09:32:18Z)

@knochenhans 

Hi, on a topic OpenCL in GIMP. I personally don't think it is worth using it in GIMP. I have done benchmarks using `babl` on CPU and on OpenCL (Fury X), and almost never GPU OpenCL method was faster. Only few complex transforms (which are rarely used) were observably faster. So I personally wouldn't bother if you have 8 cores on CPU, as it will be equally fast. In fact OpenCL in babl is mostly unmaintained, with only part of filters and transforms being even implemented, and the ones that were written log time ago, are poorly tested or tuned.

Still, having said that, I spotted few bugs and miscalculations when using `babl` with OpenCL on ROCm 3.9. Which I assume are bugs in the OpenCL driver / compiler. I didn't experience GPU hangs or crashes, tho. But I have a different GPU, so that might be different. That I would considered even bigger issue.

Crash on startup is even more weird.

If you could run gimp under `gdb`, and provide a stack trace (`bt full`), that would be quite useful probably.

Also, do you experience similar crashes in other programs using OpenCL?
