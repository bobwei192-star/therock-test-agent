## 描述

Environment: Ubuntu `22.10` with linux kernel `5.15.0` and amdgpu-pro `22.40-1538781.22.04` and rocm `5.4.3`.

Kernel command line options:

```

amdgpu.si_support=1
amdgpu.cik_support=1
amdgpu.exp_hw_support=1
amdgpu.gpu_recovery=1
amdgpu.ppfeaturemask=0xffffffff
```

As described [there](https://github.com/RadeonOpenCompute/ROCm/issues/1624#issuecomment-1501290112), both `rocminfo` and `clinfo --list` now lists the gfx7 device, but complete `clinfo` hangs, and I get an error printed in `dmesg`, and if I interrupt `clinfo` with `Ctrl+C` I get more error logs in `dmesg`. At the moment I interrupt `clinfo` I get a screen freeze for a short time.

The fact OpenCL applications will hang is a problem.

### 评论 #1 — illwieckz (2023-04-10T02:58:47Z)

In `amd/amdkfd/cik_event_interrupt.c` I read:

```c
        /* If there is no valid PASID, it's likely a firmware bug */
        pasid = (ihre->ring_id & 0xffff0000) >> 16;
        if (WARN_ONCE(pasid == 0, "FW bug: No PASID in KFD interrupt"))
                return false;
```

That reminded me something… @fxkamd, [here](https://github.com/RadeonOpenCompute/ROCm/issues/1624#issuecomment-988460780) I read from you:

> It also depends on custom firmware that was never pushed upstream due to quality regressions in the graphics driver.

Would you have a download link for that custom firmware? 🙂️

### 评论 #2 — fxkamd (2023-04-10T14:21:51Z)

The fix I implemented in Dec 2021 was to basically disable ROCm on Hawaii because the firmware for it was not public and had conflicts with graphics functionality. You're still using a 5.15 kernel that doesn't have my fix. I would recommend you update your Ubuntu kernel to linux-generic-hwe-22.04, which is currently at version 5.19 and does include my fix. This should do the trick:

```
$ apt install linux-generic-hwe-22.04
```
