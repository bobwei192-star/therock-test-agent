`sudo apt install linux-image-5.4.0-67-generic linux-headers-5.4.0-67-generic`

`sudo dkms install amdgpu/4.0-23 -k 5.4.0-67-generic`

### 评论 #2 — ROCmSupport (2021-03-22T09:24:31Z)

Meanwhile if you wish to try with ROCm 4., follow the below way.

You can install linux-5.4 or 5.6 kernel and uninstall linux-5.8 manually. Reboot and re-install rocm-dkms.
Like this:
_sudo apt install linux-image-5.4.0-67-generic linux-modules-extra-5.4.0-67-generic linux-headers-5.4.0-67-generic_
_sudo apt remove linux-image-5.8.0-44-generic linux-modules-extra-5.8.0-44-generic linux-headers-5.8.0-44-generic_

Hope this helps and clarifies.
Thank you.

### 评论 #3 — incardon (2021-03-22T10:00:54Z)

Thanks. For the fast answer. Installing linux-modules-extra-5.4.0-67-generic, removing 5.8 using the instruction above and reinstalling dkms made it works. To assemble a set of instruction for ROCM 4.0

`sudo apt install linux-image-5.4.0-67-generic linux-modules-extra-5.4.0-67-generic linux-headers-5.4.0-67-generic`
`sudo apt remove linux-image-5.8.0-44-generic linux-modules-extra-5.8.0-44-generic linux-headers-5.8.0-44-generic rocm-dkms`
`sudo apt autoremove`
`sudo reboot`
`sudo apt install rockm-dkms rockm-dev`
