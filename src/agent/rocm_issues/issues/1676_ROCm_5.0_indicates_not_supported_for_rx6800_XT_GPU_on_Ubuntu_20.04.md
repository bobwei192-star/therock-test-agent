## 描述

Installed ROCm 5.0 along with Radeon 21.50 drivers and getting error 'hipErrorNoBinaryForGpu: Unable to find code object for all current devices' when testing with Tensorflow.

Steps:

1. Removed all older gpu drivers and ROCm 4.5 from system & restarted.
2. Download installer script for AMD 21.50 drivers and ROCm 5.0 tools per guide: https://rocmdocs.amd.com/en/latest/Installation_Guide/Installation_new.html
3. Ran amdgpu-install --usecase=graphics,rocm,hip
4. Installed rccl library: apt-get rccl
5. Added user to 'video' & 'render' groups
6. Set LD_LIBRARY_PATH=/opt/rocm/lib/:/opt/rocm/rocblas/lib/:/opt/rocm/rccl/lib/:/opt/rocm/hsa/lib/:/opt/rocm/hip/lib/:/opt/rocm/miopen/lib/:/opt/rocm/hipfft/lib/:/opt/rocm/rocrand/lib/

In python console, run:

1. import tensorflow as tf
2. sess = tf.compat.v1.Session()

Result:

2022-02-13 14:06:45.471445: I tensorflow/stream_executor/platform/default/dso_loader.cc:44] Successfully opened dynamic library libamdhip64.so
"hipErrorNoBinaryForGpu: Unable to find code object for all current devices!"

See for reference:
https://github.com/RadeonOpenCompute/ROCm/issues/1617

I'm testing this on an older project using python 3.7 and Tensorflow 1.15.

See for reference: https://github.com/RadeonOpenCompute/ROCm/issues/1617
