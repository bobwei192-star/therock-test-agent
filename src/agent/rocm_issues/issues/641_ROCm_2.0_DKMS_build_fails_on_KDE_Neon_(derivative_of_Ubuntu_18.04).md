### 评论 #1 — krrk (2018-12-20T17:30:19Z)

I found the issue. I am running KDE Neon which is based on Ubuntu 18.04. The file `/etc/os-release` is not identical to Ubuntu's so the amdgpu dkms Makefile sets `OS_NAME` to "neon" instead of "ubuntu". This results in incorrect values for `DRM_VER` and `DRM_PATCH` which causes the build to fail.

The contents of `/etc/os-release` from my machine are shown below. I'm not sure if other ubuntu derivatives change the ID in this file, but if they do perhaps a way to fix this is to check ID_LIKE instead of ID and look for debian or ubuntu.

/etc/os-release:
```
NAME="KDE neon"
VERSION="5.14"
ID=neon
ID_LIKE="ubuntu debian"
PRETTY_NAME="KDE neon User Edition 5.14"
VERSION_ID="18.04"
HOME_URL="http://neon.kde.org/"
SUPPORT_URL="http://neon.kde.org/"
BUG_REPORT_URL="http://bugs.kde.org/"
PRIVACY_POLICY_URL="https://www.ubuntu.com/legal/terms-and-policies/privacy-policy"
VERSION_CODENAME=bionic
UBUNTU_CODENAME=bionic
```

### 评论 #3 — krrk (2018-12-20T19:47:20Z)

Yes I noticed that `ID_LIKE` is just `"debian"` on Ubuntu proper as well, but as you said I'm sure there is a way to modify the logic. At least grepping for "ubuntu" ought to work for both ubuntu proper and ubuntu derivatives or just checking for ubuntu in `ID` and `ID_LIKE`. Also in case you're interested I've been using ROCm since version 1.6 on KDE Neon without any issues.

For now and if anybody else comes by needing a workaround I've hacked the makefile in `/usr/src/amdgpu-2.0-89/Makefile` to change line 8 to `OS_NAME = "$(shell sed -n 's/^ID_LIKE=\(.*\)/\1/p' /etc/os-release | tr -d '\"' | cut -d" " -f1)"` and installation is successful.
