### 评论 #3 — harkgill-amd (2026-01-22T21:08:01Z)

It's worth getting a baseline with a fresh install to see if `https://repo.radeon.com/amdgpu/7.2/ubuntu ` repo is still getting pulled in. Could you try removing your current install with,

```
sudo amdgpu-install --uninstall --rocmrelease=all
sudo apt purge amdgpu-install
sudo apt autoremove
```
Then reinstalling ROCm 7.2 with, 
```
wget https://repo.radeon.com/amdgpu-install/7.2/ubuntu/noble/amdgpu-install_7.2.70200-1_all.deb
sudo apt install ./amdgpu-install_7.2.70200-1_all.deb
sudo apt update
amdgpu-install -y --usecase=graphics,rocm
```

### 评论 #7 — harkgill-amd (2026-01-26T19:20:02Z)

Hey @Danny3, there are currently ROCm packages hosted in the Debian repositories though they're lagging far behind the latest ROCm release - Debian Stable is on ROCm 5.7. Debian Unstable is on ROCm 6.4. 

Going forward, we'll be working with the Canonical team to help package and maintain newer versions of ROCm, see https://canonical.com/blog/canonical-amd-rocm-ai-ml-hpc-libraries, so you will be able to `apt install rocm` and get up and running without having to add any additional repositories. Development for Ubuntu 26.04 is ongoing and developers are in the process of updating to ROCm 7.1. The updates are based on the packages in the Bullwinkle PPA: https://launchpad.net/~bullwinkle-team/+archive/ubuntu/rocm-devel.

For Debian specifically, the metapackages for installing all of ROCm are under review https://ftp-master.debian.org/new/rocm_6.4.3-4~exp1.html.
