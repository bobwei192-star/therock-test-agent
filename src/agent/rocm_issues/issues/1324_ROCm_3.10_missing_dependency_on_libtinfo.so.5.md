## 描述

ROCm 3.10, on Ubuntu 20.04, Linux kernel 5.9.12.

I installed rocm-opencl3.10.0, set up groups (video & render), set up kfd rights, set up LD_LIBRARY_PATH:
export LD_LIBRARY_PATH=/opt/rocm-3.10.0/opencl/lib

Now when I run

```
~$ /usr/bin/clinfo 
dlerror: libtinfo.so.5: cannot open shared object file: No such file or directory

~$ /opt/rocm-3.10.0/opencl/bin/clinfo 
dlerror: libtinfo.so.5: cannot open shared object file: No such file or directory
ERROR: clGetPlatformIDs(-1001)
```

I tried to install manually on ubuntu libtinfo-dev, but that installed /usr/lib/x86_64-linux-gnu/libtinfo.so.6 and did not fix the problem.

Unless I'm doing something wrong, this seems a serious issue, affecting one of the supported OSes (Ubuntu 20.04). I'd like to have at least a workaround.

I belive the culprit, might be this file:

`/opt/rocm-3.10.0/opencl/lib/libamdocl64.so`:


This file is indeed in `rocm-opencl` / `rocm-opencl3.10.0` package:

```
root@debian:~# dpkg -S /opt/rocm-3.10.0/opencl/lib/libamdocl64.so
rocm-opencl: /opt/rocm-3.10.0/opencl/lib/libamdocl64.so
root@debian:~# 
```

However this package doesn't depend directly on libtinfo5:

```
root@debian:~# apt-cache show rocm-opencl | grep Depends
Depends: libelf-dev, comgr, hsa-rocr-dev
root@debian:~# 
```

I also checked transitive dependencies, and they don't provide or depend on `libtinfo5` either.

Fix would be to add `Depends: libtinfo5` to `rocm-opencl*` package.

@preda As a workaround in the mean time please do `sudo apt update; sudo apt install libtinfo5; sudo ldconfig;` this should help.
