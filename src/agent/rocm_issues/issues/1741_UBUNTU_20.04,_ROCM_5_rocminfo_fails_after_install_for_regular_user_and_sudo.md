### 评论 #5 — aoolmay (2022-05-31T15:13:13Z)

@mwibral From my experience, it won't. You have to go down the release tree and find the latest version that works for you, along with proper requirements (older Linux distributions, older kernel) and that forces you to use older and outdated(?) software stacks that are of interest to you. Due to GPUs being phased out and introduced in such a long time spans it's unlikely you will find many configuration supporting mixed GPU model families.

If your use case is ML for example i can point you to PlaidML, it will run on just OpenCL and is limited to rather ancient Keras frontend, but it held me over until they included limited and unofficial Navi support. For example, on ROCm 4.x(don't remember which one exactly) i was able to run mixed Navi/Vega just for PlaidML.
