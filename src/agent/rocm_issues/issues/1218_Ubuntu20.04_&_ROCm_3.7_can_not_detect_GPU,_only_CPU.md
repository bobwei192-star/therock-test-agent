### 评论 #3 — xuhuisheng (2020-09-14T02:58:16Z)

Here is the source code for requiring atomics on gfx803(polaris10)
https://github.com/RadeonOpenCompute/ROCK-Kernel-Driver/blob/master/drivers/gpu/drm/amd/amdkfd/kfd_device.c#L242
https://github.com/RadeonOpenCompute/ROCK-Kernel-Driver/blob/master/drivers/gpu/drm/amd/amdkfd/kfd_device.c#L566

AMD required atomics for gfx803 in the kernel drivers. If your cpu or motherboard didnot support atomics, the kernel driver will reject registering this device at the boot time.
Now I cannot find where the AtomicOps used in the source code. Maybe there is some configs in the microcode of firmware. so cannot make sure what will happen if just modify the kfd_device config from true to false. Need AMD to check this issue.

BTW: Now, if you want to use gfx803(RX580), need to buy a new cpu and motherboard which supporting atomicsOp PCIe3.
But, gfx803 maybe not supported by AMD offcially in the future, it is real an old product series. AMD said that Vega56/64 and RadeonVII didnot require atomicsOp which is newer and more expensive.
Recently, the price of card is raising, I want to wait for a few month to see if there is chances to buy a vega gpu to test rocm.
