## 描述

The latest ROCm-1.6 kernel exits `./vector_copy` at `hsa_init()` with error code 1008. The problem seems to be a failing ioctl call:
```
strace ./vector_copy
...
brk(0x18c6000)                          = 0x18c6000
brk(NULL)                               = 0x18c6000
ioctl(3, _IOC(_IOC_READ|_IOC_WRITE, 0x4b, 0x1d, 0x10), 0x7ffdfef45e30) = -1 EINVAL (Invalid argument)
close(3)                                = 0
fstat(1, {st_mode=S_IFREG|0664, st_size=116960, ...}) = 0
write(1, "Initializing the hsa runtime fai"..., 43Initializing the hsa runtime failed: 1008.
) = 43
exit_group(1)                           = ?
+++ exited with 1 +++
```
With ROCm-1.5 kernel (and everything else identical) that ioctl call succeeds:
```
...
brk(0x1c16000)                          = 0x1c16000
brk(NULL)                               = 0x1c16000
ioctl(3, _IOC(_IOC_READ|_IOC_WRITE, 0x4b, 0x1d, 0x10), 0x7ffc45762420) = 0
open("/sys/module/amdgpu/parameters/vm_size", O_RDONLY) = 4
fstat(4, {st_mode=S_IFREG|0444, st_size=4096, ...}) = 0
read(4, "128\n", 4096)                  = 4
close(4)                                = 0
...
+++ exited with 0 +++
```
Any ideas?

For 1.6 check to see if the /dev/kfd file exists, and has the correct permissions, crw-rw-rw- 1 root root. If the defice doesn't exist, you will need to examine dmesg to see if you can find a problem with loading the kfd driver: 'sudo dmesg | grep kfd".
