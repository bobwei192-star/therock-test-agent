### 评论 #16 — gstoner (2018-05-06T22:48:03Z)



@zpodlovics 

The issues you raised are base Linux driver issues to support graphics.  They flow into a driver that is supported by ROCm Userland team.   Understand we ride on top of Linux driver which Linux Graphics driver team build for us. 

The big difference from a standard driver it takes the AMDGPU driver and it supports KFD+  Thunk, also change in the base driver to better support Memory Allocation needed in Servers space and few other changes.  This now going upstream to simplify future driver. 

Now an APU has other issues, the SBIOS of the board needs two things. First is to support a  CRAT table for Topology Discovery,  and it has to support  IOMMUv2.   We saw ODM for  Carizzo system were spotty in supporting CRAT table in SBIOS. Which why a system was hit or miss with ROCm.   I was told there is Bristol Ridge system that supports ROCm.   

There is work to get Raven running it just not in ROCm 1.8.  The team has been working on it internally but we are working on a number of issues.
