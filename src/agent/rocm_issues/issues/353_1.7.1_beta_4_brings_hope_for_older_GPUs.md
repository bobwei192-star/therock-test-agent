## 描述

My R9 390 (GFX7 Hawaii) works for the first time on 1.7.1 beta 4 (kernel 4.13.0-36-generic). Prior to this version, all of the compute programs would freeze or crash and dmesg would be full of errors like "cp queue timeout". 

The betas are posted at http://repo.radeon.com/misc/archive/beta/. If you have one of these old GPUs, you will need to change the amdgpu boot parameters in grub or modprobe. My grub cmdline is: "amdgpu.dc=0 amdgpu.dpm=1 amdgpu.cik_support=1 amdgpu.si_support=1".

### 评论 #2 — pmc01 (2018-03-08T00:16:58Z)

dc=0 disables the new display engine. This engine is only needed for newer cards are will likely never work on older cards.

dpm=1 without this the system will freeze with a black screen on boot or have other problems. 

cik_support=1 si_support=1 Linux support for Sea Islands and Southern Islands has always been iffy so you have to explicitly enable it. Technically you only need the one for your card but it helps other people to include both.
