## 描述

### Problem Description

On a APU (PCI ID 1002:150e, gfx11), ROCr init fails with:

```
hsa api call failure at .../rocminfo.cc:1304
HSA_STATUS_ERROR_OUT_OF_RESOURCES
```

when using AMDGPU 6.14.14 + amdgpu-dkms-firmware 30.10.1 on Ubuntu 24.04.
The same hardware and ROCm user-space works when the kernel module/firmware is AMDGPU 6.12.12.

This appears to be a regression in the newer driver/firmware combination rather than a missing KFD/device support issue.

I have four nodes with the same APU. I'm happy to test anything or provide any additional information. 



### Operating System

Ubuntu 24.04.3 LTS (Noble Numbat)

### CPU

AMD Ryzen AI 9 HX PRO 370 w/ Radeon 890M

### GPU

Radeon 890M gfx1150

### ROCm Version

ROCm 7.0.1

### ROCm Component

_No response_

### Steps to Reproduce

1. Enable Secure Boot, enroll MOK; kernel 6.14.0-29-generic.
2. Add AMD repo via amdgpu-install_7.0.1.70001-1_all.deb.
3. amdgpu-install --usecase=dkms,graphics,rocm --accept-eula -y
4. This installs amdgpu-dkms 6.14.14 and amdgpu-dkms-firmware 30.10.1.
5. Ensure memlock unlimited (PAM + systemd) and re-login.
6. Run:

```
rocm-smi           # device shows
rocminfo           # FAILS with HSA_STATUS_ERROR_OUT_OF_RESOURCES
```

### Expected

rocminfo should enumerate HSA agents and clinfo should show an OpenCL device on gfx11 APU (0x150e) with the current AMDGPU 6.14.14/30.10.1 stack.

### Actual

ROCr init fails early with HSA_STATUS_ERROR_OUT_OF_RESOURCES, despite KFD attaching and rocm-smi working.

### Workaround

Downgrading to AMDGPU 6.12.12 + matching firmware resolves the issue immediately on the same hardware and user-space:

```
cat /sys/module/amdgpu/version  # 6.12.12
rocminfo                        # OK
```

### 评论 #2 — harkgill-amd (2025-09-25T17:37:55Z)

Hi @initiate6, when using ROCm on Ryzen (APUs), the in-kernel drivers are expected to be used not the `amdgpu-dkms` driver packaged with the release. This is highlighted in the 6.4.4 [ROCm on Ryzen installation steps](https://rocm.docs.amd.com/projects/radeon-ryzen/en/latest/docs/install/installryz/native_linux/install-ryzen.html#set-up-rocm-usecase),

> The --no-dkms parameter must be passed, as inbox drivers are required for ROCm on Ryzen.

Uninstalling the `amdgpu-dkms` driver will resolve the `HSA_STATUS_ERROR_OUT_OF_RESOURCES` errors when calling `rocminfo`. The drivers from the 6.14.0-29-generic kernel you have installed work just fine in my testing though the 6.14 OEM kernel is recommended/supported officially.
