## 描述

In section 12.18, the [Vega ISA guide](https://developer.amd.com/wp-content/resource/Vega_Shader_ISA_28July2017.pdf) states that vector memory operations support a flag called `lds` with the following effect:
>Data is transferred between LDS and Memory, not VGPRs.

I am interested in trying this out, but neither is this flag ever generated by the compiler (for example it will issue `global_load` followed by `ds_write` instead of `global_load … lds`) nor accepted when I write inline asm. I am using HIP/hcc.

Thanks!

### 评论 #1 — 949f45ac (2018-12-09T08:12:18Z)

Also it would be really great if there were intrinsics that allowed defining these flags (glc, slc) for vector memory operations, without the need to issue asm directly. This is because doing these via asm _efficiently_ is quite complicated, for you have to issue the `s_waitcnt` manually, too, and at the ideal position. (The compiler will place the instruction and the `s_waitcnt` as far apart from one another as possible, to give the operation maximal time to run in parallel. Hard to imitate by hand.)
