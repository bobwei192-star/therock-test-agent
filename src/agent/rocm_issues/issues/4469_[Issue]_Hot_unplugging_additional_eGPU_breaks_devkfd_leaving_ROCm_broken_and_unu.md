## 描述

### Problem Description

My setup is 1 permanent Radeon RX 7900 XTX (from Power Color) and additional Radeon RX 7900 XTX (from XFX) that is connected via TB3/TB4 as an external GPU. I would connect the eGPU when I need run intensive computations and disconnect when finish. iGPU (CPU graphics) is basically blocked/excluded from loading, destined for use in a VM - I do passthrough of the iGPU to KVM's VM. 

On a running system, I can connect eGPU and observe how 2 GPUs are being used for computations requiring ROCm. Once I hot unplug (disconnect) eGPU, leaving 1 permament GPU in the system, ROCm becomes broken and returns errors like:

![Image](https://github.com/user-attachments/assets/810990a9-e17c-4745-bab9-9c4a81ab3a4a)

Indicating there's some issue with blocked /dev/kfd

Nothing helps. Reconnecting eGPU does not bring back functionality.  Once the /dev/kfd is locked - ROCm becomes unusable and restart of the system is required.

### Operating System

24.04.1 LTS (Noble Numbat)

### CPU

AMD Ryzen 9 9950X 16-Core Processor

### GPU

2 x Radeon RX 7900 XTX

### ROCm Version

ROCm 6.3.4 / ROCm 6.3.2

### ROCm Component

ROCK-Kernel-Driver

### Steps to Reproduce



1) Having a permanent GPU connect eGPU over TB3/TB4. 
2) Run some workload requiring ROCm and observe how both GPUs are utilised.
3) Disconnect eGPU and confirm the disconnected eGPU is no longer reported by "lspci"
4) Run some workload requiring ROCm and observe how it no longer can be used. Basic commands like "rocminfo" return problems with /dev/kfd, by presenting error message "Unable to open /dev/kfd read-write: Invalid argument".
5) Attempt recover operation of ROCm without need to restart and without need to reload amdgpu driver.

I've got to the following discussion from November 2023 that is likely about the same issue with ROCm development stage: https://lore.kernel.org/all/20231106071405.121981-1-lawyiu.dev@gmail.com/T/ - Felix discusses the issue in an analogous scenario, except there the disconnection was made through software calls.
