# Problem description

I'm trying to run Folding@Home, and am getting "BAD_WORK_UNIT" errors.

I've [attached logs](https://github.com/RadeonOpenCompute/ROCm/files/2450086/runs.zip) including strace. The simulation program immediately fails with "ERROR:Discrepancy: Forces are blowing up! 1 0". This happens over and over for different tasks from the F@H servers.

The card is a Vega 56.

# Steps to reproduce

- Install ROCm 1.9 on a Debian/Ubuntu system
- Set up `/etc/profile.d/rocm.sh` with `HSA_ENABLE_SDMA=0`.
- Install FAHClient from https://foldingathome.org/start-folding/
- Create a new user account, call it "fold". Add to `video` group.
- Log in as that account, make sure env variables are ok.
- Create `config.xml` (see below).
- Run `FAHClient --config config.xml`.
