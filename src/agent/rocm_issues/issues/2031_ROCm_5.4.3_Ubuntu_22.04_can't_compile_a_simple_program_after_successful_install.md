## 描述

After successful install of rcom 5.4.3 on Ubuntu 22.04 the following simple program fails to compile.
```
int main(int, char **)
{
    return 0;
}
```
The compile command and error are:
```
$ amdclang++   hello1.cpp 
ld.lld: error: unable to find library -lstdc++
clang-15: error: linker command failed with exit code 1 (use -v to see invocation)
```
The compiler is:
```
$ amdclang --version
AMD clang version 15.0.0 (https://github.com/RadeonOpenCompute/llvm-project roc-5.4.3 23045 a29fe425c7b0e5aba97ed2f95f61fd5ecba68aed)
Target: x86_64-unknown-linux-gnu
Thread model: posix
InstalledDir: /opt/rocm-5.4.3/llvm/bin
```
I found that the above would work if I add the location of libstdc++ on this system (/usr/lib/gcc/x86_64-linux-gnu/11) to the command line. However, more complicated programs fail to locate headers from the STL. I did not find a way past that. I did notice that gcc 12 is also installed and I did not try that version since 11 is what is in the path.
```
$ gcc --version
gcc (Ubuntu 11.3.0-1ubuntu1~22.04) 11.3.0
```

Both Ubuntu 22.04 and rocm 5.4.3 were installed today from scratch. 

**Any ideas on what is wrong?**

I installed via the script method.
```
$ sudo amdgpu-install --usecase=rocm,rocmdev,hiplibsdk
```
Everything went smoothly. I validated the install as follows:
```
$ dkms status
```
