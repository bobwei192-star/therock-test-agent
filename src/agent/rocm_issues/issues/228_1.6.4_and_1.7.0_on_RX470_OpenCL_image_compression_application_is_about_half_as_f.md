## 描述

Intel i7 + RX 470 4 GB


I don't think this is related to compute, but rather to memory controllers and DMA transfer.

My application performs compression on 8,10 and 12 bit images, It allocates many OpenCL buffers and images, some residing only on device, some  with pinned host buffer for transfer of image data to and from card.

I found on windows that changing the order in which I allocate these buffers has a dramatic effect on performance. One particular order gives about twice as high performance as other orders.

On ROCm, I could get a 10% increase in performance by changing the order of allocation.

Also, interestingly, on ROCm, when I try an 8 bit image rather than 12 bit, my performance doesn't change. But, it should since 8 bit is 8/12 is big as 12 bit. 

So, it looks to me like sub-optimal DMA transfer from host to card.
