## 描述

Because of issue #768 / #769, I've been holding back with old rocm-2.2 libs so programs like Davinci Resolve and Blender would continue to work. It's been nearly a year without resolution to this issue. Only now, with the latest system upgrade to 3.1, it's a complete fail for all OpenCL applications. For whatever reason. rocm-3.1 still doesn't work with Davinci Resolve.

AMDGPU-Pro doesn't work with it. And now the latest rocm doesn't work with it. I'm downgrading again. But I wish a dev here would look into this.

### 评论 #1 — paranoidfactoid (2020-03-01T12:48:47Z)

OK. Rocm-3.1 goes down a new /opt directory tree. If you already have rocm-2.2 opencl libs, they'll be wiped and a symlink put in place for /opt/rocm -> /opt/rocm-3.1. So, the solution here is to wipe rocm entirely, including the old packages. Then reinstall rocm-3.1 from scratch. Then reinstall rocm-2.2 opencl and opencl-dev libs. And Davinci and blender will work again. This is continuing the completely broken situation that's prevailed for the last year. But at least it works for now.
