### 评论 #6 — krrk (2018-03-03T00:56:49Z)

I believe this is likely just a packaging issue. [MIOpen](https://rocmsoftwareplatform.github.io/MIOpen/doc/html/index.html) is a deep learning library by AMD which has two different backends HIP and OpenCL. Perhaps the headers for the different backends should be installed in separate locations (/opt/rocm/miopen-opencl and /opt/rocm/miopen-hip) or perhaps there are other conflicts with the backends? If there are other conflicts then the packages should explicitly mutually conflict each other.
