### 评论 #14 — ROCmSupport (2021-01-06T11:43:27Z)

Hi @Moading 
Got update on this issue:

One requirement to run test_device_execution on ROCm is that SBIOS places the GPU's doorbell physical address to 44bit address space. if the system places it to 46bit address space, then the system doesn't support running test_device_execution on ROCm. Looks like your SBIOS shows the similar output and so device_enqueue is not supported on your system.

For example:
