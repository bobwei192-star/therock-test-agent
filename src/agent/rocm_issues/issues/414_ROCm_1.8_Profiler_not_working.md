### 评论 #6 — jlgreathouse (2018-12-21T02:03:37Z)

Hi all,

Note that we recently released a new profiling framework, [rocprofiler](https://github.com/ROCm-Developer-Tools/rocprofiler). This is a new mechanism that will sit under other profilers to allow them to get access to our GPUs' and runtimes' performance monitoring features. For example, this will allow tools like PAPI to get access to AMD GPU hardware performance counters.

You may also be interested in it because it includes tools for directly gathering counters yourself. Because `rocprofiler` is primarily intended as a framework, these tools may not be as clean and easy to use as things like our CodeXL GUI-based programs. However, if you're interested in performance monitoring on ROCm, you may find it interesting to explore this tool.

#497 includes some extra discussion about this.

### 评论 #7 — jlgreathouse (2018-12-21T21:51:04Z)

Looks like I responded and closed just slightly too quickly.

Please note that with the release of ROCm 2.0 and [RCP 5.6](https://github.com/GPUOpen-Tools/RCP/tree/v5.6), we have now integrated RCP (a.k.a. `rcprof` discussed earlier in this thread) to work by using `rocprofiler`. This integration should have cleared up a number of the previous issues you may have had with profiling GPU kernels on the ROCm software stack. You should be able to install it by installing the `rocm-profiler` package on Ubuntu, CentOS, and RHEL.

Please try it out!
