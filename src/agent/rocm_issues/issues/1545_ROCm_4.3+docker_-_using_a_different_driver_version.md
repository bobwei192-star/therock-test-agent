I would like to use ROCm 4.3 with docker, but using a different driver that's usually provided for bare-metal (non-docker) machines, e.g., the 21.20 driver https://www.amd.com/en/support/kb/release-notes/rn-amdgpu-unified-linux-21-20

Hi @jammm 
Thanks for reaching out.
As per the definition of docker, all user space/mode components can be kept in a docker container, because docker container also runs on kernel. So we can not install/use kernel components/packages inside docker container, which is not possible.
So recommend to use stock kernel + install any kernel driver like amdgpu-pro or rocm kernel (rock-dkms and rock-dkms-firmware) on top of it and then run the docker container.
hope this helps.
Thank you.

### 评论 #3 — bd4 (2021-08-04T15:37:45Z)

Alternatively, you could run ROCm in a KVM virtual machine instead of in docker, and passthrough the AMD GPU to the KVM instance with vfio-pci. This is fairly involved though, and on top of what is usually required e.g. for passing through an nvidia gpu to CUDA, it also requires code changes and some monkeying with setpci after booting the virtual machine, described here: https://github.com/RadeonOpenCompute/ROCK-Kernel-Driver/issues/100. If you can, installing the desired driver on host and using it from docker is much simpler.
