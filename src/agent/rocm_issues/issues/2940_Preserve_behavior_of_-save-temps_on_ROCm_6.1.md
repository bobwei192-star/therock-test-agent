### Problem Description

On Ubuntu 22.04 w. linux kernel 6.7.5, Radeon Pro VII.
The behavior of the OpenCL compilation flag -save-temps seems altered on ROCm 6.1 vs. all previous ROCm versions.

The expected behavior: with -save-temps=folder/
the "temps" (.s, .so, .i etc) are written to that folder.

New observed behavior in 6.1: the value of the -save-temps=something is ignored, and some temps are written to the current folder, seemingly with overwriting names (when compiling multiple OpenCL programs in sequence).

Please restore the normal behavior of -save-temps in ROCm 6.1.

-save-temps is an important debug tool that has been around for decades with the AMD OpenCL stacks.

Caveat: ROCm 6.1 has not been released yet ATM.

### 评论 #3 — jamesxu2 (2024-06-24T13:59:50Z)

Hi @preda, 

The **clang-ocl** tool has been deprecated as of December 2023. Please use the hipcc compiler instead (which also uses the clang backend). 

You can use it like:
``` hipcc <file.cpp> -o <file> -lOpenCL -save-temps=<arg>``` where **arg** controls the output directory and must be either **cwd** (default, if you just pass -save-temps without an argument) or **obj** (save temp files in the build folder). The explanation of these arguments can be found in the [official clang documentation](https://clang.llvm.org/docs/ClangCommandLineReference.html#cmdoption-clang-save-temps). clang does not accept the name of a folder for this argument, but will prefix the name of your temporary files with the name of your output file and should solve your temp file self-overwrite issue.

For example, these files are generated by the -save-temps flag:
```
HelloWorld.cpp-hip-amdgcn-amd-amdhsa.hipfb     HelloWorld-hip-amdgcn-amd-amdhsa-gfx1030.o                   HelloWorld-hip-amdgcn-amd-amdhsa-gfx1030.s     HelloWorld-host-x86_64-unknown-linux-gnu.o
HelloWorld-hip-amdgcn-amd-amdhsa-gfx1030.bc    HelloWorld-hip-amdgcn-amd-amdhsa-gfx1030.out                 HelloWorld-host-x86_64-unknown-linux-gnu.bc    HelloWorld-host-x86_64-unknown-linux-gnu.s
HelloWorld-hip-amdgcn-amd-amdhsa-gfx1030.hipi  HelloWorld-hip-amdgcn-amd-amdhsa-gfx1030.out.resolution.txt  HelloWorld-host-x86_64-unknown-linux-gnu.hipi 
```

Additionally, you can ```export HIPCC_VERBOSE=1``` to see more details of the command passed to the compiler.
