## 描述

The definition of a const MAX_TEST_CASES in rdc.h causes gcc to fail with the error below.
One solution is illustrated by:
```
sed -i -e 's/^const uint32_t MAX_TEST_CASES =  RDC_DIAG_TEST_LAST - RDC_DIAG_TEST_FIRST + 1;/#define MAX_TEST_CASES (RDC_DIAG_TEST_LAST - RDC_DIAG_TEST_FIRST + 1)/' $(RDC_INC)/rdc/rdc.h
```

the error is:
```
/opt/rocm-4.5.0/rdc/include/rdc/rdc.h:479:28: error: variably modified 'diag_info' at file scope
     rdc_diag_test_result_t diag_info[MAX_TEST_CASES];
                            ^
```

Whether or not this is an excessive error detection by gcc I cannot say, but on my production hpc clusters gcc is what we compile with.

### 评论 #4 — morrone (2022-10-05T22:30:18Z)

This is still a problem in rocm-5.2.1.

Here is a stackoverflow entry describing why this doesn't work with gcc in more detail.

https://stackoverflow.com/questions/21592494/initializer-element-is-not-constant-error-for-no-reason-in-linux-gcc-compilin

The long and short of it is that "const" just means "read only". It does not mean determined-at-compile-time. It is still a variable. The use of MAX_TEST_CASES needs to be a a fixed value at compile time, not merely a read-only variable (a const).

The problem goes away if MAX_TEST_CASES is a #define, as @baallan pointed out.
