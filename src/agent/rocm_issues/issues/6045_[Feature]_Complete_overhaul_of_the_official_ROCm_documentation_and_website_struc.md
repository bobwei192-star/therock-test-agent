### Suggestion Description

Who exactly designed and structured the official ROCm website and documentation? It honestly feels like the pages were generated by ChatGPT v0.1 with zero human quality control. The entire structure is a complete mess, making it an absolute nightmare to find any specific, practical information.

To make matters worse, the documentation doesn't even index properly in search engines. When developers Google specific ROCm errors or manuals, we find absolutely nothing useful.

Is it really that massive of a problem to just sit down, properly sort the manuals, and organize the documentation into a logical, readable structure? The current state of the website is actively driving developers away from the AMD ecosystem. Please fix this.

Hi @supersfera-online,
 
Thanks for the feedback. We take documentation quality seriously and want to make sure we're addressing the right things. 
 
We have a couple different ROCm documentation spaces currently,
 
ROCm Documentation: https://rocm.docs.amd.com/
ROCm for Radeon/Ryzen Documentation: https://rocm.docs.amd.com/projects/radeon-ryzen
 
Both of these will be merged into the same space in a future ROCm release but you can already get a glimpse of what that'll look like with our preview release docs https://rocm.docs.amd.com/en/7.11.0-preview/. Beyond these two main spaces, each component also has their own documentation project, for example, https://rocm.docs.amd.com/projects/HIP.
 
Having said this, it'd be helpful if you could narrow down which of these spaces you're referring to in regards to poor structuring. We're constantly working on improving documentation quality control and usability so any specifics on your workflow and where the documentation fell short would provide good insights for our team.
