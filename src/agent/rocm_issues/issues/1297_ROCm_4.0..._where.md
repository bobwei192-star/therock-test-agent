### 评论 #11 — Spacefish (2020-12-05T20:43:44Z)

> While this is happening, the default ROCm repository for Ubuntu 20.04 here is completely broken & won't install things like rocm-dkms and my machine rebooted into a mesa software driver... On the Vega FE here though.

Had the same issue.. The kernel in 20.04 isn´t supported by the "pro" out of tree drivers (amdgpu works fine though). the installer which should install the dkms package fails, but a blacklist entry for the amdgpu module is added..

Delete the blacklist file in /etc/modprobe.d  not sure whether it´s in the "blacklist" or there is a seperate file, just grep for "amdgpu" in that folder. and remove the entry in the file.

You can load the blacklisted module manually via "sudo modprobe amdgpu", this will end you current Gui session though and you have to re-login.

AMD should just remove the "pro" drivers dependency for Ubuntu versions with moderately recent kernels which already support the required interfaces..
