### 评论 #2 — jlgreathouse (2019-01-12T04:07:08Z)

I believe I know what the problem is. Could you try modifying the following line and then rebuilding the ROCm OpenCL runtime? (You can find scripts in the [Experimental ROC project](https://github.com/RadeonOpenCompute/Experimental_ROC/tree/roc-2.0.0) for rebuilding the OpenCL runtime)

In [rocdefs.hpp](https://github.com/RadeonOpenCompute/ROCm-OpenCL-Runtime/blob/roc-2.0.0/runtime/device/rocm/rocdefs.hpp#L61), the line with `HSA_HAWAII_ID` looks like the following:
```
    /* TARGET HAWAII */ {HSA_HAWAII_ID, "", "hawaii", "", "Hawaii", 4, 16, 1, 256, 64 * Ki, 32, 0, 0, false},
```
Could you change it to instead say the following?
```
    /* TARGET HAWAII */ {HSA_HAWAII_ID, "", "hawaii", "gfx701", "Hawaii", 4, 16, 1, 256, 64 * Ki, 32, 0, 0, false},
```

If, for example, you are using Ubuntu 18.04, you should be able to run the following Experimental ROC scripts to rebuild the ROCm OpenCL runtime:

1. `distro_install_scripts/Ubuntu/Ubuntu_18.04/src_install/component_scripts/01_07_opencl.sh --get_code_only --source_dir ~/rocm_src/` This will download the source code for the ROCm OpenCL runtime
2. Modify `runtime/device/rocm/rocdefs.hpp` based on what I described above
3. `distro_install_scripts/Ubuntu/Ubuntu_18.04/src_install/component_scripts/01_07_opencl.sh --required` to install the software needed to build the ROCm OpenCL Runtime
3. `distro_install_scripts/Ubuntu/Ubuntu_18.04/src_install/component_scripts/01_07_opencl.sh --source_dir ~/rocm_src/ --package ~/rocm_packages/ --local` This will build the modified ROCm OpenCL runtime .deb packages and put them into `~/rocm_packages`. You can choose to install them with `dpkg` after that.
4. Alternately, you could run `distro_install_scripts/Ubuntu/Ubuntu_18.04/src_install/component_scripts/01_07_opencl.sh --source_dir ~/rocm_src/`, which will install the modified ROCm OpenCL Runtime into `/opt/rocm/` on top of anything else you installed there, but it will **not** create/install this a .deb package

To help other people having issues with their Hawaii GPUs on ROCm 2.0:

**I could fix my R9 390 test setup at home with a rollback to ROCm 1.9.2**

### Rollback to ROCm 1.9.2

- Remove any parts of current ROCm setup
```sudo apt autoremove rocm-dkms rocm-dev rocm-utils```

- Clone experimental project
```git clone https://github.com/RadeonOpenCompute/Experimental_ROC.git```

- cd into project directory
```cd Experimental_ROC```

- switch to 1.9.2
```git checkout roc-1.9.2```

- run preparation script
```distro_install_scripts/Ubuntu/Ubuntu_18.04/deb_install/00_prepare_system_ubuntu_18.04.sh```
_will prompt to reboot after finish -> confirm with 'y'_

- run setup
```distro_install_scripts/Ubuntu/Ubuntu_18.04/deb_install/01_install_rocm_ubuntu_18.04.sh```
_Will prompt to execute script 02 after finish -> confirm with 'y'_

- reboot again and you're back to ROCm 1.9.2

**HINT:** 
If apt complains about unresolved dependencies, there might be some packages left from newer setup.
You can remove them with
```sudo apt autoremove hip_* hsa*```
