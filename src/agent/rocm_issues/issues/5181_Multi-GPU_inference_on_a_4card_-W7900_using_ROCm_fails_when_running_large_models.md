## 描述

### Problem Description

The issue was initially encountered when quantizing Qwen2.5-32B-Instruct on a W7900 GPU using PyTorch 2.7.1 + ROCm 6.2.4 with multi-GPU, resulting in a quantization failure error.
 
The minimal reproducible case is: using the Transformers library to load the Qwen2.5-7B model, single-GPU inference works fine, but multi-GPU inference produces garbled output. 
Rocm 6.4.3, After trying different ROCm versions of PyTorch:
ROCm 6.1: the process hangs
ROCm 6.2: produces garbled output
ROCm 6.3 and 6.4: the following error occurs:
:0:rocdevice.cpp            :2993: 7226916028 us:  Callback: Queue 0x7fd1e8200000 aborting with error : HSA_STATUS_ERROR_EXCEPTION: An HSAIL operation resulted in a hardware exception. code: 0x1016

following is  Qwen2.5-7B model, single-GPU and  multi-GPU inference result:
Rocm 6.3.4  torch:2.7.1+rocm6.2.4
multi gpu 4card
```
from transformers import pipeline
pipe = pipeline("text-generation", model="../../../../../model/Qwen2.5-7B-Instruct",device_map="auto")
message = [{"role":"user", "content":"who are you?"}]
a = pipe(message)
print(a)
```
