### 评论 #12 — srinivamd (2020-08-23T14:04:50Z)

FWIW, please collect the logs using `rocm_techsupport.sh` utility (available https://github.com/srinivamd/rocminstaller ) for analysis and debug.
You may also try to use `rocminstall.p`y script to install ROCm - it installs all available ROCm packages of the specific release version - it is also available at https://github.com/srinivamd/rocminstaller .
Instead of using `-lhip_hcc` use `-lamdhip64` The symbolic link libhip_hcc.so (only valid until 3.3 ROCm release) to libamdhip64.so (ROC CLR runtime starting in 3.5) has been removed. 
Use hipcc to build automatically adds the correct library, excerpt from hipcc verbose mode: 
`-lhsa-runtime64 --enable-new-dtags --rpath=/opt/rocm-3.7.0/hip/lib:/opt/rocm-3.7.0/lib -lamdhip64`
HTH

### 评论 #21 — srinivamd (2020-08-26T13:35:05Z)

> Can someone update instructions to install package "rccl"?

When installing ROCm, use the rocminstall.py script (available at https://github.com/srinivamd/rocminstaller) simple to use, and installs with no hassle in one step.
`sudo ./rocminstall.py --rev 3.7`
will install all of ROCm 3.7.0 released packages. (Kernel packages `rock-dkms` and rock-dkms-firmware` will be installed only if none are installed already, i.e. it will not overwrite.)
Just run it and forget about ROCm install, just use ROCm~

### 评论 #22 — srinivamd (2020-08-26T13:58:47Z)

If you use `rocminstall.py` from https://github.com/srinivamd/rocminstaller there would NO need to uninstall ROCm to pick up/install a new ROCm release! Simply use `rocminstall.py` to install the new release.
