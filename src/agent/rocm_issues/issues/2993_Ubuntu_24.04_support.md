### 评论 #32 — harkgill-amd (2024-11-20T15:10:34Z)

@chris-hatton, I'm not seeing any issues with the Ubuntu 24.04 quick start install instructions on my end. It's possible there are remnants from a previous ROCm install that are causing the dependency errors you're encountering. Could you please try removing the existing install with the below commands prior to installing with the quick start instructions?
```
sudo amdgpu-install --uninstall --rocmrelease=all
sudo apt purge amdgpu-install
sudo apt autoremove
```

For reference, the AMD repos (amdgpu and ROCm) are registered during the amdgpu-install .deb installation. 
```
sudo apt install ./amdgpu-install_6.2.60204-1_all.deb
```
After this step, you should see `amdgpu.list` and `rocm.list` present under `/etc/apt/sources.list.d/`.

If you do encounter any further issues with the install, please provide updates over at https://github.com/ROCm/ROCm/issues/4042. Thanks!

### 评论 #33 — harkgill-amd (2024-11-25T14:55:27Z)

@chris-hatton, did some more digging and was able to reproduce these exact dependency errors when using the Ubuntu 22.04/jammy amdgpu-install on Ubuntu 24.04/noble. See the difference in the paths for the two installers below.

```
wget https://repo.radeon.com/amdgpu-install/6.2.4/ubuntu/jammy/amdgpu-install_6.2.60204-1_all.deb
```
```
wget https://repo.radeon.com/amdgpu-install/6.2.4/ubuntu/noble/amdgpu-install_6.2.60204-1_all.deb
```
Please be sure to use the correct installer according to the OS that you have installed. As for next steps, you can remove the existing amdgpu-install with the commands from my previous comment and install the 24.04/noble packages with
```
wget https://repo.radeon.com/amdgpu-install/6.2.4/ubuntu/noble/amdgpu-install_6.2.60204-1_all.deb
sudo apt install ./amdgpu-install_6.2.60204-1_all.deb
sudo apt update
sudo apt install amdgpu-dkms rocm
```
